项目/资源与获取

资源与获取

当前没有 Noemion 软件可以下载。你现在可以阅读规范、研究与机器可读示例；正式发行后，这里才会提供经过完整性和供应链检查的安装资源。

尚无软件发行公开研究资料未来发行需验证

现在可以获取什么

当前公开的是项目设计、规范和示例，不是可运行产品。不要执行网上出现的同名 endem 命令，也不要根据设计文档编写安装脚本。

  • 网站、设计记录与研究资料已公开

    用于理解项目目的、责任边界、当前策略和待定问题；除文件另有说明外，本仓库源码与文档采用 Apache License 2.0。

  • 核心规范、实验格式与机器可读示例草案已公开

    用于理解条款、案例和预期分类之间的关系。精确版本见机器可读登记，案例见向量源；它们不是稳定 ABI 或组件运行结果。

  • producer、inspector、runner 与 endem尚未开发

    当前只有职责、信任边界和未来验证条件，没有生产源码、可执行程序、包名、版本号或安装命令。

未来下载出现时怎样判断它可以使用

一个文件出现在下载页上并不等于它适合安装。正式版本必须让开发者逐项核对对象、来源、许可、完整性和支持范围。

必须提供的信息开发者应能完成的检查单独出现时不能证明
版本、平台、文件大小、摘要和签名确认下载对象与发布记录完全一致，再开始解析或安装签名有效不能证明软件安全、适用或获得运行授权
SBOM、逐对象许可证和来源锁定查明代码、依赖、模型相关对象和资料分别来自哪里，允许怎样使用仓库使用 Apache-2.0 不能自动覆盖第三方依赖、模型权重、数据或商标
固定构建环境和可复现步骤在两个独立环境重建并比较实际输出相同字节不能证明设计正确，不同字节也不能自动选择可信一方
安全、模糊、资源限制和独立读取结果区分结构有效、两条读取路径一致、策略允许和业务接受测试通过不能扩大证据覆盖，也不能替代真实部署观察
支持范围、撤回、漏洞响应、密钥轮换和归档判断升级、停用和长期验证路径是否完整“最新版”不能同时代表稳定版、开发快照、协议版本和依赖状态

“公开”需要说明公开了什么

软件自由、模型材料可得、服务接口可用和用户控制实际执行是不同问题。页面必须逐项说明，不能用“开放”“本地”或“可导出”概括全部权利。

资源事实准确说法不能自动声称
只能远端调用托管 API 可用，并列出服务条款、数据路径和退出条件用户拥有软件、模型参数或那一次服务执行的控制权
模型参数可以取得参数可下载，并列出精确许可、配套代码与数据说明它已经满足 Open Source AI、自由软件或可复现模型的全部条件
源文件可以阅读源码可见；还要核对许可证允许运行、修改和再分发中的哪些动作允许任何用途，或服务端实际运行的就是这份源码
代码、参数与资料较完整逐项列出架构、参数、推理代码、训练代码、数据说明、数据和文档一个“开放模型”标签可以替代对象清单和逐项许可

更完整的对象边界见模型开放性与软件自由研究，执行、数据和迁移边界见托管人工智能服务与用户控制研究。两份材料都是非规范研究，不代表项目已经拥有模型或服务。

从设计资料到正式发行

状态读者可以取得进入下一状态前必须补齐
当前设计项目范围、ADR、规范草案、开放问题和验证方案继续明确责任、反例、失败条件和声明上限
未来实验规范快照、向量和获准的原型源码可重复构建、安全测试、许可证清单和明确的实验限制
未来候选版候选软件与互操作材料摘要、签名、SBOM、独立读取、模糊测试和复现报告
未来稳定版软件、规范、示例与迁移说明漏洞响应、撤回、长期归档，以及正式名称、读音和权利复核

名称发布条件：Endem 目前只是设计阶段名称。它尚未完成商标、近音、首次朗读和口头区分验证。任何验证失败都应在接口和 ABI 固定前通过 ADR 一次性改名，不保留旧命令、格式别名或兼容发音。现行名称与候选读法见术语简化决定。

现在从哪里开始

先理解项目解决什么问题用一次依赖升级分开目标、权限、外部任务、证据和最终决定。 查看计划中的 Endem 工作流程按形成、组合、独立检查和受限运行的顺序阅读手册；其中没有虚构的安装命令。 查看项目当前进展了解已经形成的资料、正在研究的边界和仍待验证的条件。
