为什么暂不提供下载

项目尚未进入组件代码开发阶段，因此不存在 Ktisor、Theor、Drasor、endem CLI、安装包或二进制。当前公开的是架构、规范和验证方案；把这些资料包装成下载会错误暗示组件已经实现。

正式版本

暂未发布；没有有效的安装命令、包名或版本号。

资源状态

  • 网站与设计资料已公开

    项目门户、ADR、架构、规范草案与验证路线；除文件另有说明外，本仓库源码与文档采用 Apache License 2.0。

  • 核心、格式与规范向量实验草案已公开

    已有 END-CORE、END-FMT、10 个 Endem 语义 JSON 向量、6 个 END-P0 结构字节和 14 个 END-P1 字节；稳定 ABI 仍待形成。

  • P0-W4 语言实验证据已公开

    C/Rust 结构读取原型、机器可读结果和 Linux 模糊测试工作流可供复核；它们不是可安装程序或生产解析器。

  • Endem Ktisor 与独立 Theor尚未开发

    当前只有职责、信任边界、候选语言和未来验证条件；没有实现仓库、生产源码或可执行程序。

  • 发行制品等待验证

    需要双环境复现、摘要、外部签名、SBOM、供应链声明与撤回机制。

“公开”不等于同一种权利

当前公开仓库不包含模型参数、训练数据或托管模型服务。Apache-2.0 只覆盖实际置于该许可证下的仓库材料，不自动覆盖未来模型权重、数据、第三方依赖、外部服务或商标。

资源事实准确说法不能自动声称
只能远端调用托管 API 可用，并列出服务条款用户拥有软件、参数或修改权
参数可以取得模型参数可下载，并列出精确许可与限制Open Source AI、自由软件或可复现模型
源文件可以阅读源码可见；再核对许可证授予的动作允许任何用途、修改和再分发
代码、参数与资料较完整逐项列出架构、参数、推理/训练代码、数据说明、数据和文档用一个“开放模型”标签替代对象清单

未来若引用 GNU 自由软件、OSI Open Source AI 或 Model Openness Framework，必须给出定义版本、覆盖对象、逐项证据和复核日期。详细边界见非规范模型开放性与软件自由研究提案。
