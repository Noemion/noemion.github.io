从提示词工程开始

今天，引导大语言模型（LLM）的主要方式仍是向模型提供 Prompt 和上下文。提示词工程把角色、目标、限制、示例、工具说明、输出格式与验收要求组织成自然语言，让模型更可能生成符合预期的结果。

这种方式直观、灵活，也不会因为 Noemion 出现而消失。问题在于：当任务变得复杂、长期或高风险时，目标、权限、证据和验收条件仍然埋在文本中，只能由模型临时解释，难以被独立工具检查和复用。

真正的缺口：意义还没有成为工程对象

提示词解决“怎样向模型表达”，Noemion 进一步追问“被表达的目标和意义，能否脱离某次对话，成为可验证、可组合和可治理的计算对象”。

目标与措辞绑定

同一意图可以有多种说法，但现有系统往往把具体文字当作目标本身，难以区分表达方式与真正要满足的条件。

约束不能独立检查

“必须做到什么”和“绝不能发生什么”常与背景文字混在一起，无法在运行前由确定性工具完整验证。

依赖不能稳定链接

知识、工具、权限和子任务通过上下文临时拼接，缺少明确符号、类型化引用、版本关系和重定位机制。

结果难以复现与治理

模型、上下文或调用顺序变化后，失败原因难以定位，也难回答执行时究竟采用了哪些目标、证据与权限。

Noemion 的出发点：不替代 Prompt，而是把需要长期保存和严格处理的语义，从一次性文本上下文中提取出来，交给确定性工具链管理。

思想来源：从“思考的行为”到“被思考的对象”

Noemion 不是生硬缩写，而是从哲学术语 Noema 衍生出的独立名称。以下思想用于解释问题和建立设计视角，不等于把某套哲学理论直接编码成技术规范。

现象学：Noesis 与 Noema

在胡塞尔的意向性分析中，Noesis 指理解、判断或思考的行为，Noema 指与该行为相关的意义内容或对象。Noemion 借此区分“形成理解的过程”和“形成后可被处理的表征对象”。

分析哲学：表达不等于所指

弗雷格关于意义与指称的区分提醒我们：一段表达、它呈现意义的方式，以及它所指向的对象不能简单混为一体。工程上，这启发我们把 Prompt 的表面措辞与任务的语义身份分开。

形式语义：说明满足条件

目标不能只依赖“读起来像什么”，还需要说明什么状态算成功、哪些约束不可违反、证据如何支持结论，以及仍有哪些歧义尚未解决。

概念背景可参见 Stanford Encyclopedia of Philosophy 的 Edmund Husserl、Intentionality 与 Gottlob Frege 条目。Noemion 对这些概念的使用属于工程类比，而不是哲学解释的定论。

工程转译：把意义送入确定性工具链

传统编译器把源程序转化为中间表示和对象文件，再经过链接、装载和执行。Noemion 借鉴这条工程路径，但处理的不是传统机器指令，而是生成式任务的目标、约束、证据、权限、歧义和验收条件。

在这个对应关系中，编译过程类似形成理解的 Noesis，GSIR/GOBJ 类似形成后可被分析和组合的 Noema。这只是帮助理解的映射；真正的行为由规范、数据结构、不变量和测试定义。

Noemion 实际要建设什么

GSIR：目标语义

表达目标、解空间、硬约束、软偏好、歧义、证据要求、权限和验收条件，回答“什么结果才算满足要求”。

GOBJ 与 SSO：对象工程

组织语义记录、符号、类型化引用、重定位、依赖闭包与完整性信息，使对象能够被链接、分发和受控装载。

确定性工具链

建设 Compiler Core、链接器、验证器、对象检查工具和安全 Reader/Writer；Strict 模式不调用模型并保持可复现。

受约束 Runtime

Runtime 在已验证对象规定的边界内调用生成式能力并执行 Lowering；模型可以提出候选，但不能决定对象布局或绕过约束。

边界：我们不在做什么

  • 不是提示词二进制化：把 Prompt、文本或 Skill 包换一种编码，不会自动获得类型、约束、链接和验证语义。
  • 不是取代大语言模型：模型仍负责擅长的生成式求解，但其输出始终是不可信候选。
  • 不是让模型生成对象格式：只有确定性的 Compiler Core 能生成 GSIR/GOBJ，模型不得决定二进制布局。
  • 不是已经完成的标准：当前名称、字段、ABI、命令行和扩展名仍需规范、ADR、原型和一致性测试逐步冻结。

当前状态

设计与安全核心阶段：目前没有可供安装的编译器、Runtime 或正式规范版本。网站展示的是问题定义、设计约束和证据路线，不是已完成能力。

第一阶段只推进规范、确定性 Compiler Core、安全 Reader/Writer、验证器和测试基础，不引入模型、LLVM、MLIR 或大型框架。性能、跨模型等价、互操作性和产业价值必须等待可复现的实现与实验验证。
