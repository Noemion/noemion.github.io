# Noemion 首页设计规范

状态：Endem 词汇版本
适用范围：`index.html` 与门户专用样式/交互

## 首屏任务

首屏必须在不要求读者了解编译器术语的情况下回答三件事：

1. Noemion 让自然语言目标成为可确定性处理的持久制品。
2. 这个最小制品叫 Endem，而不是带品牌前缀的传统对象缩写。
3. 当前处于规范和安全核心阶段，没有可安装软件。

主行动进入入门指南，次行动进入 Endem 生命周期。首屏图形只显示 `say/aim/must/done/open`、`form` 和 Endem，不显示虚构监控数据、模型拟人形象或未冻结字段编号。

## 图形语义

- Endem 面板使用五条短记录表达语义组，不表现为传统机器段表。
- 数据流标签使用 `SAY / AIM / MUST / DONE / OPEN` 与 `FORM`。
- Endem、Weave、Frame、Witness 的关系使用状态/闭包/运行证据区分，不画成四个同构文件盒。
- 颜色只区分来源、边界、未决、验收和状态；动画不暗示真实吞吐、成功率或发布状态。

## 版式与动效

- 桌面主画布约 1200px，正文阅读列约 700–760px；移动端按语义顺序单列。
- 门户标题 48–64px，章节标题 30–42px，正文 17–18px，约 1.75 行高。
- 进入与悬停只过渡明确属性；禁止 `transition: all`、持续视差和不可中断交互动画。
- `prefers-reduced-motion` 下取消进入与循环动画，内容和状态仍完整可读。
- 所有行动入口有可见键盘焦点和至少 44×44px 触控区域。

## 验收

1. 桌面与 390px 移动视口均无页面级横向溢出。
2. 首屏直接出现 Endem 定义、项目阶段与阅读入口。
3. 首页不存在历史术语、历史路由、23 个工具承诺或模型可写规范字节的暗示。
4. 源码测试、Jekyll 构建、成品测试、脚本语法与真实浏览器检查全部通过。
