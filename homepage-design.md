# Noemion 首页设计规范

状态：已实施，桌面端与移动端视觉验收通过
范围：`index.html` 项目门户及其必要的共享外壳、样式和交互

## 1. 设计目标

首页要在保留 Noemion 工程严谨性和成熟度边界的前提下，以清晰的空间节奏、内容层级和交互手感，让第一次访问的读者依次理解：现实缺口是什么、Noemion 的直白结论是什么、对象工具链如何回应、当前做到哪一步、下一步应阅读什么。

首页采用以下设计关系：

- 约 1180–1200px 的居中长页面画布，桌面端保留稳定外侧留白。
- 顶部横向导航、左右分栏首屏、双栏论点区、居中章节标题、横向入口行和三列网格的交替节奏。
- 大字号但克制的标题、细分隔线、近白纸面、少量高饱和强调色和低圆角几何表面。
- 导航、入口行、箭头、图片和按钮在 150–300ms 内完成的快速反馈。
- 移动端按同一阅读顺序折叠为单列，而不是缩小桌面布局。

所有抽象对象图形、流程图、内容和色彩系统都必须服务于 Noemion 自身语义。

## 2. 首页在全站系统中的角色

首页承担全站唯一的完整“叙事型门户”，但不再是唯一采用该设计语言的页面。全站共享顶部导航、连续画布、编辑式排版和交互语言；列表、文章、工具和文档页面分别采用适合其职责的页面范式，详细映射统一维护在 README。首页不承担规范定义，也不把候选方案、未来能力或研究愿景写成已完成成果。

首页首屏必须先用普通语言回答现实问题，再出现 GSIR、GOBJ、SSO、Strict 和 Lowering 等术语。所有能力描述继续标注成熟度；当前仍明确处于“设计与安全核心阶段”，没有可安装编译器、Runtime 或正式规范版本。

### 3.2 首页导航信息架构

桌面顶部导航保持一行，按读者任务而不是仓库目录机械排列：

1. 品牌 `Noemion` 返回首页。
2. `为什么` 指向 `about/background.html`。
3. `如何工作` 指向 `architecture/object-lifecycle.html`。
4. `规范` 指向 `specifications/index.html`。
5. `工具` 指向 `tools/index.html`。
6. `文档` 指向 `docs/index.html`。
7. 右侧状态入口只显示当前阶段的 3–4 字短值，以卡片内部动画表达正在进行，并指向 `development/current-stage.html`。

移动端只保留品牌、状态短标和菜单按钮。菜单展开后显示同一组入口，不复制另一套链接数据；键盘焦点必须被清晰看见，关闭后焦点回到菜单按钮。

### 3.3 图形资产语义

| 图形 | 构图 | 必须表达 | 禁止表达 |
| --- | --- | --- | --- |
| 首屏主图 | 右侧纵向“语义频谱”中悬浮一组可链接对象切片，节点从自然语言片段逐步收敛到带符号和重定位标记的对象 | 表达过程与对象分离、歧义被保留、确定性核心决定布局 | 人脑、机器人、魔法粒子、虚构产品界面或人物肖像 |
| 背景与边界缩略图 | 多段文字指向同一目标身份，但硬约束独立成层 | 措辞不等于语义身份 | 把 Prompt 描述成无价值或应被取代 |
| 生命周期缩略图 | Source、Compiler Core、Object、Linker/Loader、Runtime 五段路径 | 确定性边界和不可信输入边界 | 模型直接连接 GOBJ 输出 |
| GSIR/GOBJ 缩略图 | 左侧目标/约束图，右侧 Section/符号/重定位对象表 | 语义层与对象格式分工 | 冻结尚未决定的字段编号或 ABI |
| Strict 缩略图 | 相同输入两次生成相同摘要，模型路径被隔离在候选区 | 无模型、确定、可复现 | 宣称已有性能、兼容性或安全认证结果 |

图形优先使用自有 SVG，并沿用现有 `glass-mint-underlay.svg`、如意云纹和 favicon 的几何语言；可以重新组合视觉语汇，但不直接把现有装饰资产当作技术证明。

“背景与边界缩略图”，也就是“表达方式，不等于语义身份”卡片左侧图形，内部可见标签统一使用简洁英文，例如 `CREATE REPORT`、`WRITE ANALYSIS` 和 `SAME GOAL IDENTITY`。该约束不适用于首屏主图；首屏仍可通过中英文组合表达输入语义，同时保留中文无障碍说明。

## 4. 视觉系统

### 4.1 画布与网格

- 桌面主画布上限 1200px，整体居中，外侧背景与纸面通过极轻色差区分。
- 门户内容区不再套现有 1000px 白色正文卡片；分区直接构成连续画布，并以 1px 低对比分隔线建立节奏。
- 首屏与关键介绍区使用 1:1 或接近 5:4 的双栏；内容行采用“图形 24% / 文本弹性区 / 44px 行动按钮”。
- 三列网格在宽桌面等分；中等宽度改为两列；移动端全部单列。
- 桌面分区纵向留白以 80–112px 为主，内容行以 28–40px 内边距为主；移动端分区留白收敛到 48–80px。

### 4.2 字体与层级

- 延续系统无衬线字体，标题使用 600–700 字重，正文使用 400–500 字重，避免依赖外部商业字体。
- 桌面主标题约 56–64px，移动端约 44–48px；章节标题桌面约 34–40px，移动端约 28–32px。
- 标题使用 `text-wrap: balance`，中短正文使用 `text-wrap: pretty`；根元素保持 macOS 字体抗锯齿。
- 状态数字或动态计数才使用等宽数字；静态版本号和装饰数字不强制使用。

### 4.3 色彩与表面

- 以现有墨绿、纸白和薄荷色为基础，加入 Noemion 的“语义频谱”：薄荷、青蓝、琥珀和朱橙只用于首屏图形、关键短语和分区细线。
- 卡片和入口行保持近方形或小圆角，不使用大面积胶囊卡片；圆形箭头是少量明确的行动符号。
- 深度表面优先使用透明多层阴影，内容分区仍使用细分隔线。图片或渲染图在浅色模式使用 `rgba(0, 0, 0, 0.1)` 内描边，深色模式使用 `rgba(255, 255, 255, 0.1)`。
- 图形资产必须服务于 GSIR、GOBJ、符号、重定位、链接或可信装载等真实概念，禁止仅为装饰堆叠无意义科技纹理。

## 5. 动效契约

动效目标是形成快速、直接和可中断的手感，同时提高可访问性与性能。

| 场景 | 行为 | 时长与曲线 |
| --- | --- | --- |
| 首屏进入 | 标题、说明、行动区和右侧图形分组进入；透明度、4px 模糊和 12px 下移逐项归位 | 360–420ms，组间约 90ms，单次关键帧 |
| 顶部菜单 | 桌面悬停/聚焦改变底色；移动菜单和分组面板可反向中断 | 180–200ms，`cubic-bezier(.2,0,0,1)` |
| 内容行悬停 | 纸面轻微提亮，图形最多放大 1.015，箭头向右移动 4px | 150–200ms，仅过渡具体属性 |
| 三列卡片悬停 | 图片亮度或对比度轻微增加，标题和分隔线强调 | 150ms |
| 按压反馈 | 可点击按钮和入口缩放到 0.96，释放后平滑恢复 | 150ms，可中断 transition |
| 滚动进入 | 只对较大的叙事分区执行一次 12px/透明度进入；不做持续视差 | 360ms，分区间不连锁拖延 |
| 状态图标切换 | 两个图标同时保留并以透明度、0.25→1 缩放和 4px→0 模糊交叉切换 | 300ms，`cubic-bezier(.2,0,0,1)` |

禁止使用 `transition: all`、不可中断的交互关键帧、横向页面抖动、持续跟随鼠标的 3D 倾斜和无信息价值的循环动画。`will-change` 只在真实出现首帧卡顿时用于 transform、opacity 或 filter。`prefers-reduced-motion: reduce` 下取消进入、滚动和图形循环，仅保留即时状态变化。

## 6. 响应式与可访问性

- 桌面导航在移动端变为品牌加菜单按钮；按钮触控区域至少 44×44px。
- 首屏移动端先显示标题与状态，再显示对象图形；双栏论点变为标题在前、解释在后。
- 横向内容行在移动端变为“图形 / 标题与摘要 / 行动按钮”的纵向卡片，不依赖横向滚动。
- 主要断点以内容是否拥挤决定，建议在约 1100px、760px 和 480px 校验，而不是复制某个设备清单。
- 所有动效后的最终内容必须在无 JavaScript和减少动态效果设置下可读；图形使用有意义的替代文本或被标记为纯装饰。
- 焦点样式、键盘导航、标题层级、跳到正文链接和深浅色对比继续通过现有质量基线验证。

## 7. 实施边界

方案确认并由用户明确解除当前实现限制后，预计只改动以下范围：

- `index.html`：重组门户语义结构和内容顺序。
- `_includes/site-header.html`：为门户提供顶部导航变体，内页结构保持不变。
- `assets/style.css`：增加以 `body[data-page-role="portal"]` 为作用域的门户视觉和响应式规则。
- 必要时新增一个门户专用、无外部依赖的轻量交互脚本或自有 SVG 图形；不引入大型框架或动画依赖。
- `README.md`、测试和内容审查记录：同步门户角色、设计规则与验收结果。

不改变现有正式 HTML 路由，不为旧首页保留别名或兼容样式，不把门户样式泄漏到规范、工具和手册页面。

## 8. 验收标准

1. 1440px 左右桌面视口下，页面呈现约 1200px 居中长画布，顶部导航、首屏双栏、论点双栏、入口行和三列网格的节奏与参考页明显同源。
2. 390px 左右移动视口下，阅读顺序正确，首屏和全部内容块单列化，无横向溢出，菜单与行动按钮触控区域合格。
3. 首屏第一屏能直接回答 Noemion 解决什么问题、当前处于什么阶段，并提供至少一个明确阅读入口。
4. 所有图形和内容都与 Noemion 的对象工具链直接相关。
5. 交互反馈集中在 150–300ms，进入动效克制且支持减少动态效果；不存在 `transition: all`。
6. 首页仍明确区分已确认原则、设计提案、开放问题和未来能力，没有虚构版本、下载、背书、客户或实验结果。
7. `python3 tests/site_quality_test.py`、Jekyll 构建后的质量测试以及桌面/移动浏览器实测全部通过。
