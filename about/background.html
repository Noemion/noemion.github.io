背景与边界

Noemion 研究怎样把人用自然语言表达的目标形成具有独立身份和明确责任边界的工程制品，供人工智能系统安全使用。

项目目的开发者问题当前策略术语读音待验证

先分清项目研究什么

传统编译器把程序员书写的形式代码转成机器能够执行或检查的对象。人工智能系统则经常从自然语言出发，并可以调用工具、修改文件或操作外部服务。

Noemion 研究的是目标形成，不是自然语言生成代码。形成制品保留原始表达、已确认意义、期望结果、接受条件和尚未解决的问题，让开发者能够检查从来源到目标结构的转换。

未来发布制品将按明确的裁剪关系移除原始自然语言与可逆重建内容，只保留已经确认的目标结构。含来源形成制品与未来裁剪发布制品具有不同精确身份；内容一旦改变就必须产生新身份，不能静默继承原对象的签名、证据、授权或接受状态。

发布 Profile、字段映射和正反向量尚未定义，当前也没有可执行编译器、稳定格式或生产兼容性。因此，这里的“编译”只表示一条正在形成规范与验证方法的研究路径。

用一次依赖升级定位缺失事实

假设用户要求 Agent“把支付服务依赖升级到经过批准且没有已知高危漏洞的版本”。执行过程中会出现多种成功信号，但每种信号只回答自己的问题。

系统给出的信号它可以证明什么还缺少什么事实
用户的自然语言请求保存了这次请求的来源表达。“经过批准”、漏洞资料截止点、允许修改的仓库和接受者仍可能不明确。
MCP 工具调用成功客户端与服务器完成了一次协议调用，工具返回了结果。调用是否具有业务授权，返回是否覆盖目标条件，以及结果是否应被接受。
A2A TASK_STATE_COMPLETED远端 Agent 报告该 A2A Task 已经成功结束。远端完成定义是否等同于本地目标的满足判据。
包管理器和 CI 成功依赖解析、构建或已选择的测试在对应环境中完成。候选版本是否由正确权威批准，漏洞资料是否满足截止要求，未运行的检查是否仍是缺口。
日志、追踪与扫描结果记录了具名操作、观察和遥测属性。观察覆盖是否足够，以及谁按哪项规则作出接受、拒绝或未知决定。

Noemion 不替代协议、构建、授权、遥测或决定系统。它固定目标内容及其仍未解决的缺口；外部系统产生的调用、观察和决定继续保留来源，并按各自规则进入受限会话。

为什么目标必须成为独立对象

独立对象的价值不在于比普通文本、提示词文件或 AI Skill 保存得更久。它的价值在于固定精确身份、版本化的解释规则和失败责任，使模型改写、协议转发、会话恢复或供应商切换不能静默改变目标。

开发者需要始终回答下面六个问题。直白职责是阅读入口；短词只是现行设计字段，正式发行前仍需完成人类读音、听写和相邻词区分验证。

相邻系统各自回答什么

相邻系统主要责任不能证明什么
模型栈与外部控制平面生成候选、规划步骤、选择模型和装配上下文。候选流畅或置信度高不能证明意义已经确认，也不能修改规范制品。
MCP、A2A 1.0与工作流系统传输工具、消息和远端任务，安排动作、重试、取消与终止。协议能力、调用成功和任务终态不能证明业务授权、目标满足或最终接受。
身份、政策、同意与凭据系统决定谁可以在什么条件下对哪个资源执行哪项动作。动作授权不能确认自然语言意义，也不能证明动作取得了预期结果。
受限运行器及其控制平面执行动作，限制能力、资源、副作用和会话生命周期。会话完成不能替代目标判断；取消也不能证明外部副作用已经回滚。
OpenTelemetry、扫描器、测试系统与 Iknem 生产者记录实际观察、方法、完整性、支持范围和限制。更多日志或测试不能自行选择满足判据，也不能扩大证据覆盖范围。
确定性规则与具名权威分别确认意义、评估证据、形成满足判断和作出最终决定。任何单一主体都不能仅凭自己参与了执行，就越过其授权范围自我验收。

NIST AI Agent Standards Initiative也把协议互操作、身份基础设施和安全评价作为不同支柱。这一趋势支持责任分层，但不构成对 Noemion 设计或实现的认可。

GNU make通过 goal、target 和 prerequisite 显式组织构建依赖，但目标文件更新不能证明业务意义、动作授权或最终接受。GNU 手册规范要求资料按读者问题组织；Noemion 因此先解释开发者要作出的判断，再提供字段、组件和 ADR。

当前开发者可以依赖什么

  • 当前策略：Endem 分开六项语义职责；Synem、Dromen 和 Iknem 分别处理组合闭包、一次会话与有范围证据。
  • 已有成果：CORE 草案、实验 Profile、ADR、威胁模型和规范向量已经定义资料层的规则与失败边界。
  • 正在研究：未来发布 Profile、现行专名的必要性、发行拼写、主要语言读音和口头区分，以及外部协议到本地责任的精确 Profile。
  • 限制条件：当前没有 Ktisor、Theor、Drasor、CLI、运行时或协议适配器；资料构建和向量通过不能证明组件安全或互操作。

当前策略已确定 Endem 所指的最小制品职责；现行拼写和读音仍须通过发行名称审查。职责稳定不等于名称已经适合发布。
