思想与方法基础

从现象学、语言哲学和逻辑中提取工程问题，同时解释为什么工程词汇最终必须脱离哲学术语独立成立。

PhenomenologySemanticsEngineering Limits

为什么阅读这些著作

Noemion 要把自然语言中的目标和约束编译为机器对象，因此必须先面对一个比对象格式更早的问题：一段表达的意义来自什么，什么使它指向同一对象、构成一个行为或满足一个条件。

哲学著作不会替我们给出 Endem 字段、规范编码或 Praxor 算法。它们的价值是揭示容易被工程直觉掩盖的分歧，并提供反例：表面语法可能遮蔽逻辑形式；同一句话可能执行不同的行为；语境推论不等于字面内容；规则不能只靠继续解释规则来固定；名称也不等于一组可变描述。

研究方法与采用边界

  1. 先还原问题：作者试图解释或反驳什么，而不是先寻找与 Noemion 相似的词。
  2. 再提炼区分：记录表达/所指、命题/行为、字面内容/语用推论、对象语言/元语言等关键差异。
  3. 形成工程问题：把哲学区分翻译为可被规范、类型、诊断或测试回答的问题。
  4. 保留反方限制：同时记录该理论不能覆盖什么，以及后续著作如何修正前一理论。
  5. 经过独立验证：只有形成 ADR、规范条款、反例语料和一致性测试后，思想启发才能成为工程设计。

Noemion 名称怎样形成

Noemion 的名称受到现象学中“意向相关项”讨论的启发，但它不是缩写。本项目采用“意向相关项”和“意向活动”作为工作性译法，用来区分意识活动与其相关项；相关概念存在竞争读法，因此该译法只是工程类比，不宣称解决哲学争议。概念背景：https://plato.stanford.edu/entries/husserl/

Noemion 借用这组关系来提出工程问题：来源句子可以变化，但目标身份、约束和验收条件需要独立表示。工程体系创造 Endem（end + -eme）表示最小期望终态单元，并从古希腊语词根再造短词；每个短词都必须同时给出直白工程副标题。

“视域”继续提醒我们：背景信息总是多于当前披露的信息。但工程上只使用 scope 表示一次运行可见的能力过滤视图，它不是新的文件类型。

“充实”继续提示目标与实际结果之间需要核对。工程上由 krin 预先固定比较规则和决定权威，由 Tekmor 封装可与 skena 对齐的结构化观察 phain，再形成 met、unmet、agno 或 fault 判断。

《逻辑哲学论》与工程设计相关的命题

本次按命题编号逐项核对路德维希·维特根斯坦著、贺绍甲译《逻辑哲学论》（商务印书馆，2009 年第 1 版）与开放编号校本。以下内容只说明哪些结构约束进入工程设计，不构成对该书的完整解释。开放编号文本：https://www.wittgensteinproject.org/w/index.php/Tractatus_Logico-Philosophicus_%28English%29

  • 1–2.063：世界由事实和事态而不是事物清单构成。Endem 因此要求 skena 表达对象之间成立或不成立的关系。
  • 2.1–3.1432：图像与命题由元素按确定方式结合，并通过投影关系指向可能事态。Endem 因此把来源 rhem、投影 semion 与中性事态 skena 分开。
  • 4.002–4.0031：日常语言会掩饰思想的形式，表面语法不保证真实结构。Poiet 不能把模型摘要或六段文本直接当作规范意义。
  • 4.022–4.024、4.431：理解命题关联到它为真时事情如何及其真值条件。Endem 由 skena 描述可能事态，telis 单独描述目标方向，krin 规定怎样把 phain 与 skena 比较；这些工程结构不等于哲学真值理论。
  • 2.172、4.12–4.1212：逻辑形式由结构显示而不是再被说出。Endem 不设置自填的 logical_form 或 valid 字段，Poiet 与独立 Theor 直接验证关系拓扑。
  • 4.112–4.116、6.5–7：划界要求把无意义、可表达但未决、意义已定但观察未知分成 aseme、apor 和 agno；格式不声称编码不可说之物。

反向约束：这本书的图像论、原子事实和早期语言观不能单独覆盖请求、承诺、授权、时间、概率、价值判断与开放式生成。ADR-0010 只采用可测试的结构约束；后期维特根斯坦、语用学、言语行为、翻译不确定性和推理语义学继续提供必须共同考虑的反例。

Endem 语义核与后续验证

ADR-0010 已采用原生语义核；物理编码、更多言语行为和求值语言仍待规范向量与反例验证：

已采用：六个语义面

rhem / semion / skena / telis / krin / apor 分别承担来源、意义投影、中性事态、目标方向、满足判据和未决边界。

已采用：结构显示形式

semion、skena 与 phain 共享符号和关系位置；规范载荷不能通过自填字段自证有效。

已采用：方向不改写事态

telis 只冻结 kine 与 mene；禁止事项由 skena 的显式否定表达，不再重复编码。

待实现：满足求值

krin 必须在显式变量绑定、Tekmor、版本、能力和 Praxor 配置中产生 met、unmet、agno 或 fault；不得读取未登记背景。

思想采用的验证要求

  • 每项哲学启发必须能被重写为不依赖哲学术语的工程问题。
  • 必须同时给出支持案例、反例、未覆盖场景和至少一个竞争设计。
  • 进入 Endem 规范前必须有 ADR，说明采用、修改或拒绝哪些部分。
  • 规范条款必须由机器可执行的一致性测试验证，不能用书籍权威替代证据。
  • 后续阅读如果推翻现有映射，应直接修订提案，不为保持叙事完整而保留错误对应。
