思想与方法基础

分析语言、意义、指称、规则、行为与推理承诺，并说明它们怎样约束 Noemion 的工程设计。

Analytic PhilosophySemanticsEngineering Limits

为什么阅读这些著作

Noemion 要把自然语言中的目标和约束编译为机器对象，因此必须先面对一个比对象格式更早的问题：一段表达的意义来自什么，什么使它指向同一对象、构成一个行为或满足一个条件。

哲学著作不会替我们给出 GSIR 字段、GOBJ 布局或 Runtime 算法。它们的价值是揭示容易被工程直觉掩盖的分歧，并提供反例：表面语法可能遮蔽逻辑形式；同一句话可能执行不同的行为；语境推论不等于字面内容；规则不能只靠继续解释规则来固定；名称也不等于一组可变描述。

阅读与采用方法

  1. 先还原问题：作者试图解释或反驳什么，而不是先寻找与 Noemion 相似的词。
  2. 再提炼区分：记录表达/所指、命题/行为、字面内容/语用推论、对象语言/元语言等关键差异。
  3. 形成工程问题：把哲学区分翻译为可被规范、类型、诊断或测试回答的问题。
  4. 保留反方限制：同时记录该理论不能覆盖什么，以及后续著作如何修正前一理论。
  5. 经过独立验证：只有形成 ADR、规范条款、反例语料和一致性测试后，思想启发才能成为工程设计。

Austin 与 Searle陈述/行为，命题内容/言外之力，成功条件/真假条件，权限/情境目标是否应区分询问、承诺、授权、请求和执行？行为对象是否记录行动者、权限、前置条件、效果和撤销边界？不把人类制度中的言语行为直接等同于 Runtime 指令。
Grice说出的内容/会话含义，共享目标/背景知识编译器能否标明哪些语义来自字面内容，哪些是可撤销的语境推论？推论需要什么来源和确认权限？不把“通常会这样理解”升级为硬约束。
Kripke名称/描述，指称固定/属性变化，必然性/先验性符号身份能否跨版本和重定位保持稳定，而显示名称、描述和属性可以独立演进？不把刚性指称直接变成对象 ID 规则，也不以稳定 ID 证明语义等价。
Brandom表征/推理角色，断言/承诺，蕴涵/不相容目标和约束是否应记录它们授权哪些推论、排斥哪些状态、由谁承担何种承诺？不把社会规范理论直接实现为约束图，也不把推理网络当作全部语义。

《逻辑哲学论》的第一批检查点

当前使用商务印书馆贺绍甲译本进行编号正文阅读。以下内容是设计检查点，不是对该书的完整解释：

  • 1–2：“世界是事实的总体”促使对象模型关注成立或不成立的关系状态，而不是只收集名称和资源。
  • 2.1–3：图像、事实和逻辑形式提醒我们，任何机器表征都必须明确说明它怎样对应任务状态；仅凭结构相似不能证明语义正确。
  • 4.024：理解一个命题与知道它为真时事情如何相关。这为 Artifact Contract 和 Evaluation Graph 提供问题来源，但生成式验收不必等同于命题真值。
  • 4.111–4.112：澄清思想和划定边界支持规范先行、术语分层和拒绝无意义输入的方向。
  • 6.5–7：可说与不可说的边界提醒规范必须能表示“尚不能判定或表达”，而不是让模型补出虚假确定性。

反向约束：这本书的图像论、原子事实和早期语言观不能单独成为 Noemion 的语义基础。后期维特根斯坦、语用学、翻译不确定性和推理语义学提供了必须共同考虑的反例。

对 GSIR 的设计提案

以下内容仍是研究提案，尚未成为规范：

四层语义身份

区分 Source Expression、Compiled Meaning、Reference Binding 与 Satisfaction Contract，避免把原文、编译结果、外部所指和验收规则混为一个字段。

语义来源等级

至少区分明示内容、结构蕴涵、语境推论、工具推导和模型候选；每一层具有不同的确认权限与可撤销性。

行为类型

目标不仅描述世界，也可能要求询问、承诺、授权或执行。行为类型需要行动者、权限、前置条件、效果和失败语义。

满足环境

约束是否满足取决于明确的变量绑定、证据、版本、能力和 Runtime Profile；验证器不得使用未登记的背景假设。

思想采用的验证要求

  • 每项哲学启发必须能被重写为不依赖哲学术语的工程问题。
  • 必须同时给出支持案例、反例、未覆盖场景和至少一个竞争设计。
  • 进入 GSIR/GOBJ 规范前必须有 ADR，说明采用、修改或拒绝哪些部分。
  • 规范条款必须由机器可执行的一致性测试验证，不能用书籍权威替代证据。
  • 后续阅读如果推翻现有映射，应直接修订提案，不为保持叙事完整而保留错误对应。
