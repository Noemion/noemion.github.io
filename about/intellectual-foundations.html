思想与方法基础

哲学帮助 Noemion 发现问题；工程规范、数据结构和测试负责给出可执行答案。

PhenomenologySemanticsEngineering Limits

为什么阅读这些著作

在设计对象格式之前，Noemion 必须先回答：一段表达怎样取得意义，怎样稳定地指向对象，又怎样说明某种事态已经满足。

哲学著作不会替我们给出 Endem 字段、规范编码或 Praxor 算法。它们帮助项目识别容易混淆的边界，例如表面语法与关系结构、字面内容与语境推论、规则说明与规则实践。工程设计随后把这些边界写成类型、不变量、诊断和测试。

研究方法与采用边界

  1. 先还原问题：作者试图解释或反驳什么，而不是先寻找与 Noemion 相似的词。
  2. 再提炼区分：记录表达/所指、命题/行为、字面内容/语用推论、对象语言/元语言等关键差异。
  3. 形成工程问题：把哲学区分翻译为可被规范、类型、诊断或测试回答的问题。
  4. 保留反方限制：同时记录该理论不能覆盖什么，以及后续著作如何修正前一理论。
  5. 经过独立验证：只有形成 ADR、规范条款、反例语料和一致性测试后，思想启发才能成为工程设计。

核心思想与工程问题

思想来源关键区分对 Noemion 的工程问题不得直接推出
胡塞尔现象学意向活动/意向相关项、对象同一性/被给予方式、意向/充实、主题/视域系统怎样区分来源表达、编译活动和被表达的对象？同一对象通过不同表达出现时，哪些关系必须保持，哪些背景可以稍后披露？不把软件编译等同于意识活动，不用哲学术语替代字段、算法或测试。
早期维特根斯坦事物/事实、记号/命题、命题/可能事态、说出/显示、意义/真值条件Endem 是否区分来源记号、意义投影、可能事态和满足观察？关系结构能否由两个实现独立验证，而不是靠载荷自称有效？不采用逻辑原子主义作为世界模型，不假定自然语言与任务世界简单同构，也不把开放生成式目标缩成二值命题逻辑。
弗雷格与罗素表达/意义/指称，表面语法/逻辑形式，名称/摹状词源码措辞、语义身份、符号绑定和外部对象是否分层？唯一性、存在性与作用域是否显式？不把每个目标压成唯一指称，也不把自然语言正规化误认为语义已经确定。
Carnap 与 Tarski外延/内涵，对象语言/元语言，真/满足，原子/复合递归Endem、规范配置、验证器和评价环境怎样分层？复合约束的满足关系能否由组成部分递归定义？不把开放生成式任务简化为封闭二值真值系统。
后期维特根斯坦与 Quine意义/使用，规则/实践，翻译/不确定性语义是否需要操作配置、使用场景和可观察行为？无法唯一翻译的内容怎样保留为歧义而不是静默确定？不以“意义依赖使用”为理由放弃形式化，也不宣称存在脱离实现和场景的万能语义对象。
Austin 与 Searle陈述/行为，命题内容/言外之力，成功条件/真假条件，权限/情境目标是否应区分询问、承诺、授权、请求和执行？行为对象是否记录行动者、权限、前置条件、效果和撤销边界？不把人类制度中的言语行为直接等同于 Praxor 指令。
Grice说出的内容/会话含义，共享目标/背景知识编译器能否标明哪些语义来自字面内容，哪些是可撤销的语境推论？推论需要什么来源和确认权限？不把“通常会这样理解”升级为硬约束。
Kripke名称/描述，指称固定/属性变化，必然性/先验性符号身份能否跨版本和重定位保持稳定，而显示名称、描述和属性可以独立演进？不把刚性指称直接变成对象 ID 规则，也不以稳定 ID 证明语义等价。
Brandom表征/推理角色，断言/承诺，蕴涵/不相容目标和约束是否应记录它们授权哪些推论、排斥哪些状态、由谁承担何种承诺？不把社会规范理论直接实现为约束图，也不把推理网络当作全部语义。

Noemion 名称怎样形成

Noemion 的名称受到现象学中“意向相关项”讨论的启发，但它不是缩写。本项目采用“意向相关项”和“意向活动”作为工作性译法，用来区分意识活动与其相关项；相关概念存在竞争读法，因此该译法只是工程类比，不宣称解决哲学争议。概念背景见 Stanford Encyclopedia of Philosophy 的 Husserl 词条。

Noemion 借用这组关系来提出工程问题：来源句子可以变化，但目标身份、约束和验收条件需要独立表示。工程体系创造 Endem（end + -eme）表示最小期望终态单元，并从古希腊语词根再造短词；每个短词都必须同时给出直白工程副标题。

“视域”继续提醒我们：背景信息总是多于当前披露的信息。但工程上只使用 scope 表示一次运行可见的能力过滤视图，它不是新的文件类型。

“充实”继续提示目标与实际结果之间需要核对。工程上由 krin 预先固定比较规则和决定权威，由 Tekmor 封装可与 skena 对齐的结构化观察 phain，再形成 met、unmet、agno 或 fault 判断。

《逻辑哲学论》与工程设计相关的命题

以下短引文已按命题编号核对路德维希·维特根斯坦著、贺绍甲译《逻辑哲学论》。它们只用来澄清 Noemion 面对的问题，不构成对全书的完整解释，也不直接决定软件规范。编号可对照 Wittgenstein Project 开放文本。

命题范围经核对的问题结构Noemion 的工程映射采用限制
2.01事态由对象的结合构成。skena 必须保存对象、关系、角色与组合，不能只是实体清单。不接受逻辑原子主义作为现实世界的完整本体。
2.1–2.15图像通过要素与对象的对应及要素之间的关系描述可能情况。semion 固定记号、符号和指称投影；skena 保存可比较关系结构。自然语言、软件状态和社会事实不必与单一图像简单同构。
2.17–2.172图示者与被图示者必须共享可对应形式，而图像不再把这种形式当作另一个对象陈述。逻辑形式由关系拓扑和角色位置显示；载荷不得用 logical_form、true 或 valid 自证。“显示”在这里转化为可由两条读取路径独立检查的结构，不是神秘属性。
3.203–3.21名称指向对象，记号配置对应对象在情况中的配置。rhem 保存提交后的来源记号，semion 记录稳定符号身份、指称来源和关系位置。稳定 ID 不自动证明现实指称正确，也不消除翻译和语境不确定性。
4.022命题显示其意义，并宣称事情就是它为真时所显示的样子。候选命题的意义结构、真假判断和谁有权确认该候选必须分别核对；授权边界由 AUT-CORE 定义。命题结构不能决定组织权威、同意、委托或多人冲突，这些制度关系必须另行规范。
4.024 与 4.063理解一项陈述需要知道它在何种情况下成立；作出真假判断前必须先规定判定条件。krin 必须在观察之前固定 phain 类型、比较关系、必需 Tekmor 和决定权威。开放生成目标不能被压成无语境的二值真值表；价值判断仍需具名外部权威。
4.062–4.064肯定与否定针对同一现实关系，否定不是另一个被指称对象。禁止事项由 skena 的显式否定关系表达；telis 只区分达到成立与保持成立。这不等于所有规范性禁止都能只靠命题否定解释；权限与制度条件仍须另行建模。
5.6 与 7语言能够表达的边界限制了可陈述内容。来源无法支持、指称无法授权或条件无法表达时保留 aseme 或 apor，不伪造确定性。Noemion 不声称编码“不可说之物”；它只记录明确的表达缺口、候选与责任边界。

“理解一个命题意味着知道若命题为真事情该是怎样的。”

命题 4.024 的工程启发：目标结构与判断条件必须分开。skena 说明哪种事态，telis 说明目标方向，krin 说明哪些实际观察足以支持判断；任何一层都不能替另一层自证。

工程上，Noemion 还必须区分四种结果：没有允许意义的 aseme、仍待授权选择的 apor、观察不足的 agno 和求值故障 fault。这些错误与状态由 ADR-0010 和规范定义，不是从原书直接推导出来的分类。

反向约束：这本书的图像论、原子事实和早期语言观不能单独覆盖请求、承诺、授权、时间、概率、价值判断与开放式生成。ADR-0010 只采用可测试的结构约束；后期维特根斯坦、语用学、言语行为、翻译不确定性和推理语义学继续提供必须共同考虑的反例。

Endem 语义核与后续验证

ADR-0010 已采用原生语义核，ADR-0011 至 ADR-0014 又依次采用实验容器、END-P1 载荷和首个来源清单。更多言语行为、量化、时间和求值语言仍待规范向量与反例验证：

已采用：六个语义面

rhem / semion / skena / telis / krin / apor 分别承担来源、意义投影、中性事态、目标方向、满足判据和未决边界。

已采用：结构显示形式

semion、skena 与 phain 共享符号和关系位置；规范载荷不能通过自填字段自证有效。

已采用：方向不改写事态

telis 只冻结 kine 与 mene；禁止事项由 skena 的显式否定表达，不再重复编码。

待完善：满足求值规范

krin 仍需补齐变量绑定、统计窗口、单位、容差、Tekmor 范围和决定状态转换；当前没有求值组件。

思想采用的验证要求

  • 每项哲学启发必须能被重写为不依赖哲学术语的工程问题。
  • 必须同时给出支持案例、反例、未覆盖场景和至少一个竞争设计。
  • 进入 Endem 规范前必须有 ADR，说明采用、修改或拒绝哪些部分。
  • 规范条款必须由机器可执行的一致性测试验证，不能用书籍权威替代证据。
  • 后续阅读如果推翻现有映射，应直接修订提案，不为保持叙事完整而保留错误对应。
