ADR-0033
文本与标识符标准命名

用完整职责名称消除文件格式误读，同时保持文本、标识符、显示与模型输入边界不变。

当前策略2026-07-13Direct replacementNo new format

直接结论

跨制品文本与标识符标准采用 TEXT-IDENTIFIER-CORE，条款和配套资料使用 TEXT-*。

这个名称明确说明规范同时约束文本槽和机器标识符。它不是 .txt 文件格式，不对应某份原始文本文件，也不创建文本处理器、词法分析器或 Unicode 对象。

迁移规则：TXT-CORE 容易被读成普通文本文件，已经直接退出机器登记、文件、向量、测试和公开路由。项目不保留旧路径、别名、重定向、双写或兼容垫片。

为什么必须改名

读法它会暗示什么实际职责
把旧 ID 读成 .txt规范只描述普通文本文件或保存原文规范约束所有已登记字符串槽、结构标识符、比较、范围、显示与模型输入
把它读成原始文件层解码后的 rhem.content 等于输入文件的原始字节原始字节、解码文本和变换结果必须分别绑定
把它读成文本组件项目已经拥有规范化器、编辑器或输入网关当前只有条款、威胁、场景、提案向量和资料一致性检查

名称已经让读者误判对象边界，就不应依赖注释反复补救。标准 ID 应先揭示职责，再由条款定义精确行为。

候选比较

候选核对结果决定
TXT-CORE与 .txt 的视觉和读音关联过强，已经产生“是否对应原始文件”的实际误解。退役
TEXT-COREInteredition Text Core 已用于范围标注文本模型。这个候选既发生精确工程重名，也会加强“通用文本文件核心”的误读。拒绝
TIB-CORESemEval-2025 LLMs4Subjects 使用 tib-core 数据集训练和评估技术文献主题标注，处于相邻的文本与人工智能发现空间。拒绝
LEX-CORElex 通常指词法分析、词典或词法语义，只覆盖当前职责的一部分，并会把显示、来源与模型输入错误收窄为 token 处理。拒绝
TEXT-IDENTIFIER-CORE直接列出两个治理对象；虽比缩写长，却不会伪装成新领域词、文件格式或组件品牌。采用

思想来源与采用限制

维特根斯坦在《逻辑哲学论》3.321 指出，同一个记号可以被不同符号共有。这里的工程启发只是：相同字母不能保证读者理解同一职责。它不能决定标准 ID、UTF-8、标识符语法或迁移策略；这些仍由命名证据、规范条款、登记和测试共同约束。
