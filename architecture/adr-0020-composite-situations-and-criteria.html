项目/架构设计/架构决策/ADR-0020

ADR-0020
复合事态
判据组合

让一个根事态可以表达彼此不可分的多个关系，同时固定四结果传播、决定性短路和证据覆盖，避免用布尔脚本掩盖未知与故障。

Accepted DesignComposite CriteriaNo Wire Schema ChangeNo Component Code

直接结论

第一阶段只允许用 all_of 和 any_of 组合判据。它们组合的是已经独立得到的叶级 met / unmet / agno / fault，不是原始数值、自由文本或可执行表达式。

只有共同描述同一个终态、共享同一目标方向和验收责任、不能被独立替换或接受的关系，才能进入一个 Endem 的复合根。能够独立实现、版本化、授权、接受或失败的目标必须拆成多个 Endem，再由 Synem 组合。

本决定冻结抽象语义，不给 END-P1 增加字段，也不表示 Poiet、Theor、Praxor、CLI 或求值器已经实现。

哲学启发与采用限制

“所有命题都是基本命题的真值运算结果。”

路德维希·维特根斯坦《逻辑哲学论》5.3

命题 5 及 5.3 提供了一个重要问题结构：复合陈述的形式必须能够从其组成部分和组合方式中显示出来。Noemion 因此拒绝把“同时满足这些条件”藏在备注、脚本或模型推理中。

这里不直接采用《逻辑哲学论》的二值真值函数。现实系统还会遇到观察不足和求值故障，所以 Noemion 必须保留四种判断结果；安全优先级、短路记录和 Endem 边界也都由工程规范决定。

先判断一个 Endem 还是多个 Endem

问题一个复合 Endem多个 Endem 与 Synem
终态身份所有叶共同描述同一个终态每一部分都有自己的终态
目标方向共享一个 kine 或 mene方向或时间责任可以不同
生命周期只能共同实现、替换和验收可以独立版本化、替换或停止
权威与失败责任共享同一最终决定边界可以由不同主体独立接受或失败

自然语言中出现“并且”或“或者”不能机械决定边界。例如“接口可达并且响应模式正确”可以是同一个服务终态；“发布报告完成并且服务已经部署”则是两个可独立失败的目标。

结构必须同时出现在事态与判据中

  1. skena 使用有限、无环的关系叶和组合节点显示一个根事态。
  2. 每个组合节点只能是 all_of 或 any_of，且至少包含两个不同子节点。
  3. krin 为每个关系叶给出一个叶判据，并以同样的组合拓扑组合叶结果。
  4. phain 仍与叶关系和角色位置对齐；原始观察先按叶判据判断，再组合结果。

子节点顺序不承载语义，重复引用和环都必须拒绝。未来物理 Profile 必须另行定义规范排序、引用和深度预算；当前 END-P1 不编码复合结构。

四结果组合矩阵

操作决定性结果没有决定性结果时
all_of任一叶为 unmet，总体为 unmet有 fault 则 fault；否则有 agno 则 agno；全部为 met 才是 met
any_of任一叶为 met，总体为 met有 fault 则 fault；否则有 agno 则 agno；全部为 unmet 才是 unmet

fault 只在没有决定性满足或反驳时优先于 agno。这个优先级不允许删除叶结果：一次故障即使没有控制总体结果，也必须留在求值记录中。

短路只能减少工作不能改写事实

all_of 发现有效 unmet，或 any_of 发现有效 met 后，可以停止求值其余叶。结果必须保存决定依据 decisive-basis，以及由已求值叶、所用 Tekmor、未求值叶清单和停止原因组成的 evaluation-coverage。

未求值叶不是 agno，因为它没有得到判断。不同求值顺序可以改变覆盖范围，但不能改变已经出现决定性结果后的总体分类；已遇到的故障也不能被短路记录抹掉。

为什么不开放任意表达式

  • not 不作为组合操作；否定仍由关系叶的显式极性表达，避免与 ADR-0017 重复。
  • if / iff 暂不支持。条件目标需要另行定义适用性、激活和“不适用”边界，不能用 agno 冒充。
  • xor / xone 暂不支持。它们需要额外的计数、故障与未知传播规则。
  • 不得嵌入 shell、CEL、Rego、SPARQL、任意算术或模型代码。ADR-0019 的已登记测量程序只在单个叶内产生结果。

外部技术提供的对照

  • W3C SHACL Recommendation定义 and、or、not 和 xone 约束，并把数据不符合与验证过程失败分开。Noemion 采用显式组合与失败分离，不采用 RDF Shape 或二值符合性作为制品语义。
  • SHACL 1.2 Core截至 2026 年仍是 Working Draft；其中 intent 与 agentInstruction 不得影响验证结果。这一发展方向支持把 AI 指令和形式判据分开，但不能作为稳定依赖。
  • GNU Coreutils test用不同退出状态区分假与错误，说明“条件不成立”和“检查失败”不应合并。
  • GNU Bash Lists展示 && 与 || 的决定性短路。Noemion 只借鉴减少不必要工作的机制，不采用 shell 退出码、求值副作用或左结合语义。
  • NIST AI 800-2 Initial Public Draft要求先说明评估目的与可直接观察的测量标准。组合判据因此只能连接已经明确的叶判断，不能用总分掩盖叶定义。

规范影响与验证

  • END-CORE 新增 END-CMP-001 至 END-CMP-004。
  • END-SCEN 新增四个案例；威胁模型新增复合判断洗白。
  • 十二个提案向量覆盖六个允许分类和六个确定拒绝；检查器只是规范一致性检查，不是求值器。
  • END-P1 不变；物理字段、规范字节和组件测试必须等待新 Profile 与用户明确开启代码阶段。

仍未冻结的内容

条件适用性、排他析取、复合否定、跨叶变量绑定、时间和量化组合、规范化、物理字段、规范排序、深度预算与规范字节仍需真实消费者、威胁分析和新 Profile。
