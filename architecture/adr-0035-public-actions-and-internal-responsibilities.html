ADR-0035
公开动作与内部职责收敛

只把具有独立用户任务、稳定输入输出和明确失败责任的职责保留为公开动作。

当前策略2026-07-14Five ActionsNo Compatibility Alias

直接结论

endem 的公开动作收敛为 ktise、elenk、pleko、theor 与 drase。

先前为裁剪与发布、签名响应核对、符合性测试设置的三个动作名不再属于当前词表、CLI 或未来发行承诺。对应工程责任没有消失：它们分别回到制品生产者、独立外部签名集成和一致性验证；这些责任不是公开动作。当前不保留旧入口、别名、重定向、双写或兼容垫片。

当前状态：五个公开动作只是职责边界，不是五个已实现子命令。第一条未来实现切片仍只考虑 ktise、elenk、独立 theor 与一致性验证；pleko 和 drase 要等待真实消费者。

判定方法

证据采用什么不能推出什么
GNU Binutils
GNU readelf		链接、转换、裁剪与只读检查可以有不同职责；独立读取路径能减少共同故障。GNU 有一个程序，就要求 Noemion 也有一个同构公开动作。
GNU objcopy
GNU strip		只有输入、输出、允许损失和失败条件明确时，变换才构成可辩护用户任务。尚无物理 Profile 与消费者的“裁剪”已经是稳定产品职责。
GNU Coding Standards：Names名称应传达有用含义，短词不能以晦涩换取整齐。为了形成对称词表而保留难读或无消费者的专名。
MCP Tasks
A2A Specification		任务、消息、产物与轮询状态是外部协议对象，适配时必须保留来源和状态域。远端 Task、Artifact 或完成状态自动成为 Endem 动作、Iknem 或满足结果。
OpenAI Agents SDK handoffs
tracing		handoff 与 trace 可用于运行编排、调试和监测。一次交接或 trace 成功就是符合性证据、授权或最终接受。

公开动作必须同时具有独立用户任务、可固定输入、可识别输出、唯一失败责任和不能由相邻职责承担的价值。维护者需要某项检查，或外部工具恰好有同名程序，都不足以通过这项审查。

五个公开动作

动作独立用户任务边界状态
ktise从受控来源确定性形成一个 Endem唯一规范写入入口；模型只提供候选未来首条切片
elenk按层验证实际制品字节检查通过不等于满足或接受未来首条切片
pleko把至少两个精确 Endem 解析为封闭 Synem不使用环境搜索或权限扩大等待真实组合消费者
theor以独立只读路径检查不可信字节不复用 Ktisor 的形成侧解析器，不写回未来首条切片
drase在最小能力域建立一次有界 Drase 会话不能改写制品、扩权或自判 accepted等待运行消费者和隔离证据

撤下公开动作后，责任落在哪里

历史职责现行归属重新获得接口的前置条件
封装、裁剪与发布派生物理封装由对应生产者承担：Endem 归 ktise，Synem 归 pleko。裁剪或迁移只作为 ID-CORE 的类型化派生关系；输出具有新身份。真实消费者、物理 Profile、允许损失、保留关系、正反向量和失败原子性全部锁定。
签名请求与响应核对独立外部签名集成；私钥、证书、撤销和时间保持在 Ktisor 与 Drasor 之外。待签陈述、包络 Profile、算法政策、验证截止点、撤销来源和真实发布消费者锁定。
差分、恶意输入、复现与互操作检查一致性与发行验证；规范注册表使用中性的 conformance: 引用，不映射为 CLI。若未来出现最终用户必须直接运行、且不能由测试或发行流程承担的任务，再重新提出 ADR。

撤下名称不合并信任域。Ktisor、独立 Theor、隔离 Drasor 与密钥权限域仍必须具有可测试的实现、进程或权限分离。

案例、反例与威胁

支持案例

用户需要形成、验证、组合、独立查看或有界运行一个精确制品。这五类任务各自有可说明的输入、输出和失败责任。

反例

维护者要在 CI 中跑恶意语料。它是不可省略的验证责任，但没有形成独立最终用户任务，因此不应增加公开命令。

待确认场景

未来出现必须离线签署同一类精确包络的真实发布者。项目应先锁定包络与消费者，再判断它属于现有发布系统还是需要新接口，不能预留名字占位。

主要威胁

最危险的退化是把外部任务完成、签名有效、trace 完整或测试通过提升为目标满足、授权或最终接受。结果域继续按 ADR-0015、ADR-0022、ADR-0027 与 ADR-0029 分开。

迁移与验证

历史动作名 tasse、sphra 与 peira 只在本 ADR 和名称审计中保留，用来解释一次性迁移；其他现行页面、规范、向量、测试引用、路线图和导航必须删除这些名称。没有旧路由，也不接受旧拼写。

高读音风险的历史签名动作被删除，是职责审查的结果，不是“改个更顺口的名字”。其余五个动作仍须通过 ADR-0034 的目标语言读音、听辨和写回验证；本决定不证明它们已经适合发行。
