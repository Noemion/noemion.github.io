项目/架构设计/架构决策/ADR-0035

ADR-0035
用户任务不是
内部职责清单

Noemion 只在开发者需要独立发起任务、理解结果并处理失败时暴露动作；测试、签名和发布派生不会因为重要就自动成为命令。

当前策略2026-07-16五项公开职责发行拼写待定

开发者现在会看到什么

Noemion 计划只提供一个顶层入口 endem，并在其下保留形成、生产检查、组合、独立检查和有界运行五项公开职责。

form、lint、compose、inspect 与 run 是五项职责的现行设计标识，不是已经发布的子命令。项目目前没有可安装的 endem，参数、选项、退出状态与稳定诊断接口同样尚未定义。

直接结论：五项职责说明未来用户需要完成什么，不表示系统必须有五个组件，也不表示五项职责具有相同实现优先级。现行拼写仍须在首次发行前通过真实人类验证。

一项责任怎样成为公开动作

开发者提出独立任务
固定输入与输出
指出唯一失败责任
证明存在真实消费者
检查信任与权限边界
再决定是否暴露接口
审查问题可以进入公开动作的条件不满足时怎样处理
用户任务开发者能说明要完成的工作，而不必先知道内部流水线只保留为实现步骤、库接口或发布流程责任
封闭契约输入、输出、允许的副作用和资源上限可以分别定义继续完善规范；不能先用名字占位
失败责任失败能够定位到该动作，并给出安全的停止与恢复方式归回拥有实际失败事实的相邻职责
真实消费者至少存在一个必须独立调用该任务的开发者流程消费者尚不存在时不承诺接口
安全边界暴露动作不会让写入、密钥、运行或决定权限被顺带扩大拆开权限域，或拒绝暴露

只有五项条件同时成立，一项责任才有资格成为公开动作。代码中存在函数、测试中需要某个步骤、外部协议提供同名操作，或者词表看起来更对称，都不是独立理由。

五项公开职责分别交付什么

直白职责现行设计标识交付结果与停止边界当前状态
形成目标制品form从受控来源和已确认意义形成来源保留的 Endem；意义、引用或根事态未闭合时停止未来首条实现切片
检查生产侧字节lint分别报告结构、Profile、内容与策略层；通过不等于目标满足、授权或最终接受未来首条实现切片
形成完整组合闭包compose把至少两个精确 Endem 解析为完整传递闭包；缺失引用、冲突、循环或权限扩大时停止等待真实消费者与物理 Profile
独立只读检查inspect从实际字节产生带范围的视图、差分和诊断；不复用形成侧解析器，不修复或写回未来首条实现切片
建立有界会话run重新核对目标、授权、能力与预算后建立一次会话；会话完成不等于目标满足或最终接受等待运行消费者与隔离证据

动作名称不等于实现优先级。compose 和 run 若长期没有独立消费者，就应继续停留在职责设计中，而不是因为已经列入五项职责就进入实现或发行。

重要责任为什么仍然不是命令

必要责任现行归属为什么不建立公开动作何时重新审查
从形成版派生发布版对应制品的确定性生产者当前没有发布 Profile、允许损失清单或可执行的裁剪契约；直接增加 strip 会把未定义变换伪装成产品能力来源闭包、保留/删除规则、新身份、伴随关系和正反向量全部确定后
签署陈述并核对响应独立外部签名集成与密钥权限域私钥、算法政策、证书、撤销和时间不属于目标制品生产或运行权限待签陈述、包络 Profile、验证截止点和真实发布消费者确定后
差分、恶意输入、复现与互操作检查一致性测试与发行验证它们是不可省略的质量责任，但通常由 CI、测试工具或发布流程调用；登记中的 conformance: 引用不是 CLI最终用户出现无法由现有测试或发布流程承担的独立任务时
授权、满足判断与最终接受具名政策、判断责任与决定权威把决定包装成普通动作会让调用成功被误读为有权执行、目标成立或已经批准只可为精确决定协议定义接口，不得并入制品动作

撤下接口不等于删除工程责任。相反，责任留在最接近实际事实和权限的位置，才能在失败时知道由谁停止、由谁修复，以及哪些结果不能继续向上推导。

一个入口不能抹平三条信任路径

责任边界可以拥有不能取得
确定性生产边界（deterministic producer）ktise / elenk / pleko 的形成侧逻辑与规范写入外部签名私钥、独立检查结论、运行能力或最终决定权
独立检查边界（independent inspector）inspect 的独立解析、只读视图、差分和诊断生产写入器、修复权限、密钥或运行权限
有界运行边界（bounded runner）run 的会话准入、能力控制、预算和观察责任改写精确制品、自行扩权、给证据自填有效或宣布最终接受
外部签名与决定域密钥、陈述、验证政策、截止点、撤销和具名决定借签名有效替代动作授权、目标满足或发布接受

endem 只是统一的人类入口。未来实现可以在入口后调用不同进程、权限域或独立程序；不能为了命令行整齐而链接共享解析器、共用密钥或把运行器变成写入器。

发行名称与停止条件

现行五个设计标识仍须通过 ADR-0034 的首次朗读、听写回填、职责匹配和词表内成对混淆审查。发行术语去专名化研究提案建议把它们同 form / lint / compose / inspect / run 比较；该提案不是 CLI、别名、重定向或迁移决定。

  • 没有真实消费者，或用户不会独立发起该任务时，停止把责任推进为公开动作。
  • 相邻职责能够承担同一输入、输出和失败责任时，合并用户入口，不为词表对称保留名称。
  • 名称持续造成职责误判、听写混淆或越权暗示时，重新命名；工程含义不得随拼写一起漂移。
  • 未来接受新拼写时一次性替换现行标识，并同步手册、导航、诊断操作名与测试；不保留旧入口、别名、重定向、双写或兼容垫片。

当前状态与继续阅读

已有成果：五项用户职责、三条信任路径、外部签名域、内部一致性责任和停止条件已经分开描述。

限制条件：当前没有可执行 endem、五项动作实现、稳定 CLI 契约或人类读音证据。资料检查只能证明页面与设计边界一致，不能证明组件安全、接口可用或发行名称已经确定。
