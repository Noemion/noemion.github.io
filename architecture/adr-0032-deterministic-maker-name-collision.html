ADR-0032
确定性制作名称冲突

用 Ktisor 与 ktise 消除推理引擎同名冲突，同时保持规范制品的唯一写入责任不变。

Accepted2026-07-13Direct replacementNo compatibility

直接结论

Ktisor 取代旧确定性制作组件名，公开动作改为 ktise。

Ktisor 仍是 Endem 与 Synem 规范字节的唯一写入边界。它确定投影、规范顺序、引用闭包、物理布局和内容身份；模型、Theor、Drasor 与签名权限域都不能绕过它写入或改写规范制品。

迁移规则：项目尚未进入组件代码阶段，也不考虑向后兼容。旧组件页、图像名、动作、路径和正文直接退出；不保留别名、重定向、双写或兼容垫片。

遗漏了什么

ADR-0031 检查了组件名，却没有把大小写无关的公开动作作为独立发布面复核。这使一个精确冲突漏过门禁。

退役名称可核对的外部使用冲突判断
旧公开动作PFA Open Inference Engine 使用相同字母组合，公开说明其参考实现用于执行预测与分析模型的评分计算。大小写不能形成可靠区分；双方都位于人工智能工程与模型处理的发现空间，已触发名称审计的停止条件。
旧组件名组件名与旧动作来自同一词族，继续保留会让文档、搜索和未来安装界面维持错误关联。组件与动作必须共同退出，不能只改命令拼写。

这项判断只说明工程发现与语义相邻性，不能判断商标权、在先权利或法律可用性。

动作名称不等于实现优先级

五个公开动作目前只是应用职责，不是五个已经证明必要的组件。保留名称不能替代真实消费者、失败责任和验证证据。

成熟度动作当前边界
首条切片有直接消费者ktise、elenk、theor分别承担确定性形成、生产校验和独立只读解释；仍没有实现授权。
依赖后续消费者pleko、drase只有在组合和受限会话出现真实消费者后才进入实现范围；发布与签名责任按 ADR-0035 外置。
内部质量责任内部符合性门禁一致性与互操作验证不可省略，但 ADR-0035 已决定不扩大公开 CLI。

本 ADR 只接受 Ktisor 与 ktise 的名称替换，不自动冻结五个动作的最终产品拓扑。

不改变的边界

本次更名不修改 END-CORE、END-P1、END-FMT、SYN-CORE、内容身份、结果域、字节布局、唯一 endem CLI、Theor 独立读取路径、Drasor 最小能力域或签名权限隔离。旧词只保留在本 ADR 与名称审计中作为冲突证据。

读音状态

Ktisor 与 ktise 通过的是工程冲突初筛，不是口头传播门禁。词首 kt 的实际朗读与听写仍需按 ADR-0034 验证；若持续需要插入不一致的元音、丢失辅音或额外解释，必须重新审查拼写。
