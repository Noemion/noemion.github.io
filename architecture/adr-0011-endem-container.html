ADR-0011
Endem 实验性容器格式

采用固定前导、显式记录目录和确定性载荷子集，为 Ktisor 与独立 Theor 建立同一组可对抗的规范字节。

当前草案END-FMT 0.1ABI 未稳定

决定

第一阶段采用 END-FMT 0.1.0-draft：64 字节固定前导、48 字节定宽目录项、六个关键语义记录，以及 RFC 8949 确定性 CBOR 的严格子集。配套 END-P0 Profile 给出有限资源上限，但当前字节不承诺向后兼容。

格式只解决安全定位、确定性编码和独立解释。结构合法不等于语义合法，更不等于目标为真、可执行或已经完成。

为什么现在必须作出决定

当前问题没有格式草案的后果这项决定提供什么
Ktisor 与 Theor 需要解释同一原始字节两条路径只能比较内部 JSON，无法发现偏移、长度、对齐和顺序错误。共同的公开字节向量，但不共享解析代码。
所有输入都不可信资源上限和 checked arithmetic 只能停留在口号。固定乘加、范围、重叠、对齐和 Profile 检查点。
确定性必须可观察相同语义仍可能因字段顺序、时间戳或编码选择产生不同字节。固定目录顺序、最短编码、确定长度和零填充。
实现语言实验需要同一目标候选语言会各自发明输入结构，比较结果没有意义。不同语言实验可以使用同一组接受和拒绝向量。

固定前导保存格式版本、Profile、状态、目录位置、记录数和文件总大小。目录只定位记录并说明其种类、范围和对齐；六个记录的内部映射由 END-CORE 和后续 Profile 约束，首个封闭映射现由 ADR-0013 的 END-P1 定义。

有意保持简单：END-P0 不支持压缩、加密、未知记录、供应商扩展、coherent、attested 或签名包络。增加任一能力前必须先有新的 ADR、Profile、威胁分析和字节向量。

从 ELF 与 GNU 工具链借鉴什么

权威机制采用部分明确排除
ELF 固定头部与节表固定位置的路线图、显式目录和面向不同消费者的解释边界。机器地址、指令、装载段、处理器 ABI 和传统链接规则。
ELF 条目大小、数量与偏移每个结构明确宽度和范围；Endem 额外要求所有组合运算受检。扩展编号技巧、宿主结构体映射和处理器相关字节序。
GNU readelf 独立于 BFDTheor 与生产路径共享规范和向量，不共享解析器或通用内部表示。把 Theor 做成 objdump/readelf 的功能复刻。
GNU ar 确定性模式消除所有者、组、时间戳和文件模式等环境差异。把传统归档成员、路径或符号索引直接搬进 Endem。

为什么记录载荷使用受限 CBOR

重新发明整数、字符串、数组和映射编码会扩大可信解析面，也会重复解决最短整数、长度和键顺序问题。RFC 8949 已定义确定性编码基础，因此 END-P0 只采用一个更小的子集：确定长度、最短表示、无符号整数键、编码字节排序、无浮点、无标签、无重复键。

CBOR 不是 Endem 的对象模型。通用 CBOR 数据不能自动成为 Endem；记录种类、字段含义、关系位置、授权、状态和验收全部由 Noemion 规范定义。编码依据见 RFC 8949。

备选方案

方案没有采用的原因重新考虑条件
把语义向量 JSON 直接作为 .endem文本表示、数字、键顺序和解析器行为难以形成小型稳定 ABI，也容易退化成二进制化 Prompt 的反面。不重新考虑；JSON 继续用于人类可读测试外壳。
完整采用 ELF地址、段、节、符号和重定位语义围绕机器程序，错误地扩大 Endem 的机器对象负担。不重新考虑；只持续借鉴经过边界审查的机制。
只使用通用 CBOR 文件没有固定路线图、记录范围和六语义面目录，无法在解释复杂载荷前完成有界结构检查。若未来证明固定目录没有真实消费者或不能带来独立验证价值。
自创完整 TLV 类型系统会同时发明容器和数据模型，增加解析器、规范和互操作风险。只有受限 CBOR 无法满足已验证的零拷贝、流式或安全要求时。
Protocol Buffers、FlatBuffers 或大型 IR 框架引入生成代码或复杂运行时，会削弱 Ktisor/Theor 的故障独立性，并超出第一阶段小型可信核心。后续非可信适配层存在明确消费者且不进入规范字节时。

Profile 与资源上限

END-P0 把文件限制为 16 MiB、记录数限制为 64、图节点限制为 65,536、图边限制为 262,144、嵌套深度限制为 64、单字符串限制为 1 MiB、解压后总量限制为 16 MiB、累计分配限制为 64 MiB。

这些数值是第一轮跨实现实验的保守边界，不是生产规模证明。调用者可以进一步收紧；任何提高都必须使用新的 Profile 身份，不能用环境变量静默改变 END-P0。

后果与停止条件

  • Ktisor 和 Theor 可以开始针对同一字节边界做语言、模糊测试与差分实验。
  • 现有语义 JSON 向量继续验证 END-CORE；新增十六进制向量只验证结构和 Profile，不混淆两类证据。
  • 格式草案增加了规范维护成本；条款、登记、错误目录、Profile 和向量必须同步。
  • 如果两个独立解析实验不能在不共享代码的情况下稳定解释这些向量，必须修改或撤回草案，不以实现补丁掩盖规范缺陷。
  • 如果固定目录、六记录分离或受限 CBOR 没有下游消费者，或显著阻碍流式安全读取，应以新 ADR 合并或删除相应机制。

权威来源

精确字段、条款和接受语义以 END-FMT 规范源、END-P0 Profile、DIA-CAT 诊断目录和规范字节向量为准。
