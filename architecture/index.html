架构设计

用一次依赖升级定位目标、形成、运行、证据和决定的责任，并在信息不足时停止。

一次任务主线三项信任职责成功仍需核对当前尚无组件

从一次依赖升级定位架构

“把服务依赖更新到安全版本，保持测试通过，经批准后发布。”这句话包含终态、允许的变化、验收判据和决定权。任何一层都不能借上一层的成功扩大责任。

来源与意义确认保存实际来源，由有权主体确认精确解释。版本、兼容要求、允许动作或验收判据仍含糊；模型补全不能代替确认。
deterministic producer 形成按已确认且具有语义授权绑定的投影确定性形成 Endem，并执行形成侧检查。输入未封闭、Profile 不明确、字段不合规或输出不确定。
independent inspector 独立读取用不共享解析实现的只读路径直接读取同一 Endem 字节。两条读取路径分歧、资源不足或输入格式尚未定义。
bounded runner 建立会话核对精确内容、外部陈述、政策、能力、预算和截止点，再建立一次 session contract。授权、会话上限和实际能力没有交集，或外部陈述绑定内容身份失败。
外部执行把受限本地调用映射到精确协议操作，保存远端状态、产物、错误和损失。协议、对端、幂等条件或副作用不明确；远端建议不能扩大权限。
证据与满足evidence entry 保存有限主张和观察；验证者判断有效性，证据形成覆盖度，satisfaction_criteria 形成满足结果。观察缺少来源、主张越界、覆盖不足或判据要求的负观察缺失。
最终决定具名权威按政策决定接受、发布、继续或撤回。主体、作用域、委托、同意或撤销状态不明确。

来源保留的形成版分开来源表达、已确认意义投影、一个根可能事态、目标方向、满足判据和待确认意义，对应 source_expression / meaning_projection / situation / goal_direction / satisfaction_criteria / unresolved_meaning。意义确认不授予工具或运行权限。最终发布版移除原文，并由独立 Profile 重写来源绑定；两种版本具有不同精确身份，签名、验证或撤销不会原地改写内容身份。

看到成功后继续检查

协议终态、工具退出、测试通过、签名有效和模型结论回答不同问题，都不是“目标已经满足”的缩写。

看到的信号它最多证明下一项检查
A2A TASK_STATE_COMPLETED 或 MCP completed相应协议任务到达成功终态。核对精确调用、返回产物、错误层和副作用；协议终态不能直接成为 met。
HTTP 2xx、工具成功、退出码为零或测试通过调用或测试按自身成功规则结束。核对预期文件、依赖锁、测试身份、遗漏分支和未授权变化。
签名或外部陈述验证有效具名验证政策接受了精确主体上的外部陈述。核对内容摘要、作用域、时间、撤销和使用授权；有效签名不产生行动权限。
evidence entry 已记录证据一项有限主张与观察、主体、来源和限制建立了关系。证据集合相对精确 satisfaction_criteria 形成覆盖度，satisfaction_criteria 再形成满足判断，具名权威才形成最终决定。
模型或 Agent 说“已经完成”模型候选提供了一项待核对陈述。继续把它视为不可信输入；保存来源并用上面对应责任独立核对。

截至 2026-07-16，A2A 1.0分别建模 Task、Message、Artifact 和状态；MCP 2025-11-25 Tasks仍标为实验机制。NIST也把标准、协议、身份和安全评价分开推进。它们不定义 Noemion 的目标、证据或决定。

三个信任域为什么不能合并

一个发行包可以提供三类入口，但形成、读取和运行不能共享足以掩盖同一错误的代码、权限或失败处理。

deterministic producer · 确定性形成只根据封闭输入形成和检查制品；不解释来源，不持有模型、私钥或实时能力。 independent inspector · 独立只读观察直接读取当前已有实验性物理格式的 Endem；不写回输入，也不生成生产可信句柄。 bounded runner · 隔离运行建立一次会话、约束能力并保存观察；不改写目标，也不自行形成满足结果或最终决定。

形成侧检查和 inspect 独立直接读取不共享解析实现；否则同一缺陷可能同时写出并“验证”错误内容。bounded runner 必须重新验证实际字节和外部陈述，不能把生产侧状态当作运行授权。

当前策略：计划中的 endem 入口承载五项公开职责，并把请求路由到不同信任边界；五个现行标识不是已经发布的子命令。当前还没有可执行 CLI，也没有 deterministic producer、independent inspector 或 bounded runner 组件。

对象只在需要的阶段出现

对象生命周期与当前物理状态不能据此声称
Endem持久目标制品；END-P2 是来源保留的形成与评审 Profile，END-FMT 0.1.0-draft 是实验性物理容器，裁剪发布 Profile 尚未定义。形成侧检查通过不等于已经授权执行、目标满足或发布获准。
Endem closure持久组合闭包；CLOSURE-CORE 已规定完整成员、精确绑定和权限收窄，物理容器仍待确定。动态工具清单、构建依赖或一次会话选择不能改写既有闭包身份。
session contract一次会话的只读执行契约；它在准入时建立，失效或结束后销毁，永不作为持久格式序列化或恢复。检查点、记忆、Agent 会话或凭据包不能恢复旧权限。
evidence entry按 EVIDENCE-CORE 定义的有范围证据记录；物理编码、采集和归并接口仍待确定。证据存在、数量增加或外部评估有效都不自动产生覆盖、满足或接受。

当前可以证明什么

已有成果

CORE、ADR、登记、威胁模型、场景和向量已经连接对象、结果域与失败责任。资料检查只能证明已覆盖关系保持一致。

限制条件

当前没有编译器、CLI、解析器、沙箱、政策引擎、适配器或运行时；页面不能作为实现、互操作或隔离证据。

待定内容

最终裁剪发布 Profile、Endem closure 与 evidence entry 物理编码、session contract 运行 API、协议 Profile、摘要与签名算法、稳定 ABI 和迁移条件仍待确定。

名称与读音

Endem、Endem closure、session contract、evidence entry、deterministic producer、independent inspector 和 bounded runner 当前只是精确设计标识。首次朗读、口头区分和主要使用语言中的流畅度尚无人类证据，不能形成发行结论。

GNU make把 goal 定义为需要更新的 target，并沿前置条件处理其他 target；这支持显式依赖，却不把构建成功等同于业务满足。GNU readelf独立于 BFD，以免共同库缺陷同时影响生产与检查；这支持第二条读取路径，但不证明 independent inspector 已经实现。

按问题继续

一项目标怎样经过完整生命周期继续查看来源、形成、组合、会话、证据和决定在什么条件下前进或停止。 一条责任为什么这样划分按开发者问题查看现行 ADR、替代方案、反例和变更责任。 Agent、工具和协议状态放在哪里把上下文、计划、批准、外部调用、记忆、观察和决定放回各自责任域。 哪些边界仍需研究按格式、组合发布、Agent 运行、术语和实现问题进入相应资料。

精确字段、条款、Profile 和验证关系从规范参考指南继续查询。
