架构的直接结论

Noemion 不是一组仿照传统二进制工具命名的命令。它要建立的基础是：一项自然语言目标可以成为持久、可组合、可受控运行、可由证据验收的制品。

一件核心制品

Endem 保存来源表达、已确认意义投影、一个根可能事态、目标方向、满足判据和待确认意义；对应现行字段为 rhem / semion / skena / telis / krin / apor。意义确认不授予工具或运行权限。

一个命令入口

endem 提供五个固定子命令，避免每项内部操作变成独立产品。

两条读取路径

制品形成侧检查和 theor 独立直接读取，不共享会掩盖共同错误的解析实现。

一个隔离运行域

drase 在 Ktisor 外建立 Dromen，能力句柄和模型后端不能进入制品或确定性生产域。

ktise 是唯一语义成形入口；elenk 是制品形成侧检查路径；theor 是不共享解析代码的只读观察路径；pleko 只在存在多个 Endem 或外部引用时启用。drase 重新验证实际内容、适用外部陈述、当前政策和截止点，再在隔离域建立会话。签名、验证或撤销不会原地改写内容身份。

三个组件

Ktisor · 确定性制品制作域负责 ktise、生产 elenk 与 pleko；不持有私钥，也不承担外部签名。 Theor · 独立只读检查器为 theor 提供第二条有界解析与比较路径，不生成生产可信句柄。 Drasor · 隔离实现运行器重新验证精确制品与适用外部陈述，建立 Dromen，控制能力并形成 Iknem。

三者可以由同一发行包提供，但代码依赖、权限和失败责任必须保持可验证分离。没有模型、外部能力或多步反馈的任务可以使用最小 Drasor；求解后端仍不得持有实时句柄或自行验收。

制品与非制品

名称类别责任
Endem持久制品一个根可能事态和六个语义面。现行草案仍使用 nascent / coherent / attested，但外部签名、验证和撤销不能压成内容自身的单值状态。
Synem持久组合闭包按 SYN-CORE 固定完整成员、精确绑定、依赖、权限交集与激活关系；物理容器尚未定义。
Dromen一次会话的只读执行契约绑定精确制品、政策、环境、能力、预算和证据责任；不序列化或恢复权限。
Iknem有范围证据记录按 IKN-CORE 绑定主体、溯源、phain、主张和限制；有效性、覆盖与决定保持外置。

信任边界

  • 自然语言来源、模型候选、对象字节、依赖、签名包、工具返回和环境观察都从不可信状态进入。
  • Ktisor 只能证明确定性成形、结构、语义和状态不变量；它不能证明任务已经完成。
  • Theor 只提供独立观察和差分证据；可显示不等于制品形成侧检查通过。
  • Drasor 只在当前环境和策略内绑定能力；精确制品和外部陈述都不保存实时句柄，也不自动获得权限。
  • Iknem 只承担有限主张；验证政策形成有效性，krin 形成覆盖与满足判断，指定权威才形成最终决定。
  • AUT-CORE 只回答谁能在什么范围内授权；登录、签名、模型、Agent 状态和界面点击都不能越过具名政策取得决定权。
