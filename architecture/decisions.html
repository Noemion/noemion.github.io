当前权威决定

ADR-0010至 ADR-0020固定 Endem 语义、格式与判断边界；ADR-0021 至 ADR-0030固定组合、证据、内容分层、会话、诊断、适配、身份、文本、授权与伴随边界；ADR-0031至 ADR-0035处理发布名称、读音与公开动作审查。

ADR-0008 与 ADR-0009 只保存公开迁移历史。更早、无入口且与现行决策源重复的内部 ADR 副本已经删除，不能反向约束实现、路由或 ABI。

决策状态

ADR状态当前解释
ADR-0008 Endem 制品与单一命令面Superseded by ADR-0010保留单一 CLI、权限分离与安全机制；制品、组件、动作和状态名称已取代。
ADR-0009 Endem 命题投影语义核Superseded by ADR-0010保留结构显示与未确定分层；五组字段和目标力量模型已取代。
ADR-0010 原生词汇与事态分层当前策略当前六个语义面、一个根 skena、phain 比较、原生制品/组件/动作与生命周期。
ADR-0011 Endem 实验性容器格式当前草案采用固定前导、定宽目录、六记录、受限确定性 CBOR 与 END-P0；尚非稳定 ABI。
ADR-0012 未来安全核心的 Rust 基线当前策略Ktisor 与生产读取核心以 Rust 1.97.0 和安全子集作为实现前重新评审起点；当前没有实现工作区。
ADR-0013 END-P1 封闭内容 Profile当前草案END-P0 保留为结构实验；END-P1 作为首个设计 Profile 固定六记录字段、枚举、规范排序和引用闭包。
ADR-0014 首个 Ktisor 来源清单当前草案用有限来源清单连接自然语言来源与授权投影；模型不能直接生成规范对象，正式来源语言出现后删除该入口。
ADR-0015 判断与运行结果分层当前策略分开制品生命周期、满足判断、权威决定、会话终止、Iknem 有效性和证据覆盖度；不改变 END-P1 字节。
ADR-0016 mene 时间与连续性模型当前策略采用 fixed/elapsed 时间范围、strict/budgeted 连续性与显式覆盖缺口；END-P1 继续只支持 kine。
ADR-0017 否定事态与缺席证据当前策略否定保留同一关系与角色；空结果默认是 agno，只有封闭观察范围内的完整缺席可以支持负目标。
ADR-0018 量化范围与集合成员资格当前策略先固定成员权威、截止点与身份规则，再按不同成员聚合；不允许有利子集、重复见证或默认空真值。
ADR-0019 测量谓词与阈值契约当前策略分开构念、固定或泛化总体、测量程序和阈值比较；不允许用代理分数、单位、舍入或隐藏不确定度洗白结果。
ADR-0020 复合事态与判据组合当前策略只允许一个不可分终态使用 all_of/any_of；固定四结果传播、决定性短路和求值覆盖，不开放任意表达式。
ADR-0021 Synem 组合闭包与条件激活当前策略固定完整传递闭包、精确绑定、有限无环、权限交集、成员结果分离和会话激活四状态；不创建物理格式。
ADR-0022 Iknem 证据范围与评估边界当前策略固定主体范围、溯源、phain 对齐、证据类别、完整性、有效性、覆盖、决定分离与最小披露；不创建物理格式。
ADR-0023 Endem 内容标准分层当前策略由 END-CORE 定义通用内容，END-P1 定义当前封闭内容 Profile，END-FMT 定义物理容器；三层分别验证。
ADR-0024 Dromen 会话契约当前策略由 DRO-CORE 固定一次会话的精确主体、政策、环境、能力、预算、观察责任、只读失效与销毁；永远不创建 Dromen 文件。
ADR-0025 结构化诊断边界当前策略由 DIA-CORE 固定跨对象诊断身份、语境、层次、位置、主错误、恢复和安全边界；诊断不成为结果、权限或自动修复命令。
ADR-0026 外部协议适配边界当前草案由 ADP-CORE 固定协议版本、对端、能力、调用、映射、状态、产物、错误、取消、重试、交付和安全边界；具体协议 Profile 仍待确定。
ADR-0027 精确内容身份与签名边界当前草案由 ID-CORE 固定跨制品精确身份、不可变引用、签名陈述、验证材料、撤销、可复现性与派生关系；算法和物理 Profile 仍待确定。
ADR-0028 文本与标识符边界当前草案由 TEXT-IDENTIFIER-CORE 固定文本槽、严格 UTF-8、来源溯源、ASCII 标识符、规范化、比较、范围、双向显示、隐藏字符、模型输入与显示边界；不增加物理字段。
ADR-0029 权威与授权决定边界当前草案由 AUT-CORE 固定权威语境、主体资格、范围、自然语言授权、委托、同意、多人决定、撤销、重放、能力交集和结果分离；不创建新制品或政策引擎。
ADR-0030 Endem 内容与授权伴随关系当前草案修正内容身份与授权决定混合问题；P1 权威字段只作选择器，完整内容接受需要外部前置条件，授权变化不改写内容身份。
ADR-0031 发布名称冲突审查当前策略采用 Iknem 与 Drasor/drase，保持职责和不变量不变；当前阶段直接替换，不保留旧入口、别名或重定向。
ADR-0032 确定性制作名称冲突当前策略采用 Ktisor/ktise，修复对公开动作大小写无关筛查的遗漏；同时记录五个动作并不具有相同实现状态。
ADR-0033 文本与标识符标准命名当前策略采用 TEXT-IDENTIFIER-CORE，消除把跨制品文本边界误读为 .txt 文件或原始文件规范的歧义。
ADR-0034 术语读音与口头区分审查当前审查把目标语言读音和词表内听辨纳入发行审查；现行职责保持有效，具体读音与发行拼写尚待确定。
ADR-0035 公开动作与内部职责收敛当前策略公开动作收敛为五个；派生封装、外部签名和符合性验证保留责任边界，但不保留公开动作名。

当前策略

决定采用的边界直接影响
六个语义面一个 Endem 只有一个根 skena，并保存 rhem、semion、skena、telis、krin 与 apor。来源格式、投影规则、Ktisor、规范向量和错误语义。
事态与方向分离skena 保持中性；telis 只使用 kine/mene；禁止由事态否定表达。规范化、等价判断、时间字段与反例向量。
形式显示semion、skena 与 phain 共享符号和关系位置，不使用自证逻辑形式字段。结构编码、Ktisor/Theor 差分和 aseme 诊断。
同一制品状态Endem 沿 nascent → coherent → attested 演进，不建立平行对象家族。身份、摘要、发布、签名和生命周期。
组合闭包Synem 只在多个 Endem 需要解析关系时出现；权限只能收紧。pleko、依赖锁定、冲突和披露。
一次运行的只读契约Dromen 按 DRO-CORE 只绑定一个 Drase 会话；实时秘密和句柄外置，实质漂移使旧契约失效，记录不能恢复权限。Drasor、最小能力域、预算、观察责任、失效和销毁。
有限证据主张Iknem 按 IKN-CORE 保存精确主体、范围、溯源、phain、主张和限制；有效性、覆盖度与决定外置。来源、绑定、签名、运行、验收、撤销、隐私与审计。
失败可被稳定处理DIA-CORE 让机器码与消息分开，固定生产语境并确定性选择主错误；外部错误和恢复建议不能越权改写本地结果或权限。Ktisor、Theor、Drasor、协议适配器、自动化与人工审查。
文本职责保持分层TEXT-IDENTIFIER-CORE 分开来源字节、解码文本、机器标识符、人类显示与模型实际输入；当前结构标识符保持 ASCII 且不做隐式规范化。Ktisor 来源、诊断位置、身份比较、安全审查、模型候选和未来界面。
权威与授权保持有界AUT-CORE 分开认证、角色资格、语义授权、能力授予与最终决定；grant/deny/defer 只回答精确请求是否获准。自然语言候选复核、Dromen 建立、Agent 委托、同意、多人决定、撤销与重放。
唯一 CLIendem 只提供 ktise/elenk/pleko/theor/drase 五个动作。应用项目、手册、安装包和自动化接口。
名称发行审查职责、书写检索、目标语言读音和词表内口头区分分别审查；读音提示不是机器别名。全部领域词、组件、动作、手册、未来语音交互和首次正式发行。
独立与隔离theor 不共享生产解析；drase 不共享编译器权限域。仓库依赖方向、测试和安全边界。
实验性容器END-FMT 0.1 采用 64 字节前导、48 字节目录项、六种关键记录和 END-P0 有限上限。规范字节、错误分类、Ktisor/Theor 语言实验；不构成稳定 ABI。
首版核心语言实现启动前计划评估 Rust 1.97.0；禁止 unsafe 与首条切片零第三方 crate 仍是未来实现约束。未来实现的构建锁、受检算术、模糊测试和供应链边界。
首个设计 ProfileEND-P1 使用 profile 2，封闭六记录字段并只支持 nascent 与 kine。未来 Ktisor/Theor 字段解析、交叉引用和当前载荷向量；不构成稳定 ABI。
首个来源输入END-SRCM 使用 UTF-8 逐行来源清单，显式分开来源、授权投影和 apor。Ktisor 输入解析与失败定位；不是稳定源语言或对象身份。
判断与运行结果分层满足只使用 met/unmet/agno/fault；决定只使用 accepted/rejected/deferred；会话只使用 completed/failed/interrupted。Drasor、Iknem、协议适配、人工验收和未来组合向量；不新增对象或 wire 字段。
mene 时间与连续性时间范围只使用 fixed 确定区间或 elapsed 具名事件时长；连续性只使用 strict 或完整 budgeted。Ktisor、Drasor、Iknem 和未来 mene Profile；遥测缺口不能成为允许违约。
否定与缺席证据negative 作用于同一关系和角色；没有记录不等于负观察，缺席推断必须绑定封闭观察范围。Ktisor、Drasor、Iknem、END-P1 负极性向量与未来完整性声明；空日志不能成为 met。
量化范围与成员资格只支持单个被量化角色；集合、权威、截止点、身份和空集政策必须显式，基数只按不同成员计数。Ktisor、Drasor、Iknem 与未来量化 Profile；当前不改变 END-P1，也不实现求值器。
测量谓词与阈值构念、总体、数值、单位、程序、聚合器、样本、不确定度和比较器必须共同固定。Ktisor、Drasor、Iknem、AI 评估与未来测量 Profile；当前不改变 END-P1，也不实现统计引擎。
复合事态与判据一个不可分终态可用有限无环的 all_of/any_of；skena 与 krin 叶对齐，四结果确定传播，决定性短路保存覆盖。Ktisor、Drasor、Iknem、Synem 拆分与未来组合 Profile；当前不改变 END-P1，也不实现求值器。
Synem 闭包与条件激活至少两个精确 Endem 形成完整闭包；依赖唯一绑定，权限只收紧，成员结果独立，激活只选择固定成员。Pleko、Drasor、Theor 与未来 Synem Profile；当前没有物理容器、解析器或运行时。
Iknem 证据与评估记录种类与来源类别分开；有效性由外部政策评估，覆盖相对精确 krin 计算，决定只由具名权威形成。Drasor、Theor、审计者、决定权威与未来 Iknem Profile；当前没有物理格式或组件。
Endem 内容标准分层END-CORE、END-P* 与 END-FMT 分别定义通用内容、封闭能力集合与物理字节；接受结果不得跨层提升。规范作者、Profile 维护者、Ktisor、Theor、符合性报告和未来互操作；当前仍非稳定 ABI。

排除的捷径

  • 不把任意自然语言或模型 JSON 直接视为 Endem。
  • 不为查看、符号、大小、归档、裁剪、跟踪和评估各建一个产品。
  • 不使用环境路径、当前目录或网络“最新版本”解析正式依赖。
  • 不让通用对象变换器任意改写 semion、skena、telis、krin 或 apor。
  • 不把签名、Iknem 数量、模型评分或 Runtime 自述当作任务完成。
  • 不把 Dromen 序列化为可复制的权限对象。

外部协议适配不变量

AI 协议只扩大可交换信息，不扩大 Noemion 的信任。未来任何 MCP、A2A、模型供应商或遥测适配器都必须遵守以下边界。

不变量必须保留违反时怎样处理
版本固定协议主次版本、schema 摘要、扩展集合、传输与授权配置必须成为显式输入。版本未知、候选版未复核或降级不明确时拒绝适配，不按“最新版本”猜测。
身份不等于权威OAuth 主体、Agent Card 签名和服务器证书只证明声明来源；语义决定与能力授予仍由本地具名权威负责。不得因为签名有效而确认 semion、删除 apor 或授予 Drasor 句柄。
外部状态不等于本地结果Task、Message、Artifact、MCP Task 或工具结果只能成为带来源的运行事件、候选或 phain 输入。不得直接映射成 Endem 的生命周期、met/unmet、accepted/rejected、Iknem 有效性或证据覆盖度。
能力声明不等于实时句柄远端工具 schema 与 Agent Card 能力先进入控制平面，经策略、受众、范围、预算和人工升级检查。模型与远端 Agent 永远不能自行扩大能力、改写策略或绕过 Drasor。
遥测单向外送OpenTelemetry 映射位于运行与证据系统外缘，默认脱敏并绑定语义约定版本。遥测字段不得回流决定 Endem 内容身份、Iknem 身份或验收规则。
撤销与重放显式令牌撤销、Agent Card 轮换、任务取消、重试、幂等和重放窗口必须留下可定位事件。状态不明时进入 agno、apor 或失败，不得用远端“成功”覆盖本地未知。

变更规则

若要改变六个语义面、根 skena 唯一性、事态与方向分离、phain 比较、状态机、Synem 组合、Dromen 会话范围、Iknem 有限主张或唯一命令面，必须新增 ADR。策略变更后，还要同步规范、组件、生命周期、手册和测试向量。
