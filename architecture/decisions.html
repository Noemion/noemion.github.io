先从一个变更定位责任

例如，A2A Task 或 MCP 工具报告“已完成”，只说明远端任务或工具走到了某个状态。开发者还要分别保存远端实际报告了什么、它指向哪一份目标、现有证据覆盖了哪些条件，以及谁有权接受结果。外部协议、精确身份、证据、目标与授权规范分别约束这些步骤。

下面的入口按开发者要回答的问题组织。若对应物理 Profile、算法选择或权威输入仍在研究，文档只允许继续做规范与验证设计，不能把草案补成实现假设。

开发者问题先查什么停止条件
来源怎样成为确认意义，目标怎样判断？ADR-0010、ADR-0015至 ADR-0020、END-CORE语义投影、观察范围或判据不完整时，保留 unresolved_meaning、undetermined 或 fault，不生成肯定结论。
字节、Profile、来源清单和内容身份怎样分层？ADR-0011至 ADR-0014、ADR-0023、END-FMT、END-P2当前格式仍是实验草案；不得据此承诺稳定 ABI、解析器兼容或正式发行。
多个目标、一次会话和有限证据怎样组合？ADR-0021、ADR-0022、ADR-0024、CLOSURE-CORE、EVIDENCE-CORE、SESSION-CORE这些规范尚未定义物理容器或运行时；记录会话不能恢复实时权限。
失败、外部协议、身份、文本和授权怎样穿过信任边界？ADR-0025至 ADR-0030、DIA-CORE、ADP-CORE、ID-CORE、TEXT-IDENTIFIER-CORE、AUT-CORE外部状态、签名或模型输出只能成为带来源输入，不能改写本地结果、身份或能力。
术语为什么改用直白职责名称？ADR-0031至 ADR-0035、ADR-0037及术语审查结果当前机器标识已通过声明版本的关键字检查；Noemion 与 Endem 的发行读音仍待真实人类验证。
为什么形成版含原文，发布版却不含原文？ADR-0036、END-PUB-001 与 ID-REL-001发布 Profile、来源引用闭包或伴随资料关联尚未定义时，只能确定分层，不得实现裁剪或承诺发布格式。

ADR-0008 与 ADR-0009 只保存迁移历史；ADR-0010 至 ADR-0037 构成当前可查的决定记录。历史页面可以解释来路，但不能反向约束未来接口或 ABI。

常见输入怎样进入当前架构

先给每类输入一个受限角色，再说明它还缺少哪项确认。这样可以直接指导实现者接线，也避免把风险边界写成一串互不相连的禁令。

常见输入允许进入的角色仍需补足什么
自然语言、模型 JSON带来源的候选材料deterministic producer 所需的授权投影、未确定项与人工确认；候选本身不是 Endem。
独立查看实际制品字节独立只读检查职责由现行设计标识 inspect 表示；它不修复、不写回，也不产生生产检查通过结论。
发布派生、签名与符合性检查生产者、外部密钥权限域与发行验证分别负责当前都不是公开动作；在物理契约和真实消费者确定前，不用名字预占 CLI。
环境路径、当前目录、网络版本开发期定位线索正式依赖必须转换为精确身份、版本和 Profile；运行环境不能替代内容身份。
对象变换或迁移工具显式、可审计的转换步骤保存未知字段与来源，分别验证 meaning_projection、situation、goal_direction、satisfaction_criteria、unresolved_meaning 的语义不变量。
签名、证据数量、模型评分、Runtime 成功绑定来源的陈述、证据或运行事件按 evidence entry 覆盖、Endem 满足和具名权威决定分别求值，任何单一信号都不代表任务完成。
保存的会话记录审计和复盘材料新运行必须创建新的 session contract，并重新取得实时能力、秘密、预算和环境绑定。

外部资料怎样进入设计

外部资料只提供机制先例和风险证据。开发者先判断它能支持哪项工程区分，再回到 Noemion 的职责与信任边界；版本变化会触发复核，但不会自动改变 Endem。

外部协议适配不变量

AI 协议只扩大可交换信息，不扩大 Noemion 的信任。未来任何 MCP、A2A、模型供应商或遥测适配器都必须遵守以下边界。

不变量必须保留违反时怎样处理
版本固定协议主次版本、schema 摘要、扩展集合、传输与授权配置必须成为显式输入。版本未知、候选版未复核或降级不明确时拒绝适配，不按“最新版本”猜测。
身份不等于权威OAuth 主体、Agent Card 签名和服务器证书只证明声明来源；语义决定与能力授予仍由本地具名权威负责。不得因为签名有效而确认 meaning_projection、删除 unresolved_meaning 或授予 bounded runner 句柄。
外部状态不等于本地结果Task、Message、Artifact、MCP Task 或工具结果只能成为带来源的运行事件、候选或 structured_observation 输入。不得直接映射成 Endem 的生命周期、met/unmet、accepted/rejected、evidence entry 有效性或证据覆盖度。
能力声明不等于实时句柄远端工具 schema 与 Agent Card 能力先进入控制平面，经策略、受众、范围、预算和人工升级检查。模型与远端 Agent 永远不能自行扩大能力、改写策略或绕过 bounded runner。
遥测单向外送OpenTelemetry 映射位于运行与证据系统外缘，默认脱敏并绑定语义约定版本。遥测字段不得回流决定 Endem 内容身份、evidence entry 身份或验收规则。
撤销与重放显式令牌撤销、Agent Card 轮换、任务取消、重试、幂等和重放窗口必须留下可定位事件。状态不明时进入 undetermined、unresolved_meaning 或失败，不得用远端“成功”覆盖本地未知。

决定变化时要同步什么

改变六个语义面、根 situation 唯一性、事态与方向分离、structured_observation 比较、状态机、Endem closure 组合、session contract 会话范围、evidence entry 有限主张或唯一命令面时，新增 ADR，并同步对应 CORE、Profile、生命周期、开发者手册、威胁模型和测试向量。若变更涉及公开名称，还要重新执行书写冲突、目标语言读音和词表内口头区分审查。
