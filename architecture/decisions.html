架构决策

说明当前采用的原生语义、实验性容器、来源清单、结果分层、时间连续性、否定证据、量化范围、未来语言基线和组件边界。

ADR-0010 AcceptedDesign StageNo Component CodeABI Unfrozen

当前权威决定

ADR-0010定义六个语义面；ADR-0011定义实验字节边界；ADR-0012记录未来 Rust 基线；ADR-0013关闭 END-P1 设计载荷；ADR-0014定义实验来源清单；ADR-0015分开判断与运行结果；ADR-0016固定 mene 时间与连续性边界；ADR-0017分开否定事态与记录缺席；ADR-0018固定量化范围与不同成员计数。

被保留的安全原则已经重新映射到 Poiet、Theor、Praxor、Dromen 和 Tekmor。历史词只解释为什么迁移，不能反向约束实现、路由或 ABI。

ADR-0016 mene 时间与连续性模型Accepted Design采用 fixed/elapsed 时间范围、strict/budgeted 连续性与显式覆盖缺口；END-P1 继续只支持 kine。
ADR-0017 否定事态与缺席证据Accepted Design否定保留同一关系与角色；空结果默认是 agno，只有封闭观察范围内的完整缺席可以支持负目标。
ADR-0018 量化范围与集合成员资格Accepted Design先固定成员权威、截止点与身份规则，再按不同成员聚合；不允许有利子集、重复见证或默认空真值。

当前冻结的内容

决定采用的边界直接影响
六个语义面一个 Endem 只有一个根 skena，并保存 rhem、semion、skena、telis、krin 与 apor。来源格式、投影规则、Poiet、规范向量和错误语义。
事态与方向分离skena 保持中性；telis 只使用 kine/mene；禁止由事态否定表达。规范化、等价判断、时间字段与反例向量。
形式显示semion、skena 与 phain 共享符号和关系位置，不使用自证逻辑形式字段。结构编码、Poiet/Theor 差分和 aseme 诊断。
同一制品状态Endem 沿 nascent → coherent → attested 演进，不建立平行对象家族。身份、摘要、发布、签名和生命周期。
组合闭包Synem 只在多个 Endem 需要解析关系时出现；权限只能收紧。pleko、依赖锁定、冲突和披露。
运行非文件视图Dromen 只存在于会话内，不携带到磁盘或跨会话。Praxor、能力句柄、上下文和隔离。
有限证据主张Tekmor 保存可定位依据，但不自动证明结论或形成 accepted。来源、绑定、签名、运行、验收与审计。
唯一 CLIendem 提供 poie/elenk/pleko/tasse/sphra/theor/praxe/peira。应用项目、手册、安装包和自动化接口。
独立与隔离theor 不共享生产解析；praxe 不共享编译器权限域。仓库依赖方向、测试和安全边界。
实验性容器END-FMT 0.1 采用 64 字节前导、48 字节目录项、六种关键记录和 END-P0 有限上限。规范字节、错误分类、Poiet/Theor 语言实验；不构成稳定 ABI。
首版核心语言代码阶段开启后计划评估 Rust 1.97.0；禁止 unsafe 与首条切片零第三方 crate 仍是未来实现约束。未来实现的构建锁、受检算术、模糊测试和供应链边界。
首个设计 ProfileEND-P1 使用 profile 2，封闭六记录字段并只支持 nascent 与 kine。未来 Poiet/Theor 字段解析、交叉引用和当前载荷向量；不构成稳定 ABI。
首个来源输入END-SRCM 使用 UTF-8 逐行来源清单，显式分开来源、授权投影和 apor。Poiet 输入解析与失败定位；不是稳定源语言或对象身份。
判断与运行结果分层满足只使用 met/unmet/agno/fault；决定只使用 accepted/rejected/deferred；会话只使用 completed/failed/interrupted。Praxor、Tekmor、协议适配、人工验收和未来组合向量；不新增对象或 wire 字段。
mene 时间与连续性时间范围只使用 fixed 确定区间或 elapsed 具名事件时长；连续性只使用 strict 或完整 budgeted。Poiet、Praxor、Tekmor 和未来 mene Profile；遥测缺口不能成为允许违约。
否定与缺席证据negative 作用于同一关系和角色；没有记录不等于负观察，缺席推断必须绑定封闭观察范围。Poiet、Praxor、Tekmor、END-P1 负极性向量与未来完整性声明；空日志不能成为 met。
量化范围与成员资格只支持单个被量化角色；集合、权威、截止点、身份和空集政策必须显式，基数只按不同成员计数。Poiet、Praxor、Tekmor 与未来量化 Profile；当前不改变 END-P1，也不实现求值器。

排除的捷径

  • 不把任意自然语言或模型 JSON 直接视为 Endem。
  • 不为查看、符号、大小、归档、裁剪、跟踪和评估各建一个产品。
  • 不使用环境路径、当前目录或网络“最新版本”解析正式依赖。
  • 不让通用对象变换器任意改写 semion、skena、telis、krin 或 apor。
  • 不把签名、Tekmor 数量、模型评分或 Runtime 自述当作任务完成。
  • 不把 Dromen 序列化为可复制的权限对象。

外部机制的采用边界

以下资料只提供可复查的机制先例。每一项都同时写明采用部分和禁止推导，外部版本变化不能静默改变 Endem。

资料与复核基线Noemion 采用什么不能据此推出什么
ELF 对象格式
GNU Binutils 2.46.1		结构视图与装载视图分离、显式引用，以及独立于 BFD 的第二条读取路径。不继承机器地址、指令、弱符号规则，也不让 BFD 的通用表示进入第一阶段可信核心。
MCP 2025-11-25 稳定规范把协议错误与工具执行错误保留为不同来源事件，并将工具说明、注解和结果作为版本化适配输入。后续版本在正式发布并完成兼容、安全与降级复核前不成为基线；工具 schema、服务器说明、OAuth 身份和 isError 都不等于本地能力授权或满足结果。
GNU Automake 测试结果语义借鉴正常断言失败、跳过和环境硬错误的分离，避免把目标反驳、观察不足和求值故障合并。不继承 Automake 的退出码、PASS/FAIL 名称或测试协议，也不把测试成功直接映射为 Endem accepted。

A2A 1.0 版本化规范借用 Task、Message、Artifact、多绑定和主次版本协商交换跨系统候选状态与结果；签名 Agent Card 只支持来源核对。补丁号不进入协议协商；签名证明发布者关联，不证明能力描述真实，也不让外部 Task、Artifact 或 Agent Card 成为 Endem 身份、状态或验收决定。
OpenTelemetry GenAI 语义约定以后可由带版本、可脱敏的导出器映射运行观测。该约定仍处于 Development，不进入 Endem 编码、Tekmor 身份或最终决定；敏感内容不得默认导出。
NIST AI RMF 与 GenAI Profile采用风险登记、测试评估验证确认和具名责任人的治理框架，检查模型候选、远端工具和人工决定。风险框架不定义 Endem 字段、合格阈值或软件符合性；AI RMF 1.0 正在修订，不能冻结为 ABI。
智能体控制平面工程实践借鉴环境可读性、真实反馈和机械检查。实践文章不定义项目格式、能力边界、成熟度或实现证据。

复核日期：2026-07-13。适配器必须固定所用规范版本；升级时重新运行兼容、安全、重放、取消和权限测试。

外部协议适配不变量

AI 协议只扩大可交换信息，不扩大 Noemion 的信任。未来任何 MCP、A2A、模型供应商或遥测适配器都必须遵守以下边界。

不变量必须保留违反时怎样处理
版本固定协议主次版本、schema 摘要、扩展集合、传输与授权配置必须成为显式输入。版本未知、候选版未复核或降级不明确时拒绝适配，不按“最新版本”猜测。
身份不等于权威OAuth 主体、Agent Card 签名和服务器证书只证明声明来源；语义决定与能力授予仍由本地具名权威负责。不得因为签名有效而确认 semion、删除 apor 或授予 Praxor 句柄。
外部状态不等于本地结果Task、Message、Artifact、MCP Task 或工具结果只能成为带来源的运行事件、候选或 phain 输入。不得直接映射成 Endem 的生命周期、met/unmet、accepted/rejected、Tekmor 有效性或证据覆盖度。
能力声明不等于实时句柄远端工具 schema 与 Agent Card 能力先进入控制平面，经策略、受众、范围、预算和人工升级检查。模型与远端 Agent 永远不能自行扩大能力、改写策略或绕过 Praxor。
遥测单向外送OpenTelemetry 映射位于运行与证据系统外缘，默认脱敏并绑定语义约定版本。遥测字段不得回流决定 Endem 内容身份、Tekmor 身份或验收规则。
撤销与重放显式令牌撤销、Agent Card 轮换、任务取消、重试、幂等和重放窗口必须留下可定位事件。状态不明时进入 agno、apor 或失败，不得用远端“成功”覆盖本地未知。

变更规则

改变以下任一边界的提案必须新增 ADR：六个语义面、根 skena 唯一性、事态与方向分离、phain 比较、状态机、Synem 组合、Dromen 会话边界、Tekmor 有限主张或唯一命令面。采用决定后，还要同步规范、组件、生命周期、手册和测试向量。
