架构决策的适用范围

架构决策约束组件职责和后续规范方向，但不等同于已经发布的软件接口。每项决策回答“为什么这样分层、排除了哪些做法、影响哪些组件和规范”；具体字段、编码、ABI 和错误码仍须进入正式规范与一致性测试。

决定发生变化时，相关架构、组件、工具、手册和测试必须在同一次变更中更新。尚无充分证据的问题继续保留在开放问题，不能用实现便利静默替代。

已采纳的架构决策

决定采用的边界直接影响
命名与语义边界Noesis 表示确定性理解与编译活动；Noema 表示机器对象；Horizon 表示未完全展开的背景；Fulfillment 表示受约束求解。NIR、NOBJ、HOBJ、组件名、工具名与公开术语。
项目品牌与工具命令独立Noemion 不强制成为所有命令的字面前缀；工具可以采用能解释职责的独立哲学名称，但工程含义仍由输入、输出、不变量与测试定义，命令名不得进入对象格式身份。noesis、morphe、theoria、synthesis、工具路由与未来命令行界面。
Agent Harness 控制平面Harness 位于 Fulfillment Runtime 外侧，负责上下文、会话级能力、策略、观察、预算和人工升级；模型不能绕过它调用能力。Agent Harness、noemexecute、noemobserve、noemcoverage 与 noemevaluate。
工具链产物与外部签名发布链依次形成 Release Object、Release Coverage Proof、Unsigned Package Candidate、Signing Request、Signature Response 与 Signed Noemion Package；私钥只在外部签名系统。noemreduce、noemcoverage、noembundle、noemexecute 与对象生命周期。
候选、能力与验收分层Candidate Envelope 不是 NIR；Capability Requirement 不是实时句柄；Candidate Assessment、Evidence Closure 和 Acceptance Decision 是三个不同结果。Noesis Core、Object System、Agent Harness、Runtime 与所有运行证据工具。
NIR/NOBJ 结构基线自然语言先形成不可信候选；确认语义经过确定性重推导或授权绑定后进入带显式来源的 NIR 图记录。NOBJ 使用固定 Header、前置目录、长度限定 Section 和分层验证，不让模型决定记录或二进制布局。NIR/NOBJ 规范、Noesis Core、四套格式工具手册、验证器与畸形对象语料。
身份、完整性与签名域分离完整文件摘要在对象外计算；对象内 Integrity Root 不覆盖自身，未经外部摘要或签名绑定时也不是信任锚；Semantic Identity 等待等价规则冻结；发布签名属于外部封装且不证明语义正确。NOBJ Header、Integrity Section、Debug Companion、发布清单、签名范围与等价测试。
Phase 0/1 首个纵向切片先完成 Text NIR → NOBJ、生产验证器、独立直接读取器和差分一致性；工具页表示职责，不承诺 23 个独立二进制。首个实现仓库、规范条款、测试向量、Reader/Writer、theoria、noemvalidate 与 noemcertify。
外部 AI 协议只做适配MCP、A2A、JSON Schema 和 OpenTelemetry GenAI 只在 Agent Harness 外缘映射；远端工具说明、Agent Card、schema 与任务状态均是不可信声明。协议 Profile、能力策略、令牌隔离、安全回归与遥测 exporter。

已排除的捷径

  • 模型输出不能直接生成 NIR/NOBJ，也不能因结构合法被当作来源忠实语义。
  • 签名不能替代结构、语义、权限和环境策略验证。
  • Loaded State 不携带可直接执行的会话能力句柄。
  • Runtime 候选通过评价，不等于任务已经最终验收。
  • Release Coverage Proof、Evidence Closure Report、离线评估和跨工具一致性检查都不能替代发布授权或人工专属判断。

尚未冻结的内容

Candidate Envelope、Source Binding Decision、NIR 公共记录头与记录家族、NOBJ Header/Section Directory 的逻辑字段职责、非循环摘要域和首个实现切片已经形成详细设计基线。仍未冻结的是实现语言、类型枚举、字段宽度、魔数字节、Section/重定位编号、Text NIR 语法、压缩与摘要算法、协议适配 Profile，以及 Loaded State Handle、Runtime/Harness 协议、Acceptance Decision、签名封装和撤销协议。它们仍需威胁模型、跨实现原型和正反测试，不能从当前逻辑字段直接推断稳定 ABI。
