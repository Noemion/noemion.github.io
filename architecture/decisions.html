当前权威决定

ADR-0010定义六个语义面；ADR-0011定义字节边界；ADR-0012选择 Rust；ADR-0013关闭 END-P1 载荷；ADR-0014定义首个 Poiet 来源清单。

被保留的安全原则已经重新映射到 Poiet、Theor、Praxor、Dromen 和 Tekmor。历史词只解释为什么迁移，不能反向约束实现、路由或 ABI。

当前冻结的内容

决定采用的边界直接影响
六个语义面一个 Endem 只有一个根 skena，并保存 rhem、semion、skena、telis、krin 与 apor。来源格式、投影规则、Poiet、规范向量和错误语义。
事态与方向分离skena 保持中性；telis 只使用 kine/mene；禁止由事态否定表达。规范化、等价判断、时间模型和反例向量。
形式显示semion、skena 与 phain 共享符号和关系位置，不使用自证逻辑形式字段。结构编码、Poiet/Theor 差分和 aseme 诊断。
同一制品状态Endem 沿 nascent → coherent → attested 演进，不建立平行对象家族。身份、摘要、发布、签名和生命周期。
组合闭包Synem 只在多个 Endem 需要解析关系时出现；权限只能收紧。pleko、依赖锁定、冲突和披露。
运行非文件视图Dromen 只存在于会话内，不携带到磁盘或跨会话。Praxor、能力句柄、上下文和隔离。
有限证据主张Tekmor 保存可定位依据，但不自动证明结论或形成 accepted。来源、绑定、签名、运行、验收与审计。
唯一 CLIendem 提供 poie/elenk/pleko/tasse/sphra/theor/praxe/peira。应用项目、手册、安装包和自动化接口。
独立与隔离theor 不共享生产解析；praxe 不共享编译器权限域。仓库依赖方向、测试和安全边界。
实验性容器END-FMT 0.1 采用 64 字节前导、48 字节目录项、六种关键记录和 END-P0 有限上限。规范字节、错误分类、Poiet/Theor 语言实验；不构成稳定 ABI。
首版核心语言代码阶段开启后计划评估 Rust 1.97.0；禁止 unsafe 与首条切片零第三方 crate 仍是未来实现约束。未来实现的构建锁、受检算术、模糊测试和供应链边界。
首个设计 ProfileEND-P1 使用 profile 2，封闭六记录字段并只支持 nascent 与 kine。未来 Poiet/Theor 字段解析、交叉引用和当前载荷向量；不构成稳定 ABI。
首个来源输入END-SRCM 使用 UTF-8 逐行来源清单，显式分开来源、授权投影和 apor。Poiet 输入解析与失败定位；不是稳定源语言或对象身份。

排除的捷径

  • 不把任意自然语言或模型 JSON 直接视为 Endem。
  • 不为查看、符号、大小、归档、裁剪、跟踪和评估各建一个产品。
  • 不使用环境路径、当前目录或网络“最新版本”解析正式依赖。
  • 不让通用对象变换器任意改写 semion、skena、telis、krin 或 apor。
  • 不把签名、Tekmor 数量、模型评分或 Runtime 自述当作任务完成。
  • 不把 Dromen 序列化为可复制的权限对象。

变更规则

改变以下任一边界的提案必须新增 ADR：六个语义面、根 skena 唯一性、事态与方向分离、phain 比较、状态机、Synem 组合、Dromen 会话边界、Tekmor 有限主张或唯一命令面。采用决定后，还要同步规范、组件、生命周期、手册和测试向量。
