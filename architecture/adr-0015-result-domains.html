ADR-0015
判断与运行结果分层

把目标满足、权威决定、会话终止和证据状态分开，避免外部智能体或运行器用一个成功词覆盖全部判断。

当前策略结果域不改变线格式尚未实现

直接结论

一次运行顺利结束，不等于目标已经满足；目标满足，也不等于具名权威已经接受；一份记录有效，也不等于证据已经覆盖全部判断条件。

现行设计采用五个互不替代的结果域。每个结果都保存自己的主体、依据和原因；适配器只能记录映射，不能把外部协议状态直接改写成 Endem 状态或最终决定。

五个结果域

结果域允许值只回答什么绝不推出什么
制品生命周期nascent / coherent / attestedEndem 或 Synem 已形成、已收敛或已绑定发布见证。不推出当前环境可运行、目标满足或权威接受。
满足判断met / unmet / agno / faultskena 相对固定 krin、适用 phain 和 Iknem 的比较结果。不推出最终决定或会话是否正常结束。
权威决定accepted / rejected / deferred具名规则或主体是否在声明范围内作出最终决定。不改写原满足结果，也不证明制品、证据或运行无误。
会话终止completed / failed / interrupted一次 Drase 会话按终止规则结束、因运行故障失败或被外部条件中断。completed 不等于 met 或 accepted；failed 不等于 unmet。
证据状态valid / invalid / revoked
sufficient / insufficient		Iknem 的身份与适用性是否有效，以及证据集合是否覆盖当前 krin。单份有效记录不自动形成充分覆盖或最终接受。

满足判断怎样形成

结果成立前提禁止的替代推导
met必需观察齐全、有效、适用且比较完整执行，结果支持目标事态。不能从工具成功、日志存在、模型评分或外部 Task 完成推出。
unmet同样满足观察与求值前提，但比较结果反驳目标事态。不能从没有日志、缺少权限、超时或验证器崩溃推出。
agno意义和判据已经确定，但观察缺失、过期、越出范围或覆盖不足。不能用它保存语义歧义，也不能静默降成 unmet。
fault求值器、规则、依赖或观察适配过程没有按声明契约完成。不能把目标本身标为未满足，也不能用重试后的成功覆盖故障记录。

每项满足结果必须绑定 Endem 身份、krin 版本、观察截止点、所用 Iknem 和稳定原因。新观察或规则版本会产生新结果，不能原地改写旧记录。

权威决定怎样形成

决定最低条件边界
acceptedmet、必需 Iknem 有效且覆盖充分，以及 krin 指定的权威确实作出决定。任何缺项都不得由模型、Drasor 或外部协议补写。
rejected具名权威依据 unmet 或预先登记的拒绝政策作出否定决定。拒绝原因必须保留原满足结果；证据不足不自动等于目标为假。
deferred尚无获授权的最终决定，或 agno、fault、人工判断与授权缺口仍未解决。它不是隐式接受、普通失败或待模型自行选择。

非一一映射：met 可以因额外政策而被 rejected 或 deferred；agno 可以在预注册的关闭失败政策下被 rejected。任何决定都必须保留输入满足结果和决定权威，不能只保存最后一个状态词。

GNU 与智能体协议提供的反例

GNU Automake 测试语义区分正常断言失败、跳过与测试环境硬错误。这支持 Noemion 把 unmet、agno 与 fault 分开，但 Automake 的退出码和 PASS/FAIL 名称不会进入 Endem ABI。

MCP 工具错误规范区分协议错误与工具执行错误；A2A 1.0 任务状态又区分完成、失败、拒绝、取消以及等待输入或授权。这些状态只描述外部调用和任务生命周期，适配器不得把 completed 映射为 met 或 accepted。

W3C PROV-O把实体、活动、主体、生成、使用和失效保持为可追溯关系。Iknem 借鉴这种来源分离，但 Noemion 自行定义证据有效性、覆盖度与决定规则。

后果与验证路线

  • END-CORE 新增满足结果、权威决定和结果域分离条款；本决定不新增 END-P1 字段，现有字节保持不变。
  • Drasor、生命周期和 Iknem 页面删除跨层状态混用；外部 MCP/A2A 状态只进入带来源的运行事件。
  • 十二个结果域提案向量执行六个允许组合与六个确定拒绝，只证明当前矩阵和条款一致。
  • 未来一致性验证必须覆盖“会话 completed 但结果 unmet”“求值 fault 但目标未知”“结果 met 但决定 deferred”等交叉组合。
  • 当前只有规范、场景和计划验证，没有 Drasor、决定引擎、状态编码或组件测试。

待定内容

结果事件的物理编码、时间域、重放键、聚合规则、决定类 Iknem 格式、会话恢复和协议状态映射表仍需独立 Profile、威胁分析与正反向量。当前策略只确定语义分层和禁止推导。
