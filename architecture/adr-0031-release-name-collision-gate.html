ADR-0031
发布名称冲突门禁

用 Iknem 与 Drasor 消除相邻领域冲突，同时保持证据和受限执行职责不变。

Accepted2026-07-13One-time migrationNo compatibility

直接结论

Iknem 取代旧证据对象名，Drasor 取代旧受限执行域名，对应公开动作改为 drase。

Iknem 仍表示绑定精确主体、有限主张、溯源、观察、方法、结果和限制的证据记录。Drasor 仍负责在一次会话中重验精确输入、收窄能力、控制副作用并封存 Dromen。更名不改变任何结果域、权限边界、身份规则或规范内容。

迁移规则：当前尚未进入组件代码阶段，也不承诺向后兼容。旧术语不再进入现行页面、规范 ID、诊断域、路由、向量或动作；不保留别名、重定向、双写或兼容垫片。

为什么必须更名

退役名称可核对的外部使用冲突判断
PraxorPraxor Lab研究大模型推理、解释性和强化学习；Praxor.ai提供 Agentic AI 系统；praxor.dev提供软件工程与开发工具服务。与受限执行组件处于同一人工智能和软件工程发现空间，不能依靠说明文字消歧。
Tekmortekmor.xyz把同名用于隐私身份、零知识证明、TEE、确定性构建和链上 attestation。与范围证据、完整性、验证和证明职责直接重叠，达到停止条件。

这些证据证明工程发现冲突，不判断商标权或在先权利。首次正式发行仍需完成 WIPO、目标法域数据库、软件包坐标和合格专业人员复核。

证据对象候选比较

候选检索与可读性决定
Iknem2026-07-13 工程初筛没有发现同名的活跃 AI、证明、协议或开发工具。公开结果中的课程缩写、航空航路点和零散人名不承担相邻工程职责；读音与听辨尚未验证。采用其当前设计拼写与证据职责；含义只由 IKN-CORE 定义，不暗示记录真实、有效或充分。发行读音门禁见 ADR-0034。
Vekem字形简短，但与面向 AI Agent 的记忆软件 vemem 仅差一个字母，口头和搜索区分度不足。拒绝
Pistem容易与 PiSTEM 教育组织及使用 AI 的透明能源模型项目发生大小写和拆词混淆。拒绝
evidence-record直白，但过于通用，适合解释职责，不适合作为需要稳定引用的对象名、规范 ID 和诊断域。保留为释义，不作为术语

受限执行候选比较

候选检索与可读性决定
Drasor / drase组件与动作关系清楚。2026-07-13 初筛没有发现同名的活跃 AI、Agent、运行时或开发工具；DraSOR 酶缩写、房地产主体和早期 FORTRAN 子程序属于不相邻用途。采用；不得由词源推导权限或执行成功。
Dranor / dranDranor 是已发表游戏角色；dran 已是活跃 PyPI 科学软件名，DRAN 也用于生成式 AI 研究。拒绝
Drast已有同名 AI 销售自动化产品，处于直接相邻领域。拒绝
execution-controller容易理解，却把会话封存、最小能力域和失败责任压扁成通用控制器，也不适合作为简洁动作。保留为释义，不作为术语

一次性迁移面

  • 证据标准改为 IKN-CORE，规范 ID 改为 ikn-core，诊断域和向量目录使用 iknem.* 与 vectors/iknem/。
  • 受限执行组件改为 Drasor，唯一 CLI 动作集合中的对应动作改为 drase。
  • Dromen 继续是 Drasor 为一次 Drase 会话封存的只读执行契约；Iknem 继续只支持自身声明范围。
  • 规范、ADR、场景、威胁模型、页面、导航、测试、项目提示词和登记项同步替换；旧入口直接消失。
  • 历史冲突证据只保留在本 ADR 和名称审计中，不构成旧接口或兼容承诺。

仍然存在的停止条件

Iknem、Drasor 与 drase 只通过了开发阶段的工程初筛。正式发行前必须再次查询 PyPI、npm、crates.io、主要系统包、GitHub、协议与标准、人工智能产品、域名、WIPO 和目标法域商标。若出现活跃相邻用途，直接重新命名，不以兼容成本为理由保留冲突名称。
