项目/架构设计/架构决策/ADR-0031

ADR-0031
搜索没有冲突
不等于名称可以发行

evidence entry、bounded runner 与 drase 已替代发生相邻工程冲突的旧名称，但它们仍只是当前设计标识；正式发行还需要包坐标、权利、目标语言读音和完整词表口头区分证据。

当前策略设计标识已采用发行读音待定不保留兼容

先区分设计采用与发行通过

名称治理不是一次搜索。开发阶段先排除明显冲突并保持规范可读，首次可安装发行前再用新的时间点、目标市场和完整词表证明名称仍然可用、可读、可听写且可取得。

固定工程职责
排查书写与相邻用途
比较直白名称
验证读音与听写
核对权利与分发坐标
形成发行决定
标识当前设计职责当前结论
evidence entry绑定精确主体、有限主张、溯源、方法、结果和限制的有范围证据记录；规范为 EVIDENCE-CORE已替代旧名；没有人类读音证据，不得称为发行名称
bounded runner在一次会话中重验输入、收窄能力、控制副作用并封存 session contract 的受限执行域已替代旧名；组件尚未实现，也未完成发行名称审查
drase对应受限运行职责的当前公开动作标识已替代旧动作；口述不能选择动作、参数或权限
直白职责名scoped evidence record、bounded runner、run 等非规范候选必须与保留专名方案一同进入人类职责匹配，不能提前当作别名

当前策略：保留 evidence entry、bounded runner 与 drase 作为现行设计标识，同时明确其发行资格尚未成立。搜索为空只说明某个时间点未发现结果，不产生包名保留、商标权或官方读音。

为什么旧名称必须一次性退出

退役名称可核对的相邻用途为什么文案不能消歧
PraxorPraxor Lab 研究大模型推理、解释性与强化学习；Praxor.ai 提供 Agentic AI；praxor.dev 提供软件工程服务与受限执行域处在同一 AI 与开发工具发现空间，名称本身已无法稳定定位 Noemion 职责
Tekmortekmor.xyz 使用同名描述隐私身份、零知识证明、TEE、确定性构建和链上 attestation与证据、完整性、验证和证明职责直接重叠，达到工程停止条件
保留旧名并补说明会让搜索、Issue、依赖说明和口头讨论继续混入外部相邻对象解释文字不能改变名称冲突，也不能为未来兼容成本制造既成事实

该判断只证明工程发现冲突，不判断外部主体的商标权、在先权利或法律优先顺序。采用新设计标识时不保留别名、重定向、双写或兼容垫片；历史 ADR 只保存迁移原因，不恢复旧接口。

为什么现行名称仍不能称为低风险

证据轴截至 2026-07-16 的观察声明上限
PyPI 与 npm 精确入口iknem、drasor、drase 均返回 404只说明本次公开入口没有精确项目；不保留名称，也不覆盖近似拼写
crates.io本次自动查询返回 403，无法取得可解释结果证据缺失，必须在发行审查中按站点允许方式重新核对并实际取得坐标
产品与工程检索未发现足以推翻既有设计决定的精确相邻 AI、Agent、证明、运行时或开发工具结果搜索引擎会遗漏新发布、地区限制和未索引对象，不能证明不存在冲突
商标数据库尚无目标法域、类别、近音、拆词和视觉近似的合格正式报告WIPO 明确说明全球数据库未命中不等于名称可用
读音与听写尚未执行真实人类发现阶段和发行审查阶段不能发布 IPA、官方发音或全球传播性结论
职责必要性非规范提案提出直白职责名，尚未完成与专名方案的盲测比较必须先证明专名有独立价值，不能因已经写入资料就默认保留

一次正确朗读、TTS、ASR、模型转写、团队习惯或维护者自拟 IPA 都不能补足人类证据。名称必须在中文和明确登记的英语变体中，与 Endem、Endem closure、session contract、deterministic producer、independent inspector 及全部公开动作共同测试。

名称进入首次发行前必须通过什么

检查必须保存的证据停止条件
职责必要性专名与直白职责名的对象选择、职责匹配、记忆与误解比较直白名称已经准确表达职责，而专名只增加学习和听写负担
工程生态PyPI、npm、crates.io、主要系统包、GitHub、GNU、协议、标准、AI 产品和开发工具的查询日期、查询式与命中活跃相邻用途导致搜索、依赖或职责混淆
书写与口头区分目标语言、说话者、条件、首次朗读、手写回填、完整词表选择和职责匹配矩阵误选其他现行名称、匹配错误职责、无法判断或形成稳定替代拼写
分发与社区坐标实际取得的软件包、命令、仓库、域名和发布者控制关系只能依赖误导拼写、历史包接管或不受控下载入口
权利复核WIPO、目标国家或地区官方数据库、商品与服务类别、近音与视觉近似，以及合格专业意见有效相邻权利存在可能混淆，或证据不足以作出决定
变更触发新市场、新语言、新词表成员、新职责、语音入口和重大外部命中的复核记录旧证据范围不覆盖新条件时，发行资格失效并重新审查

正式发行前必须再次查询全部工程、权利和口头区分范围。WIPO Global Brand Database 也建议同时检查国家或地区知识产权局数据库，并在需要时咨询商标专业人员；全球数据库不是全世界所有权利的完整证明。

重新命名怎样保持工程边界不变

迁移面必须同步不得发生
证据对象对象名、EVIDENCE-CORE、规范 ID、诊断域、向量目录、页面与登记名称变化暗示证据自动真实、有效或充分
受限执行域组件名、公开动作、session contract 关系、页面、导航、测试和职责说明词源或名称变化扩大能力、权限或会话生命周期
机器接口命令、路径、格式、Profile 和示例在同一次变更中保持唯一保留旧命令、隐藏转换、双写或模糊诊断
规范语义逐项验证字段、结果域、身份规则、授权和失败语义未改变用改名掩盖语义迁移，或沿用旧身份表示新内容
公开解释先写直白职责，再给出当前精确设计标识和状态把候选名、临时读法或迁移历史写成兼容入口

当前没有组件实现，因此采用一次性迁移成本最低。未来如果名称再次失败，也必须创建新的命名 ADR 并同步全部现行资料，不能以兼容成本为理由保留冲突名称。

当前状态与继续阅读

evidence entry、bounded runner 与 drase 已完成开发阶段的直接迁移，但发行名称、人类读音、分发坐标和法律复核均未完成。现阶段会议和文档应先说“有范围证据记录”“受限执行域”“受限运行动作”，再给出精确书面标识；口述近似音不能触发操作。

ADR-0034 · 读音与口头区分查看首次朗读、完整词表听写、职责匹配和发行停止条件。 ADR-0037 · 术语改用直白职责名称查看现行名称、被替换名称及不保留兼容入口的决定。 ADR-0034 · 读音与口头区分审查查看为什么自动语音工具和一次听写不能成为发行门槛。 ADR-0032 · 命令只改大小写，仍然是名称冲突继续核对 deterministic producer、ktise 与旧名 POIE 的相邻冲突。

读音状态：当前没有合格发行读音表。evidence entry、bounded runner 与 drase 的职责已确定，读音和发行拼写仍须由独立人类证据决定；悦耳或词源贴切不能代替职责、听写和权利验证。
