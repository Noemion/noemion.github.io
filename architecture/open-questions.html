开放问题

记录 Endem 体系中尚未通过规范向量、威胁模型、验证方案或正式权利检查回答的问题。

哪些问题已经关闭

ADR-0010 与 ADR-0015 已经采用现行制品、六个语义面、生命周期和结果域，ADR-0031 至 ADR-0033 又直接替换冲突名称与易误读的标准 ID。它们仍是当前规范基线，但后续非规范审查已经重新打开具体发行拼写，并质疑 attested 是否把外部签名与证明关系误写成内容状态；形成新 ADR 前不能把这项边界称为最终关闭。

END-CORE 与 END-FMT 已建立 Endem 内容和实验性容器设计。ADR-0015 至 ADR-0020 固定结果域、时间、否定、量化、测量和复合判断；ADR-0021 至 ADR-0030 固定 Synem、Iknem、内容标准分层、Dromen、结构化诊断、协议适配、身份、文本、授权以及内容与外部决定的伴随边界；ADR-0031 至 ADR-0035 固定名称迁移、口头区分与公开动作门禁。

当前没有 Theor、权威目录、政策求值器或其他组件实现。稳定 ABI、时间与组合物理字段、嵌套或多变量量化、Synem 与 Iknem 物理容器、正式来源语言、Unicode 标识符、原始来源字节绑定、摘要和签名物理 Profile、授权决定与最终决定事件编码仍未冻结。

以下问题只讨论编码、互操作、性能、安全与正式发布条件，不能借实现便利重新引入旧对象或 23 个命令家族。

Endem 编码与语义

实验来源清单怎样升级为正式语言？

ADR-0014 只关闭首个 .ends 输入。正式语言仍需解决注释、包含、版本演进、歧义、错误恢复和工具支持；在正反语料与真实消费者出现前，不扩展当前清单。

END-P1 之后还缺哪些字段？

ADR-0023 已关闭内容标准分层，但没有预先冻结未来内容。END-P1 已关闭未来首条实现路径评审所需字段，并已有正负极性字节；时间、缺席、单变量量化、测量、组合、条件适用性和统计登记仍需真实消费者、新 Profile 与正反字节向量。ID-CORE 已固定摘要与签名的抽象边界，但它们是否进入容器、采用何种物理 Profile 仍未决定。

条件与排他组合怎样表达？

ADR-0020 只允许单个 Endem 使用 all_of / any_of；ADR-0021 已把跨 Endem 条件固定为 Synem 会话激活。xor / xone、形成时条件编码和复杂守卫语言仍未冻结，不得嵌入自由脚本。

一个根 skena 的语义等价怎样判断？

当前非规范语义等价与迁移研究提案已经否定跨 Profile 的通用等价布尔值，并把精确身份、封闭结构同构、有范围观察等价、迁移、强化/弱化与模型相似度分开。它提出十六个案例、十三类威胁和 Semantic Key 的进入条件，但仍在等待用户接受；在此之前只能比较显式符号、关系和条件，不能发明语义哈希。

Unicode 标识符与原始来源字节何时进入 Profile？

TEXT-IDENTIFIER-CORE 已固定当前 ASCII 标识符和来源文本不做隐式规范化，也明确 END-SRCM 不能证明原始文件逐字节保真。只有国际化标识符或原始字节审计出现不可替代消费者，并能固定 Unicode 版本、比较、双向、同形、身份、变换和正反字节向量时，才建立新 Profile。

权威目录与政策语言怎样获得物理 Profile？

AUT-CORE 已固定主体、范围、委托、同意、多人冲突、撤销、重放与结果分离。ADR-0030 又规定 END-P1 的权威字段只是选择器，不能自证授权。权威目录、角色与政策语言、授权伴随关系、事件编码、同意 UI Profile 和外部协议映射仍需真实消费者、独立威胁分析、规范字节与实现阶段授权。

连续或模糊要求怎样进入六个语义面？

ADR-0019 已关闭可量化构念、总体、单位、聚合器、不确定度和阈值的抽象边界。软偏好、感知质量、风格、模型裁判校准和不可完全判定条件仍需保留评价环境，不能为了机器处理伪造精确阈值。

kine/mene 会成为正式发行拼写吗？

不会按现状直接发行。kine 已有稳定而无关的英语读音和词义，两个词也不能让首次读者恢复目标方向职责，因此已经未通过桌面门禁。当前非规范 telis 发行词候选提案建议先用独立人类证据验证 reach/maintain。候选不是现行枚举；END-TEL-001、END-P1 和向量暂不修改，也不建立别名。若测试与用户决定接受替换，再以单独 ADR 一次性迁移。

Synem 与发布

绑定标识和版本约束采用什么语法？

SYN-CORE 已要求最终绑定唯一精确内容身份，ID-CORE 又固定完整引用的必需语境，并禁止环境搜索和网络最新版本。TEXT-IDENTIFIER-CORE 进一步规定当前结构标识符保持 ASCII、大小写敏感且与 locale 无关。可读名称、版本范围的候选语法、摘要算法基线、国际化标识符和绑定记录物理格式仍需新 Profile。

ELF 的符号与重定位思想怎样用于 Synem？

当前非规范 GNU 与 ELF 适用性研究提案只采用显式引用、精确绑定、形成映射和失败责任。它拒绝地址修补、弱符号、默认版本、环境搜索、顺序选择和运行期语义漂移。物理符号视图、绑定表或版本语法只有在出现真实 Synem 消费者、同名冲突反例和规范向量后才能提出 ADR。

什么时候允许依赖循环？

SYN-GRF-001 已规定第一阶段拒绝依赖与激活循环。未来若支持，必须先定义固定点语义、披露与验收顺序、预算上限、失败原子性和跨实现向量。

Synem 是否需要独立物理容器？

在出现多 Endem、流式读取或离线分发的真实消费者前，内容寻址清单和现有确定性归档可能已经足够。不能只为形式完整提前创造格式。

派生制品可以删除哪些信息？

ADR-0035 已拒绝预留通用裁剪动作。ID-REL-001 要求裁剪结果与调试伴随文件各有精确身份，并记录变换、保留内容和损失。当前没有裁剪等价规范；哪些属性允许删除、伴随关系采用什么物理格式，仍需真实消费者。

外部签名采用什么算法与撤销策略？

ADR-0035 已把签名保留为外部集成责任，而不是公开动作。ID-CORE 固定有类型待签陈述、外置包络、权威分离、截止点和撤销解释；算法基线、多签政策、密钥体系、证书 Profile、时间来源、透明日志、撤销分发和离线包络物理编码仍需独立 ADR 与证据。

Dromen、Drasor 与 Iknem

这些横切问题的已接受边界、等待决定研究和外部运行事实集中在Agent 系统边界图。以下条目继续保存尚未关闭的具体决定。

Dromen 的运行 API 和平台隔离怎样实现？

DRO-CORE 已固定主体、政策、环境、能力、秘密、预算、激活、观察、只读与销毁边界。未来仍需在代码阶段选择进程模型、沙箱组合、凭据代理、事件编码、预算计数器与销毁机制，并用平台级对抗证据证明，而不是把 Linux、容器或模型 SDK 名称当作隔离结论。

模型上下文怎样装配而不让数据取得指令权？

TEXT-IDENTIFIER、AUT、ADP 与 DRO 已分别约束模型实际输入、授权、外部来源和会话上限，但尚未统一输入选择、角色、顺序、截断、缓存和损失。当前非规范研究提案已经补齐十四个案例、唯一责任归属、八类威胁和失败域矩阵，主张由控制平面形成可审计装配说明，并把未来条款归还现有规范。它不创建新制品、命令、组件或哲学专名；在用户明确接受设计前仍不进入 ADR、CORE、登记或向量。

Agent 声明的能力何时才真正可调用？

Agent Card、工具列表、schema 与签名只能形成有来源的能力声明；协议初始化只确定双方支持的机制，AUT 决定精确动作是否获权，Dromen 固定本会话上限，调用前还要核对端点、预算、配额、输入和依赖。当前非规范能力发现与协商研究提案用十二个案例和变化矩阵说明：列表新增不能扩大旧 Dromen，移除或实质 schema 漂移必须阻止陈旧调用，一次成功也不能成为长期权限、met 或 accepted。它不创建 CAP-CORE、能力制品、组件或新专名；责任分配等待用户接受。

目标、计划和重规划怎样分开？

Endem/Synem 固定终态，Dromen 固定会话上限，计划只是在上限内可变的控制平面状态。当前非规范目标、计划与重规划研究提案用十二个案例、重规划矩阵和十类威胁说明：步骤、handoff、MCP/A2A Task 和工具完成都不能直接成为目标满足或最终接受。它不创建计划制品、命令、组件或 CORE；在用户接受责任分配前不进入登记或向量。

终态成立是否表示某个行动者实现了目标？

不能。现有 krin 能比较终态，却没有固定前态、动作发生、竞争行为者或因果方法。当前非规范状态变化与因果归因研究提案把终态满足、动作发生、状态转变、因果归因、授权责任和最终决定分开，并建议把 kine 澄清为目标方向。该建议仍等待用户接受，不修改 END-TEL-001、不进入登记或向量，也不创建新对象、命令或组件。

预览或 dry-run 通过后能否直接批准并认定完成？

不能。预览只说明决定者看到了什么，模拟只说明声明条件下预计什么，授权只允许精确请求尝试。当前非规范预览、模拟与批准研究提案用 GNU Make 的 dry-run 例外、MCP/A2A、OpenAI Agent 审批和 NIST 人机风险说明：执行前仍须核对漂移，执行后仍须独立观察。它不新增制品、结果域、命令、组件或 CORE；责任分配等待用户接受。

保存检查点后能否恢复原会话、权限和完整记忆？

不能。会话历史只是候选输入，压缩摘要必须披露变换与损失，检查点只描述一个截止点，外部 Task 句柄只定位协议状态。当前非规范记忆、检查点与恢复研究提案用 OpenAI Agents SDK、MCP、A2A、GNU Make 与 GNU Guix 分开继续、恢复、重试、重放、回滚和补偿。旧 Dromen、秘密与权限不能复活；恢复必须重新验证并建立新会话。提案不新增制品、命令、组件或 CORE；责任分配等待用户接受。

Drasor 怎样隔离模型和能力适配器？

需确定进程、沙箱、取消、超时、幂等、网络、文件、凭据和资源限额。后端失败不能让 Drasor 静默扩大权限或修改 attested 制品。

Iknem 怎样获得物理编码？

IKN-CORE 已固定 phain 对齐、有限无环溯源、证据类别、有效性、覆盖、决定分离和最小披露。字段编号、容器、摘要与签名算法、透明日志、撤销分发、跨生产者时钟归并、隐私策略和规范字节仍需真实消费者与新 Profile。

mene 时间怎样进入未来 Profile？

ADR-0016 已决定 fixed/elapsed、strict/budgeted 与覆盖结果；字段编号、数值宽度、亚秒精度、闰秒、循环日历、时区数据库封装、跨重启关联和多生产者归并仍保持开放。

授权与最终决定事件怎样编码与撤销？

AUT-CORE 已定义 grant/deny/defer 的授权决定内容、重放、时效和多人规则；ADR-0015 与 IKN-DEC-001 则规定 accepted/rejected/deferred 必须保存原满足结果与具名权威。两类事件仍保持正交，其字段、物理编码、撤销传播和外部协议映射仍需独立 Profile 与规范字节。

MCP 与 A2A 的具体 Profile 怎样冻结？

ADP-CORE 已固定版本、对端、能力、调用、状态、产物、取消、重试、交付和安全不变量。具体字段映射仍需等待精确稳定版本、真实消费者、协议专属威胁、正反协议向量和实现阶段授权；不得用通用 SDK 对象替代 Profile。

结构化诊断怎样获得物理编码与协议映射？

DIA-CORE 已固定机器身份、生产语境、层次、位置、主错误、恢复和安全边界。JSON、CBOR、SARIF、Problem Details、MCP、CLI 退出状态、日志与遥测字段仍需真实消费者、版本规则、披露政策、规范字节和跨实现证据；任何映射都不能改写本地结果域或授权。

实施与发布条件

独立 Theor 使用什么实现边界？

未来 Theor 必须拥有自己的数据结构、解码器、验证器和错误路径，只与 Ktisor 共享公开规范和字节向量。当前没有实现；是否采用 Rust 或第二种语言，必须在用户开启代码阶段后根据共同故障风险和维护成本重新确认。

独立 Theor 还缺什么证据？

当前只有规范向量、资料检查器和未来验证条件。用户开启代码阶段后，才建立依赖审计、差分测试、长时 fuzz、故障注入和跨平台运行，用来寻找两条独立路径的共同输入假设。

哪些现行名称还值得成为发行专名？

尚未决定。当前非规范发行术语去专名化提案建议只保留 Endem 进入人类验证，并比较对象、角色与动作的直白候选；后续语义面与观察词提案又保留七项职责，但建议以直白字段比较现行短词。最新生命周期与结果词提案还发现 attested 把外部多值签名和证明关系误写成内容单值状态，并比较 formed/resolved/undetermined/no_allowed_projection/stopped。三组候选都不是现行规范值、字段、CLI、别名或重定向；只有用户接受方向、独立人类验证通过并形成迁移 ADR 后，才一次性替换旧入口。

名称何时满足正式发布条件？

ADR-0031 至 ADR-0033 只完成工程初筛、当前名称选择与标准 ID 可读性治理；ADR-0034 进一步要求为目标语言登记读音，并用首次朗读、听写回填和成对混淆记录证明词表内口头区分。具体任务、样本边界和停止规则见术语与读音验证指南，当前尚未执行。kine/mene 已在此前先未通过桌面门禁，reach/maintain 也只处于候选状态。公开发行前仍需重新检查相邻 AI 与开发工具、目标法域商标、近似类别、包管理器、命令和域名，并完成法律复核。任一门禁失败都必须重新命名或给出经验证的新候选，审查完成前不得声称名称已具备发行条件。

怎样证明构建可复现？

ID-REP-001 已要求精确输入闭包、工具与参数、环境约束、至少两条独立产出路径和逐字节输出比较。仍需在代码阶段建立真实隔离构建、缓存独立性、差异归因和发布证据；build ID、签名和一次重跑都不能替代这些证据。

问题关闭规则

设计问题可以由权威规范或 ADR、正反向量和明确失败语义关闭。涉及组件行为、性能或互操作的问题必须保持开放，直到用户明确开启代码阶段并出现相应实现证据；论文、模型指标、概念演示或单一实现行为不能单独冻结 ABI。
