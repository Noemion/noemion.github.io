开放问题

记录 Endem 体系中尚未通过规范向量、威胁模型、验证方案或正式权利检查回答的问题。

哪些问题已经关闭

ADR-0010 已经采用现行制品、组件、六个语义面、三个制品状态和八个动作。这些名称与职责不再作为开放问题反复讨论。

END-CORE 与 END-FMT 已建立 Endem 内容和实验性容器设计。ADR-0015 至 ADR-0020 固定结果域、时间、否定、量化、测量和复合判断；ADR-0021 与 SYN-CORE 固定 Synem 抽象闭包；ADR-0022 与 TEK-CORE 固定 Tekmor 证据与评估边界；ADR-0023 固定 END-CORE 内容标准、END-P* 内容 Profile 与 END-FMT 容器的分层。

当前没有 Theor 或其他组件实现。稳定 ABI、时间与组合物理字段、嵌套或多变量量化、Synem 与 Tekmor 物理容器、正式来源语言和结果事件编码仍未冻结。

以下问题只讨论编码、互操作、性能、安全与正式发布条件，不能借实现便利重新引入旧对象或 23 个命令家族。

Endem 编码与语义

实验来源清单怎样升级为正式语言？

ADR-0014 只关闭首个 .ends 输入。正式语言仍需解决注释、包含、版本演进、歧义、错误恢复和工具支持；在正反语料与真实消费者出现前，不扩展当前清单。

END-P1 之后还缺哪些字段？

ADR-0023 已关闭内容标准分层，但没有预先冻结未来内容。END-P1 已关闭未来首条实现路径评审所需字段，并已有正负极性字节；时间、缺席、单变量量化、测量、组合、条件适用性、统计登记、摘要、签名和 Synem 的物理表达仍需真实消费者、新 Profile 与正反字节向量。

条件与排他组合怎样表达？

ADR-0020 只允许单个 Endem 使用 all_of / any_of；ADR-0021 已把跨 Endem 条件固定为 Synem 会话激活。xor / xone、形成时条件编码和复杂守卫语言仍未冻结，不得嵌入自由脚本。

一个根 skena 的语义等价怎样判断？

布局、来源注释或调试记录变化后，哪些 semion、skena、telis、krin 和 apor 属性必须保持，仍需规范化规则和反例。规则冻结前只能比较显式符号、关系和条件，不能发明语义哈希。

自然语言候选由谁授权决定？

需定义决定主体、作用范围、撤销、重放、高风险语义面和多人冲突。模型置信度、结构合法和单次界面点击都不能单独成为永久授权。

连续或模糊要求怎样进入六个语义面？

ADR-0019 已关闭可量化构念、总体、单位、聚合器、不确定度和阈值的抽象边界。软偏好、感知质量、风格、模型裁判校准和不可完全判定条件仍需保留评价环境，不能为了机器处理伪造精确阈值。

Synem 与发布

绑定标识和版本约束采用什么语法？

SYN-CORE 已要求最终绑定唯一精确内容身份，并禁止环境搜索和网络最新版本。可读名称、版本范围的候选语法、摘要算法和绑定记录物理编码仍需新 Profile。

什么时候允许依赖循环？

SYN-GRF-001 已规定第一阶段拒绝依赖与激活循环。未来若支持，必须先定义固定点语义、披露与验收顺序、预算上限、失败原子性和跨实现向量。

Synem 是否需要独立物理容器？

在出现多 Endem、流式读取或离线分发的真实消费者前，内容寻址清单和现有确定性归档可能已经足够。不能只为形式完整提前创造格式。

tasse 可以删除哪些信息？

只有不改变六个语义面、组合、能力上限和运行披露的内容才能外置。还需证明裁剪前后关系等价，并用完整摘要绑定调试伴随记录。

sphra 的算法与撤销策略是什么？

需确定待签消息、算法协商、多签、密钥轮换、撤销、时间来源和离线验证。签名必须位于被签载荷之外，且不能替代结构或环境授权。

Dromen、Praxor 与 Tekmor

Dromen 的最小只读接口是什么？

需要定义 semion 关系、skena 子图、telis、krin、apor、成员披露、预算和能力需求怎样进入一次运行，同时确保 Dromen 不含实时句柄、凭据或可跨会话权限。

Praxor 怎样隔离模型和能力适配器？

需确定进程、沙箱、取消、超时、幂等、网络、文件、凭据和资源限额。后端失败不能让 Praxor 静默扩大权限或修改 attested 制品。

Tekmor 怎样获得物理编码？

TEK-CORE 已固定 phain 对齐、有限无环溯源、证据类别、有效性、覆盖、决定分离和最小披露。字段编号、容器、摘要与签名算法、透明日志、撤销分发、跨生产者时钟归并、隐私策略和规范字节仍需真实消费者与新 Profile。

mene 时间怎样进入未来 Profile？

ADR-0016 已决定 fixed/elapsed、strict/budgeted 与覆盖结果；字段编号、数值宽度、亚秒精度、闰秒、循环日历、时区数据库封装、跨重启关联和多生产者归并仍保持开放。

最终决定事件怎样编码与撤销？

ADR-0015 与 TEK-DEC-001 已规定 accepted/rejected/deferred 必须保存原满足结果与具名权威，decision-record 只记录决定而不创造权限。事件字段、重放键、撤销传播、多权威冲突和聚合规则仍需 Profile 与规范字节。

MCP 与 A2A 怎样映射到 Praxor 外缘？

需冻结协议版本、工具或 Agent 声明的信任等级、令牌受众、取消与重放语义。外部任务状态和产物不能直接成为 Endem 状态或最终决定类 Tekmor。

实施与发布条件

独立 Theor 使用什么实现边界？

未来 Theor 必须拥有自己的数据结构、解码器、验证器和错误路径，只与 Poiet 共享公开规范和字节向量。当前没有实现；是否采用 Rust 或第二种语言，必须在用户开启代码阶段后根据共同故障风险和维护成本重新确认。

独立 Theor 还缺什么证据？

当前只有规范向量、资料检查器和未来验证条件。用户开启代码阶段后，才建立依赖审计、差分测试、长时 fuzz、故障注入和跨平台运行，用来寻找两条独立路径的共同输入假设。

名称何时满足正式发布条件？

项目内部命名已经冻结，但公开发行前仍需检查目标法域商标、近似类别、包管理器、命令和域名，并完成法律复核。审查完成前不得声称商标可用。

怎样证明构建可复现？

对象字节、工具二进制和签名真实性是三个不同属性。需要双路径构建、固定环境输入、逐字节比较和独立签名验证。

问题关闭规则

设计问题可以由权威规范或 ADR、正反向量和明确失败语义关闭。涉及组件行为、性能或互操作的问题必须保持开放，直到用户明确开启代码阶段并出现相应实现证据；论文、模型指标、概念演示或单一实现行为不能单独冻结 ABI。
