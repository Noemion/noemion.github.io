开放问题

记录 Endem 体系中尚未通过规范向量、威胁模型、验证方案或正式权利检查回答的问题。

Names FrozenABI UnfrozenEvidence Required

哪些问题已经关闭

ADR-0010 已经采用现行制品、组件、六个语义面、三个制品状态和八个动作。这些名称与职责不再作为开放问题反复讨论。

END-CORE 与 END-FMT 已建立 Endem 内容和实验性容器设计。ADR-0015 至 ADR-0020 固定结果域、时间、否定、量化、测量和复合判断；ADR-0021 至 ADR-0030 固定 Synem、Tekmor、内容标准分层、Dromen、结构化诊断、协议适配、身份、文本、授权以及内容与外部决定的伴随边界。

当前没有 Theor、权威目录、政策求值器或其他组件实现。稳定 ABI、时间与组合物理字段、嵌套或多变量量化、Synem 与 Tekmor 物理容器、正式来源语言、Unicode 标识符、原始来源字节绑定、摘要和签名物理 Profile、授权决定与最终决定事件编码仍未冻结。

以下问题只讨论编码、互操作、性能、安全与正式发布条件，不能借实现便利重新引入旧对象或 23 个命令家族。

Endem 编码与语义

实验来源清单怎样升级为正式语言？

ADR-0014 只关闭首个 .ends 输入。正式语言仍需解决注释、包含、版本演进、歧义、错误恢复和工具支持；在正反语料与真实消费者出现前，不扩展当前清单。

END-P1 之后还缺哪些字段？

ADR-0023 已关闭内容标准分层，但没有预先冻结未来内容。END-P1 已关闭未来首条实现路径评审所需字段，并已有正负极性字节；时间、缺席、单变量量化、测量、组合、条件适用性和统计登记仍需真实消费者、新 Profile 与正反字节向量。ID-CORE 已固定摘要与签名的抽象边界，但它们是否进入容器、采用何种物理 Profile 仍未决定。

条件与排他组合怎样表达？

ADR-0020 只允许单个 Endem 使用 all_of / any_of；ADR-0021 已把跨 Endem 条件固定为 Synem 会话激活。xor / xone、形成时条件编码和复杂守卫语言仍未冻结，不得嵌入自由脚本。

一个根 skena 的语义等价怎样判断？

布局、来源注释或调试记录变化后，哪些 semion、skena、telis、krin 和 apor 属性必须保持，仍需规范化规则和反例。规则冻结前只能比较显式符号、关系和条件，不能发明语义哈希。

Unicode 标识符与原始来源字节何时进入 Profile？

TXT-CORE 已固定当前 ASCII 标识符和来源文本不做隐式规范化，也明确 END-SRCM 不能证明原始文件逐字节保真。只有国际化标识符或原始字节审计出现不可替代消费者，并能固定 Unicode 版本、比较、双向、同形、身份、变换和正反字节向量时，才建立新 Profile。

权威目录与政策语言怎样获得物理 Profile？

AUT-CORE 已固定主体、范围、委托、同意、多人冲突、撤销、重放与结果分离。ADR-0030 又规定 END-P1 的权威字段只是选择器，不能自证授权。权威目录、角色与政策语言、授权伴随关系、事件编码、同意 UI Profile 和外部协议映射仍需真实消费者、独立威胁分析、规范字节与实现阶段授权。

连续或模糊要求怎样进入六个语义面？

ADR-0019 已关闭可量化构念、总体、单位、聚合器、不确定度和阈值的抽象边界。软偏好、感知质量、风格、模型裁判校准和不可完全判定条件仍需保留评价环境，不能为了机器处理伪造精确阈值。

Synem 与发布

绑定标识和版本约束采用什么语法？

SYN-CORE 已要求最终绑定唯一精确内容身份，ID-CORE 又固定完整引用的必需语境，并禁止环境搜索和网络最新版本。TXT-CORE 进一步规定当前结构标识符保持 ASCII、大小写敏感且与 locale 无关。可读名称、版本范围的候选语法、摘要算法基线、国际化标识符和绑定记录物理格式仍需新 Profile。

什么时候允许依赖循环？

SYN-GRF-001 已规定第一阶段拒绝依赖与激活循环。未来若支持，必须先定义固定点语义、披露与验收顺序、预算上限、失败原子性和跨实现向量。

Synem 是否需要独立物理容器？

在出现多 Endem、流式读取或离线分发的真实消费者前，内容寻址清单和现有确定性归档可能已经足够。不能只为形式完整提前创造格式。

tasse 可以删除哪些信息？

ID-REL-001 已要求裁剪结果与调试伴随文件各有精确身份，并记录变换、保留内容和损失。哪些内容可以删除、怎样证明语义等价、伴随关系采用什么物理格式，仍需真实消费者与正反字节向量。

sphra 的算法与撤销策略是什么？

ID-CORE 已固定有类型待签陈述、外置包络、权威分离、截止点和撤销解释。摘要与签名算法基线、多签政策、密钥体系、证书 Profile、时间来源、透明日志、撤销分发和离线包络物理编码仍需独立 ADR 与证据。

Dromen、Praxor 与 Tekmor

Dromen 的运行 API 和平台隔离怎样实现？

DRO-CORE 已固定主体、政策、环境、能力、秘密、预算、激活、观察、只读与销毁边界。未来仍需在代码阶段选择进程模型、沙箱组合、凭据代理、事件编码、预算计数器与销毁机制，并用平台级对抗证据证明，而不是把 Linux、容器或模型 SDK 名称当作隔离结论。

模型上下文怎样装配而不让数据取得指令权？

TXT、AUT、ADP 与 DRO 已分别约束模型实际输入、授权、外部来源和会话上限，但尚未统一输入选择、角色、顺序、截断、缓存和损失。当前非规范研究提案主张由控制平面形成可审计装配说明，并把未来条款归还现有规范；它不创建新制品、命令、组件或哲学专名。

Praxor 怎样隔离模型和能力适配器？

需确定进程、沙箱、取消、超时、幂等、网络、文件、凭据和资源限额。后端失败不能让 Praxor 静默扩大权限或修改 attested 制品。

Tekmor 怎样获得物理编码？

TEK-CORE 已固定 phain 对齐、有限无环溯源、证据类别、有效性、覆盖、决定分离和最小披露。字段编号、容器、摘要与签名算法、透明日志、撤销分发、跨生产者时钟归并、隐私策略和规范字节仍需真实消费者与新 Profile。

mene 时间怎样进入未来 Profile？

ADR-0016 已决定 fixed/elapsed、strict/budgeted 与覆盖结果；字段编号、数值宽度、亚秒精度、闰秒、循环日历、时区数据库封装、跨重启关联和多生产者归并仍保持开放。

授权与最终决定事件怎样编码与撤销？

AUT-CORE 已定义 grant/deny/defer 的授权决定内容、重放、时效和多人规则；ADR-0015 与 TEK-DEC-001 则规定 accepted/rejected/deferred 必须保存原满足结果与具名权威。两类事件仍保持正交，其字段、物理编码、撤销传播和外部协议映射仍需独立 Profile 与规范字节。

MCP 与 A2A 的具体 Profile 怎样冻结？

ADP-CORE 已固定版本、对端、能力、调用、状态、产物、取消、重试、交付和安全不变量。具体字段映射仍需等待精确稳定版本、真实消费者、协议专属威胁、正反协议向量和实现阶段授权；不得用通用 SDK 对象替代 Profile。

结构化诊断怎样获得物理编码与协议映射？

DIA-CORE 已固定机器身份、生产语境、层次、位置、主错误、恢复和安全边界。JSON、CBOR、SARIF、Problem Details、MCP、CLI 退出状态、日志与遥测字段仍需真实消费者、版本规则、披露政策、规范字节和跨实现证据；任何映射都不能改写本地结果域或授权。

实施与发布条件

独立 Theor 使用什么实现边界？

未来 Theor 必须拥有自己的数据结构、解码器、验证器和错误路径，只与 Poiet 共享公开规范和字节向量。当前没有实现；是否采用 Rust 或第二种语言，必须在用户开启代码阶段后根据共同故障风险和维护成本重新确认。

独立 Theor 还缺什么证据？

当前只有规范向量、资料检查器和未来验证条件。用户开启代码阶段后，才建立依赖审计、差分测试、长时 fuzz、故障注入和跨平台运行，用来寻找两条独立路径的共同输入假设。

名称何时满足正式发布条件？

项目内部命名已经冻结，但公开发行前仍需检查目标法域商标、近似类别、包管理器、命令和域名，并完成法律复核。审查完成前不得声称商标可用。

怎样证明构建可复现？

ID-REP-001 已要求精确输入闭包、工具与参数、环境约束、至少两条独立产出路径和逐字节输出比较。仍需在代码阶段建立真实隔离构建、缓存独立性、差异归因和发布证据；build ID、签名和一次重跑都不能替代这些证据。

问题关闭规则

设计问题可以由权威规范或 ADR、正反向量和明确失败语义关闭。涉及组件行为、性能或互操作的问题必须保持开放，直到用户明确开启代码阶段并出现相应实现证据；论文、模型指标、概念演示或单一实现行为不能单独冻结 ABI。
