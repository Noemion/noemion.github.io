开放问题

记录 Endem 体系中尚未通过规范向量、威胁模型、实现原型或正式权利检查回答的问题。

Names FrozenABI UnfrozenEvidence Required

哪些问题已经关闭

Endem、Weave、Frame、Witness、open → bound → sealed、唯一 CLI 与八个子命令由 ADR-0008 采用；一个根 case 与 say/mean/case/when/open 由 ADR-0009 采用。它们不再作为候选名称或工具数量问题反复讨论。

以下问题只讨论编码、互操作、性能、安全与正式发布条件，不能借实现便利重新引入旧对象或 23 个命令家族。

Endem 编码与语义

Source Card 的最小表面语法是什么？

需要比较严格声明式文本、带自然语言来源的结构化文档和 API 输入。任何方案都必须明确表达五组投影语义、来源范围、关系位置和未知状态，并能在无模型环境确定性解析。

Endem 的物理编码怎样支持安全随机访问？

需确定魔数、版本、目录、字段宽度、记录长度、对齐、关键/可选扩展、压缩和资源上限。首要目标是有界读取和跨实现一致，不是兼容 ELF 或某种通用容器。

一个根 case 的语义等价怎样判断？

布局、来源注释或调试记录变化后，哪些 mean、case、when 和 open 属性必须保持，仍需规范化规则和反例。规则冻结前只能比较显式符号、关系和条件，不能发明语义哈希。

自然语言候选由谁授权绑定？

需定义决定主体、作用范围、撤销、重放、高风险字段和多人冲突。模型置信度、结构合法和单次界面点击都不能单独成为永久授权。

连续或模糊要求怎样进入五组投影语义？

软偏好、感知质量、风格和不可完全判定条件需要保留不确定性和评价环境，不能为了机器处理伪造精确阈值。

Weave 与发布

绑定标识和版本约束采用什么语法？

需要同时满足内容寻址、可读命名、版本兼容和可重放解析；环境搜索顺序和网络最新版本不能成为正式输入。

什么时候允许依赖循环？

第一版拒绝必需循环。未来若支持，必须先定义固定点语义、披露与验收顺序、预算上限、失败原子性和跨实现向量。

Weave 是否需要独立物理容器？

在真实多 Endem、流式读取或离线分发消费者出现前，内容寻址 manifest 和现有确定性归档可能已经足够；不能为形式完整提前创造格式。

pack 可以删除哪些信息？

只有不改变 mean、case、when、open、绑定、能力上限和运行披露的内容才能外置。需要先定义关系级等价检查和 Debug Companion 摘要绑定。

seal 的算法与撤销策略是什么？

需确定待签消息、算法协商、多签、密钥轮换、撤销、时间来源和离线验证。签名必须位于被签载荷之外，且不能替代结构或环境授权。

Frame、Runner 与 Witness

Frame 的最小只读接口是什么？

需要定义 mean 关系、case 子图、when、open、成员披露、预算和能力需求怎样进入一次运行，同时确保 Frame 不含实时句柄、凭据或可跨会话权限。

Runner 怎样隔离模型和能力适配器？

需确定进程、沙箱、取消、超时、幂等、网络、文件、凭据和资源限额。后端失败不能让 Runner 静默扩大权限或修改 sealed 制品。

Witness 采用什么事件与完整性结构？

需确定序列、时间域、父记录、摘要、脱敏、丢失、采样、签名和多生产者归并，同时避免循环自证。

谁形成最终决定？

确定性规则、外部评价器与人工权威需要明确输入、权限和失败语义。Witness 齐全只说明依据可用，不能自动把状态提升为 accepted。

MCP 与 A2A 怎样映射到 Runner 外缘？

需冻结协议版本、工具/Agent 声明的信任等级、token audience、取消与重放语义。外部 Task 状态和 Artifact 不能直接成为 Endem 状态或 Decision Witness。

实施与发布条件

首版实现语言和仓库结构是什么？

需要用同一规范向量比较内存安全、checked arithmetic、模糊测试、跨平台、FFI、依赖和复现成本。第一阶段不引入 LLVM、MLIR、BFD 或模型框架。

独立 Reader 怎样证明实现独立？

需禁止共享生产解析库、生成代码和错误分类实现，并通过差分、变异和故障注入证明它能发现生产路径的共同假设。

名称何时通过正式发布 gate？

项目内部命名已经冻结，但公开发行前仍需目标法域商标、近似类别、包管理器、命令、域名和法律复核。未通过 gate 不得声称商标可用。

怎样证明构建可复现？

对象字节、工具二进制和签名真实性是三个不同属性。需要双路径构建、固定环境输入、逐字节比较和独立签名验证。

问题关闭规则

开放问题只有在权威规范或 ADR、正反测试向量、至少一个实现原型和明确失败语义同时存在时才关闭。论文、模型指标、概念演示或单一实现行为不能单独冻结 ABI。
