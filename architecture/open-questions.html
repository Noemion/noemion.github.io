开放问题

开放问题：对象与语义

对象与总体格式最终叫什么？

工程阶段暂用 GOBJ；GRO、SRO、ARO 以及 AELF 等均只是候选名称，不是已确认规范。

GSIR 最小核心类型集是什么？

需要用真实语料和实现原型确定最小集合；“约 20～30 种”只是规划估计。

连续语义残差如何标准化？

Encoder 指纹、维度、码本、量化、相似度与跨模型稳定性仍需证据和互操作测试。

评价器如何成为可链接对象？

确定性、模型与外部评价器的接口、权限和失败边界尚待设计。

四层语义身份是否足够且正交？

Source Expression、Compiled Meaning、Reference Binding 与 Satisfaction Contract 是研究提案；需要以同义改写、跨版本绑定、语境推论和复合验收反例检验是否遗漏或重复。

语义来源和满足环境如何进入对象？

明示内容、结构蕴涵、语境推论、工具推导与模型候选的确认权限尚未定型；评价环境的证据、能力和背景假设也须决定是对象数据、可链接依赖还是 Runtime Profile。

开放问题：语言、ABI 与工具面

GSL 采用何种表面语法？

严格声明式 DSL 是当前优先方向，自然语言化语法糖仍是后续候选。

披露与 Runtime ABI 如何定型？

状态哈希、预算单位、继续句柄、检查点、Prompt Renderer 槽位和消费者确认协议均未冻结。

CLI 与重定位标识是什么？

尚未规范化的命令示例和 R_GSIR_* 名称只表示候选职责，不构成稳定 CLI 或 ABI。冻结前必须确定命名空间、版本策略、未知类型处理和一致性测试向量。

行为请求需要哪些最小字段？

询问、承诺、授权与执行是否属于核心行为类型，以及行动者、权限、前置条件、效果和撤销边界怎样跨 Compiler、Linker 与 Runtime 保持一致，仍需威胁模型和 ADR。

开放问题：实现与模型

首版实现语言是什么？

Rust 与 C++ 各有工程优势，尚未形成 ADR；第一阶段也不引入 LLVM/MLIR。

NSFE 选用哪个语言底座和训练方案？

候选底座、蒸馏范围、量化方法、阈值与数值门槛必须经过来源隔离评测，并以质量、校准、资源和许可证据决定，不能由未经验证的目标值直接决定。

覆盖证书采用哪些自动门槛？

需通过真实 Skill/Prompt 语料确定 Source Unit 分类、等价性测试与人工确认风险分级。

未来阶段：设备升级条件

当前 M2 Pro、16GB 设备可承担规范、安全 Reader/Writer、Strict 前端、链接器、验证器、Fuzz 与最小 Runtime 的开发。只有 NSFE 训练、多 VM 并行或大型 Debug 构建成为实际瓶颈时才触发升级评估；优先考虑统一内存容量。

决策方法与关闭条件

开放问题不能仅靠偏好或命名讨论关闭。每项问题应记录负责人、影响的规范和工具、候选方案、现有技术或先例、评价指标、原型或实验、风险、知识产权与互操作影响，以及最终 ADR。

优先级判定标准最迟关闭阶段
阻断型影响安全解析、确定性、对象兼容或第一阶段范围进入相关开发阶段前
架构型影响多个组件、规范层次或长期演进计划评审或首个纵向切片前
接口型影响 CLI、ABI、Profile 或互操作对外候选发布前
优化型只影响性能、体验或后续模型能力基线与瓶颈证据形成后
