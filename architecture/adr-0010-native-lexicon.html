ADR-0010：原生词汇与事态分层

现行决定：用六个短词分开来源、意义、事态、目标方向、判断和未决信息。

Accepted2026-07-12Native LexiconABI Unfrozen

直接结论

Endem 保留；原有五组英文短词、通用制品名和编译器式组件名被一次性取代。新语义核采用 rhem / semion / skena / telis / krin / apor，并把中性的可能事态与目标方向彻底分开。

结构底线：一个 Endem 只有一个根 skena。semion、skena 与观察中的 phain 使用可比较的符号和关系位置；形式由结构显示，制品不能用自填标签证明自己有效。

六个语义面

词词根工程职责
rhemrhēma · 说出的内容保存来源表达、主体、媒介、范围、版本和可重定位位置。
semionsēmeion · 记号保存记号到符号、指称、关系、角色和作用域的已授权投影。
skenaskēnē · 场景保存一个根、中性且可比较的可能事态图。
telistelos · 趋向的终点保存目标对事态的变化或保持方向，不改写事态内容。
krinkrinein · 区分与判断规定观察、结构比较、证据强度、未知处理和决定权威。
aporaporia · 疑难保存仍可表达但尚未获授权决定的有限缺口。

事态、目标和观察不再混合

rhem · 来源
semion · 投影
skena · 可能事态
telis · 目标方向
krin · 比较契约
phain · 结构化观察
Tekmor · 有范围证据
  • skena 只描述对象怎样关联，不携带“希望、保持、禁止”等力量。
  • telis 第一阶段只区分 kine（使事态达到成立）和 mene（使事态持续成立）。
  • 禁止事项由 skena 的显式否定表达，不再与独立 avoid 值重复编码。
  • phain 是 Tekmor 内的结构化观察；无法与 skena 比较的自由文本或模型评分不能独自支持满足判断。

制品、组件与动作

类别现行名称直白含义
系统名词Endem / Synem / Dromen / Tekmor最小目标制品、组合闭包、非文件会话实现态、有范围证据。
组件Poiet / Theor / Praxor确定性制作、独立观察、隔离实行。
动作poie / elenk / pleko / tasse / sphra / theor / praxe / peira依次负责形成、检查、组合、封装、签名核对、只读观察、受控实现和一致性试验。
生命周期nascent / coherent / attested结构已形成、依赖已收敛、发布载荷已外部见证。

每个专名旁都必须保留直白工程副标题。词源只提供记忆路径，真正含义仍由数据结构、不变量、权限和测试决定。

名称审计

本次对照 C/C++、Rust、Go、Python、Java、ECMAScript、Swift 与 Kotlin 的关键字集合。项目也查询了 PyPI、npm 与 crates.io 的精确包名。

现行领域词没有关键字冲突，查询时也没有形成跨三大注册表的强占用。该结果只降低工程混淆，不构成商标或未来包名可用性保证。
