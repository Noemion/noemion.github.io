Endem 生命周期

跟随一项目标从来源表达进入 Endem，再经过组合、发布、受控实现和证据验收。

nascentcoherentattestedDromenIknem

生命周期不是信任等级流水线

生命周期不是一条“越往后越可信”的流水线。每个阶段只回答一种问题，任何阶段都不能替代其他阶段。

  • ktise 说明目标契约已经确定性编码。
  • pleko 说明引用和冲突已经解析。
  • 独立外部签名集成说明发布载荷与签名响应一致；它不是公开动作。
  • drase 说明当前环境已经建立受控会话。
  • Iknem 只说明判断依据可定位，并限定这项依据的适用范围。

关键边界：Endem 在 nascent、coherent、attested 三个状态之间演进。apor 是语义面，不是生命周期状态；Dromen 是一次 Drase 会话的密封只读执行契约，Iknem 是有限证据记录。

nascent → coherent

  1. elenk 验证 nascent Endem 的结构和语义，不因存在 apor 项就称对象损坏。
  2. pleko 只从显式依赖锁定、授权决定和绑定策略取得候选。
  3. 每个 apor 项必须变成已解析、获授权决定或失败；观察不足只能在运行时产生 agno，不能用 agno 保留语义歧义。
  4. 跨 Endem 引用形成 Synem；没有跨对象消费者时保持单一 coherent Endem。
  5. skena 的极性、telis 的 kine/mene 边界和权限只能保持或收紧，krin 不能因组合变得更容易满足。

coherent → attested

  1. 对应制品生产者形成确定性物理封装；若产生派生物，必须保存新身份、变换、保留内容与损失。
  2. 待签载荷锁定后生成 Signing Request；私钥始终留在外部签名系统。
  3. 外部签名集成同时读取原候选、原请求与签名响应，先验证绑定，再附加签名包络。
  4. 签名封装不得改写被签载荷；任何载荷变化都要求新身份和新签名。
  5. attested 只说明发布身份和字节范围，不证明当前环境可运行。

attested → Dromen → Iknem

  1. Drasor 对实际输入字节重新执行所需 elenk，不复用过期结论。
  2. Drasor 按 ADR-0024 重新验证精确对象，并把政策、环境、能力、预算和证据责任求交后封存 Dromen；实时句柄只保留在独立最小能力域。
  3. 后端读取 Dromen 的非秘密只读描述，返回候选或类型化能力请求，不能直接取得外部能力。
  4. Drasor 验证请求、执行或拒绝，再把真实观察、规范化变换与 phain 分开记录为符合 IKN-CORE 抽象边界的 Drase 证据。
  5. 若 telis 为 mene，Drasor 还必须按 ADR-0016 绑定 fixed 或 elapsed 范围，记录时钟域、覆盖区间、缺口和违约预算。
  6. 求值器按照执行前固定的 krin 比较 skena 与 phain，形成 met / unmet / agno / fault 中的一个满足结果。
  7. 验证者先在精确政策、参考值、环境与截止点下评估 Iknem 有效性，再相对当前 krin 计算覆盖度；具名权威最后形成 accepted / rejected / deferred 中的一个决定。会话另行记录 completed / failed / interrupted。
  8. 会话结束、失败、中断或实质漂移后，Dromen 与绑定实时能力都不可再用；允许保留的运行记录只进入有范围的 Iknem 或会话事件，不能恢复权限。

稳定状态与失败状态

类别允许状态含义
Endemnascent / coherent / attested语义与发布生命周期状态。
满足判断met / unmet / agno / faultskena 相对 phain、krin 与求值规则的工程满足结果。

Drase 会话completed / failed / interrupted只描述一次运行怎样终止，不说明目标是否满足或已被接受。
Elenkpass / fail / not-evaluated每个请求验证层的独立结论；不能压成含义不明的成功布尔值。
Iknem 有效性valid / invalid / revoked验证者针对精确政策、参考值、环境和截止点形成的外部评估；不等同于 claim 为真。
证据覆盖度sufficient / insufficient一组有效 Iknem 是否覆盖当前 krin；不与单份记录有效性合并。

这五个结果域由 ADR-0015 分开。外部 Task 的 completed、工具返回成功或会话正常结束都不能跨域改写满足结果与最终决定。

mene 的时间范围、连续性政策与覆盖分类由 ADR-0016 定义。当前没有 mene 线格式、时钟适配器或持续求值实现。

Iknem 的主体、范围、溯源、观察类别、有效性、覆盖度、决定和最小披露由 ADR-0022 与 IKN-CORE 定义。当前没有物理 Iknem 格式、验证器、归并器、撤销服务或决定引擎。

未来实现顺序

当前没有组件实现。第一条切片限制为“受控来源表达 → 确定性投影 → ktise → nascent Endem → elenk/theor → 差分与复现验证”。最小格式、两条读取路径、恶意语料和字节复现稳定后，项目才评估组合、发布与受控运行。
