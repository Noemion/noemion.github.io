项目/架构设计/架构决策/ADR-0036

ADR-0036
来源保留与裁剪发布边界

形成版保留原始自然语言，最终发布版移除原文；两者是身份、验证和披露边界不同的制品。

当前策略2026-07-15形成版含来源发布版不含原文

直接结论

Endem 的形成版保留输入的原始自然语言和可重定位范围，供形成过程、人工复核与受控诊断使用。面向运行和分发的最终发布版经过来源裁剪，不携带原文，也不携带可无损重建原文的等价材料。

这与本地程序先产生包含调试信息的完整制品、再生成不含调试信息的发布制品具有相似的工程分层，但 Noemion 不继承 ELF、DWARF 或 GNU 工具的字段和链接语义。类比只用于说明“同一形成过程产生用途不同的独立制品”。

当前状态：END-P1 是来源保留的形成与评审 Profile，不是最终发布 Profile。发布 Profile 的物理字段、编码和来源伴随关联仍在研究；当前没有组件，也没有可执行的裁剪命令。

三个用途边界

对象包含什么用于什么不能获得什么
来源保留的形成制品原始自然语言、来源范围、已确认语义结构与未决投影确定性形成、人工复核、受控诊断不能因含有原文而自动获得发布许可或运行权限
来源裁剪的发布制品发布 Profile 允许的已确认目标结构、判据和授权边界分发、加载、验证与受限运行不能仅凭自身证明“发布结构忠实于原始表达”
受控来源伴随资料按政策保留的形成制品或来源材料，以及与发布制品的类型化关系审计、争议复核和高权限来源一致性检查不是公开包的必需部分，也不是运行能力、接受决定或信任凭据

从 GNU 工具链采用的工程原则

权威资料采用的原则不照搬的部分
GNU GDB：Separate Debug Files完整制品、裁剪制品和伴随资料可以分开保存，并通过明确关系重新关联。.gnu_debuglink、CRC、build ID 和文件搜索路径不自动成为 Endem 格式。
GNU strip删除调试信息是有损变换；删除范围必须明确，错误裁剪可能使制品不可用。不把“删除某些 section”直接映射为 Endem 字段删除规则。
GNU objcopy先保留完整资料、再裁剪发布对象、最后建立显式伴随关系，是可检查的生产顺序。不据此预留公开 strip 动作；未来由对应制品生产者按 Profile 承担。

规范决定

Profile 分离

END-P1 保留原文，只服务形成和评审。最终发布必须使用另一封闭 Profile，明确保留字段、删除或降级字段、资源上限与失败条件。

引用闭包

发布版不得简单删除 rhem.content 后保留悬空 source_ref。新 Profile 必须定义不泄露原文的来源绑定，或重写、移除依赖来源区间的引用，并证明引用闭包。

新身份与新验证

形成版、发布版和伴随资料分别拥有精确字节身份。发布版不得继承形成版的签名、Iknem、接受决定或能力；面向发布字节的声明必须重新生成和验证。

有限证明

没有受控伴随资料时，验证者只能证明发布字节符合发布 Profile 以及外部发布声明有效，不能独立证明它忠实投影了某段原始自然语言。

安全威胁与停止条件

威胁必须阻止的退化
来源泄露发布对象、公开清单或错误信息携带原文、可逆编码、敏感范围或足以恢复原文的材料。
引用失效裁剪后仍保留指向不存在来源内容的引用，读取器却把它当作已闭合结构。
身份漂白裁剪结果复用形成版身份、签名、证据、接受状态或权限。
过度证明只验证发布对象就宣称已验证原始表达与发布语义之间的忠实性。
摘要猜测把低熵原始文本的裸摘要公开为“安全来源链接”，使攻击者可以离线枚举原文。
伴随资料越权运行路径自动读取受控来源伴随资料，或把能够读取它解释为执行权限。

在发布 Profile、引用重写规则、伴随关系、披露政策、正反向量和真实消费者锁定前，不得开发裁剪实现、稳定 ABI 或公开命令。删除原文这一用途边界已经确定；如何编码仍保持开放。

开发者应怎样理解“可验证”

形成阶段可以直接对照原始表达检查语义投影；发布阶段检查的是发布字节、发布 Profile、授权边界与外部声明。若审计任务需要回答“发布结构是否忠实于原始表达”，验证者必须在获准后取得受控伴随资料。三种检查不能合并成一个模糊的“来源可验证”。
