项目/架构设计/架构决策/ADR-0036

ADR-0036
移除原文
就得到另一份制品

形成版保留自然语言和确认路径；最终发布版移除原文，只保留经过确认的目标结构，并取得新的身份和验证关系。

当前策略2026-07-16形成版含来源发布 Profile 待定

先把三种对象分开

形成版保留原始自然语言；最终发布版移除原文；受控伴随资料在需要审计时另行提供。三者不是同一文件的三种显示方式。

对象保存什么主要使用者不能据此获得
来源保留的形成制品原始自然语言、来源范围、已确认结构、确认记录和显式未决项形成责任、语义复核者和受控诊断流程发布许可、运行权限或最终接受
来源裁剪的发布制品发布 Profile 允许的已确认目标结构、判据和必要非秘密选择器分发、加载、独立读取和有界运行责任原文内容、语义忠实性证明或永久授权
受控来源伴随资料形成制品、原始来源、复核记录，以及与发布制品的有类型关系获准的审计、争议复核和高权限验证者默认运行输入、能力凭据或公开包身份

当前状态：END-P1 只是来源保留的形成与评审 Profile。项目尚未定义最终发布 Profile，也没有可执行的裁剪命令。

一次发布怎样从形成版得到新制品

以“更新服务依赖，并确认当前版本可以发布”为例。形成版保存原句、服务范围、目标版本、允许变化、发布判据和仍未确认的问题。发布版只保留最终确认的目标结构。

删除、重写、保留和外置不是一回事

处理当前确定的职责发布 Profile 仍要定义什么
删除移除原始自然语言、精确来源区间、可逆重建材料和不应公开的候选内容逐字段删除清单，以及如何检查文本、日志和诊断残留
重写所有依赖来源位置的 source_ref 必须改成发布版可闭合的关系，或从发布对象移除新引用类型、排序、规范化、未知引用处理和失败码
保留只保留已经确认且运行消费者确实需要的目标结构、判据和安全约束物理字段、枚举、资源上限和独立读取视图
外置形成制品、来源、复核记录、派生记录和签名包络保持独立身份伴随关系、存储与访问政策、保留期限和删除责任

“移除原文”不是对 END-P1 直接删除 rhem.content。若其他字段仍依赖来源区间，生产者必须重写整个引用闭包。关键未决意义也不能被静默删掉。

新字节为什么必须取得新身份

关系必须记录禁止继承
形成版身份身份域、Profile、精确字节范围、算法和完整摘要不能把路径、文件名或短摘要当作发布对象身份
发布版身份新的身份域、发布 Profile 和实际发布字节不能复用形成版身份，即使目标含义保持不变
派生关系
ID-REL-001		两个完整身份、变换版本、生产者、保留内容、损失和验证条件不能用“derived-from”文字标签替代可检查关系
发布约束
END-PUB-001		原文与可逆材料已移除，引用闭合，发布结构仍符合 END-CORE不能因裁剪成功就产生目标满足或动作授权
外部陈述面向发布版新身份重新生成的签名、来源或发布声明形成版签名、Iknem、接受决定、能力和授权状态

新身份与新验证是发布派生的底线。任何字节、Profile 或身份域变化都必须关闭旧绑定，再为新对象建立适用关系。

获准的派生验证者形成版、发布版、精确变换规则和派生记录发布版由指定形成版按指定规则确定性产生语义确认者有权授权动作，或目标已经满足
语义复核者原始来源、形成版投影、未决项和确认记录结构是否忠实反映给定来源与具名决定发布时的授权仍然有效，或运行结果必然正确
运行加载者发布版、当前外部陈述、政策、环境、能力与预算本次会话是否可以加载及调用有限能力自动读取受控伴随资料，或把读取权变成执行权

没有受控伴随资料时，接收者仍可验证发布对象本身，却不能验证原始表达到发布结构的忠实转换。页面因此不再用一个模糊的“来源可验证”概括四种任务。

移除原文降低暴露面，但不产生安全许可

自然语言来源可能包含秘密、个人信息、错误解释或恶意指令。运行时默认读取这些内容，会扩大披露与间接提示注入面。发布版因此不把原文作为模型上下文或运行输入。

NIST AI 600-1说明直接与间接提示注入可能使生成式人工智能系统产生非预期行为，并影响相连系统。这个风险支持把不可信原文留在受控形成与复核路径，但不证明裁剪后的结构天然安全。

安全边界：裁剪只能移除一类输入和披露风险。发布结构仍须检查内容身份、当前授权、能力范围和使用环境；人工智能系统不能因原文已删除就执行未经授权的越权行为。

GNU 与供应链资料提供哪些工程先例

权威资料Noemion 采用的纪律不照搬什么
GNU GDB：Separate Debug Files完整对象、裁剪对象和伴随资料可以分开保存，并用明确关系重新定位.gnu_debuglink、CRC、build ID、搜索路径和自动下载机制
GNU strip删除信息是有损变换；选错删除范围可能使输出不可用原地改写文件的默认行为，以及 section 和符号删除规则
GNU objcopy明确指定输入、输出、保留内容和变换选项，并把完整资料另行保存BFD 对象模型、ELF/DWARF 字段和公开 strip 动作
SLSA 1.2 Provenance来源作为外部可验证信息，说明制品在何处、何时以及怎样产生SLSA Provenance 不定义自然语言语义、Endem Profile、授权或目标满足

GNU 的类比接近“带调试资料的形成产物与裁剪后的发行产物”，但原始自然语言不是 DWARF 调试信息。Noemion 必须自行定义语义保留、来源泄露和派生身份。

哪些失败必须停止发布

失败必须阻止的退化
来源泄露发布字节、清单、诊断或日志保留原文、可逆编码、敏感范围或模型候选
引用失效删除来源后留下悬空引用，读取器却报告结构已闭合
身份漂白发布版复用形成版身份、签名、证据、接受状态或权限
摘要猜测公开低熵原文的裸摘要，使接收者可以离线枚举来源内容
未决项消失为得到“干净”发布版而删除影响含义或安全的 apor
伴随资料越权加载器自动寻找、下载或向模型暴露受控来源材料
非确定或部分成功同一输入得到不同发布字节，或失败后留下可被误用的部分输出
过度证明只验证发布字节就宣称已经核对原意、授权、满足或最终接受

当前状态与继续阅读

已有成果：形成版、发布版、受控伴随资料、新身份、派生关系和四类验证任务已经分开。

限制条件：当前没有发布 Profile、字段映射、保留/删除清单、伴随资料格式、裁剪实现或稳定 ABI。资料一致不证明原文已经安全移除，也不证明发布组件存在。
