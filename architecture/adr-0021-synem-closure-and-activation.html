项目/架构设计/架构决策/ADR-0021

ADR-0021
组合闭包
条件激活

固定 Synem 的完整闭包、精确绑定、权限收敛、成员结果与会话激活边界，让多目标关系可以重放而不依赖环境和模型猜测。

Accepted DesignSYN-CORE 0.1.0-draftNo Wire FormatNo Component Code

直接结论

Synem 是两个或更多 Endem 的完整、有限、精确解析闭包。它不是文件包、搜索结果、模型上下文或运行时发现的工具集合。

每个必需引用必须绑定唯一内容身份；权限只能取交集或继续收紧；成员判断与决定保持独立。条件激活属于 Synem 的会话关系，不进入单个 Endem 的 all_of / any_of 根。

本决定建立 SYN-CORE 0.1.0-draft，但不创建 Synem 文件扩展名、魔数、物理字段、版本范围语法或实现。本次工作不表示 Poiet、Theor、Drasor、CLI、解析器或运行时已经实现。

哲学启发与采用限制

“每一个关于复合物的陈述可以分解为关于其各组成部分的陈述”

路德维希·维特根斯坦《逻辑哲学论》2.0201

命题 2.0201 和 5.5423 都把复合物的可理解性放在组成部分及其关联方式上。Synem 因此必须同时显示成员和关系；只列名字或文件，不能完整说明组合对象。

这里采用的是设计问题，不是把哲学命题直接变成软件定理。成员身份、无环限制、权限交集、激活状态和失败责任仍由威胁模型、规范条款与正反向量定义。

Synem 与 Endem 复合根的边界

问题Endem 复合根Synem

闭包形成的六条规则

  1. SYN-CLS-001：至少两个不同 Endem，并包含全部必需传递依赖。
  2. SYN-RES-001：每个必需引用绑定恰好一个精确内容身份，且与输入顺序无关。
  3. SYN-GRF-001：第一阶段依赖图有限无环；可选项缺失不能削弱语义和判据。
  4. SYN-AUT-001：能力上限只由成员、组合策略和会话政策的交集形成。
  5. SYN-STA-001：成员结果保持成员身份，不自动产生一个总体满足或接受结果。
  6. SYN-ACT-001：会话条件只激活固定闭包成员，不改变身份或授权。

形成时选择与会话期激活必须分开

阶段作用身份影响失败处理
形成时选择决定某 Endem 是否进入成员闭包改变 Synem 闭包身份必须在形成身份前解析；不确定则拒绝形成
会话期激活决定固定闭包中的成员本次是否生效不改变制品身份记录独立激活状态；依据变化则重新检查或使会话失效

这一区分阻止“运行到一半再下载最新目标”改变正式闭包，也阻止把一次会话分支误写成新的制品事实。

激活不是第六种满足结果

激活结果域固定为 active / inactive / unresolved / error，四个值只回答成员在本次会话是否生效。

激活状态含义不得映射为
active固定守卫已匹配，本成员本次进入后续政策检查met 或能力授权
inactive守卫已确定不匹配，本次不求值该成员met / unmet / agno
unresolved守卫依据尚不可用满足结果 agno
error守卫检查契约失败满足结果 fault

独立名称不是为了增加词汇，而是为了阻止不同问题被同一个状态词洗平。激活状态必须携带自己的结果域，不能脱离上下文只传一个裸字符串。

GNU 与开放规范提供的对照

  • GNU ld 文件命令允许当前目录、-L 和 SEARCH_DIR 参与搜索，且归档位置会影响解析。Synem 反向采用这一经验：正式绑定必须记录精确身份，不能把环境搜索结果当闭包。
  • GNU Guix 参考手册把引用图作为显式构建输入，并可用 allowed-references 拒绝输出引用清单外对象。Synem 借鉴可审计的传递闭包和允许引用边界，不继承 Guix Store 路径或包管理模型。
  • GNU make 条件指令决定 make 在读取时看到哪些文本，不能控制执行时 recipe。Synem 同样分开形成时成员选择和会话期激活。
  • W3C SHACL规定停用的 Shape 产生空验证结果。Synem 采用“未激活成员不产生满足判断”，但不把空结果解释为符合或满足。
  • MCP 2025-11-25 工具规范允许服务器通知工具列表变化。这说明动态能力发现是现实需要，但它必须留在控制平面，不能改变 Synem 闭包或直接授予权限。

规范影响与验证

  • 新增 SYN-CORE 0.1.0-draft 六条规范条款。
  • 新增六项 Synem 威胁和十个非规范自然语言场景。
  • 十二个提案向量覆盖六个允许分类与六个确定拒绝；检查器只证明资料矩阵一致。
  • END-CORE、END-FMT、END-P1 与 .endem 字节保持不变。

仍未冻结的内容

物理 Synem 容器、文件扩展名、版本约束语法、符号与重定位、执行顺序、并行调度、受限循环、远程对象仓库和规范字节仍保持开放。没有真实消费者和新 Profile 前，不为了形式对称创建格式。
