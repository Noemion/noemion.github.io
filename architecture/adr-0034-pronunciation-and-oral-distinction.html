项目/架构设计/架构决策/ADR-0034

ADR-0034
读得出，不等于
听得清、写得回

发行名称必须让目标读者恢复正确拼写和工程职责，并能同完整词表稳定区分；维护者自拟读音、一次顺利朗读或语音模型转写都不能代替真实人类证据。

当前审查发行读音待定人类证据必需不建立语音接口

先区分四项独立审查

一个名称可以容易朗读，却仍然无法写回、容易听成另一项，或根本无法说明职责。发行审查必须分别回答四个问题；任一失败都不能由其他结果补偿。

审查轴要回答的问题不足以通过的证据
职责透明首次接触者能否把名称匹配到正确对象、消费者、信任和失败责任词源优美、维护者熟悉或页面已有大量解释
书写与检索区分精确拼写、大小写和近似形式能否同相邻软件、标准、命令和权利稳定区分包入口暂时为空、搜索第一页未命中或只改大小写
目标语言读音稳定首次朗读是否形成可解释的断词、重音和允许变体维护者指定 IPA、TTS 读得顺或一次 ASR 转写正确
词表内口头区分听者能否从完整词表唯一选择、手写拼回并匹配正确职责孤立音节可辨、看到上下文后猜对或团队内部形成习惯

读音流畅性是传播条件，不是语义豁免。名称若暗示错误对象、错误权限或错误结果，即使悦耳也必须退出；直白职责名同样需要在真实句子和目标语言中验证。

读音待定时怎样协作

开发场景当前安全表达必须停止的情况
会议中讨论对象先说“有范围证据记录”，再在文字频道给出 Iknem听者不能立即写回，却继续依靠近似声音讨论字段或证据范围
讨论形成边界先说“确定性生产边界”，再展示 Ktisor 的精确拼写参与者对词首辅音是否发音没有一致理解
引用命令或规范从受控书面资料复制动作、参数、Profile 与规范 ID依靠会议录音、自动转写或个人记忆恢复机器值
代码评审与 Issue使用可点击路径、代码样式标识和直白职责，不只写专名临时读法被登记为别名、纠错规则或第二个机器标识符
跨语言协作说明本次使用的语言范围和仍未覆盖的口音、地区与环境把一种英语变体或普通话结果写成全球统一读音

口头交流用于定位书面标识，不取代书面标识。没有合格读音表时，项目不得发布“官方发音”、语音词典或名称已经适合全球传播的结论。

现行词表为什么仍未通过

优先复核项当前可见风险下一步
Ktisor / ktise词首 kt 可能被省略辅音或插入不同元音；组件与动作又需要口头区分与“确定性生产边界”及 form 候选共同做首次朗读、听写回填和职责匹配
Iknem词首分段和首个元音没有人类证据；“短”不表示“听得清”与“有范围证据记录”比较对象选择、拼写恢复和证据职责误解
kine / mene前者已有稳定英语读音和无关词义，二者又不能让首次读者恢复达到与保持的区别现行规范值暂时不变；优先验证 reach / maintain 等直白候选
Endem / Synem共同结尾显示词族，也可能在弱信号、口音和句中位置造成成对混淆在完整词表和远程会议条件中建立成对混淆矩阵
Noemion、Dromen、Theor、Drasor 与其余动作没有真实使用者记录，未发现明显难点也不能自动通过先判断是否需要专名，再逐项登记目标语言、自然读法、替代拼写和职责结果

风险等级只安排验证顺序，不是保留决定。若直白职责名在对象选择、记忆、听写和职责匹配上更好，项目应优先去专名化，而不是为既有拼写补造发音规则。

发行审查怎样运行

先证明专名有必要
未接触者暴露自然读法
形成候选读音与风险对
独立听者完成三项任务
按语言与条件分析
保留、改名或重新测试
阶段材料与任务能够形成的结论
桌面排除检查既有词义、工程冲突、职责恢复和明显近音只排除不值得招募测试的候选，不能让替代名称自动通过
发现阶段每种目标语言至少 24 名首次接触者看名称与直白职责，朗读、断词、标重音并指出近音形成自然读法、风险对和改名候选；24 人是发现预算，不是统计阈值
发行审查另一批独立听者在清晰录音和普通远程会议音频中完成名称选择、手写拼写和职责匹配每种语言、名称和条件至少 60 个首次判断；结果只适用于已登记范围
记录保存 BCP 47 标签、参与者范围、刺激、说话者、随机化、原始响应、分类和数据边界允许复核决定和未覆盖总体，不创建新的 Noemion 制品或接口

60 个判断中零次关键混淆，只能把真实关键混淆率的单侧 95% 上界约束在约 5%，不能证明错误率为零。具体招募、材料平衡、隐私、保留和删除方案由术语与读音验证指南规定；指南尚未被实际执行。

什么结果会停止发行

观察结果为什么不能继续处理
误选成另一个现行名称声音已经跨越对象、动作或信任职责当前候选失败；分析原因并优先重命名
匹配到另一个职责即使拼写正确，工程理解仍然错误当前候选失败；不能靠补充词源说明修复
选择“无法判断”传播证据不足，不能把不确定性算作正确停止发行并重新设计名称或材料
形成稳定替代拼写声音无法唯一恢复书面接口进入重新命名审查，不增加隐藏纠错或机器别名
只在某一语言或清晰录音中通过证据范围不覆盖另一目标语言或真实协作条件限定结论并补齐范围；不得外推全球适用性
新增主要受众、名称、职责或语音入口旧混淆矩阵不再覆盖完整词表和使用情境相关发行资格失效，使用全新参与者重新测试

零关键混淆是 Noemion 的发行风险政策，不是 ISO、W3C、ITU、NIST 或 GNU 规定的通用阈值。它针对的是听错对象或动作可能改变信任与权限边界；普通口音差异可以保留，只要仍唯一指向同一拼写和职责。

语音系统为什么仍需书面确认

外部机制它提供什么不能替代什么
ISO 704:2022 与 GNU Coding Standards：Names对象、概念、定义、指称和名称可读性的治理原则不能替项目选择名称、读音或发行阈值
ISO 9241-11:2018要求在具体用户、目标、资源和环境中理解可用性结果不能只看拼写推断口头可用性，也不规定唯一测试方法
RFC 5646 / BCP 47为口语、书面语或手语等交流语言标记适用范围语言标签不证明口音覆盖、同形安全或名称可听辨
W3C Pronunciation Lexicon Specification 1.0把书写形式映射到一个或多个读音，并支持 IPA 表示读音词典不能证明名称职责正确或人类混淆率合格
OpenAI Realtime API 转写指南音频转写产生增量与完成事件，并用项目标识关联输入；不同语音轮次的完成顺序不保证转写是有来源的异步系统输出，不是人类证据、语义授权或动作授权

本决定不创建语音界面，也不建立语音接口、发音组件、运行词典、命令别名或新规范字段。未来若允许口述动作，系统必须安全显示识别出的精确动作、对象、参数、目的、权限范围和截止点，并取得适用授权；只有声音、转写、模型置信度或会话成功都不能触发执行。

当前状态与继续阅读

Noemion 已确定职责优先、书写区分、读音稳定和词表口头区分四项独立门槛，但尚未执行人类研究，也没有合格发行读音表。现行拼写继续作为设计标识；任何“官方发音”、全球传播性或名称已经通过的主张都没有证据。

术语与读音验证指南查看两阶段招募、材料、任务、阈值、研究数据和删除边界。 ADR-0031 · 搜索没有冲突，不等于名称可以发行查看工程检索、分发坐标、权利和口头证据怎样共同构成发行门槛。 ADR-0032 · 命令只改大小写，仍然是名称冲突查看 Ktisor 与动作名称为什么仍需专名必要性和读音复核。 ADR-0035 · 公开动作与内部职责继续判断哪些名称需要成为用户接口，哪些只应留在内部责任分工。

当前协作规则：先说直白职责，再展示精确书面标识；听者不能唯一写回时立即切换到文字确认。口述近似值不得选择命令、规范版本、对象、参数或权限。
