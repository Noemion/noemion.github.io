ADR-0008：Endem 制品与单一命令面

保存 Endem 与单一 CLI 的形成历史；其余制品、组件、动作和状态名称已由 ADR-0010 取代。

Superseded by ADR-00102026-07-12Historical VocabularyABI Unfrozen

决定摘要

Noemion 不再按传统二进制工具的数量组织产品。系统以 Endem 为最小、可编译的自然语言目标制品，以 Weave 表示多个 Endem 已完成解析的组合闭包，以 Frame 表示一次运行中建立的只读求解视图，以 Witness 保存可定位的观察和判断依据。

后续决定：本 ADR 除 Endem、`.endem` 与单一 CLI 外的现行名称均已取代。当前语义、制品、组件、动作和生命周期由 ADR-0010：原生词汇与事态分层定义。

直接结论：公开命令只有 endem。form、check、bind、pack、seal、see、run 与 test 是它的固定职责入口，不再建设 23 个并列应用。

制品与运行术语

名称直白工程副标题规范边界
Endem最小目标制品一个 Endem 必须且只能有一个根 aim；它保存 say、aim、must、done 与 open 五组语义。
Weave多目标解析闭包由两个或更多 Endem 经过显式绑定产生；所有必需引用、约束冲突、能力上限和验收关系都有确定结论。
Frame运行时只读视图由 Runner 针对一次会话建立，不是文件、对象种类、包成员或可跨会话复用的句柄。
Witness可定位证据记录绑定对象、会话、环境、观察、规则和判断；它证明“记录了什么依据”，不宣称数学证明、事实必然为真或任务自动通过。

Endem 的五组语义

字段组回答的问题必须保留的边界
say用户或权威来源实际表达了什么？精确文本或内容摘要、语言、来源范围和版本；来源身份不证明解释正确。
aim什么结果才算目标？唯一根目标、结果种类和预期产物；多个独立根目标必须拆成多个 Endem。
must哪些边界不得违反？硬约束、能力上限、禁止事项、适用范围、冲突与未知处理；不得静默降级为偏好。
done怎样判断目标已经满足？验收条件、结果结构、必需 Witness、失败状态和决定权威。
open哪些语义还没有被确认？替代解释、冲突、缺口、风险、允许的解决方式和有权确认的主体。

自然语言可以进入 say，但模型解释只能成为候选。只有可由版本化规则重新推导、经过明确授权绑定，或仍以未决状态保留的内容，才能进入同一 Endem 的其他字段组。Endem 不保存思维链、运行计划、模型权重、实时能力句柄、凭据或私钥。

同一制品的状态

open · 仍含未决引用或语义
bound · 引用和冲突已解析
sealed · 发布载荷已冻结并绑定外部签名
  • open 是合法但不可直接发布执行的状态；它必须准确保存未决事项，不能假装失败已经消失。
  • bound 表示必需语义、依赖与能力上限已经解析，不表示来源可信、环境授权或结果满足。
  • sealed 表示不可变发布载荷与外部签名响应一致，不表示语义正确，也不表示当前环境允许运行。

状态是同一 Endem 的生命周期属性，不通过复制成 Open Object、Release Object 或 Signed Object 等平行对象家族来表达。任何状态变化都产生新的精确文件摘要，并保留前一状态的来源绑定。

唯一命令面

入口职责主要失败责任
endem form把受控目标卡或已授权候选确定性形成 open Endem。来源无法定位、根 aim 不唯一、约束或验收不完整、候选越权升级。
endem check使用生产读取路径分层检查结构、语义、状态、完整性和策略。越界、溢出、悬空引用、未知关键记录、状态与内容不一致。
endem bind解析 Endem 之间的引用和约束，形成 bound Endem 或 Weave。缺失依赖、循环、冲突、权限扩大、依赖身份不稳定。
endem pack冻结发布闭包、可选调试伴随资料和待签消息。覆盖缺口、不可复现元数据、发布裁剪改变语义、闭包不完整。
endem seal核对外部 Signature Response，并在被签载荷外附加签名封装。请求、载荷、算法、签名者或策略版本不匹配。
endem see以独立直接读取器查看、比较和导出有界结构视图。畸形输入、未知关键结构、资源超限；不得生成生产 Verified Handle。
endem run在隔离域重新验证 sealed Endem/Weave，建立 Frame，受控调用后端与能力并形成 Witness。签名、策略、能力、后端、预算、状态漂移、证据缺失或需要人工判断。
endem test运行规范向量、生产/独立读取差分、恶意语料和复现检查。实现与条款不一致、两条读取路径分歧、拒绝行为或字节复现失败。

see 与其他子命令共享发行入口，不共享生产解析实现；run 共享用户入口，不共享编译器权限域。命令行的一致外观不能消除内部的实现独立性或进程隔离。

命名检索与正式商标检查

Endem、Weave、Frame、Witness 与 endem 已作为本项目工程术语冻结，用于替换旧对象和工具家族。冻结只说明项目内部不再继续发散命名，不构成商标可注册、域名可取得或第三方权利无冲突的法律结论。

  1. 开发阶段执行字面与近似检索：操作系统命令、主流包管理器、GitHub 仓库、语言生态包名、域名和同类开发工具。
  2. 公开发布前建立正式 gate：目标法域商标数据库、相同及近似类别、公司与产品名称、开源包身份和律师或合格知识产权人员复核。
  3. gate 未通过时必须在首个稳定 ABI 前一次性调整术语、命令和扩展名，并用新 ADR 保持五字段语义与身份规则不变；不得在发布后依靠别名掩盖冲突。

外部技术只提供机制先例

  • GNU readelf 独立于 BFD 的做法支持 see 采用第二条直接读取路径：https://www.sourceware.org/binutils/docs/binutils/readelf.html
  • BFD 的多格式 canonical form 可能丢失格式特有信息，因此不进入 Endem 第一阶段可信核心：https://sourceware.org/binutils/docs/bfd.html
  • MCP 的工具与资源声明只作为 Runner 外缘适配输入，不构成权限或事实：https://modelcontextprotocol.io/specification/2025-11-25
  • A2A 的 Task 与 Artifact 只承担跨系统交换，不定义 Endem、Frame 或最终验收：https://a2a-protocol.org/v1.0.1/specification/
  • OpenAI 的 Harness Engineering 提供环境可读性、反馈循环和机械边界的实践依据，不决定本项目格式或成熟度：https://openai.com/index/harness-engineering/
