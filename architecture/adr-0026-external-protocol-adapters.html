ADR-0026
外部协议适配边界

让 MCP、A2A、HTTP 与未来协议进入受限会话，同时不让外部状态、身份和权限改写 Noemion 的工程语义。

Accepted DraftADP-CORE 0.1.0-draftProtocol IndependentNo Adapter Code

直接结论

Noemion 不需要一个万能 Agent 对象，也不应把 MCP 或 A2A 变成内部语义。每个外部协议只通过受 Dromen 限制的适配绑定连接到一次精确调用。

适配必须保留协议原始事实、收窄能力、声明信息损失并控制副作用。外部任务完成、工具成功、HTTP 状态和模型停止原因都不能直接成为目标满足、最终接受、会话终止或证据有效。

为什么现在只冻结抽象不变量

截至 2026 年 7 月 13 日，MCP 2025-11-25 仍是稳定基线，2026-07-28 是包含破坏性变化的候选版本。候选版把核心改为无状态并把 Tasks 移到扩展，说明握手、会话与任务字段仍在快速变化。A2A 最新发布规范已经进入 1.0，但它的 Task、Message、Artifact 和 Agent Card 仍属于外部协议对象。

因此 ADP-CORE 冻结版本、身份、能力、状态、取消、重试和交付边界，不冻结具体 JSON、Protocol Buffers、SSE 或 SDK 字段。未来每个协议 Profile 必须绑定精确稳定版本和真实消费者。

映射和重试的硬边界

GNU BFD 说明不同对象格式进入统一 canonical form 时可能丢失原格式信息。外部协议适配面临同样问题，因此每次映射必须保留原始协议身份、映射版本和有限损失清单，不能用一个通用 success 布尔值掩盖差异。

自动重试只适用于只读、协议级幂等或具备去重证据的调用。连接中断和没有响应不能证明副作用没有发生；所有重试共享原调用身份、Dromen 预算、截止点和取消状态。

安全边界

  • 凭据留在独立能力域，并绑定主体、受众、scope、期限和本次会话。
  • 禁止把上游令牌透传给下游，也禁止把静态秘密写入 Agent Card、工具 schema、消息或诊断。
  • 网络目标必须经过 SSRF、重定向、Origin、DNS 重绑定和私有地址政策。
  • 流式、Webhook、推送和轮询必须保存授权、序列、去重、乱序、缺口、终结与资源证据。

权威资料与采用限制

  • MCP 2025-11-25 规范变更记录提供当前稳定版本基线。
  • MCP 2026-07-28 候选版说明展示无状态核心、扩展与授权变化；项目只把它作为版本漂移证据。
  • A2A 1.0 版本化规范定义 Agent Card、Task、Message 与 Artifact；这些对象保持在外部状态域。
  • RFC 9110 HTTP Semantics提供幂等与自动重试边界；Noemion 进一步要求共享会话预算和调用身份。
  • GNU BFD canonical format 说明揭示统一映射可能丢失信息；Noemion 因而要求显式损失清单。

后果与当前状态

  • ADP-CORE 0.1.0-draft定义 12 条协议无关适配义务。
  • 12 类威胁、18 个设计场景和 24 个正反向量覆盖当前抽象边界。
  • 当前没有协议 Profile、适配器 API、网络服务、凭据代理、重试引擎、Webhook 服务或 Praxor 实现。
