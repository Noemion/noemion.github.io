项目/架构设计/架构决策/ADR-0016

ADR-0016
mene 时间
连续性模型

为持续目标固定时间范围、时钟来源、区间边界、允许违约和观测覆盖，避免用系统当前时间或零散采样伪造连续成立。

当前策略Mene SemanticsNo Wire Change尚未实现

直接结论

mene 不只是“保持”标签。一个可判断的持续目标必须同时说明从何时到何时、由哪种时钟解释、是否允许短暂违约，以及证据覆盖了哪些时间。

当前策略只确定抽象语义和禁止推导，不增加 END-P1 字段。END-P1 继续只接受 kine；未来支持 mene 必须建立新 Profile、线格式字段和独立正反字节向量。

两类时间范围

范围必须保存适用场景拒绝条件
fixed开始时刻、结束时刻、具名决定权威和原始时间表达的来源绑定。发布窗口、禁止变更时段、维护时段等跨系统共享的确定区间。缺少偏移或时区决定、开始不早于结束、使用当前机器默认时区，或仍含“今天”“明天”等相对词。
elapsed具名起始事件、正的固定长度时长、单调时钟域、时钟生产者与重启边界。部署完成后保持十分钟、能力授予后等待三十秒等会话内经过时长。起始事件未绑定、使用日历月或年、使用可回拨的民用时钟测量时长，或跨重启复用未声明的单调时钟值。

fixed 的规范时刻采用 RFC 3339 的 UTC Z 形式，并在投影阶段把经授权的本地时间解析为确定瞬间。RFC 9557 时区名、原始偏移、解析规则与数据库版本保留在来源或决定依据中，不以运行机器当前时区重新解释。

区间边界与时间表达

  • 目标范围统一解释为半开区间 [start, end)：包含开始时刻，不包含结束时刻。相邻区间因此既不重叠也不留下边界空洞。
  • fixed 第一版只接受秒值 00–59 的规范 UTC 瞬间；闰秒词法、循环日历规则、开放端点、无限区间和本地日历重复仍保持待确认。
  • elapsed 时长必须是正的、精确的固定长度单位；“一个月”“一个工作日”和模糊近似不能按秒数猜测。
  • 来源可以保留“下周一凌晨”等原句，但 Ktisor 只有在具名权威把它解析为上述范围后才能确认 semion；否则进入 apor。
  • 时钟分辨率、同步方式、最大不确定度、回拨或重启事件属于 Iknem。没有这些信息时，比较器不能假定时间顺序精确。

两类连续性政策

政策含义必须声明不能怎样解释
strictskena 在整个目标区间都必须成立。目标范围、覆盖方法、可接受时钟与观察不确定度。几个健康采样点不能证明采样之间也成立。
budgeted允许有限违约，但只在显式预算内仍视为满足。最大累计违约时长、最大单次违约时长和最大违约次数。遥测缺口不是允许违约；未观察到的时间不能消耗预算后自动通过。

“采样”是证据方法，不是第三种连续性政策。若方法只能证明离散时刻，未覆盖区间必须显式保留为观察缺口。

观测覆盖怎样决定结果

事实满足结果原因
覆盖完整、时钟有效，且没有违约或违约仍在预算内met目标区间每一部分都有适用依据。
覆盖完整且观察到严格违约，或显式违约预算被超过unmet有效观察反驳持续目标。
存在未覆盖区间、只有离散采样、时间不确定度跨越关键边界agno目标和判据明确，但适用观察不足。
时钟、同步、适配器或求值器违反声明契约fault求值过程没有可靠完成，不能推出目标为假。

覆盖范围使用同一半开区间规则形成规范并集。重叠、乱序和重复可以在不改变含义时规范化；空洞必须保留，不能插值成事实。观测方法及其保证必须进入 Iknem，单个时间戳只证明一个时刻，不证明一段区间。

外部资料提供的边界

RFC 3339提供可互操作的互联网时刻表示；RFC 9557补充时区等关联信息。Noemion 采用确定瞬间和显式时区来源，不采用缺省本地时区，也不把整个 RFC 类型空间直接变成 Endem ABI。

GNU C Library 时钟说明指出单调时钟适合测量经过时长，却不能表示绝对时间；GNU Coreutils 相对日期说明也记录了夏令时和模糊月运算风险。Noemion 因此拒绝把 now、tomorrow 或默认时区写进规范时间范围。

W3C OWL-Time区分瞬间、区间、时长与时间参考系统；OpenTelemetry Metrics 数据模型明确记录时间窗口、重置和数据缺口。它们支持概念分层与缺口显式化，但 OWL 类、OTLP 时间戳和遥测聚合规则都不会直接进入 Endem。

A2A 1.0的 Task 时间戳和状态只作为外部运行事件。远端 Agent 声称任务持续多久或何时完成，不会自动成为本地时钟权威、Iknem 覆盖或 mene 满足结论。

后果与验证路线

  • END-CORE 新增时间范围、连续性政策和观测覆盖三条义务；END-P1 字段和字节保持不变。
  • 十二个 mene 提案向量执行六个允许分类和六个确定拒绝，只证明抽象矩阵与条款一致。
  • 未来 Profile 必须登记时间字段、数值范围、时长单位、时区来源、闰秒策略和资源上限，并建立独立正反字节向量。
  • 未来 Drasor 必须以单调时钟测量经过时长，保留民用时钟调整、观测空洞和不确定度；当前没有计时器、监控器或求值组件。

待定内容

物理字段、数值宽度、亚秒精度、闰秒时间尺度、循环日历、开放区间、跨重启单调时钟关联、时区数据库封装、观测压缩和多生产者时间归并仍需新 Profile、威胁分析与实现证据。
