项目/架构设计/Agent 系统边界图

Agent 系统边界图

从一次工具调用出发，分开目标、行动者、授权、会话、外部状态、证据与最终决定。

开发者可以据此判断一个框架事件应进入哪个责任域，以及还缺少哪项证据。

非规范说明不增加对象尚无运行组件复核于 2026-07-15

先按一次调用分开判断

Agent 框架会报告模型生成、工具调用、委托、暂停和恢复等事件。Noemion 不把它们压成一个“成功”状态。开发者先回答下面六个问题，再决定事件应进入哪个责任域。

开发者问题责任域需要保留的事实
期望终态是什么？Endem / Synem目标身份、依赖闭包、判据和仍未解决的意义；计划与步骤留在控制平面。
目标只要求当前结果，还是要求动作、转变或因果？END / IKN结果型目标只判断后态；更强主张还要保存动作、同一对象的前后态、竞争活动、方法与限制。
谁代表谁执行什么动作？ID / AUT模型、Agent 定义、工作负载、运行实例、实际行动者、被代表主体、对象、动作、目的和截止点。
本次运行最多能做什么？DRO / ADP已核对的协议、对端、环境、能力交集、预算、凭据域和证据责任；能力声明只作为输入。
外部系统实际报告了什么？ADP / DIA精确调用、Task 状态、返回、错误、取消、交付和已知副作用，不提前改写本地结果。
什么证据支持哪项结论？IKN / END / AUT观察范围、方法、限制、满足判断和具名权威决定；每项结论保留自己的输入。

模型身份不等于实际行动者，认证也不等于逐动作授权。模型输出是候选，不是规范内容。工具成功或远端 Task 完成，也不证明目标满足。

这份说明不增加 ADR、CORE、Profile、对象、命令或组件。现行要求仍以对应 ADR、CORE 和登记为准；相邻主题只作为正在研究的输入。

用依赖升级看清边界

假设一个 Agent 要把仓库依赖升级到经过允许的新版本。这个例子不需要新的 Noemion 对象，却能展示每一层怎样提供不同答案。

  1. 目标：Endem 描述期望版本、兼容条件和验证判据；它不保存“编辑文件、安装依赖、运行测试”等可变步骤。
  2. 行动者与授权：AUT 绑定当前仓库、分支、依赖、允许的版本范围、实际行动者和截止点。登录成功或 Agent 名称不能代替这项决定。
  3. 会话上限：Dromen 只允许本次会话使用已经核对的文件、网络目标、工具和预算。工具列表后来新增的能力不进入旧会话。
  4. 外部事实：包管理器退出码、远端 Task 状态和生成的锁文件分别记录。退出码为零只说明该次调用怎样结束。
  5. 证据与决定：锁文件差异、构建结果、测试覆盖和来源限制进入有范围的 Iknem；krin 形成满足判断，具名权威再决定是否接受。

如果并行尝试多个版本，分支只产生候选。提交路径仍要重新核对当前分支、授权、预算与外部副作用；取消慢分支也不证明它没有留下远端状态。

运行事实先按责任归类

开发者不需要为每个 SDK 名词建立新对象。先按事实的职责归类，再检查它能支持什么结论。

事实类别主责任可以支持不能自动推出

找不到精确归属时先记录未知范围。产品名称、协议状态和机制标签都不能替开发者补齐缺失事实。

先判断是哪一种越级

候选输入取得指令权

检索片段、工具返回或远端描述只因为进入上下文，就被模型当作授权指令。

显示名称或声明替代授权

Agent 名称、工作负载身份、Agent Card、工具列表、schema、scope 或签名覆盖实际行动者、代表关系和逐动作决定。

运行状态替代目标判断

步骤、handoff、HTTP、MCP、A2A 或 SDK 的 completed 被直接写成 met、证据充分或最终接受。

记忆、检查点或轨迹替代当前事实

过期指导、恢复状态、Span Ok 或日志数量被当作现行政策、持续权限、完整因果链或充分证据。

隔离机制名称替代有效限制

容器、seccomp、超时或“本地服务”标签被用来概括文件、网络、秘密、资源、终止和信息流控制。

单项信号替代完整结论

模型评分或反馈替代评测与发布；参数可下载、AGPL、导出或删除响应替代完整对象范围。

行业变化只改变外部机制

截至 2026-07-15，主流 Agent 框架和协议正在把委托、审批、状态、任务和观察显式化。显式机制便于追踪，但仍不能跨越本地目标、授权、证据和决定边界。

开发者面对的变化官方资料说明什么Noemion 保留的边界
委托与人工批准OpenAI Agents SDK handoffs把控制权转交和“作为工具调用”分开；OpenAI Agents SDK human-in-the-loop允许暂停、序列化并在审批后恢复运行。答复所有权不等于动作授权。审批必须绑定精确请求；handoff 不能隐藏实际行动者或扩大 Dromen。
会话、记忆、工作区与轨迹OpenAI Agents SDK sessions区分多种对话状态策略；OpenAI Sandbox Agents 跨运行记忆又与消息历史、resume 和 snapshot 分开；OpenAI Agents SDK tracing记录生成、工具、handoff 和 guardrail 事件。这些状态各有身份和生命周期。记忆与快照不恢复旧权限；trace 是有范围观察，不自证完整性、因果或证据覆盖。
协议任务与动态能力MCP 2025-11-25仍是 Current；MCP 2026-07-28 候选版计划于 7 月 28 日定稿并含破坏性变化。A2A 1.0 版本化规范分别定义 Task、Message、Artifact、取消和交付。Profile 必须固定正式版本。能力变化不扩写旧 Dromen；Task、Message 和 Artifact 继续留在外部域。
Agent 身份与授权基础设施NIST AI Agent Standards Initiative把安全、身份和互操作列为重点；NIST 软件与 AI Agent 身份授权概念工作继续研究非人主体与逐动作权限。模型、Agent 定义、工作负载、运行实例、凭据、实际行动者和被代表主体必须分开。认证只形成授权输入。
自动评测与模型更新NIST AI 800-2 初稿把测量目标、执行、分析和报告分开；NIST AI 600-1与NIST SP 800-218A要求把模型生命周期、数据与安全开发责任显式化。模型评分是有范围的候选测量，反馈记录不是训练资格。训练完成、多个模型一致或点估计不能替代独立评测与发布决定。
模型开放性与托管服务控制OSI Open Source AI Definition 1.0与Model Openness Framework按对象说明开放范围；GNU 对他人服务替代用户计算的分析则把软件权利与正在运行的服务控制分开。开放权重、公开代码、许可标签、API 可用和自主管理部署是不同事实。任何单项都不能概括整个系统或托管实例。
数据使用、保留与删除NIST Privacy Framework 1.0与NIST SP 800-88 Rev. 2支持把处理活动、删除请求和介质清除分别记录。工程分类不产生法律结论。策略到期、逻辑删除、工具完成或供应商收据不能证明备份、快照和派生物已经不可恢复。

研究问题按开发者问题路由

相邻研究只在改变当前评审问题时进入这份指南。完整案例、反例、威胁和现有规范归属保留在各自提案中。

开发者需要回答的问题继续阅读当前结论上限
哪些来源进入模型，目标与计划怎样分开，迁移后是否仍是同一语义？模型上下文装配、目标、计划与重规划、语义等价与迁移研究中；候选输入、计划和等价主张继续由现有身份、文本、目标、会话与证据责任处理。
谁在行动、当前能做什么、并行分支怎样提交、隔离失效时谁负责？能力发现、协商与调用、软件 Agent 身份、委托与责任链、并行、推测执行与提交、模型、适配器与能力域隔离研究中；能力、身份、授权、会话和外部协议仍保持分开，不创建通用 Agent 权限对象。
动作是否发生、预览是否可信、恢复了什么、模型评审能支持什么？状态变化与因果归因、预览、模拟与批准、记忆、检查点与恢复、模型参与评测与裁判边界研究中；动作、后态、因果、恢复状态、候选测量和最终决定继续使用各自证据。
反馈能否训练，更新后的模型怎样发布，数据何时真正删除？模型训练与更新边界、软件 Agent 数据使用、保留与删除边界研究中；反馈、用途授权、模型派生、独立评测、发布、保留和清除不能压成一条状态线。
模型开放到什么程度，用户能否控制实际服务与数据路径？模型开放性与软件自由边界、托管人工智能服务与用户控制边界研究中；逐对象权利、复现范围、实例控制、数据路径和退出能力分别陈述。

这些研究共用一个停止条件：现有责任能够形成唯一主条款时，不创建新的 CORE。只有真实消费者证明责任无法由现有规范承担，才重新讨论独立标准。

怎样完成一次边界评审

  1. 写下期望终态，并说明它只要求结果，还是还要求动作、状态转变或因果。
  2. 固定请求主体、实际行动者、被代表主体、对象、动作、目的、受众和截止点。
  3. 封闭本次 Dromen 的环境、能力交集、预算、凭据域和证据责任。
  4. 把工具、handoff、Task、取消、重试、交付和副作用保存为外部事实，不先翻译成本地成功。
  5. 让 Iknem 只支持声明范围内的结论，再由 krin 和具名权威分别形成满足结果与最终决定。
  6. 某项事实不可见时记录未知范围；机制名称、模型评分和产品声明都不能补齐它。

什么时候才值得增加新对象

只有一项责任同时具备现有对象无法表达的身份与不变量、独立生命周期、真实生产者和消费者，以及可重现的失败责任，才进入新对象讨论。支持案例、反例、威胁、资源上限和正反向量还必须能够验证它。当前列出的 Agent 研究主题都没有达到这项条件。

架构决策查看当前策略和待定接口。 开放问题查看研究提案仍需关闭的决定与证据。 外部协议适配查看版本、对端、能力、任务、重试和交付边界。 权威与授权决定查看主体、委托、同意、撤销和能力交集。 Dromen 会话契约查看一次运行怎样封闭环境、能力、预算和证据责任。
