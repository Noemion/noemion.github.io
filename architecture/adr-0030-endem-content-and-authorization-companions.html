ADR-0030
Endem 内容与授权伴随关系

让目标文件稳定回答“对象是什么”，让外部决定单独回答“谁在什么范围和时间内允许怎样使用它”。

当前草案END-CORE ClarificationIdentity StableNo New Artifact

直接结论

`.endem` 中的权威名称不能证明授权已经发生。内容身份只取决于指定 Profile 的规范字节；授权、签名、证据和最终决定作为外部伴随关系解释这些字节。

END-P1 仍只包含六个语义记录。它可以单独通过容器和 Profile 检查，却不能单靠文件内字符串通过完整内容接受。

  • 规范字节表达六个语义面和封闭引用。
  • 内容身份只标识这些精确字节，不证明它们正确或获准。
  • 授权绑定连接来源、候选、语义位置、显示视图、主体、范围与决定。

  • 授权成功仍不能替代现实观察、满足判断或最终接受。

为什么采用外部伴随关系

  • in-toto Attestation Framework 1.2把谓词、主体绑定、认证包络和集合分层。Noemion 采用职责分离，不复制其 JSON 格式。
  • The Update Framework 1.0.27让角色签署目标元数据，而不是让目标文件自带信任。Noemion 同样把对象与信任判断分开。
  • GNU Guix 的替代物认证依赖本地授权密钥清单。服务器名称或签名存在都不能替代本地信任政策。
  • MCP 2025-11-25 的信息征询明确区分用户接受交互与交互已经完成。界面动作不能越级成为最终事实。
  • ELF 未知节处理规则要求拒绝需要未知特殊知识的关键节。Endem 也不能静默忽略未绑定的关键语义。

语言哲学边界

《逻辑哲学论》4.022 区分命题显示的意义与它对事情如此的断言。工程上还要再分一步：即使命题结构清楚，也不能从措辞推出组织中的批准权。

授权属于制度、主体、范围和时间关系。它必须由规范结构与外部证据显示，不能藏在一个听起来可信的名称里。

验证材料

  • SCN-028 至 SCN-030 覆盖名称自证、精确伴随绑定和身份稳定性。
  • 语义向量 v2 显式声明文件外测试前置条件，不把它们写进 Endem 输入。
  • 缺失与错绑授权分别产生稳定诊断。
  • 两个内容完全相同、外部授权证据不同的向量必须进入同一身份等价组。

待定内容

授权伴随关系、权威目录、政策语言、规则登记、决定事件、撤销分发和符合性报告还没有物理 Profile。当前改动只修正规范边界，不创建新文件、包络、服务或组件。
