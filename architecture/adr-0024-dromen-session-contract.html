项目/架构设计/架构决策/ADR-0024

ADR-0024
权限只属于
这一次会话

可长期保存的目标不携带永久权限。每次尝试都要重新固定精确目标、当前政策与环境，再把能力、预算和观察责任收窄到本次会话。

当前策略SESSION-CORE 0.1.0-draft仅限一次会话尚未实现

用一次受限发布会话理解 session contract

某个 Endem closure 固定了构建制品与服务健康目标的完整闭包，但它没有授予部署权，也没有指定本次运行可以访问哪个环境。bounded runner 只有重新核对全部会话条件，才能建立一次会话的只读执行契约（设计阶段名称 session contract）。

选择目标精确 Endem closure 身份、Profile 和全部成员闭包文件名、分支名、缓存记录或远端 latest
核对前置条件适用外部陈述、验证记录、政策截止点、撤销状态与依赖方判断单一可信布尔值、签名外观或上次通过
关闭运行边界仅限预发布环境、具名适配器、协议版本、时钟和后端当前目录、机器默认值或工具自行发现的环境
分配权限与预算仓库只读、预发布环境写入、十分钟、二十次调用、两次重试和一个子任务操作者的全部权限、每个子任务一份新配额或模型提出的扩权
运行与观察每项 satisfaction_criteria 的观察者、方法、时间、披露和决定责任工具成功、Task 完成或日志数量直接代表目标满足
结束会话保留有范围事件和 evidence entry，销毁实时能力与秘密用检查点复活旧权限，或把旧 session contract 用于下一次运行

session contract 的“只读”表示建立后不能原地修改，并不表示会话没有计划、事件、重试或终止状态。任何需要扩大权限或改变实质绑定的继续运行，都必须重新建立会话。

目标、会话契约、运行者和证据各自负责什么

对象开发者用它回答什么不得承担什么
Endem 或 Endem closure期望哪种终态，单个目标或多个目标怎样形成固定闭包不携带当前机器、实时凭据或一次会话权限
session contract这个精确目标在本次尝试中受哪些政策、环境、能力、预算和观察责任约束不保存可变计划、工具返回、实时秘密、最终结果或可恢复权限
bounded runner谁重新验证输入、建立契约、控制能力并执行一次会话不改写目标、不自行扩权，也不替验收权威作出决定
evidence entry 与决定记录实际观察支持什么有限主张，以及具名权威作出什么决定存在记录不表示会话可恢复，也不能反向改变目标或契约

这个边界使持久目标与临时权力分离：Endem 和 Endem closure 可以长期保存，session contract 只属于一次授权上下文中的尝试。运行历史可以成为下一次会话的输入，却不是下一次会话的准入证明。

现行 SESSION-CORE 只接受精确的 resolved Endem 或 Endem closure。内容状态不能证明会话准入；适用外部陈述、验证政策、截止点、撤销状态与依赖方判断必须逐项绑定并重新核对。

现行草案限制：这些关系已经同步进入条款、登记、威胁、场景和正反向量，但仍没有 session contract 文件格式或运行实现。

MCP 2025-11-25 授权规范要求令牌面向目标资源并按需请求权限；其安全实践进一步说明会话标识不是认证，令牌也不得穿透转发。Noemion 采用资源、受众和最小权限的边界，但不把 OAuth 令牌、MCP Session ID 或 Task ID 放进 session contract。

预算耗尽只说明本次尝试不能在现有上限内继续，不自动产生目标 unmet。资源限制、会话终止、证据充分性、目标满足和最终接受仍使用各自的结果域。

漂移、取消和恢复为什么必须开始新会话

模型版本、适配器、协议、政策、能力、决定权威或隔离配置发生实质变化时，旧 session contract 失效。bounded runner 保存已经发生的事件和有范围证据，再按预先固定的策略中断或失败；不能把新值覆盖到旧契约上。

检查点、外部任务句柄和脱敏报告可以成为新会话的候选输入，但恢复仍要重新核对主体、外部陈述、政策、环境、能力和剩余预算。取消请求也只是一项事件：在远端状态或副作用无法确认时，系统必须保留未知，而不是声称工作从未发生。

外部机制可以借鉴什么不能据此推出什么
ELF gABI 程序装载区分持久文件与系统据此建立的进程映像session contract 不是段、内存映像、进程或可执行格式
Linux capabilities拆分特权并以能力上界限制执行后可获得的权限Linux 能力不可移植，单一能力也不能证明完整隔离；过宽的 CAP_SYS_ADMIN 尤其不能作为最小权限证明
Linux no_new_privs 与 Landlock提供执行后不新增权限和进程自我收窄的候选机制两者都不是完整沙箱，也不覆盖网络、模型、协议和外部服务权限
GNU C Library 资源限制区分软上限与硬上限，并说明进程怎样继承和收窄限制进程限制不覆盖工具调用、模型令牌、供应商成本或分布式任务预算
GNU make 中断处理中断后删除可能只完成一部分的目标，避免把半成品误当成最新结果清理动作本身可能失败；发出取消或销毁请求不证明副作用已经消失
MCP 2025-11-25 Tasks持久任务必须绑定授权上下文，并受期限、并发和清理规则约束外部 Task 可恢复，不表示 Noemion 会话或实时权限可以恢复
A2A 1.0Task、Message、Artifact、状态与取消语义保持分离Task 终态、取消结果或上下文标识不能直接成为本地满足、证据充分或授权决定

复核日期：2026-07-16。外部协议和操作系统机制只提供设计证据；正式实现仍需固定平台、版本、威胁假设、降级行为和可重复测试。

当前还不能承诺哪种运行时或沙箱

本决定建立 SESSION-CORE 0.1.0-draft 的解释边界，但不建立 session contract 文件、序列化格式、API、运行时、进程模型、沙箱、凭据代理、预算器、事件编码、恢复格式或销毁实现，也不表示 bounded runner 已经实现。

现有十条条款、十项威胁、十五个自然语言场景以及十个允许和十个确定拒绝向量，只检查规范资料的一致性。它们不能证明隔离有效、秘密已经销毁、取消成功、协议互操作或任何平台达到生产安全要求。

名称与口头边界：session contract、Drase 与 bounded runner 在连续讲解时近似，而且“只读”容易被误解为“没有运行状态”。首次出现应先写“一次会话的只读契约”“本次会话”或“会话执行器”，再给出设计阶段名称；日志不得用裸露的 active=true 合并目标激活、会话存活和权限有效。发行拼写与读音仍须通过真实使用者朗读、听写和职责匹配验证。
