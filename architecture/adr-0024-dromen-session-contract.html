ADR-0024
Dromen 会话契约

固定持久目标制品进入一次受控运行前必须绑定的主体、政策、环境、能力、预算、观察责任与销毁边界。

Accepted DesignDRO-CORE 0.1.0-draftSession OnlyNo Component Code

直接结论

Dromen 不是可执行目标文件，也不是运行中的全部可变状态。它是 Drasor 在一次 Drase 会话开始前封存的只读执行契约。

这份契约把精确 attested Endem 或 Synem，与当时有效的政策、权威、环境、能力上限、预算和证据责任绑定在一起。运行观察进入 Iknem；能力句柄留在独立最小权限域；满足结果、最终决定和会话终止继续使用各自结果域。

本决定建立 DRO-CORE 0.1.0-draft。它不建立 Dromen 文件、序列化格式、运行时 API、沙箱实现或凭据代理，也不表示 Drasor 已经实现。

为什么需要独立边界

对象回答的问题不能承担的职责
Endem 或 Synem期望什么终态，多个目标怎样形成固定闭包不能携带当前机器、实时凭据或一次会话权限
Dromen这个精确制品在这一次会话中依据什么边界可以运行不能记录持续变化的观察、工具返回或最终结果
Drasor谁重新验证、建立契约、隔离能力并执行会话不能改写制品、扩大授权或替代验收权威
Iknem实际发生了什么，证据在什么范围内适用不能保存实时秘密或仅凭存在就恢复会话

若省略 Dromen，Drasor 只能把运行环境和能力保存在隐式配置里。这样既无法说明某次运行实际采用了哪组边界，也无法判断中途变化应当继续、重验还是关闭会话。

销毁实时能力

只有完成全部检查并封存后的对象才叫 Dromen。政策、模型、适配器、能力范围或其他实质绑定变化时，旧 Dromen 失效；Drasor追加漂移事件并按预先声明的规则中断或失败。扩大权限必须形成新的 Drase 会话，而不是修改原契约。

日志可以保存经脱敏的绑定摘要和失效原因，但这些记录只能作为 Iknem 或会话事件。它们不能重新创建 Dromen、恢复凭据或继续旧会话。

能力描述与实时能力必须分开

Dromen 只记录非秘密能力描述，例如允许读取哪个资源、令牌面向哪个受众、调用上限是多少。Bearer token、刷新令牌、私钥、文件描述符、socket 和供应商原生句柄留在独立最小能力域，并绑定这一会话。

MCP 2025-11-25 授权规范要求令牌绑定目标资源，并强调按需请求范围；MCP Task 安全条款要求任务绑定授权上下文。Dromen 采用相同问题边界，但不复制 OAuth、Task ID 或协议对象。协议凭据与远端任务始终位于 Drasor 外缘。

从 ELF 与 GNU Linux 借鉴什么

  • ELF gABI 的程序装载章节明确区分静态文件和系统据此建立的动态进程映像。Noemion 借鉴“持久制品不等于运行表示”，但 Dromen 不是 Segment、内存映像或可执行格式。
  • Linux capabilities把传统超级用户权限拆成可独立限制的能力集合，并以 bounding set 限制执行后可获得的上界。Dromen 同样只允许能力求交和收窄，但规范不预先指定 Linux 实现。
  • Linux no_new_privs提供执行后不能获得新增权限的单向约束；Landlock允许进程继续限制自己的环境权限。这些机制是未来候选证据，不是“使用 Linux 就已经安全”的证明。

面向 Agent 时代的直接影响

  • 模型、MCP 工具和 A2A Agent 可以提出计划、能力请求和候选观察，但不能修改 Dromen。
  • 模型切换、协议降级、工具列表变化和远端权限提升都是环境或能力变化，必须由绑定规则判断是否使会话失效。
  • 子 Agent、重试和长时任务不能获得新的预算；它们只能消费会话上限的严格子集。
  • 外部 Task 的成功、失败和取消保留为带来源事件，不直接成为目标满足、Iknem 充分或权威接受。
  • 当前 Agent 协议越来越强调协商、授权上下文和长任务生命周期，这使显式会话契约成为底层需要，而不是附加日志功能。

规范影响与验证

  • 新增 DRO-CORE 0.1.0-draft 十条规范条款。
  • 新增十项 Dromen 威胁和十五个非规范自然语言场景。
  • 二十个提案向量覆盖十个允许契约与十个确定拒绝；检查器只证明资料矩阵一致。
  • END-CORE、END-FMT、END-P1、SYN-CORE、IKN-CORE 与任何规范字节保持不变。
  • 项目仍未进入组件代码阶段；当前不存在 Drasor、装载器、沙箱、凭据域、预算器或 Dromen API。

仍未冻结的内容

运行 API、进程模型、沙箱组合、凭据存储、能力代理、事件编码、预算计数器、恢复策略和销毁机制仍保持开放。Dromen 永远不获得文件格式；若未来实现需要持久审计，只能定义 Iknem 或会话事件，而不能把 Dromen 转化为可恢复权限的对象。
