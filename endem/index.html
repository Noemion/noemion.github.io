项目/Endem

endem 目标制品应用

计划中的 endem 是单一命令入口。开发者围绕同一目标完成形成、检查、组合和受限运行准备；当前没有可安装软件。

应用设计按任务阅读名称仍在审查尚未发布

先确认当前可用性

当前不能安装或运行 endem。项目尚未进入组件代码开发阶段。开发者现在可以审查目标制品、动作职责、信任边界、失败条件和未来验证要求，但不能把规范资料或测试向量当作程序。

现在可以取得
CORE 草案、ADR、威胁模型、场景、规范向量与开发者手册
现在还不存在
Ktisor、Theor、Drasor、endem CLI、安装包和可执行文件
正式发行前必须确定
裁剪发布 Profile、命令参数、退出状态、稳定 ABI、签名与供应链资料

END-P1 字节只用于说明来源保留的形成 Profile。它们不是组件产出，也不是可运行的最终发布制品。

用一次依赖升级理解应用边界

假设开发者要求“把服务依赖更新到批准的安全版本，保持测试通过，经批准后发布”。endem 的设计目标不是替 Agent 完成这项工作，而是让目标、实际文件、运行权限、观察证据和最终决定各有明确责任。

开发者要回答的问题应用边界内的工作信息不足时怎样处理
目标实际表示什么？保存来源，把已经确认的意义和待确认事项分开，再形成来源保留的目标制品意义没有确认时停止形成，不让模型选择一个最可能的解释
实际文件是否符合精确规则？生产路径检查真实字节；独立路径重新读取并给出有范围的视图与差异任一路径失败或两条解释不一致时停止发布，保留最小复现并调查规则或实现
是否需要多个目标共同交付？只有真实消费者需要多个可独立失败的目标时，才固定成员、引用、冲突与共同权限引用缺失、闭包不完整或权限扩大时拒绝组合
这次运行可以做什么？使用精确发布对象、当前政策、环境、能力和预算建立一次只读会话上限授权、环境或对象发生漂移时不开始或不恢复会话
工作是否达到目标？保存工具、协议和环境产生的有范围观察，再分别形成满足判断与最终决定外部 completed、工具成功或测试通过只保留为各自事实

按工作选择设计入口

开发者工作现行设计标识读取什么交付什么
从已确认意义形成目标制品ktise受控来源、候选投影、具名意义决定和精确 Profile来源保留的初始 Endem；意义未确认或 Profile 不匹配时不写出对象
从生产侧检查实际字节elenk实际制品、请求层次、限制和信任材料分层结论与诊断；内部检查引用不跨入独立读取域
独立读取和比较实际字节theor当前为实际 Endem 字节、精确规范/Profile、视图和预算；其他对象等待物理格式带范围的只读视图、差分和诊断；不产生生产检查通过结论
固定多个目标与依赖pleko两个或更多 Endem、组合策略和依赖锁抽象 Synem 闭包；物理格式尚未确定
建立一次受限运行会话drase精确发布制品、适用外部陈述与验证记录、运行策略、环境和能力目录Dromen、运行记录和有范围证据；最终决定由具名权威另行作出

名称仍是研究对象：ktise、elenk、pleko、theor 与 drase 是现行设计标识，不是已经发布的命令参数。它们的必要性、发行拼写、读音流畅度和口头区分仍需独立人类证据。

一条任务怎样经过不同信任边界

确认来源与意义
形成含来源目标
生产侧检查
独立读取实际字节
按需固定依赖
派生并核对发布对象
重新决定运行授权
记录观察并作出决定
信任职责现行设计名称不能越过的边界
确定性目标制品生产边界Ktisor消费已经确认的意义并写入规范字节；不替人选择意义，不持有私钥或实时能力
独立制品检查边界Theor不复用形成侧解析器，不修改输入，不把可读视图变成生产接受结论
受限运行边界Drasor不拥有制品写入器；每次会话重新核对对象、授权、环境、能力和预算
签名、授权与最终决定外部系统与具名权威外部签名响应由独立集成核对，不作为 Ktisor 输入；签名、授权和接受互不替代

什么时候必须停止或交接

遇到的事实本地处理下一责任方
来源存在多种可表达解释，或没有主体有权确认意义保留待确认事项，不形成新的规范对象能够说明意义的具名权威
生产检查与独立读取结果不一致阻断发布，保存实际字节、规则版本和最小复现规范维护者与两条实现路径的负责人分别调查
最终裁剪发布 Profile 尚未确定保留来源形成版，不自行删除字段或复用原身份版本化 Profile、身份与披露边界的规范工作
动作授权、环境或能力目录已经变化停止启动或恢复旧会话，重新计算本次能力交集授权系统、环境提供者和受限运行控制面
协议、工具或测试返回成功保存原始状态与有范围观察，不直接写成目标满足满足判断方法与具名最终决定者

外部先例只提供局部纪律

GNU 手册规范要求按使用者面对的问题组织资料，而不是按程序结构或命令库存展开。GNU readelf独立于 BFD 读取实际对象，支持把独立读取与生产路径分开；它不定义 Endem 字段或 Theor 接口。

MCP负责上下文交换、能力协商和工具调用，A2A 1.0负责 Task、Message、Artifact 与协议状态。两者都可以连接未来的受限会话，但协议成功不能代替本地意义确认、动作授权、目标满足或最终决定。
