测试与放行门

确定性、Golden Cases 与冲突矩阵

noemld Phase 4 / Phase 5

测试目标与证据等级

测试不仅证明实现“能链接”，还要证明它在输入排列、并发、缓存、平台和恶意数据变化下仍遵守同一契约。示例测试用于解释设计，规范测试用于判断一致性，安全测试用于验证拒绝边界；三者不得互相替代。

等级回答的问题典型证据
单元与性质局部算法和不变量是否成立。checked arithmetic、稳定排序、映射双射、合并代数性质。
Golden 与差分给定规范输入是否得到预期对象，独立实现是否一致。字节产物、Link Map、诊断快照和语义归一化结果。
集成与系统上下游工具和 Loader 是否遵守共同契约。编译—归档—链接—验证—装载的封闭流水线。
安全与鲁棒性恶意或超限输入是否安全、快速、原子地失败。模糊测试、变异语料、资源上限和故障注入。
复现与互操作不同环境和实现是否产生规范等价结果。跨平台重复构建、独立实现和版本矩阵。

放行门

  • 链接顺序不改变规范允许之外的结果。
  • 全部强依赖解析且弱依赖缺失行为明确。
  • 符号、ID 与重定位 Golden Cases 全部通过。
  • 冲突矩阵覆盖类型、约束、权限、Contract 和 Disclosure。
  • 相同输入、Profile 与锁文件产生可复现结果。

最低测试矩阵

  • 符号：强弱、版本、可见性、类型、归档拉入、别名、未解析与重复定义。
  • 结构：空对象、边界大小、未知可选与必需特性、重复 Section、错误索引和截断记录。
  • 重定位：每种类型的正常值、边界值、溢出、错误目标类别和悬空引用。
  • SSO：硬约束、权限收敛、软偏好、Contract、Disclosure Graph 和依赖循环。
  • 确定性：输入排列、线程数、哈希种子、缓存冷热、绝对路径和支持平台变化。
  • 失败原子性：每个阶段注入故障，确认没有可误用的部分产物或残留权限。

研究与标准化证据

论文实验应预先声明假设、基线、数据生成器、规范版本、硬件环境和统计方法，并公开失败案例。专利材料要区分新颖机制与 ELF/传统链接器已有策略；软件著作权材料不能替代原创性或正确性证明。进入行业标准候选前，还需要规范测试套件版本化、独立实现结果和跨组织互操作报告。

放行记录

每次阶段门应保存测试套件摘要、输入语料版本、通过与失败清单、覆盖的规范条款、未覆盖风险和批准人。覆盖率只能说明被执行的结构，不能独立证明语义完整或安全。
