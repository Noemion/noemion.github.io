工具简介

noemdis 把运行 Section 反汇编为可再次汇编的规范文本，并按请求附带调试、覆盖和来源注释。

问题背景：对象必须能被独立审计、评审和重建；只显示字节或摘要不足以验证语义。反汇编器提供对象到规范文本的可追溯映射，同时明确哪些信息已被 strip 或无法恢复。

当前状态

设计阶段：当前未发布可执行程序；命令行接口、参数和文件扩展名尚未冻结。已确认内容限于工具职责、预期输入输出、安全边界和阶段门；具体调用方式以未来正式版本为准。

阶段门：依赖安全 Reader、规范文本语法和稳定命名规则；放行要求合法对象可规范化再汇编、损坏对象关闭失败、注释视图不改变运行文本语义。

主要能力

结构验证
读取语义记录
建立稳定名称
分离运行与注释
输出规范文本
  • 反汇编多层 GSIR、符号引用、重定位和 Artifact Contract
  • 使用稳定名称替代布局相关 ID
  • 可选显示 Source Map、Coverage Ledger 和 Disclosure 注释
  • 输出 canonical、pretty 或 JSON 视图

输入与产物

方向对象作用
输入Object通过结构验证的 GOBJ/SSO
输入Layer Filterhierarchical/constraint/evaluation/binary 或 all
输入Annotation Policydebug/coverage/source/relocation 注释开关
输出Text GSIR可被 noemas 接受的规范或易读文本
输出Symbol Map对象 ID 到稳定文本名称映射
输出Diagnostics无法表示或损坏记录的位置与原因

处理边界

  • 不恢复已被 strip 删除的源文本
  • 不把调试注释混入运行语义
  • 不将损坏对象猜测为合法 GSIR
  • 不宣称反汇编摘要等同原始自然语言

所有对象输入均视为不可信；偏移、长度、计数、索引和对齐计算必须使用 checked arithmetic。输出必须标记不可表示记录、未知扩展和信息损失，不能用占位文本制造可再次汇编的假象。
