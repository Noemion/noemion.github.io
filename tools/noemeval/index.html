noemeval · NSFE 模型评估

工具简介

为什么平均分不够

NSFE 的主要风险不是单纯“预测错”，而是在证据不足时给出错误确定结论，或在关键语言、结构与风险切片上悄然退化。单一平均准确率会掩盖这些问题，也无法说明改进来自模型结构、数据变化还是评测污染。

noemeval 以错误确定率、覆盖、校准、OOD 拒绝、跨块一致性和端侧性能为核心，对训练与量化候选执行可复现的分层评估，并把研究主张映射到可重复的指标、切片和失败样本。

当前状态

设计阶段：当前未发布可执行程序；命令行接口、参数和文件扩展名尚未冻结。页面仅说明已经确认的工具定位、预期输入与产物以及安全边界。

评估放行门

任何候选进入正式评估前，必须冻结 Evaluation Suite、指标定义、切片、阈值、基线与统计方法，并把盲测数据与训练调参隔离。放行报告只能回答预先声明的问题；探索性分析产生的新假设必须进入下一轮独立实验。

主要能力

  • 评估跨度、关系、模态、歧义、路由和封闭任务头，并保留逐样本证据
  • 测量错误确定、拒绝质量、校准和覆盖账本影响，而非只统计命中率
  • 执行长度、领域、语言、结构和风险切片，暴露总体指标下的局部退化
  • 在相同协议下比较基座、消融、教师、量化与部署后端

概要处理流

  1. 核验候选：确认模型、数据、配方、环境和评测套件的不可变指纹。
  2. 冻结执行：在隔离环境生成逐样本预测和运行记录。
  3. 分层计量：计算切片指标、校准、拒绝质量、置信区间与资源表现。
  4. 对照分析：与批准基线和预声明消融比较，定位差异来源。
  5. 形成决策：输出通过、拒绝或人工复核结论，并精确列出失败门和受影响切片。

输入与产物

方向对象作用
输入Model Candidatecheckpoint 或量化模型及其完整 Manifest
输入Evaluation Suite冻结数据、切片、指标、阈值和执行后端
输入Baseline Registry已批准基线、历史版本和可比条件
输出Evaluation Bundle逐样本预测、指标、置信区间与环境指纹
输出Gate Decision通过、拒绝或需复核及精确失败切片
输出Regression Report相对基线的质量、性能和校准变化

处理边界

  • 不以平均准确率掩盖高风险切片退化，不省略样本规模和不确定区间
  • 不在评测集上调参后继续把同一结果作为盲测结论
  • 不把模型指标等同于 Compiler Core、GOBJ 或 Runtime 的正确性证明
  • 不允许缺少数据血缘、许可、版本或污染检查的基准进入放行门
  • 不把统计显著性自动解释为工程价值、原创性或标准化成熟度

模型输出和模型包一律视为不可信；它们不能决定二进制布局，也不能直接生成 GOBJ。评估结论只在指定数据、阈值、后端和环境范围内成立，外推必须另行验证。

专题文档拆分条件

当评估协议进入评审时，应拆分指标与校准、数据分区、统计置信、回归门和结果解释专题，并明确哪些指标用于论文、工程放行或标准一致性测试。
