工具简介

noemcov 逐 Source Unit 证明输入已结构化、精确保留、显式丢弃或阻断发布，模型置信度不能替代覆盖证据。

问题背景：生成式编译最危险的失败之一是静默遗漏条件、例外或安全要求。字符覆盖率无法说明语义去向，noemcov 因此以稳定 Source Unit、映射状态和风险策略审计发布闭包。

当前状态

当前状态：尚未发布可执行版本，命令行界面、参数和文件扩展名仍在设计中。当前公开信息包括计划用途、输入输出和安全边界；安装与调用说明将在首个可用版本发布时提供。

开发计划：先稳定 Source Graph、Coverage Ledger、Source Map 与精确 Artifact Contract；随后用人工 Gold 数据验证跨文件条件、否定、例外、重复、冲突和不可安全摘要材料。

主要能力

锁定源与对象
对齐 Source Unit
检查语义去向
应用风险策略
输出缺口与证明摘要
  • 关联 Source Graph、Coverage Ledger 与最终链接对象
  • 检查目标、条件、例外、硬约束和权限是否有去向
  • 验证精确 Artifact 的内容哈希、范围和消费 Contract
  • 生成风险分级、缺口列表和发布证明摘要

输入与产物

方向对象作用
输入Source Package原始文件、资源与稳定 Source Graph
输入Compiled/Linked Object含 Coverage Ledger 和 Source Map 的对象
输入Coverage Policy风险级别、允许丢弃类别、人工确认和 Artifact 规则
输出Coverage Report每个 Source Unit 状态、目标节点和证据
输出Gap Report未覆盖、冲突、可疑摘要与跨文件断链
输出Coverage Proof可签名的源哈希、对象哈希、policy 与结论摘要

处理边界

  • 不证明任意自然语言与 GSIR 数学等价
  • 不把 token/字符覆盖率当语义覆盖率
  • 不自动批准有损摘要
  • 不允许模型高置信掩盖未分类 Source Unit

Coverage Proof 证明给定 Policy 下的映射与审计过程，不证明源文本和 GSIR 的普遍数学等价。未解决、冲突或高风险有损映射必须阻断对应发布 Profile。
