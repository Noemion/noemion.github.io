它解决什么问题

为什么需要数据工程边界

Horizon Engine 的研究结论不仅取决于模型结构，也取决于样本从哪里来、允许怎样使用、经历了哪些变换，以及训练集是否污染了评测集。若数据谱系、许可或切分关系不可复核，即使指标很高，也无法支撑论文复现、专利实施例、软件成果登记或未来一致性测试。

noemdataset 的职责是把来源各异的 Skill、提示词、规范与负例转化为有许可、有血缘、可切分、可复现的 Horizon Engine 数据集和来源图样本。它不是通用爬虫，也不负责用规模掩盖来源不确定性。

当前状态

当前状态：尚未发布可执行版本，命令行界面、参数和文件扩展名仍在设计中。当前公开信息包括计划用途、输入输出和安全边界；安装与调用说明将在首个可用版本发布时提供。

实现准备

开发前需要明确 Source Registry、许可与隐私分类、样本结构定义、Gold 仲裁规则、文档族切分策略和撤回机制。训练快照生成前还要验证训练、验证和盲测分区不存在同源文档族或模板级泄漏。

它怎样工作

  • 导入、规范化、去重、分片和许可过滤，并保留原始来源指纹
  • 构造层级来源图、跨度标注和封闭任务标签，显式保存冲突与未决项
  • 隔离 Gold、Silver、合成与对抗样本，防止不同证据等级被混合解释
  • 检测训练评测泄漏、近重复和来源污染，并生成可追踪到样本族的报告

概要处理流

  1. 登记来源：记录所有者、许可、用途、敏感等级、获取方式和内容指纹。
  2. 准入过滤：拒绝许可不清、无法撤回或超出研究授权范围的来源。
  3. 规范化与去重：保留原文映射，在文档族和模板层面识别近重复。
  4. 构图与标注：生成来源图、任务标签、冲突记录和人工仲裁状态。
  5. 冻结与审计：按内容寻址生成快照、血缘清单、分区证明和泄漏报告。

它读取什么，产生什么

方向对象作用
输入Source Registry位置、许可、用途、敏感级别和获取时间
输入Annotation Package人工标签、教师候选、冲突和审核状态
输入Dataset Recipe结构定义、过滤、切分、采样和随机种子
输出Dataset Snapshot内容寻址、不可变、按任务分片的数据集
输出Lineage 清单每条样本的来源、变换、许可和审核链
输出Leakage Report跨 split 重复、基准污染和模板泄漏证据

它不会做什么

  • 不自行创造缺少来源的权威标签；人工裁决也必须记录依据和复核者
  • 不把教师模型输出、合成样本或高置信预测自动视为 Gold
  • 不跨越许可、隐私、地域或用途边界收集和复用数据
  • 不按随机记录切分同源文档族，不用表面去重代替泄漏审计
  • 来源撤回或授权变化时，必须能定位受影响样本、快照、模型与评估结论

模型输出和模型包一律视为不可信；它们不能决定二进制布局，也不能直接生成 NOBJ。数据规模、覆盖率和自动化程度同样不能替代来源合法性与证据可追溯性。

尚未冻结的数据契约

来源与许可治理、标注与仲裁、切分与泄漏检测、血缘与撤回仍需分别形成规范。上述契约获批前，任何样本结构定义、目录布局和报告格式都只属于候选接口。
