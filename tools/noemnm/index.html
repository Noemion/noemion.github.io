工具简介

noemnm 显示导入、导出、弱强绑定、版本、可见性、Contract 与未解析引用，使用与 noemld 相同的符号模型。

问题背景：符号不仅是名称，还关联类型、版本、契约、权限和依赖。缺少独立符号视图时，链接失败与 ABI 漂移只能在 noemld 内部排查，难以形成可审计快照。

当前状态

当前状态：尚未发布可执行版本，命令行界面、参数和文件扩展名仍在设计中。当前公开信息包括计划用途、输入输出和安全边界；安装与调用说明将在首个可用版本发布时提供。

开发计划：与 GOBJ 和 noemld 共用符号 Schema、版本比较与可见性规则；随后验证对象、归档和 SSO 的稳定排序、未知扩展、损坏索引和 ABI Snapshot 回归。

主要能力

读取对象或归档
解析符号与版本
关联类型/契约/权限
筛选和稳定排序
输出清单与快照
  • 读取 GOBJ、SSO、归档与发布包的符号
  • 显示定义/未定义、local/global、weak/strong 和版本
  • 关联符号的类型、Section、Contract、权限与依赖
  • 提供稳定文本、POSIX-like 和 JSON 输出

输入与产物

方向对象作用
输入ArtifactGOBJ、archive、SSO 或 noempkg
输入Selectiondefined/undefined/exports/imports/versions/contracts
输入Display Profile排序、demangle、radix 和格式
输出Symbol Listing稳定排序的符号与属性
输出Undefined Summary未解析引用及来源成员
输出ABI Snapshot导出表、版本和 Contract 哈希

处理边界

  • 不解析或修改符号
  • 不预测链接器未给定 profile 时的最终选择
  • 不把符号存在等同于依赖可用
  • 不解码敏感 Artifact 内容

所有对象输入均视为不可信；ABI Snapshot 只是给定 Profile 下的导出观察结果，不自动证明语义兼容。无法解析的版本或 Contract 必须显式标记。
