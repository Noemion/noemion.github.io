工具简介

为什么训练必须可证伪

训练系统若只产出 checkpoint，而不能回答“使用了什么、为何这样训练、失败过什么、能否复现”，其结果就难以作为科研证据或工程基线。NSFE 还承担歧义保留、知识路由和封闭任务头等特定目标，不能仅用通用语言模型损失证明有效。

noemtrain 围绕预训练双向语言底座、层级 Source Graph 聚合、知识路由、稀疏语义图和封闭任务头组织可复现训练与教师蒸馏实验。它的核心产物是可复查的实验谱系，checkpoint 只是该谱系中的一个候选结果。

当前状态

设计阶段：当前未发布可执行程序；命令行接口、参数和文件扩展名尚未冻结。已确认内容限于工具职责、预期输入输出、安全边界和阶段门；具体调用方式以未来正式版本为准。

进入实验前的阶段门

训练实验必须先绑定已批准的数据快照、基线模型、任务假设、评估计划、资源预算和停止条件。语言底座从头预训练只有在现有底座无法满足已量化研究问题且独立评审通过证据门时才可立项，不能把“自主训练”本身当作研究贡献。

主要能力

  • 验证底座、数据、结构配置、任务头和 tokenizer 的契约兼容性
  • 编排 DAPT/TAPT、监督微调、多任务与蒸馏阶段，并支持独立消融
  • 控制冻结、解冻、采样权重和优化器状态，记录所有影响复现的随机源
  • 记录完整谱系、指标、资源、异常和失败状态，避免只保留成功实验

概要处理流

  1. 验证输入：核对配方、数据、基座、许可和评估计划的指纹与兼容性。
  2. 建立基线：先运行可比较基线，确认任务和度量链路有效。
  3. 分阶段训练：按预声明顺序训练、蒸馏或消融，并在阶段门保存状态。
  4. 记录谱系：持续记录代码、环境、随机性、资源、异常和人工决策。
  5. 提交候选：仅把满足停止条件的 checkpoint 连同完整 Manifest 交给独立评估。

输入与产物

方向对象作用
输入Training Recipe架构、阶段、损失、超参数、种子和停止条件
输入Dataset Snapshot由 noemdata 冻结且通过许可与泄漏审计
输入Base Model / Teacher版本、许可、tokenizer、结构和输出协议明确
输出Checkpoint Set权重、优化器状态、tokenizer 和结构配置
输出Training Manifest输入指纹、代码版本、环境、随机性和资源记录
输出Experiment Record分阶段曲线、异常、消融和候选选择证据

处理边界

  • 不自动决定从头预训练，也不在缺少独立证据门时扩大模型规模
  • 不使用盲测集反复调参，不以单一总体分数替代分层评估
  • 不允许训练产物直接生成 GOBJ，或绕过确定性的 Compiler Core
  • 不隐藏教师数据、教师行为、基座许可或外部服务依赖
  • 不删除失败、异常终止和负面消融记录；这些同样属于研究证据

模型输出和模型包一律视为不可信；它们不能决定二进制布局，也不能直接生成 GOBJ。训练可复现性也不等同于模型正确性，候选仍须接受冻结评测与下游安全验证。

尚未冻结的训练契约

配方 Schema、复现实验、蒸馏协议、故障恢复和资源核算仍需规范化。候选超参数、阶段顺序和默认训练策略不得在缺少对照实验与 ADR 时成为兼容性要求。
