工具简介

noemverify 按结构、语义、覆盖和信任四个独立层验证对象；任何未执行层都不得被表示为通过。

问题背景：格式可解析、语义有效、来源覆盖和密码学可信是不同命题。把它们压成一个布尔值会隐藏未执行检查和错误信任，因此验证器必须保存分层结论与证据。

当前状态

设计阶段：当前未发布可执行程序；命令行接口、参数和文件扩展名尚未冻结。页面仅说明已经确认的工具定位、预期输入与产物以及安全边界。

阶段门：Phase 1 先形成结构层与资源限制；语义和覆盖层依赖 GSIR/GOBJ 条款；信任层依赖签名、Merkle、撤销和 Profile 规范。只有请求层全部实现并通过对抗测试后，才能产生 Verified Handle。

处理边界

  • 不执行模型或 Runtime 来证明对象安全
  • 不自动下载缺失依赖
  • 不接受部分成功替代请求层的完整通过
  • 不根据扩展名、路径或签名存在本身建立信任

所有对象输入均视为不可信；偏移、长度、计数、索引和对齐计算必须使用 checked arithmetic。验证报告必须包含请求层、实际执行层、限制值、信任材料版本和失败位置，避免跨环境复用过期结论。
