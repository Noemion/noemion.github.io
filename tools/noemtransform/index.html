noemtransform · 对象复制与变换工具

执行 Section 复制、抽取、重命名和受控删除，并在每次变换后重建索引、对齐、哈希与签名状态。

Phase 1 / Phase 5Object Tools结构保持的对象变换Design Stage

它解决什么问题

noemtransform 执行 Section 复制、抽取、重命名和受控删除，并在每次变换后重建索引、对齐、哈希与签名状态。

对象变换会级联影响索引、引用、布局、完整性和签名；直接编辑字节容易产生表面可解析但语义或信任已损坏的对象。noemtransform 把允许的变换表达为可审计计划。

当前状态

当前状态：尚未发布可执行版本，命令行界面、参数和文件扩展名仍在设计中。当前公开信息包括计划用途、输入输出和安全边界；安装与调用说明将在首个可用版本发布时提供。

开发顺序：先稳定 Section/引用规则、语义哈希和对象写入器；每种变换都将验证前置条件、影响分析、原子提交、往返转换，以及失败时不产生部分对象。

验证源对象

验证与报告
  • 复制完整对象或选定 Section 到新对象
  • 抽取调试、来源、产物或模型元数据为补充对象
  • 重命名允许重命名的 Section 与局部符号
  • 规范化对齐、排序、内容寻址和构建元数据

它读取什么，产生什么

方向对象作用
输入Source Object已通过结构验证的 NOBJ/HOBJ
输入Transform Plancopy/extract/rename/remove/normalize 操作及顺序
输入Target 配置对象版本、字节序、对齐和发布/开发封装
输出Transformed Object重建目录、索引、引用和哈希的新对象
输出Change Manifest操作、受影响引用、旧签名失效状态和规范属性比较
输出Supplement Object通用抽取结果；当用于发布裁剪时必须采用 noemreduce 定义的 Debug Companion 身份、访问策略和消费者契约

它不会做什么

  • 不做语义级 strip 决策
  • 不直接修补任意偏移或原始字节
  • 不保留已被变换破坏的旧签名为有效状态
  • 不允许删除运行闭包必需 Section

所有对象输入均视为不可信；偏移、长度、计数、索引和对齐计算必须使用 checked arithmetic。变换必须先形成新对象并完成验证，再原子交付；失败时保留原对象不变。
