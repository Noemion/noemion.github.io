noemformat · 规范格式化工具

对 NSL 和文本 NIR 做纯表现层规范化，提供稳定格式作为评审、缓存和差异比较基础。

Phase 2 / Phase 3Text IRNSL / 文本 NIR 规范形式Design Stage

它解决什么问题

noemformat 对 NSL 和文本 NIR 做纯表现层规范化，提供稳定格式作为评审、缓存和差异比较基础。

无关的空白、顺序和字面量形式会制造评审噪声并破坏缓存与复现；但格式化器如果改变约束作用域或声明顺序，也会产生隐蔽语义变化。noemformat 必须证明只改变表现层。

当前状态

当前状态：尚未发布可执行版本，命令行界面、参数和文件扩展名仍在设计中。当前公开信息包括计划用途、输入输出和安全边界；安装与调用说明将在首个可用版本发布时提供。

开发顺序：先稳定语法树、注释附着和可交换声明规则；随后验证幂等性、格式化前后语义树等价、宽度与注释属性，以及不同环境产生相同输出。

它怎样工作

解析完整语法树
保留语义顺序
规范字面量与注释
稳定打印
幂等与差异检查
  • 统一缩进、换行、数字、字符串、标识符和注释位置
  • 按规范规则排序可交换声明，同时保留有语义顺序的结构
  • 提供 check、write、stdin/stdout 和差异输出模式
  • 识别 NSL 与 NIR 或接受显式语言参数

它读取什么，产生什么

方向对象作用
输入Source TextNSL 或文本 NIR
输入Format 配置行宽、规范版本和允许的语法扩展
输入Modecheck/write/diff/stdout
输出Formatted Text语义等价的规范表现
输出Format Diffcheck 失败时的最小文本差异
输出Syntax Diagnostics阻止安全格式化的解析错误

它不会做什么

  • 不重命名符号或改变绑定
  • 不删除看似冗余的约束
  • 不修复类型错误或补全缺失字段
  • 不把格式化成功解释为语义有效

源文本、include 和配置均视为不可信并受大小与深度限制。无法建立完整语法树时应报告诊断，不以局部重排覆盖错误区域。
