项目/工具/morphe

morphe · NIR 汇编器

把规范化文本 NIR 编码为可重定位 NOBJ，严格解析显式语义而不补全缺失意图。

Phase 2Text IR文本 NIR → NOBJDesign Stage

它解决什么问题

morphe 把规范化文本 NIR 编码为可重定位 NOBJ，严格解析显式语义而不补全缺失意图。

文本 IR 是规范评审、基准样例测试和对象复现的可读桥梁；若汇编器自行推断缺失意图，文本将无法作为确定性对象的权威输入。morphe 只编码显式声明并报告缺口。

当前状态

当前状态：尚未发布可执行版本，命令行界面、参数和文件扩展名仍在设计中。当前公开信息包括计划用途、输入输出和安全边界；安装与调用说明将在首个可用版本发布时提供。

开发顺序：先确定文本 NIR 语法、规范化规则、最小对象结构定义和重定位类别；随后验证文本—对象—文本规范化往返转换、确定性字节和完整负例语料。

它怎样工作

解析规范文本
绑定局部符号
检查类型与限制
生成重定位
确定性写入对象
  • 解析文本 NIR 的层、节点、边、约束、符号和产物声明
  • 解析局部符号并为外部符号生成重定位
  • 生成字符串、符号、NIR、重定位和调试映射 Section
  • 执行语法、类型、范围和限制检查

它读取什么，产生什么

方向对象作用
输入Text NIR Source PackageUTF-8 规范文本、include 受控源、内容身份与来源根
输入Assembly 配置对象版本、命名空间、目标 ABI 和限制
输入Debug Options源位置、注释和构建注记保留策略
输出Relocatable NOBJ带符号、重定位和可选调试信息的对象
输出Assembly Evidence Ledger每项文本声明到 NIR 节点、对象记录与调试映射的确定性对应
输出Build Manifest + Dependency File输入摘要、配置、工具版本、include、结构定义和本体库依赖
输出Diagnostics稳定错误码、源范围和修复提示

它不会做什么

  • 不从自然语言推断节点或关系
  • 不自动选择唯一歧义解释
  • 不链接外部定义
  • 不产生绕过规范 writer 的手工二进制布局

所有源依赖和生成对象均视为不可信；include 必须受根目录、内容摘要和循环/深度限制约束，偏移、长度、计数、索引和对齐计算必须使用 checked arithmetic。
