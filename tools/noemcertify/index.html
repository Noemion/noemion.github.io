noemcertify · 规范一致性检查工具

运行格式、语义、基准样例、畸形样例和跨工具一致性测试，并生成可以复查的一致性报告。

Phase 0 / Phase 8Conformance规范套件与一致性报告Design Stage

它解决什么问题

noemcertify 把格式规范、语义规则、基准样例 Corpus、畸形样例 Corpus 和跨工具一致性要求转化为可重复检查的结果。

单个工具“能运行”不能证明工具链遵守同一规范；若没有条款级测试、独立语料和环境指纹，实现可能把自身行为误当作标准。该工具负责把规范、实现与可复现证据连接起来。

当前状态

当前状态：尚未发布可执行版本，命令行界面、参数和文件扩展名仍在设计中。当前公开信息包括计划用途、输入输出和安全边界；安装与调用说明将在首个可用版本发布时提供。

开发顺序：先建立规范条款 ID、能力配置、语料许可和报告结构定义，再实现可执行规范子集。验证阶段将加入独立实现或独立验证器的交叉结果；这些能力目前尚未完成。

它怎样工作

选择规范/配置
锁定工具与环境
运行正反语料
关联条款与诊断
生成证据包
  • 加载带版本的规范清单并选择适用测试套件
  • 运行对象、文本 IR、链接、strip、信任和可复现构建的一致性用例
  • 把每个失败绑定到规范条款 ID、输入位置和稳定诊断码
  • 输出人类可读摘要与机器可读 JSON/CBOR 一致性报告

它读取什么，产生什么

方向对象作用
输入Conformance 清单规范版本、目标平台、必选/可选能力与工具指纹
输入Test Corpus基准样例、畸形样例、边界值、历史回归和互操作样本
输入Toolchain Build待检查的 noesis、synthesis 和对象工具及其构建信息
输出Conformance Report条款级通过、失败、跳过、证据和环境指纹
输出Reproduction Bundle最小失败输入、命令、期望值与实际值
输出Certification Summary能力矩阵、阻断项和发布资格证据；名称沿用工具产物术语，但不构成第三方认证、发布授权、签名授权或会话验收

它不会做什么

  • 不修复不合规对象或源码
  • 不替代 noemvalidate 的单对象验证
  • 不根据实现行为反向修改规范
  • 不允许跳过必选测试后仍标记为 conformant

所有对象输入均视为不可信；偏移、长度、计数、索引和对齐计算必须使用 checked arithmetic。一致性结论必须声明规范版本、配置、未执行项和环境；“部分通过”不能简写为“兼容”，也不构成第三方认证或发布授权。
