noemexecute · 可信装载与执行

验证 HOBJ 包、建立受策略约束的装载映射，并把已验证的 NIR 交给 Fulfillment Runtime 求解与实现化；拒绝任何越过对象边界或披露策略的执行。

Phase 6Release & Fulfillment RuntimeHOBJ Package → Run ResultDesign Stage

它解决什么问题

noemexecute 验证 HOBJ 包、建立受策略约束的装载映射，并把已验证的 NIR 交给 Fulfillment Runtime 求解与实现化；拒绝任何越过对象边界或披露策略的执行。

验证通过的对象仍需要与具体模型、外部能力、预算和会话状态安全绑定。noemexecute 是“已验证静态对象”进入“受控动态求解”的信任转换点，必须防止运行期绕过对象契约。

当前状态

当前状态：尚未发布可执行版本，命令行界面、参数和文件扩展名仍在设计中。当前公开信息包括计划用途、输入输出和安全边界；安装与调用说明将在首个可用版本发布时提供。

开发顺序：先稳定装载器、运行配置、披露 ABI、能力授权、状态哈希与 Fulfillment Runtime 契约；随后测试篡改、回滚、越权披露、预算耗尽、状态漂移和模型或外部能力失败。

它怎样工作

验证发布包
建立装载与权限映射
创建隔离会话
暴露类型化能力
受控披露与求解
验收结果并记录证据
  • 验证包签名、清单、Section 摘要和依赖闭包
  • 按 Segment 与权限建立只读、可执行和私有运行映射
  • 为 Agent Harness 建立隔离会话、能力句柄、预算和人工升级条件
  • 执行渐进式披露，维护来源、观察和会话状态
  • 调用 Fulfillment Runtime 求解并记录结构化结果

它读取什么，产生什么

方向对象作用
输入Signed HOBJ Package由 noembundle 生成且可完整验证的发布包
输入运行配置能力、预算、目标环境、披露、停止、人工升级与审计策略
输入Fulfillment Runtime Context显式传入的任务上下文和允许访问的产物
输出Run Result结构化状态、结果对象和失败分类
输出Session State披露游标、预算消耗、能力绑定、观察和派生产物引用
输出Run Report验证链、装载映射、能力调用、策略决定和运行摘要

它不会做什么

  • 不修复损坏或不兼容的对象
  • 不绕过 noemvalidate 的拒绝结论
  • 不让模型修改二进制布局或加载策略
  • 不把运行期产物静默写回签名包

所有对象、模型输出、外部能力结果和运行上下文均视为不可信。运行失败、未满足结果和策略拒绝必须分类记录，不能把“模型生成了内容”当作验收契约已满足。
