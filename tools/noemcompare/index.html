noemcompare · 语义差异工具

比较两个对象或文本 IR，并把差异分类为布局、调试、运行语义、依赖、权限和信任元数据变化。

Phase 2 / Phase 5Text IR对象与 IR 的分类差异Design Stage

它解决什么问题

noemcompare 比较两个对象或文本 IR，并把差异分类为布局、调试、运行语义、依赖、权限和信任元数据变化。

字节不同不一定意味着运行语义变化，字节相近也不能证明权限、依赖或约束等价。发布、复现和研究实验需要一个明确区分“相同、改变、未知”的语义差异工具。

当前状态

当前状态：尚未发布可执行版本，命令行界面、参数和文件扩展名仍在设计中。当前公开信息包括计划用途、输入输出和安全边界；安装与调用说明将在首个可用版本发布时提供。

开发顺序：先完成规范化文本、语义哈希、稳定符号和差异分类规则；随后验证 ID 重排、等价布局、真实语义变化、权限扩大、未知扩展和资源受限图匹配。

它怎样工作

识别输入层次
规范化稳定标识
匹配图与依赖
分类差异
给出等价/改变/未知
  • 比较 NSL、文本 NIR、NOBJ、HOBJ 与发布包
  • 通过稳定符号和图同构消除 ID、顺序和偏移噪声
  • 输出 layout/debug/runtime/trust 分类差异
  • 提供发布门使用的允许差异策略

它读取什么，产生什么

方向对象作用
输入Left/Right 产物文本或二进制对象，可为不同封装
输入Diff 配置关注层、允许差异、规范版本和资源限制
输入Symbol Hints可选的重命名映射或链接 map
输出Classified Diff带路径、旧值、新值和严重度的差异
输出Semantic Verdictequivalent/changed/unknown
输出Machine ReportCI、noemreduce 和发布审计可以直接读取

它不会做什么

  • 不把未证明等价报告为等价
  • 不忽略硬约束、权限或依赖变化
  • 不自动合并冲突对象
  • 不以文本 diff 替代语义比较

所有对象输入均视为不可信；图匹配、递归和候选映射必须有资源上限。无法在配置允许范围内证明等价时输出 unknown，而不是以超时或启发式相似度替代结论。
