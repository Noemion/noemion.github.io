theoria · 对象检查器

以只读方式检查对象的链接视图、装载视图、NIR 图、覆盖与披露元数据，承担 Noemion 的 readelf/objdump 角色。

Phase 1Object ToolsNOBJ / HOBJ 结构视图Design Stage

它解决什么问题

theoria 以只读方式检查对象的链接视图、装载视图、NIR 图、覆盖与披露元数据，承担 Noemion 的 readelf/objdump 角色。

二进制对象如果只能由编译器和 Fulfillment Runtime 解释，开发者、安全评审者和研究者无法核对布局、引用、来源和策略。theoria 提供稳定、受限且不执行对象的观测面。

当前状态

当前状态：尚未发布可执行版本，命令行界面、参数和文件扩展名仍在设计中。当前公开信息包括计划用途、输入输出和安全边界；安装与调用说明将在首个可用版本发布时提供。

开发顺序：先确定最小 Header、Section、Symbol 结构定义和安全读取器契约；随后验证畸形对象下的有界输出、字段级基准样例视图和结构化输出结构定义一致性。

它怎样工作

受限读取
选择视图
解析引用
限制深度与预算
稳定输出
  • 显示 Header、Section、Segment、String、Symbol、Relocation 和 Note
  • 按规范定义的高层、规范化、后端适配或行为视图查看节点、边、约束与验收契约
  • 显示 HOBJ 依赖闭包、披露图、模型与策略指纹
  • 支持稳定文本、JSON 和限定深度的图输出

它读取什么，产生什么

方向对象作用
输入NOBJ / HOBJ / Signed Noemion Package本地文件、stdin 或签名包内具名成员；Package 必须明确是最终签名包
输入View Selectionheader/sections/symbols/relocs/nir/coverage/disclosure
输入Resource Limits最大节点数、递归深度、字符串与输出预算
输出Text View面向开发者的稳定字段与偏移视图
输出Structured ViewJSON/CBOR 结构数据，字段名与结构定义对齐
输出Graph Extract受限 DOT 或节点边清单

它不会做什么

  • 默认不执行、装载或修复对象
  • 不把可解析等同于验证通过
  • 不绕过 noemvalidate 给出信任结论
  • 不在损坏计数驱动下进行无界内存分配

所有对象输入均视为不可信；偏移、长度、计数、索引和对齐计算必须使用 checked arithmetic。输出必须区分“原始声明”“解析推导”和“验证结论”，避免查看工具无意中为损坏对象背书。
