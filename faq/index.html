名称与定位

Noemion 与 Endem 有什么区别？

Noemion 是项目和生态品牌；Endem 是项目创造的工程术语，表示最小、独立有效、可验证的期望终态单元。工具和格式不需要批量携带 Noemion 前缀。

为什么不用“某某 OBJ”？

因为目标、约束、歧义和验收不是机器指令对象的另一种载荷。Endem 借鉴可链接制品思想，但用新名词建立新本体，避免名称反过来把设计限制在传统目标文件模型里。

Endem 这个词怎样形成？

它由 end 与表示最小区别单位的构词成分 -eme 合成，工程定义为最小期望终态单元。正式软件发行前仍需完成专业商标检索。

制品与应用

Endem 包含什么？

五组显式语义：say 来源表达、aim 唯一根终态、must 硬边界、done 验收、open 未决项。它不是提示文本的二进制包装。

Weave、Frame 和 Witness 是什么？

Weave 是已解析的 Endem 与固定依赖闭包；Frame 是一次运行的不可变加载态，不是文件；Witness 是绑定对象、环境和策略的证据与决定记录。

为什么只保留一个 CLI？

现有职责没有 23 个独立用户、权威或生命周期。唯一入口 endem 用短子命令组织职责；只有 Reader 与 Runner 因独立证据和权限边界保持独立实现/进程。

一个 CLI 会不会破坏安全隔离？

不会。endem see 由单独构建的 Reader 实现，endem run 启动隔离进程。统一的是用户入口，不是解析代码、verified handle 或实时能力句柄。

安全与模型

模型能直接生成 Endem 吗？

不能。模型只能生成带来源和未决状态的候选；Core 依据确定性规则或具名 Source Binding Decision 形成规范字节。

模型能调用工具或宣告完成吗？

模型只能提出能力请求和候选结果。Harness 的确定性策略决定实际调用，done 预先固定验收，Witness 记录真实证据，指定权威形成最终决定。

Deterministic 路径会调用模型吗？

不会。形成、验证、绑定、布局、摘要、签名范围和规则型验收都必须可在无模型环境复现。

状态与实施

现在可以安装吗？

不可以。当前处于规范与安全核心设计阶段，没有正式版本或有效安装命令。

第一阶段实现什么？

只实现 form、check、独立 see 和 test，证明双 Reader、畸形输入关闭失败、资源有界和双环境逐字节复现。

会自建模型训练平台吗？

默认不会。数据、训练、评估和量化优先适配成熟外部栈；只有出现 Noemion 特有且不可替代的需求，并通过增益实验，才新增研究适配面。
