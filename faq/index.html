常见问题

直接回答 Noemion 的项目定位、Endem 等工程名词、应用拓扑、安全边界和当前成熟度。

NoemionEndemSecurity

名称与定位

Noemion 与 Endem 有什么区别？

Noemion 是项目、新领域与社区的总名；Endem 是 Noemion 定义的工程术语，表示最小、独立有效且可验证的期望终态单元。工具和格式不需要批量携带 Noemion 前缀。

为什么不用“某某 OBJ”？

因为目标、约束、歧义和验收不是机器指令对象的另一种载荷。Endem 借鉴可链接制品思想，但用新名词建立新本体，避免名称反过来把设计限制在传统目标文件模型里。

Endem 这个词怎样形成？

它由 end 与表示最小区别单位的构词成分 -eme 合成，工程定义为最小期望终态单元。正式软件发行前仍需完成专业商标检索。

制品与应用

Endem 包含什么？

六个语义面：rhem 来源记号、semion 意义投影、skena 一个根中性可能事态、telis 目标方向、krin 结构比较契约、apor 未决边界。它不是提示文本的二进制包装。

Synem、Dromen 和 Iknem 是什么？

Synem 是多个 Endem 解析引用后形成的组合闭包。Dromen 是 Drasor 为一次 Drase 会话封存的只读执行契约，不是文件、凭据包或可恢复权限。Iknem 是绑定主体、环境和声明范围的证据，其中 phain 用与 skena 对齐的关系结构记录观察。

为什么只保留一个 CLI？

现有职责没有 23 个独立用户、权威或生命周期。唯一入口 endem 用短子命令组织职责；只有 Theor 与 Drasor 因独立证据和权限边界保持独立实现/进程。

一个 CLI 会不会破坏安全隔离？

设计上不会，但当前还没有 CLI 或组件实现。未来 endem theor 必须由单独构建的 Theor 承担，endem drase 必须进入隔离的 Drasor 进程；统一的只能是用户入口，不能是解析代码、生产验证句柄或实时能力句柄。

安全与模型

模型能直接生成 Endem 吗？

不能。模型只能生成带来源和未决状态的候选；Ktisor 依据确定性规则或具名语义决定形成规范字节。

模型能调用工具或宣告完成吗？

模型只能提出能力请求和候选结果。控制平面的确定性策略决定实际调用；krin 预先固定比较条件和决定权威；Iknem 以 phain 记录结构化观察。最终决定仍由指定权威作出。

自然语言候选由谁确认？

只有落在 AUT-CORE 精确政策、对象、动作和范围内的具名权威或版本化确定性规则可以确认。模型置信度、结构合法、签名、登录和单次点击都不够；委托、多人批准、同意、撤销与重放也必须单独核对。

文件里写了权威名称为什么还不能证明授权？

因为名称只能定位待核对的权威，不能证明该权威针对这份精确内容作出了有效决定。独立 .endem 最多通过容器和 Profile 检查；完整内容接受还要核对外部授权决定的政策、主体、对象、范围、时效与精确绑定。这个外部决定也不会改变 Endem 的内容身份。

确定性路径会调用模型吗？

不会。形成、验证、绑定、布局、摘要、签名范围和规则型验收都必须可在无模型环境复现。

状态与实施

现在可以安装吗？

不可以。当前处于规范与安全核心设计阶段，没有正式版本或有效安装命令。

第一阶段什么时候开始并计划实现什么？

当前不会开始。只有用户明确开启代码开发阶段并重新确认仓库与范围后，首条实现路径才计划限制在 ktise、elenk、独立 theor 和必需的内部一致性门禁。

会自建模型训练平台吗？

默认不会。数据、训练、评估和量化优先适配成熟外部栈；只有出现 Noemion 特有且不可替代的需求，并通过增益实验，才新增研究适配面。
