项目定位

Noemion 是什么？

Noemion 是面向生成式计算的表征对象与工具链设计，目标是让目标和约束可被机器分析、验证、重定位和执行。对普通使用者，它意味着任务要求能够被明确保存和检查；对开发者，它意味着可定义对象、符号、链接、装载和验证契约。更多背景见项目背景。

Noemion 当前是产品、研究课题还是标准？

当前首先是规范与工程研究项目，处于概念细化和安全二进制核心设计阶段。它可以发展为博士研究、论文、专利、软著或标准化候选，但这些方向都需要独立证据和审查，不能把愿景表述为已经取得的成果。

架构设计

NIR、NOBJ 和 HOBJ 分别负责什么？

NIR 表达目标、约束、歧义、证据和验收条件；NOBJ 把这些语义封装为可重定位、可验证的对象；HOBJ 保存可按依赖、权限和任务逐步披露的共享内容。三者的关系见架构设计，精确约束以规范与成熟度页面所指向的权威文件为准。

Agent Harness 和 Fulfillment Runtime 有什么区别？

Agent Harness 是运行外侧的控制平面，管理上下文、能力、策略、观察、预算和人工升级；Fulfillment Runtime 在已验证对象与运行配置下生成并评价候选结果。模型不能绕过 Harness 直接调用能力，Runtime 也不能修改对象约束。可分别阅读 Agent Harness 与 Fulfillment Runtime。

尚未决定的架构事项在哪里？

对象命名、ABI、模型工程和硬件条件等未决事项保留在开放问题中。

使用与安装

现在可以下载安装吗？

不可以。项目处于最小规范与安全二进制核心设计阶段，尚未开始核心实现；下载页会如实标注资源状态。

安全与确定性

Deterministic Profile 会调用模型吗？

不会。Deterministic Profile 必须无模型、确定且可复现；具体约束以规范与成熟度页面所指向的权威文件为准。

模型能直接生成 NOBJ 吗？

不能。一切模型输出均视为不可信，模型不得决定二进制布局或直接生成 NOBJ；规范依据见规范与成熟度页面。

开发与贡献

当前优先开发什么？

当前正在完成最小规范与安全二进制核心设计，下一步开发 Checked 读取器/对象写入器、结构验证器和对应的恶意对象测试。实时状态见当前项目阶段。

项目按什么顺序开发？

先完成规范和安全二进制核心，再开发 Deterministic 编译链接、发布签名、可信装载、Agent Harness 与 Fulfillment Runtime，最后进入模型工程。每一步都会明确交付内容和验证结果，具体顺序见开发路线图。

论文、专利和标准材料可以直接复用网站内容吗？

可以把网站作为问题、架构和追溯入口，但引用时必须记录版本和成熟度，并核对权威规范、ADR、实验或测试证据。专利公开前需要新颖性与权属审查；标准提案需要可测试条款、互操作案例和安全分析。

常见误解

NIR 是二进制化提示词或 Skill 包吗？

不是。NIR 必须表达可机器分析、验证、重定位和执行的目标与约束；规范定义见规范与成熟度页面。

借鉴 ELF 是否意味着兼容 ELF 或执行机器指令？

不是。项目借鉴 Section/Segment、符号、重定位、链接、调试和装载等成熟工程策略，但对象语义面向生成式目标与约束，不承诺传统 ELF ABI 或机器指令兼容。
