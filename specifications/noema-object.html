Noema Object

把 Noema IR 封装为可重定位、可验证和可安全装载的对象文件。

Current BoundaryUnfrozen LayoutNOBJChecked Arithmetic

对象职责

直白解释：NOBJ 类似传统工具链中的可重定位对象：它把已经检查过的语义、符号和待解析引用装进统一边界，使查看器、链接器和装载器能够在不重新猜测自然语言的前提下处理任务。

  • Noema Object（NOBJ）指类似 .o 的可重定位生成式对象；文件扩展名和版本编码仍待规范确定。
  • 对象格式组织 NIR、符号、资源、策略、调试与信任信息，并支持验证、链接、重定位和装载。
  • Noema Object System 负责链接、验证和装载；Fulfillment Runtime 而非对象格式负责生成式求解。
  • 自然语言原文只能作为可选来源或调试信息，不能成为核心执行载荷。

安全解析

所有对象输入均不可信。读取器、验证器、链接器与装载器必须对偏移、长度、计数、索引、对齐、重叠、递归深度和引用闭环执行 checked arithmetic 与显式限制；不能产生部分可信结果。

“签名有效”只证明签名范围内字节来自相应密钥持有者且未被篡改，不证明记录边界安全、语义兼容或权限合理；结构、语义与信任验证必须分层执行，不得用任一层的通过替代其他层。

详细设计基线：物理排布

NOBJ 采用前置目录与长度限定 Section。目录前置使读取器在读取大型载荷前先检查计数、偏移、长度、对齐、重叠和资源预算；Section 服务链接和对象工具，Segment 服务装载与权限映射。当前排布顺序是：

  1. 固定大小 Preamble 与 Header；
  2. 可选 Load Directory；
  3. Section Directory；
  4. 字符串、类型和结构定义字典；
  5. NIR 核心记录 Section；
  6. 符号、重定位、依赖和策略 Section；
  7. 来源映射与构建证据 Section；
  8. 可裁剪调试 Section；
  9. 完整性目录与分层摘要材料。

固定记录 Section 声明 entry_size 与 entry_count；可扩展语义记录自带 record_size。最终发布包的签名范围和外部签名回填仍属于 noembundle，不由 NOBJ Header 代替。

详细设计基线：Header 字段

字段组逻辑字段验证重点
身份magic、format_major、format_minor、header_size、object_kind魔数、格式族、兼容主版本和实际 Header 长度一致。
编码encoding_profile、flags、required_features未知必需特性失败；禁止把未声明字节序或宽度按本机默认解释。
Section 视图Section Directory 偏移、数量、表项大小、字符串表索引count × entry_size、偏移加长度和索引全部 checked。
装载视图Load Directory 偏移、数量与表项大小无装载视图时全部为规范空值；存在时 Segment 不得越权或重叠。
完整性完整性 Section 索引与完整性 Profile 标识Header 不保存递归覆盖自身的完整文件摘要；未知必需 Profile、循环覆盖或自引用必须失败。

当前实现原型候选为小端、64 位 checked offset/length 与 32 位表索引；它只是研究起点，尚未冻结为 ABI。

详细设计基线：Section Directory 字段

字段作用拒绝条件
name_ref指向受验证字符串表；名称只用于显示。索引越界、字符串编码无效或名称与类型约束冲突。
section_kind / schema_version决定载荷结构与版本。未知 required 类型或不兼容主版本。
flagsrequired、alloc、readonly、merge、debug、compressed 等属性。互斥标志组合、可写关键策略或压缩 alloc Section。
file_offset / stored_size / logical_size定位文件载荷并限定解压后大小。越界、重叠、解压放大超限或逻辑大小不一致。
alignment声明载荷和记录所需对齐。非规范值、上取整溢出或实际偏移不满足。
entry_size / entry_count交叉检查固定记录表；非表 Section 为零。乘法溢出、载荷大小不匹配或可变记录伪装成固定表。
link_section / info关联字符串表、符号表、目标 Section 或类型特定信息。错误类型关联、循环依赖或目标索引越界。
digest_ref指向完整性目录的载荷摘要。摘要缺失、算法不允许或摘要与实际字节不一致。

详细设计基线：Load Directory 字段

Load Directory 是可选装载视图，不保存传统进程虚拟地址，也不允许 Runtime 直接把对象字节映射为任意能力。每项 Segment 记录至少承担以下逻辑职责：

字段作用拒绝条件
segment_kind / schema_version选择装载视图与解释版本。未知必需种类、版本不兼容或对象种类不允许该 Segment。
flags声明 readonly、mutable-template、AI-NX 等权限与处理属性。权限比所属 Section 或策略更宽、互斥标志并存或调试内容进入运行视图。
section_set_ref引用经过验证的 Section 索引集合。成员越界、类别不允许、非法重复或通过集合绕过 required Section。
logical_size / alignment限制解码后工作集与装载对齐。累计溢出、超过预算、对齐非法或与成员载荷不一致。
policy_ref / budget_ref关联权限策略与资源上限。引用类别错误、策略缺失或 Segment 声明试图扩大授权。
integrity_ref关联该装载视图的完整性材料。摘要范围缺失、覆盖循环或实际成员与已验证集合不一致。

详细设计基线：链接与证据记录

记录最低逻辑字段验证重点
Symbolnamespace/name ref、kind、linkage class（local/export/import）、visibility、definition state、section/record/type/version ref、resolution/missing policy、size or arity、origin ref、stable-key digest名称不充当身份；可选性与缺失行为显式声明；多个定义不能按输入顺序或隐式弱绑定选择。
Relocationtarget section/record/field、kind、symbol ref、expected type、encoding/width、optional addend、overflow policy、origin ref只能改写声明字段；未知类型、错位、类型变化、溢出和重复应用失败。

Originsubject range、source digest、source unit/span、derivation class、rule/model identity、decision digest、authority、evidence refs确认语义可追溯；候选、规则推导和授权决定保持不同确认等级。
Build Evidencebuild type、input roots、parameter/config digest、resolved dependencies、toolchain identity/digest、reproducibility profile、output digest构建定义与单次运行信息分开；时间、绝对路径和 invocation ID 不污染对象语义。
Integrity Entrysubject kind/index、stored/logical coverage、algorithm/profile、digest、tree parent/path ref、coverage flags覆盖范围无循环；压缩前后摘要对象明确；摘要成功不替代结构、语义和策略验证。

字段名称表达逻辑职责，不冻结最终拼写、顺序、宽度或数值枚举。Symbol、Relocation 与 Dependency 面向链接；Origin、Build Evidence 与 Integrity 面向追溯、复现和验证，后者不能被 Runtime 当作任务语义。

详细设计基线：Section 家族

逻辑 Section职责发布处理
nir.types类型与结构定义注册表。运行或链接需要时保留。
nir.nodes / nir.edges目标、状态、行动与类型化语义关系。运行语义，必须保留。
nir.constraints / nir.ambiguities硬软约束、未知策略、替代解释与处理状态。不得因裁剪改变强度或删除未决状态。
nir.acceptance / nir.capabilities / nir.artifacts验收、能力需求和输出契约。Runtime 或策略需要时保留；不包含实时句柄。
strings / symbols / relocations显示名称、导入导出、可见性与待链接引用。链接后按对象类型保留或重建。
dependencies / policies内容寻址依赖、版本、权限和处理规则。必需依赖与安全策略必须保留。
origins确认语义的最小来源摘要、跨度和决定引用。覆盖证明完成前不得删除；发布形态至少保留可追溯摘要。
build输入摘要、配置、工具/模型指纹和已解析依赖。不进入 Runtime 语义；按发布策略保留或外置。
debug.source原始表达、注释、完整候选和详细诊断映射。可以进入 Debug Companion，但删除前必须证明不改变运行语义。
integritySection 摘要、Merkle 路径和对象根材料。由对象种类和发布封装规则决定，不能用签名替代结构验证。

这些名称描述职责，不冻结最终字节名称或 Section 编号。未知可选调试 Section 可以跳过；未知且影响语义、链接、权限、验收或装载的 Section 必须拒绝。

设计方案：记录与完整性

节点和边记录需要区分两类身份：对象内部局部索引可以在链接时重编号；跨对象、版本和重定位保持的语义身份或稳定符号不能因布局变化而改变。类型、标志、引用、模态、权重、来源关系、字段宽度、编码、对齐、版本演进、Section 名称和 Merkle 层级尚须正式格式规范与样本共同确定。

文件摘要、完整性根、语义身份与签名

四种属性服务不同消费者，不能共用一个含义模糊的“对象哈希”。完整 NOBJ 的 Exact File Digest 由对象外部对全部文件字节计算，因此不会递归写回自身；它用于内容寻址、构建比较和发布清单，不证明语义可接受。

属性覆盖与用途不得推导的结论
Exact File Digest对象外部计算的完整文件字节摘要；任何字节变化都会改变结果。不能证明两个不同布局对象语义不等价，也不能证明发布者身份。
Integrity Rootintegrity Section 按 Profile 覆盖 Header/目录职责字段和非自身 Section 的 stored/logical digest；根记录及其存储字节不覆盖自身。未由受信外部摘要或签名认证时只能证明内部自洽，不是信任锚；也不能替代范围、类型、策略与权限验证。
Semantic Identity未来基于冻结的规范化表示和等价规则建立。当前阶段不得用未定义语义哈希宣告 strip、重定位或布局变化等价。
Package Subject Digest / Signature发布层绑定不可变候选载荷、清单与外部签名身份；Signature Envelope 位于被签载荷之外。签名有效不等于对象语义正确或当前环境授权。

攻击者可以同时修改裸 NOBJ 载荷并重新计算对象内 Integrity Root。来源真实性和抗恶意篡改必须由受信锁定清单中的 Exact File Digest，或发布层 Package Subject Digest / Signature 绑定；独立验证器不得仅因内部根匹配就提升对象的信任状态。

Debug Companion 必须同时绑定主对象与伴随文件的完整内容摘要。GNU GDB 的 separate debug files 提供了分离调试资料的工程先例，但其 build ID 与 debug-link CRC 只用于关联或查找，不能替代内容摘要和签名。参考原文：https://www.sourceware.org/gdb/current/onlinedocs/gdb.html/Separate-Debug-Files.html

与 HOBJ 和工具链的关系

Noesis Core 或 morphe 通过规范对象写入路径产生可重定位 NOBJ；synthesis 可以把一个或多个 NOBJ 与已有 HOBJ 链接为 Linked Object 或新的 HOBJ。HOBJ 是保留共享依赖与披露语义的对象形态，不由 Horizon Engine 直接生成。发布前还必须经过独立验证、允许的裁剪、Release Coverage Proof 和外部签名封装。

待定事项：名称、布局与重定位

物理顺序、逻辑字段职责、Section 家族和非循环摘要域已作为详细设计基线采用；NOBJ 的文件扩展名、魔数字节、字段宽度、Section 编号、压缩与摘要算法、完整性描述符编码、ABI 数值和 R_NIR_* 等重定位标识仍未确定。当前示例只表示需要覆盖的引用类别，不能作为实现接口。参见开放问题。

设计概要：对象不变量与一致性测试

不变量必须拒绝的反例主要验证方式
边界与算术安全溢出、越界、重叠、错误对齐、递归或计数放大畸形样例语料、属性测试和覆盖引导模糊测试
引用完整错误索引、悬空 required import、类型不匹配和循环闭包分层验证器与链接冲突矩阵
确定性工作目录、归档顺序、时间戳或未锁定依赖污染字节双构建复现和规范化往返转换
语义保持重定位、strip 或布局重排改变目标、约束、权限或契约规范属性比较、差异分类和前后等价测试；语义哈希需等待等价规则冻结
完整性范围未覆盖关键 Header、策略、入口、符号或依赖签名范围测试、篡改与回滚案例
