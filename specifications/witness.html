项目/规范参考指南/Witness

Witness

记录一项来源、绑定、构建、签名、运行或决定依据，并把它绑定到精确对象和环境。

Evidence RecordNot Formal ProofTraceable

Witness 能证明什么

Witness 回答“谁在什么条件下观察或判断了什么，以及这项记录绑定到哪些对象”。它使依据可定位、可验证完整性、可比较和可撤销，但不自动证明外部事实为真，也不自动形成任务通过结论。

不得混淆：Witness 不是数学证明，不是模型自述，不是日志数量，也不是 Acceptance Decision。证据齐全只表示决定者拥有规范要求的输入；满足判断与最终结论仍由 when 中预先声明的规则和权威作出。

最低字段职责

字段职责拒绝条件
kind / version来源、form、check、bind、pack、seal、run、decision 等记录类别和结构版本。未知关键类别或不兼容版本。
subjectEndem、Weave、Frame、产物、能力调用或契约项的精确身份。摘要缺失、对象状态不匹配或引用悬空。
producer规则、工具、环境、人工或外部系统的身份与版本。无法识别生产者或把模型冒充决定权威。
claim记录实际声称的有限结论，例如“此字段通过范围检查”。结论超出生产者权限或没有可检查语义。
basis输入摘要、规则、观察、参数、时间域和关联 Witness。缺少必需输入、循环自证或把不可重放对话当作依据。
strength直接观察、确定性推导、外部声明、模型候选或人工判断等证据等级。等级与来源不一致，或通过置信度提升权限。
integrity记录摘要、序列、父记录、签名或防篡改声明。链断裂、重复序号、摘要不符或完整性能力被夸大。

Frame 与运行记录

Runner 从 sealed Endem/Weave、执行策略、授权能力和当前环境建立 Frame。Frame 只存在于会话内；Witness 只保存足以重放判断的身份、参数和观察，不序列化实时句柄、凭据、模型私有状态或整个内存视图。

一次运行可以把相关 Witness 封装为单一 Run Record，内部仍必须区分原始事件、规范化观察、完整性声明、证据闭包和最终决定。Runner 不得因为自己产生了记录就默认信任记录内容。

隐私、最小披露与撤销

  • Witness 只保存验收与审计必需的字段；敏感原文、凭据和高风险工具输出按策略摘要、脱敏或外置。
  • 脱敏必须声明损失及其对证据强度的影响，不能用“已脱敏”掩盖无法验证。
  • 来源撤回、密钥撤销、环境身份变化或规则版本变化必须使受影响的决定可定位并失效。
  • 时间戳只说明记录时间域，不进入 Endem 的确定性语义身份。
  • 模型生成的说明可以作为低等级 Witness 内容，但不能成为唯一的权限、完整性或最终验收依据。

当前成熟度

Witness 的有限主张、身份绑定、证据等级和“证据不等于证明”边界已经采用。事件编码、时间域、签名、隐私 Profile、撤销协议、跨实现归并和 Run Record 物理格式仍需威胁模型与测试。
