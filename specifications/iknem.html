项目/规范参考指南/Iknem

有范围证据记录
Iknem

记录谁在何时、用什么方法观察了哪个精确对象，并限定这项观察最多支持什么结论。

IKN-CORE 0.1.0-draft主张必须有范围尚无物理格式尚未实现

先用一次发布判断理解它

有范围证据记录（设计阶段名称 Iknem）只回答一件事：某个生产者通过已说明的方法，对一个精确对象观察到了什么，这项观察最多支持哪一句结论。

假设目标是“更新服务依赖，并确认当前版本可以发布”。锁文件、部署接口、健康探针和模型说明都能提供信息，但它们回答的问题不同。开发者应先保留各自的对象、方法和时间，再判断这些记录是否适用于当前目标。

取得的信息这项记录可以声称什么仍需补充什么
锁文件中的版本与摘要指定解析器从该文件中读到了这组依赖版本和摘要实际部署对象和运行状态
部署接口返回成功指定接口在该时刻接受了请求变更是否生效、目标实例是否正确
精确实例的健康探针响应该实例在声明方法和时刻满足了探针条件判据要求的完整观察窗口
模型对日志的解释保存为待核查的 model-candidate原始观察、变换过程和独立评估

这四项信息不能压成一个“发布成功”。Iknem 保存的是可以复核的有限主张；krin 再说明当前目标需要哪些观察，具名权威最后决定是否接受结果。

当前状态：IKN-CORE 0.1.0-draft 已定义抽象职责。物理格式和组件接口尚未形成，也没有相应的采集、验证、归并、撤销或决定实现。因此这份说明只提供规范阅读路径，不能作为可用组件或稳定互操作格式。

一项记录至少说明这些事实

记录项开发者要能回答失败风险
主体与范围哪个精确对象、事件、总体和时间段接受这项主张路径或显示名称指向了另一个对象
生产者谁以什么版本和权限产生记录工具或签名者取得了没有授予的判断权
方法与环境采用什么方法、参数、时钟、环境和参考值固定样本被外推到未知总体
溯源使用了哪些输入、活动、父记录和派生关系动态依赖被藏在环境名称中，或记录循环自证
观察与变换原始观察在哪里，经过哪些显式变换，损失了什么自由文本、工具退出状态或模型解释被当作直接观察
主张与限制记录实际声称什么，哪些路径、时段和信息仍然缺失结论越过方法和生产者能够支持的范围
完整性与披露怎样检查字节、顺序、认证或签名，哪些敏感内容被外置完整性被误写成事实正确，或脱敏删除了判断所需关系

IKN-PRI-001 要求 Iknem 不保存 Bearer token、私钥、会话 cookie、文件描述符或实时能力句柄。外置、摘要和脱敏都要记录原对象、变换规则、可见范围和对结论的影响；审计者取不到关键外置材料时，相关观察不能计入充分覆盖。

记录种类和来源类别解决不同问题

维度允许值回答的问题
记录种类observation / derivation / attestation / appraisal / decision-record这项记录执行了观察、派生、外部声明、评估还是决定事件记录
来源类别direct-observation / deterministic-derivation / external-assertion / human-judgment / model-candidate记录中的依据怎样产生

IKN-CLS-001 固定这两个分类维度。签名可以帮助认证外部声明的来源，但不会把它改成直接观察。模型分数、重复次数和自报置信度也不会提升来源类别。另一个获授权过程可以核查模型候选，并产生一项具有自身身份和范围的新记录。

四个结果必须分别保存

结果域允许值回答的问题
Iknem 有效性valid / invalid / revoked指定验证者是否在当前政策、环境和截止点下接受这项记录
证据覆盖度sufficient / insufficient适用记录是否覆盖精确 krin 要求的全部观察责任
满足判断met / unmet / agno / fault把目标要求与结构化观察比较后得到什么结果
权威决定accepted / rejected / deferred具名权威依据结果和决定规则作出什么决定

valid 是外部评估，不是记录给自己的标签。覆盖度按不同观察责任计算，不按日志、签名或重试次数计算。decision-record 只记录已经发生的决定，不得创造决定权限。政策、密钥、方法、对象或截止点发生变化后，验证者需要重新评估。

模型、Agent 协议和遥测先保持原来的身份

外部输入进入证据路径时保存什么不能直接推出什么
模型说明或模型裁判结果实际输入、模型与版本、输出、顺序、偏差探针和 model-candidate 来源类别直接观察、统计区间、事实真值或最终决定
MCP 或 A2A 任务状态精确协议版本、对端、任务身份、状态和取得时间本地目标满足、授权有效或结果接受
OpenTelemetry Span、事件或指标采用的 schema、字段稳定性、资源身份、时间与脱敏边界未观测时段、业务终态、因果责任或完整证据覆盖
签名、构建证明或透明日志记录精确主体、签名内容、验证政策、信任根、截止点和撤销状态主张正确、方法适用、运行获权或依赖方必然接受

截至 2026 年 7 月 15 日，NIST AI Agent Standards Initiative仍把互操作协议、Agent 身份与授权、安全评测分成并行工作。Noemion 采用同样的职责分离：外部系统提供带版本的输入，本地规范决定它能支持哪一项主张。

开放标准和 GNU 工具说明了哪些边界

资料可复用的工程结论Noemion 不继承的部分
W3C PROV-DM实体、活动、使用、生成、派生和责任需要显式关联；使用与生成链本身仍不足以证明派生不采用 PROV 物理编码，也不让溯源关系自行证明事实或因果
RFC 9334 RATS ArchitectureEvidence、验证者的 Attestation Results 和依赖方政策属于不同步骤不把远程证明角色直接改名为 Iknem 角色，也不让验证者替代最终权威
SLSA 1.2 制品验证消费者按既定预期核对制品身份、来源和构建事实，失败处理由使用方政策决定不把供应链证明扩展为通用意义、授权或任务满足证明
GNU Guix 的 guix challenge独立构建的逐字节比较能够发现一致或差异差异仍需判断来自不可复现、损坏还是恶意来源；工具结果不自动指定正确一方
OpenTelemetry 语义约定 1.43.0与GenAI 独立仓库Agent、模型、工具和协议遥测需要版本化词汇、资源身份和隐私边界外部字段只作为映射输入，不进入 Iknem 身份、有效性或最终决定

相邻问题按需要继续展开

IKN-CORE 只定义证据记录的共同边界。开发者提出更强主张时，再进入对应研究资料；这些提案不会在这里复制第二套规则。

开发者问题继续阅读当前上限
模型评分能否参与评测，训练反馈能否成为发布依据？模型参与评测研究与模型训练与更新研究正在研究；模型输出保持有范围候选
一条轨迹能否证明动作、状态转变或因果？状态变化与因果归因研究正在研究；现行记录只支持已声明的方法和范围
历史与检查点何时能支持证据？记忆、检查点与恢复研究保存状态只说明生产者保留了什么，不恢复旧权限或补齐未知副作用
删除收据能否证明全部副本已经清除？数据使用、保留与删除研究正在研究；请求、操作、验证、确认、备份和派生物分别记录

spec/iknem-core.md 是 Iknem 的唯一现行规范条款源；这份说明只提供阅读路径，不建立第二套字段或符合性要求。

当前状态

当前策略：IKN-CORE 已确定抽象证据与评估边界，但不锁定物理容器、扩展名、字段编号、摘要与签名算法、透明日志、撤销分发、远程查询、跨生产者时钟归并、隐私策略语言和长期归档。未来实现必须以新 Profile 和规范字节证明互操作。

名称仍在研究：Iknem 是设计阶段名称，发行拼写、读音流畅度和与 Endem、Synem、Dromen 的口头区分度尚未通过真实使用者验证。开发者文档必须先写“有范围证据记录”，不能让名称替代可验证定义。
