项目/规范参考指南/Tekmor

Tekmor

Tekmor 记录谁在什么条件下观察了什么，并明确这项证据能支持多大范围的判断。

Scoped EvidenceStructured PhainNot Formal Proof

Tekmor 能支持什么判断

Tekmor 回答“谁在什么条件下观察或判断了什么，以及记录绑定到哪些对象”。其中的 phain 与适用 skena 使用可比较的符号和关系位置。

Tekmor 让依据可定位、可检查完整性、可比较和可撤销，但不自动证明外部事实为真。

不得混淆：Tekmor 不是数学证明、模型自述、日志数量或最终验收决定。证据齐全只说明决定者拥有规范要求的输入；满足判断和最终结论仍由 krin 预先声明的规则与权威作出。

记录有效不等于证据充分

维度允许值回答的问题
Tekmor 有效性valid / invalid / revoked记录身份、完整性、生产者和适用范围当前是否可用。
证据覆盖度sufficient / insufficient一组有效 Tekmor 是否覆盖指定 krin 的全部必需观察。
满足判断met / unmet / agno / fault目标事态相对观察和比较契约的结果；不属于 Tekmor 有效性。
权威决定accepted / rejected / deferred具名权威是否作出最终决定；不能由记录数量自动产生。

同一份 Tekmor 可以完整有效，却不足以覆盖另一个范围更大的 krin。反过来，多份记录数量再多，只要生产者、范围或关系位置无效，也不能形成充分覆盖。完整分层见 ADR-0015。

持续目标需要时间覆盖

服务于 mene 的 Tekmor 除了 phain，还必须声明半开覆盖区间、观测方法、时钟域、时钟生产者、分辨率、最大不确定度、同步方式、已知空洞和重启或回拨事件。

时间覆盖语义见 ADR-0016。当前没有 Tekmor 时间编码、时钟采集器或多生产者归并实现。

Dromen 与运行记录

Praxor 从 attested Endem/Synem、执行策略、授权能力和当前环境建立 Dromen。Dromen 只存在于会话内；Tekmor 只保存足以重放判断的身份、参数和观察，不序列化实时句柄、凭据、模型私有状态或整个内存视图。

负观察与缺席证据

显式负观察必须保存与目标相同的关系身份、角色和顺序，并声明生产者、方法、范围与限制。空查询只能记录“在这些输入和条件下没有匹配”，默认支持 agno，不能伪装成负极性 phain。

若具名权威要从缺席支持负目标，Tekmor 集合还必须绑定全集、全部角色路径、时间截止、完整枚举方法、迟到规则、过滤、去重、丢失和排除项。任何未知覆盖使结论回到 agno；观察契约故障为 fault。完整边界见 ADR-0017。

隐私、最小披露与撤销

  • Tekmor 只保存验收与审计必需的字段；敏感原文、凭据和高风险工具输出按策略摘要、脱敏或外置。
  • 脱敏必须声明损失及其对证据强度的影响，不能用“已脱敏”掩盖无法验证。
  • 来源撤回、密钥撤销、环境身份变化或规则版本变化必须使受影响的决定可定位并失效。
  • 时间戳只说明记录时间域，不进入 Endem 的确定性语义身份。
  • 模型生成的说明可以作为低等级 Tekmor 内容，但不能成为唯一的权限、完整性或最终验收依据。

当前成熟度

Tekmor 的有限主张、身份绑定、证据等级、“证据不等于证明”、有效性与覆盖度分离、mene 时间覆盖以及否定证据职责已经采用。事件编码、时间与封闭声明字段、签名、隐私配置、撤销协议、跨实现归并和运行记录物理格式仍需新 Profile 与测试。
