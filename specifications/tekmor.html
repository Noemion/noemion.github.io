Tekmor
有范围证据

记录谁针对哪个精确对象、通过什么方法观察到什么，并明确这项依据能够支持和不能支持的判断范围。

TEK-CORE 0.1.0-draftScoped EvidenceNo Wire FormatNo Component Code

直接结论

Tekmor 不是把日志装进新格式，而是把证据主张限制在可复核的主体、方法、环境、时间和观察范围内。

一项记录可以完整、带签名且来源明确，却仍然不适用于当前目标。证据集合也可以全部有效，却没有覆盖 krin 要求的全部关系、主体或时间。因此 Tekmor 有效性、覆盖度、满足判断和最终决定必须分别计算。

当前状态：项目已经建立抽象 TEK-CORE 0.1.0-draft、九项威胁、十四个场景和十八个提案向量；Tekmor 物理格式与组件接口仍未冻结，也没有采集器、验证器、归并器、撤销服务或决定引擎。任何资料检查通过均不得被描述为组件实现或稳定互操作。

观察必须能与目标关系比较

decision-record 只记录具名权威已经作出的决定。创建记录不会授予决定权限，验证者或 Praxor 也不能默认成为最终决定者。

有效不等于充分

结果域允许值必须绑定
Tekmor 有效性valid / invalid / revoked记录身份、验证政策、参考值、信任根、环境和截止点
证据覆盖度sufficient / insufficient精确 Endem、krin、关系位置、主体集合、时间与观察责任

valid 是验证者在指定上下文形成的评估，不是记录对自己的声明。政策、密钥、方法、来源或截止点变化后必须重新评估；旧事件保留，但不能继续冒充当前结论。

覆盖按责任计算而不是按数量计算

覆盖度只统计当前 krin 要求且已经验证、适用、未撤销的不同观察责任。同一日志复制一百次、同一观察附加多个签名或同一主体反复重试，都不能填补新的关系路径、主体、时间区间或成员。

  • 持续目标的时间空洞产生 insufficient，不能用最后值延续或插值静默填补。
  • 负目标只有在全集、角色路径、截止点、枚举、丢失和迟到边界封闭时，缺席才可能支持判断。
  • 量化判断按不同成员身份保存成员结果，不能按事件或证据数量计数。
  • 测量判断保留原始观察、换算、聚合、不确定区间和阈值，显示舍入值不能成为依据。
  • 复合判断保留已求值叶、决定性依据、未求值叶、停止原因和已经遇到的故障。

最小披露不是信息丢失免责条款

Tekmor 不保存 Bearer token、刷新令牌、私钥、会话 cookie、文件描述符、实时能力句柄或未获授权的提示与工具正文。它可以保存不可逆摘要、发行者、受众、范围、调用身份和外置引用。

摘要、脱敏或外置必须说明原对象身份、变换规则、可见和不可见范围，以及这些省略怎样影响主张与覆盖度。若审计者无法取得外置对象，或脱敏删除了主体和关系位置，相关依据只能视为不可验证或覆盖不足。

当前成熟度

TEK-CORE 已冻结抽象证据与评估边界，但不冻结物理容器、扩展名、字段编号、摘要与签名算法、透明日志、撤销分发、远程查询、跨生产者时钟归并、隐私策略语言和长期归档。未来实现必须先由用户明确开启代码阶段，再以新 Profile 和规范字节证明互操作。
