规范与查询

先写下你要判断的事实，再从这里找到负责该问题的规范。页面会同时说明何时必须停止，以及现有资料最多能证明什么。

规范草案权威源无组件实现

怎样找到需要的规范

不必先记住全部对象名。先写出你准备作出的工程判断，再找到负责该判断的规范；外部协议状态、签名、模型输出和测试通过都不能替你跳过这一步。

你手里的问题先进入何时停止继续推导
以自然语言表达的目标应当形成什么Endem 内容规范来源、意义、期望事态、目标方向、判据或待确认项还不能唯一恢复
多个目标怎样组成完整集合Endem closure 组合规范引用未解析、闭包不完整、权限扩大或成员结果被合并
一次 Agent 会话可以调用什么session contract 会话规范、权威与授权规范主体、政策、环境、能力、预算、受众或截止点仍然缺失
外部任务完成能证明什么协议适配规范、evidence entry 证据规范协议状态尚未映射为有来源的本地观察，或证据没有覆盖当前判据
签名有效为什么仍不能发布精确身份与签名规范、权威与授权规范内容主体、陈述类型、验证政策、撤销或依赖方决定还未核对
文本看起来相同却无法匹配文本与标识符规范、结构化诊断规范来源字节、解码、比较域、显示视图和失败位置还没有分开

网页、工具返回、历史、摘要或附件进入模型时，即使自称 system 或 admin，也仍是带来源的资料。先核对文本实际怎样进入系统，再核对谁有权提供指令或授予动作。

用一个外部“已完成”状态走完整条查询链

假设外部 Agent Task 显示 completed，团队要判断服务能否发布。不要把一个状态直接翻译成“目标完成”，而要依次回答下面五个问题。

先回答查什么权威源何时停止
外部系统实际报告了什么协议版本、对端、任务状态和候选产物ADP-CORE来源或版本不明时，不做本地映射
这次运行原本允许什么主体、环境、能力、预算和证据责任SESSION-CORE外部动作超出会话上限时，拒绝继续
观察能支持多大范围的结论对象、方法、环境、截止点、覆盖和限制EVIDENCE-CORE覆盖不足时保持未知，不补写成功
目标判据得到什么结果验收条件与 met / unmet / undetermined / faultEND-CORE判据未固定或证据不适用时，不形成满足结论
谁能决定发布决定者、对象、范围、依据和截止点AUT-CORE没有适用的具名决定者时，保持待定

completed 只说明外部请求走到了某个执行状态，不直接等于目标已经满足或发布已经获准。每份规范继续只约束自己的责任。

按对象和工程问题进入

Endem · 最小目标制品END-CORE 0.1.0-draft。说明来源表达、意义确认、期望事态、目标方向、判据和待确认事项。 Endem closure · 多目标解析闭包CLOSURE-CORE 0.1.0-draft。说明引用怎样唯一绑定、闭包怎样完成以及权限怎样收窄。 session contract · 一次运行的只读契约SESSION-CORE 0.1.0-draft。说明精确主体怎样绑定政策、环境、能力、预算和证据责任。 evidence entry · 有范围的证据记录EVIDENCE-CORE 0.1.0-draft。说明观察、变换、有效性、覆盖度和决定依据怎样分开。 结构化诊断 · 可定位失败DIA-CORE 0.1.0-draft。说明机器码、失败层次、类型化位置、恢复和安全披露。 外部协议适配 · 受限连接ADP-CORE 0.1.0-draft。说明版本、对端、能力、状态、重试、交付与本地结果怎样分开。 精确内容身份与签名 · 不可变绑定ID-CORE 0.1.0-draft。说明摘要、签名陈述、验证、撤销、复现与派生关系。 文本与标识符边界 · 可核对输入TEXT-IDENTIFIER-CORE 0.1.0-draft。说明 UTF-8、来源字节、比较、范围、显示和模型实际输入。 权威与授权决定 · 有界批准AUT-CORE 0.1.0-draft。说明主体、对象、动作、范围、委托、同意、时效和能力交集。 内容身份不变，不等于授权仍然有效说明单文件声明上限、外部授权前置条件和内容身份稳定性。

每类资料只回答一种问题

实现者应以规范源与机器可读登记中的精确版本、成熟度、条款 ID 和验证责任为准。解释页帮助定位，不能取代下面的权威关系。

资料类型能够回答不能据此声称
正式条款CORE、END-FMT 和来源清单规定当前必须满足的抽象义务、格式实验和失败语义草案已经稳定发行，或尚未定义的字段与接口可以自行补齐
机器可读登记规范、条款、术语、成熟度、实现状态和验证引用之间的对应关系accepted 等于稳定发行，或计划验证已经执行
Profile封闭对象域内允许的字段、表示、限制和符合性范围一个 Profile 可以代表所有用途，或 END-P2 已是裁剪后的最终发布 Profile
设计材料ADR 保存已形成的责任边界；威胁模型、场景和研究提案保存风险、反例与候选方向非规范材料已经成为字段、命令、状态、协议 Profile 或迁移决定
验证资料已登记正反案例与草案的关系是否按预期通过或拒绝组件、互操作、性能、安全隔离或真实外部协议已经实现
实现证据未来代码阶段的独立读取器、对抗测试、跨实现比较、运行观测与发布制品当前阶段已经拥有任何解析器、运行时、政策引擎或协议适配器

截至 2026-07-15，NIST AI Agent Standards Initiative仍分别推进产业标准、社区协议以及身份认证和安全评测研究。Noemion 因此继续把协议互操作、身份、动作授权、证据与目标满足放在不同规范责任中；外部趋势不能替项目生成实现证据。

按什么顺序判断资料的权威性

  1. 先用版本化 CORE、格式条款和 Profile 确认实现必须接受、拒绝或保留什么。
  2. 再用标为“当前策略”的 ADR 查明为什么采用这条边界，以及变更会影响什么。
  3. 用威胁模型、场景和向量寻找反例、滥用方式与验证缺口。
  4. 用规范解释页、架构页和指南建立阅读路径，不从解释文字创造第二套义务。

当前策略表示项目目前采用的做法；正在研究表示已有候选但尚未形成结论；待定内容表示目前没有唯一答案。研究资料不能作为现行字段、命令、状态或互操作接口的依据；向量通过也只说明已登记案例与草案一致。

版本化规范源查 CORE、Profile、威胁模型、场景和研究提案的正式 HTML 页面。 机器可读登记核对版本、成熟度、条款、威胁和验证材料之间的关系。 架构决策按被改变的责任查找现行决定和历史迁移。 还缺什么证据确认下一步缺的是 Profile、运行证据、名称研究还是实现授权。

哪些边界还不能从规范推出

发行名称与读音

现行对象名和六个字段仍用于精确引用，但发行拼写、首次朗读和口头区分尚无人类证据。固定术语职责没有证明现行拼写或读音已经通过。

物理格式

END-FMT 0.1.0-draft 是实验性物理容器，END-P2 是其中来源保留的形成与评审 Profile；最终裁剪发布 Profile、Endem closure、evidence entry、诊断和授权事件的物理编码仍待确定。

会话契约

session contract 规范一次会话怎样建立、失效和销毁。它不是文件、凭据包、跨会话身份或恢复记录，也没有运行 API。

组件实现

当前没有 deterministic producer、independent inspector、bounded runner、解析器、沙箱、政策引擎、协议适配器、安装包或稳定 ABI。

开始一次规范查询

  1. 把准备作出的主张写成一句可以被反例推翻的话，并标出对象与使用方。
  2. 从上方问题表进入专题解释，再打开其链接的精确 CORE、Profile 或 ADR。
  3. 在机器可读登记中核对版本、成熟度、实现状态、条款和验证引用。
  4. 只运行与变更责任匹配的向量和资料检查，并保留失败结果。
  5. 声明不超过现有证据；资料一致时最多说明草案关系一致，不能写成组件已经实现。

当前策略：对象规范负责目标、组合、会话和证据；横切规范负责诊断、协议适配、精确身份、文本解释和授权。

待定内容：最终发布 Profile、发行名称与读音、对象和事件物理编码、运行 API、协议 Profile、摘要与签名算法、Unicode 标识符、权威目录、政策语言和稳定 ABI。
