先看哪些规范

先读 Endem，理解一项目标怎样成为可检查制品。再读 Synem，理解多个目标怎样解析依赖。最后读 Tekmor，理解证据能支持什么判断，又不能证明什么。

名称、职责、六个语义面、状态、.endem 扩展名、结果域、mene 抽象时间、否定缺席、量化范围、测量阈值与复合判断边界已经采用。END-P1 已关闭对象载荷，END-SRCM 已关闭首个实验来源清单；当前只有资料检查器读取这些草案，没有 Rust 组件、时钟适配器、统计引擎或求值器。mene、封闭、量化、测量与复合结构的物理字段、Synem/Tekmor 编码、摘要、签名和稳定 ABI 仍未冻结。

三项规范

Endem · 最小目标制品查看来源、意义、事态、目标方向、判据、未决边界和三个制品状态。 Synem · 多目标解析闭包查看多个 Endem 怎样解析引用、收敛权限并报告冲突。 Tekmor · 可定位证据记录查看来源、构建、绑定、签名、运行和决定依据怎样保持身份链。

规范源与可追溯性

END-CORE 0.1.0-draft 定义语义边界；END-FMT 0.1.0-draft 定义实验性容器。两者使用唯一条款 ID，并把义务连接到现有向量、计划测试或具名复核权威。

工程源用途当前边界
END-CORE 语义条款六个语义面、抽象单变量量化、授权、状态、身份、安全与验证责任。量化物理字段、嵌套或多变量量化、时间和求值语言仍未冻结。
END-FMT 格式条款固定前导、记录目录、受限 CBOR、结构实验 END-P0 与设计 Profile END-P1。END-FMT 0.1.0-draft 尚非稳定 ABI，也没有公开符合性声明。
END-P1 Profile六记录字段键、嵌套 map、枚举、规范数组顺序和资源限制。只覆盖 nascent 与 kine，不预留尚无消费者的未来字段。
END-SRCM 来源清单首个 Poiet 输入的指令、转义、基数、授权边界和 END-P1 映射。是可替换的实验接口，不是稳定源语言或 Endem 身份。
END-SCEN 场景语料二十七个自然语言场景检查方向、否定、歧义、观察、故障、权威、多根、结果域、时间、缺席、量化、测量和复合判断。是非规范设计审查材料，不是解析能力、符合性测试或实现证据。
机器可读登记规范、术语、成熟度、实现状态与验证引用。accepted 不表示稳定发行；计划测试不能写成已覆盖。
威胁模型
诊断目录		把不可信输入、资源放大和共同故障映射到条款与稳定实验错误码。END-P0 数值只支持第一轮实验，仍需规模和复杂度证据。
语义向量集多个根、事态混入目标力量、未记录歧义和模型越权等正反案例。这些 JSON 不是 .endem 物理格式。
结果域向量集十二个提案覆盖满足、决定、会话和证据状态的允许组合与确定拒绝。检查器只验证 ADR-0015 矩阵，不是 Praxor、求值器或决定引擎。
mene 时间向量集十二个提案覆盖 fixed/elapsed、strict/budgeted、覆盖缺口、违约和时钟故障。检查器只验证 ADR-0016 抽象矩阵，不是时钟、监控器、Praxor 或求值器。
否定与缺席向量集十二个提案覆盖显式负观察、空结果、封闭范围、正反例和观察故障。检查器只验证 ADR-0017 的结果矩阵，不是日志收集器、Praxor 或求值器。
量化与成员资格向量集十二个提案覆盖成员范围、空集合、不同成员计数和决定性聚合。检查器只验证 ADR-0018 的分类矩阵，不是成员目录、Praxor 或求值器。
测量与阈值向量集十二个提案覆盖构念、总体、单位、聚合、缺失、不确定度和阈值。检查器只验证 ADR-0019 的分类矩阵，不是遥测采集器、基准运行器、统计引擎或求值器。
复合事态与判据向量集十二个提案覆盖单根边界、拓扑对齐、四结果传播和决定性短路。检查器只验证 ADR-0020 的分类矩阵，不是解析器、Praxor、运行时或求值器。
字节向量集END-P0 含 1 个结构接受和 5 个结构拒绝；END-P1 含 3 个接受和 11 个分层拒绝。这些仍是实验性规范字节，不构成稳定 ABI。

不作为文件规范的概念

Dromen 是 Praxor 针对一次会话从 attested Endem 或 Synem、执行策略和当前授权建立的只读运行视图。Dromen 不得成为磁盘文件、发布对象或跨会话句柄，因此本站不为它建立对象格式规范。它的建立与销毁规则见Praxor 组件。
