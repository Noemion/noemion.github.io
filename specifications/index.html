规范参考指南

从开发者要回答的问题进入权威规范，再核对适用版本、成熟度、验证范围和仍未定义的接口。

规范草案权威源无组件实现

怎样找到需要的规范

不必先记住全部对象名。先写出你准备作出的工程判断，再找到负责该判断的规范；外部协议状态、签名、模型输出和测试通过都不能替你跳过这一步。

你手里的问题先进入何时停止继续推导
以自然语言表达的目标应当形成什么Endem 内容规范来源、意义、期望事态、目标方向、判据或待确认项还不能唯一恢复
多个目标怎样组成完整集合Synem 组合规范引用未解析、闭包不完整、权限扩大或成员结果被合并
一次 Agent 会话可以调用什么Dromen 会话规范、权威与授权规范主体、政策、环境、能力、预算、受众或截止点仍然缺失
外部任务完成能证明什么协议适配规范、Iknem 证据规范协议状态尚未映射为有来源的本地观察，或证据没有覆盖当前判据
签名有效为什么仍不能发布精确身份与签名规范、权威与授权规范内容主体、陈述类型、验证政策、撤销或依赖方决定还未核对
文本看起来相同却无法匹配文本与标识符规范、结构化诊断规范来源字节、解码、比较域、显示视图和失败位置还没有分开

需要更细的跨规范查找时，使用按工程问题组织的规范索引。它把外部 Agent Task、模型输入、研究提案和架构决策记录继续路由到精确源。

按对象和工程问题进入

Endem · 最小目标制品END-CORE 0.1.0-draft。说明来源表达、意义确认、期望事态、目标方向、判据和待确认事项。 Synem · 多目标解析闭包SYN-CORE 0.1.0-draft。说明引用怎样唯一绑定、闭包怎样完成以及权限怎样收窄。 Dromen · 一次运行的只读契约DRO-CORE 0.1.0-draft。说明精确主体怎样绑定政策、环境、能力、预算和证据责任。 Iknem · 有范围的证据记录IKN-CORE 0.1.0-draft。说明观察、变换、有效性、覆盖度和决定依据怎样分开。 结构化诊断 · 可定位失败DIA-CORE 0.1.0-draft。说明机器码、失败层次、类型化位置、恢复和安全披露。 外部协议适配 · 受限连接ADP-CORE 0.1.0-draft。说明版本、对端、能力、状态、重试、交付与本地结果怎样分开。 精确内容身份与签名 · 不可变绑定ID-CORE 0.1.0-draft。说明摘要、签名陈述、验证、撤销、复现与派生关系。 文本与标识符边界 · 可核对输入TEXT-IDENTIFIER-CORE 0.1.0-draft。说明 UTF-8、来源字节、比较、范围、显示和模型实际输入。 权威与授权决定 · 有界批准AUT-CORE 0.1.0-draft。说明主体、对象、动作、范围、委托、同意、时效和能力交集。 内容身份不变，不等于授权仍然有效说明单文件声明上限、外部授权前置条件和内容身份稳定性。

每类资料只回答一种问题

实现者应以规范源与机器可读登记中的精确版本、成熟度、条款 ID 和验证责任为准。解释页帮助定位，不能取代下面的权威关系。

资料类型能够回答不能据此声称
正式条款CORE、END-FMT 和来源清单规定当前必须满足的抽象义务、格式实验和失败语义草案已经稳定发行，或尚未定义的字段与接口可以自行补齐
机器可读登记规范、条款、术语、成熟度、实现状态和验证引用之间的对应关系accepted 等于稳定发行，或计划验证已经执行
Profile封闭对象域内允许的字段、表示、限制和符合性范围一个 Profile 可以代表所有用途，或 END-P1 已是裁剪后的最终发布 Profile
设计材料ADR 保存已形成的责任边界；威胁模型、场景和研究提案保存风险、反例与候选方向非规范材料已经成为字段、命令、状态、协议 Profile 或迁移决定
验证资料已登记正反案例与草案的关系是否按预期通过或拒绝组件、互操作、性能、安全隔离或真实外部协议已经实现
实现证据未来代码阶段的独立读取器、对抗测试、跨实现比较、运行观测与发布制品当前阶段已经拥有任何解析器、运行时、政策引擎或协议适配器

截至 2026-07-15，NIST AI Agent Standards Initiative仍分别推进产业标准、社区协议以及身份认证和安全评测研究。Noemion 因此继续把协议互操作、身份、动作授权、证据与目标满足放在不同规范责任中；外部趋势不能替项目生成实现证据。

哪些边界还不能从规范推出

发行名称与读音

现行对象名和六个字段仍用于精确引用，但专名必要性、发行拼写、首次朗读和口头区分尚无人类证据。研究候选不是别名或 Profile 输入。

物理格式

END-FMT 0.1.0-draft 是实验性物理容器，END-P1 是其中来源保留的形成与评审 Profile；最终裁剪发布 Profile、Synem、Iknem、诊断和授权事件的物理编码仍待确定。

会话契约

Dromen 规范一次会话怎样建立、失效和销毁。它不是文件、凭据包、跨会话身份或恢复记录，也没有运行 API。

组件实现

当前没有 Ktisor、Theor、Drasor、解析器、沙箱、政策引擎、协议适配器、安装包或稳定 ABI。

开始一次规范查询

  1. 把准备作出的主张写成一句可以被反例推翻的话，并标出对象与使用方。
  2. 从上方问题表进入专题解释，再打开其链接的精确 CORE、Profile 或 ADR。
  3. 在机器可读登记中核对版本、成熟度、实现状态、条款和验证引用。
  4. 只运行与变更责任匹配的向量和资料检查，并保留失败结果。
  5. 声明不超过现有证据；资料一致时最多说明草案关系一致，不能写成组件已经实现。

当前策略：对象规范负责目标、组合、会话和证据；横切规范负责诊断、协议适配、精确身份、文本解释和授权。

待定内容：最终发布 Profile、发行名称与读音、对象和事件物理编码、运行 API、协议 Profile、摘要与签名算法、Unicode 标识符、权威目录、政策语言和稳定 ABI。
