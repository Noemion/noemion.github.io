查看 Noema IR、Noema Object、Horizon Object、智能体控制面和运行语义目前具有多大约束力。

状态说明

现行设计

已被当前架构约束采用，但仍需在正式规范中给出可测试表述。

待验证设计

尚未成为规范要求，可用于原型和实验，但不得作为兼容性或实现承诺。

尚待确定

保留多个候选或证据缺口，不静默固化为规范。

后续计划

超出第一阶段范围，将在前序规范、实现和验证完成后推进。

对象规范草案

NIR · 现行概念 / 待验证结构目标、约束、歧义、类型、来源绑定与验收语义；字段尚未冻结。 NOBJ · 现行边界 / 待验证布局对象职责与安全不变量已经采用；Section、记录和 ABI 尚未冻结。 HOBJ · 待验证设计共享对象、依赖闭包与渐进式披露语义；格式和披露 ABI 尚未冻结。

架构原则与待建立接口规范

已接受的架构原则查看命名、Harness、工具链产物、能力和验收决定边界。 对象验证与装载链接、验证、Loaded State 和装载信任边界的当前入口。 智能体控制面上下文、能力、反馈、预算和人工升级的当前设计边界。 求解与候选评价Runtime Request、受约束求解、Candidate Assessment 与 Capability Request 的当前设计边界。

后三项是组件设计入口，不是已经存在的接口规范。Loaded State、Harness 协议、Runtime ABI 与 Acceptance Decision 的正式规范仍待建立。

变更纪律

格式、类型系统和执行语义的变更必须记录架构决策记录（ADR）；公开摘要见架构决策。候选命名、CLI、ABI 与重定位标识不会在规范与成熟度页面中被提升为冻结结论。

规范体系与合格证据

层次回答的问题进入下一成熟度所需证据
概念模型对象为何存在、术语如何区分、边界在哪里术语表、问题陈述、反例、与相邻技术的差异分析
规范要求实现必须做什么、何时必须失败使用 MUST/SHOULD/MAY 等一致规则、可测试条款和错误语义
对象与接口数据如何编码、引用、升级和交换结构定义、布局、版本规则、基准样例/畸形样例与 checked arithmetic 审查
互操作配置不同实现如何声明并验证兼容范围能力矩阵、交叉实现测试、安全分析和版本化测试套件

标准化候选必须能从规范条款追溯到测试与互操作证据。论文中的研究结论、专利中的创新点候选和软著中的软件边界可以引用这些材料，但不能反过来替代规范性定义。
