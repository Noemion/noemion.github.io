从一项目标、多个目标的组合，到验收证据，查看当前采用的规则和仍未冻结的接口。

先看哪些规范

先读 Endem，理解一项目标怎样成为可检查制品。再读 Synem，理解多个目标怎样解析依赖。最后读 Tekmor，理解证据能支持什么判断，又不能证明什么。

名称、职责、六个语义面、状态、.endem 扩展名和信任边界已经采用。END-FMT 0.1.0-draft 进一步给出实验性前导、记录目录与 END-P0，但完整载荷字段、Synem/Tekmor 扩展名、摘要与签名算法仍未冻结；当前也没有稳定 ABI 或可执行程序。

三项规范

Endem · 最小目标制品查看来源、意义、事态、目标方向、判据、未决边界和三个制品状态。 Synem · 多目标解析闭包查看多个 Endem 怎样解析引用、收敛权限并报告冲突。 Tekmor · 可定位证据记录查看来源、构建、绑定、签名、运行和决定依据怎样保持身份链。

规范源与可追溯性

END-CORE 0.1.0-draft 定义语义边界；END-FMT 0.1.0-draft 定义实验性容器。两者使用唯一条款 ID，并把义务连接到现有向量、计划测试或具名复核权威。

工程源用途当前边界
语义条款：https://github.com/Noemion/noemion.github.io/blob/main/spec/endem-core.md六个语义面、授权、状态、身份、安全与验证责任。完整字段 schema 与求值语言仍未冻结。
格式条款：https://github.com/Noemion/noemion.github.io/blob/main/spec/endem-format.md固定前导、记录目录、受限 CBOR、END-P0 与结构接受层次。END-FMT 0.1.0-draft 尚非稳定 ABI，也没有符合性实现。
机器可读登记：https://github.com/Noemion/noemion.github.io/blob/main/spec/registry.json规范、术语、成熟度、实现状态与验证引用。accepted 不表示实现存在；计划测试不能写成已覆盖。
威胁与诊断：https://github.com/Noemion/noemion.github.io/blob/main/spec/endem-threat-model.md
https://github.com/Noemion/noemion.github.io/blob/main/spec/endem-errors.md		把不可信输入、资源放大和共同故障映射到条款与稳定实验错误码。END-P0 数值只支持第一轮实验，仍需规模和复杂度证据。
语义向量：https://github.com/Noemion/noemion.github.io/tree/main/vectors/semantic多个根、事态混入目标力量、未记录歧义和模型越权等正反案例。这些 JSON 不是 .endem 物理格式。
字节向量：https://github.com/Noemion/noemion.github.io/tree/main/vectors/wire1 个结构接受和 5 个确定拒绝字节案例。结构接受不代表 END-CORE 语义接受。

不作为文件规范的概念

Dromen 是 Praxor 针对一次会话从 attested Endem 或 Synem、执行策略和当前授权建立的只读运行视图。Dromen 不得成为磁盘文件、发布对象或跨会话句柄，因此本站不为它建立对象格式规范。它的建立与销毁规则见Praxor 组件。
