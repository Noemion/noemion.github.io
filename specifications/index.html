规范参考指南

从一项目标、多个目标的组合、一次会话的运行边界、验收证据、结构化失败和外部协议适配，到精确内容身份与文本边界，查看当前采用的规则和仍未冻结的接口。

先看哪些规范

先读 Endem，理解一项目标怎样成为可检查制品。再读 Synem，理解多个目标怎样解析依赖。然后读 Dromen 与 Tekmor，理解一次运行怎样受限、证据能支持什么判断。最后读结构化诊断，理解失败怎样稳定定位而不越权改变结果。

名称、职责、六个语义面、结果域、复合判断、Synem 闭包、Dromen 会话契约、Tekmor 证据评估、结构化诊断、外部协议适配、精确身份以及文本与标识符边界已经采用。END-P1 已关闭 Endem 设计载荷；各 CORE 草案分别关闭抽象组合、会话、证据、诊断、协议适配、内容身份与签名以及文本解释边界。当前只有资料检查器，没有组件实现。对象与诊断的物理编码、运行 API、具体协议 Profile、摘要和签名算法、Unicode 标识符与稳定 ABI 仍未冻结。

对象与横切规范指南

Endem · 最小目标制品查看来源、意义、事态、目标方向、判据、未决边界和三个制品状态。 Synem · 多目标解析闭包查看多个 Endem 怎样解析引用、收敛权限并报告冲突。 Dromen · 一次运行的只读契约查看精确制品怎样绑定政策、环境、能力、预算和证据责任。 Tekmor · 可定位证据记录查看来源、构建、绑定、签名、运行和决定依据怎样保持身份链。 结构化诊断 · 可定位失败查看机器码、生产语境、失败层次、位置、恢复和安全边界。 外部协议适配 · 受限连接查看版本、对端、能力、状态、重试、交付与安全边界。 精确内容身份与签名 · 不可变绑定查看完整摘要、签名陈述、撤销、可复现性与伴随制品关系。 文本与标识符边界 · 可核对输入查看 UTF-8、来源字节、规范化、比较、范围、双向显示、隐藏字符与模型实际输入。

规范源与可追溯性

END-CORE 0.1.0-draft 是单个目标的通用内容标准，END-P1 是当前封闭内容 Profile，END-FMT 定义实验性容器；SYN-CORE 0.1.0-draft、DRO-CORE 0.1.0-draft 与 TEK-CORE 0.1.0-draft 定义组合、会话和证据，DIA-CORE 0.1.0-draft、ADP-CORE 0.1.0-draft、ID-CORE 0.1.0-draft 与 TXT-CORE 0.1.0-draft 定义跨对象诊断、协议适配、精确身份和文本边界。它们使用唯一条款 ID，并把义务连接到现有向量、计划测试或具名复核权威。

工程源用途当前边界
END-CORE 内容标准六个语义面、规范分层、Profile 符合性、抽象单变量量化、授权、状态、身份、安全与验证责任。量化物理字段、嵌套或多变量量化、时间和求值语言仍未冻结。
END-FMT 格式条款固定前导、记录目录、受限 CBOR、结构实验 END-P0 与设计 Profile END-P1。END-FMT 0.1.0-draft 尚非稳定 ABI，也没有公开符合性声明。
END-P1 内容 Profile六记录字段键、嵌套 map、枚举、规范数组顺序、引用、状态和资源限制。只覆盖 nascent 与 kine，不预留尚无消费者的未来字段。
END-SRCM 来源清单首个 Poiet 输入的指令、转义、基数、授权边界和 END-P1 映射。是可替换的实验接口，不是稳定源语言或 Endem 身份。
SYN-CORE 组合条款完整闭包、精确绑定、有限无环、权限交集、成员结果与条件激活。不是物理 Synem 格式，不定义扩展名、魔数、仓库或执行调度。
DRO-CORE 会话条款精确主体、政策封闭、环境绑定、能力与预算求交、观察责任、只读失效和销毁。Dromen 永远不是文件；当前不定义运行 API、沙箱、凭据代理、事件编码或恢复协议。
TEK-CORE 证据条款主体范围、溯源、观察、证据类别、完整性、有效性、覆盖、决定与最小披露。不是物理 Tekmor 格式，不定义扩展名、签名算法、撤销服务或长期归档。
DIA-CORE 诊断条款机器身份、生产语境、失败层次、类型化位置、主错误、恢复、外部来源、披露、预算与原子失败。不是结果域、权限、自动修复命令或物理编码；没有诊断运行时。
ADP-CORE 适配条款协议版本、对端、能力、调用、映射、状态、产物、错误、取消、重试、交付与安全边界。不是协议 Profile、通用 Agent 对象或适配器实现。
ID-CORE 身份与签名条款身份域、精确字节、不可变引用、算法政策、签名陈述、验证材料、撤销、可复现性与派生关系。不是新制品、密码算法、签名格式、Semantic Key 或发布实现。
TXT-CORE 文本与标识符条款文本槽、严格 UTF-8、来源溯源、ASCII 标识符、规范化、比较、范围、双向显示、隐藏字符、模型输入与显示视图。不是新制品、Unicode 标识符 Profile、文本处理库或模型输入网关。
END-SCEN 场景语料二十七个自然语言场景检查方向、否定、歧义、观察、故障、权威、多根、结果域、时间、缺席、量化、测量和复合判断。是非规范设计审查材料，不是解析能力、符合性测试或实现证据。
机器可读登记规范、术语、成熟度、实现状态与验证引用。accepted 不表示稳定发行；计划测试不能写成已覆盖。

Dromen 为什么有规范却没有文件格式

Dromen 是 Praxor 为一次 Praxe 会话封存的只读执行契约。DRO-CORE 规范它的建立、失效和销毁规则，是为了阻止隐式权限和环境漂移；它同时明确禁止文件扩展名、可移植序列化、跨会话身份和恢复机制。持久审计由 Tekmor 或会话事件承担，不能借记录重新获得权限。
