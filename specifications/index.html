规范参考指南

从开发者要回答的问题进入权威规范，再核对适用版本、成熟度和仍未定义的接口。

规范草案权威源无组件实现

怎样找到需要的规范

不必先记住全部对象名。若你从协议完成、签名失效、文本混淆、权限不足或证据缺口出发，先用按工程问题组织的规范索引定位权威源；已经知道对象名时，再从下方直接进入完整边界。

阅读约定：各规范先用直白职责解释概念，再用代码样式标出现行设计字段。rhem、semion、skena、telis、krin 与 apor 仍是精确的现行值，但尚未通过发行名称和读音审查；研究候选不是别名，也不能作为来源清单或 Profile 输入。

当前策略：对象规范负责目标、组合、会话和证据；横切规范负责诊断、协议适配、精确身份、文本解释和授权。END-P1 给出当前 Endem 载荷设计，各 CORE 草案给出对应规则。

待定内容：具体发行拼写和读音，对象与事件的物理编码、运行 API、协议 Profile、摘要和签名算法、Unicode 标识符、权威目录、政策语言与稳定 ABI。当前没有组件实现。

规范源与可追溯性

每份 CORE 草案分别固定一种对象或横切责任，END-P1 固定当前封闭内容 Profile，END-FMT 定义实验性容器。实现者应以规范源与机器可读登记中的精确版本、成熟度、条款 ID 和验证责任为准，并用下表定位相应材料。

工程源用途当前边界
END-CORE 内容标准六个语义面、规范分层、Profile 符合性、抽象单变量量化、授权、状态、身份、安全与验证责任。量化物理字段、嵌套或多变量量化、时间和求值语言仍待确定。
END-FMT 格式条款固定前导、记录目录、受限 CBOR、结构实验 END-P0 与设计 Profile END-P1。END-FMT 0.1.0-draft 尚非稳定 ABI，也没有公开符合性声明。
END-P1 内容 Profile六记录字段键、嵌套 map、枚举、规范数组顺序、引用、状态和资源限制。只覆盖 nascent 与 kine，不预留尚无消费者的未来字段。
END-SRCM 来源清单首个 Ktisor 输入的指令、转义、基数、授权边界和 END-P1 映射。是可替换的实验接口，不是稳定源语言或 Endem 身份。
SYN-CORE 组合条款完整闭包、精确绑定、有限无环、权限交集、成员结果与条件激活。不是物理 Synem 格式，不定义扩展名、魔数、仓库或执行调度。
DRO-CORE 会话条款精确主体、政策封闭、环境绑定、能力与预算求交、观察责任、只读失效和销毁。Dromen 永远不是文件；当前不定义运行 API、沙箱、凭据代理、事件编码或恢复协议。
IKN-CORE 证据条款主体范围、溯源、观察、证据类别、完整性、有效性、覆盖、决定与最小披露。不是物理 Iknem 格式，不定义扩展名、签名算法、撤销服务或长期归档。
DIA-CORE 诊断条款机器身份、生产语境、失败层次、类型化位置、主错误、恢复、外部来源、披露、预算与原子失败。不是结果域、权限、自动修复命令或物理编码；没有诊断运行时。
ADP-CORE 适配条款协议版本、对端、能力、调用、映射、状态、产物、错误、取消、重试、交付与安全边界。不是协议 Profile、通用 Agent 对象或适配器实现。
ID-CORE 身份与签名条款身份域、精确字节、不可变引用、算法政策、签名陈述、验证材料、撤销、可复现性与派生关系。不是新制品、密码算法、签名格式、Semantic Key 或发布实现。
TEXT-IDENTIFIER-CORE 文本与标识符条款文本槽、严格 UTF-8、来源溯源、ASCII 标识符、规范化、比较、范围、双向显示、隐藏字符、模型输入与显示视图。不是新制品、Unicode 标识符 Profile、文本处理库或模型输入网关。
AUT-CORE 权威与授权决定条款权威语境、主体资格、封闭范围、语义授权、委托、同意、多人决定、时效、重放、能力交集与结果分离。不是新制品、访问令牌、政策语言、身份系统或决定引擎。
END-SCEN 场景语料用支持案例、反例和边界场景检查方向、否定、歧义、观察、故障、权威、多根、结果域、时间、缺席、量化、测量、复合判断及授权伴随关系。是非规范设计审查材料，不是解析能力、符合性测试或实现证据。
机器可读登记规范、术语、状态、实现状态与验证引用。accepted 不表示稳定发行；计划测试不能写成已覆盖。

Dromen 为什么有规范却没有文件格式

Dromen 是 Drasor 为一次 Drase 会话封存的只读执行契约。DRO-CORE 规范它的建立、失效和销毁规则，是为了阻止隐式权限和环境漂移；它同时明确禁止文件扩展名、可移植序列化、跨会话身份和恢复机制。持久审计由 Iknem 或会话事件承担，不能借记录重新获得权限。
