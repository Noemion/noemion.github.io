精确内容身份与签名

定义跨 Endem、Synem、Iknem、发布陈述和伴随制品的精确字节身份、不可变引用、签名验证、撤销与可复现性边界。

ID-CORE 0.1.0-draft12 Clauses物理格式待定No Crypto Code

它解决什么问题

任何声称“这是同一个目标制品”“这份证据属于它”或“这份签名覆盖它”的系统，都必须先定义精确身份。

ID-CORE 让各对象共享同一组引用和验证不变量，又防止摘要、签名、证书、撤销、信任和验收互相冒充。它不是新制品、文件格式、命令、密码算法或 Semantic Key。

权威边界：实现义务以 ID-CORE 条款源为准。当前 SHA-256 只出现在提案向量中，不表示未来发行算法已经选定。

当前策略

ID-CORE 用十二条核心规则定义精确身份、不可变引用、有类型签名陈述、外置验证包络、截止点有效性、独立复现和制品关系。未来实现必须遵守这些规则。

摘要相同不得直接推出语义等价、事实为真、主体有权、制品已被接受或派生物继承来源身份。签名验证通过也不得越过既定权威、政策、撤销与结果域。

十二条身份与签名规则

规则直白要求必须拒绝
ID-DOM-001身份绑定对象种类、规范、表示、Profile 和身份域跨语境只比较摘要文本
ID-BYT-001摘要覆盖精确、版本化且非循环的字节范围隐式规范化、可变输入和自包含摘要
ID-REF-001安全引用携带完整身份并唯一闭合名称、路径、URL、latest、build ID 或外部 ID 替代目标
ID-ALG-001算法、参数、用途和安全状态由版本化政策登记未知算法、参数歧义、静默替换和降级
ID-DSP-001短摘要只显示，安全比较使用完整值前缀碰撞、首个匹配和 build ID 校验
ID-EQV-001精确字节身份与语义等价保持分开摘要相同推出真假或派生物继承 Semantic Key
ID-STM-001签名覆盖有类型的对象、用途、受众、主体和政策陈述裸摘要签名跨类型或跨用途重放
ID-ENV-001签名和验证材料外置并绑定精确陈述包络改变对象身份或只保存密钥提示
ID-AUT-001密码验证、主体认证、授权、适用性和决定分开签名或日志包含直接证明真、授权或接受
ID-VAL-001有效性绑定截止点、信任根、政策、撤销和状态新鲜度无限缓存成功或撤销后沿用旧授权
ID-REP-001可复现性需要独立产出和逐字节输出比较相同日志、build ID、缓存副本或单次重跑冒充复现
ID-REL-001裁剪、调试、压缩和迁移制品各有身份与显式关系派生物继承来源身份、签名、证据、接受或能力

符合性材料

材料当前覆盖不能证明
ID-THREAT12 类身份混淆、降级、重放、撤销和信任继承威胁密码实现、密钥隔离或发布系统已经安全
ID-SCEN18 个支持、反例与边界场景场景本身是签名标准或组件测试
ID-CORE 向量24 个提案包含 12 个允许案例与 12 个确定拒绝摘要器、签名器、验证器、透明日志或可复现构建已经实现

待定内容

摘要和签名算法基线、算法登记表、摘要文本语法、签名陈述物理编码、COSE、DSSE 或 Sigstore Profile、证书体系、透明日志、时间戳与撤销分发、Semantic Key、跨版本等价和调试伴随格式均保持开放。

外部训练数据、基础模型、适配器和派生权重也遵守同一身份边界。环境相同、种子相同、损失曲线相近或模型别名未变，都不能替代精确对象身份与派生关系；字节复现、行为评测和发布决定必须分开。完整反例见模型训练与更新边界研究提案。

背景、权衡和权威资料见 ADR-0027 精确内容身份与签名边界。
