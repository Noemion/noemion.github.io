外部协议适配

当 Noemion 通过 MCP、A2A、HTTP 或模型服务调用外部系统时，说明哪些信息可以带入，哪些远端说法必须留作待核对输入。

ADP-CORE 0.1.0-draft当前策略物理格式不适用尚无适配器

它解决什么问题

远端返回 completed、一件产物或一组可用工具，只能说明远端报告了什么。适配器还要保留协议版本、对端身份和来源状态；本地系统再独立核对目标、权限、证据和接受条件。

当前策略：ADP-CORE 用协议无关规则约束外部适配，外部协议不得改写本地身份、权限或结果类型。

权威边界：实现义务以 ADP-CORE 条款源为准。该规范没有物理编码，也没有创建适配器文件、媒体类型、网络协议或新组件。

用同一次发布理解一次外部调用

发布会话需要调用外部 Agent 或工具时，适配器只负责把一次已受限的本地调用连接到精确协议操作。它不能因为远端自报能力、任务完成或返回产物，就替本地系统决定身份、满足或接受。

适配器必须保持哪些边界

规则直白要求必须拒绝
ADP-PIN-001固定协议、规范、绑定、schema、传输与扩展版本latest、默认版本、未知扩展和兼容猜测
ADP-PEE-001分开发现、声明、端点、认证主体、受众、租户与政策把 Agent Card 或工具清单当作身份与授权
ADP-CAP-001有效能力取五方交集，未知能力关闭失败协商或错误建议扩大当前 Dromen
ADP-INV-001每次调用绑定主体、输入、预算、截止点、幂等与观察责任用外部 task 或 trace ID 替代本地调用身份
ADP-MAP-001保留原协议事实、映射版本和有限损失清单静默丢字段、降精度或伪造通用等价
ADP-STA-001外部状态与本地五类结果域保持正交把 completed、HTTP 200 或 tool error 直接提升
ADP-ART-001消息与产物先成为有来源、有界的不可信候选远端名称、签名或完成状态直接形成可信对象
ADP-ERR-001按 DIA-CORE 分开协议、工具、任务、业务与本地错误解析错误消息或让错误建议提权
ADP-CAN-001分开取消请求、确认、停止观察与回滚证据取消、超时或断流冒充副作用已经消失
ADP-RTY-001重试需要幂等或去重证据并共享会话预算不确定结果下自动重复非幂等副作用
ADP-DEL-001流式和异步交付绑定授权、序列、去重、缺口与终结证据流关闭、通知缺失或轮询超时冒充完成
ADP-SEC-001凭据、网络目标、披露和资源保持最小化令牌透传、SSRF、跨租户枚举和无界保留

材料当前覆盖不能证明

协议基线怎样选择

开发者不能从“最新”链接、SDK 包版本或服务器默认值推断适配版本。每个 Profile 都要固定协议修订、绑定、schema、传输和扩展，并把外部对象留在它原来的结果域。

待定内容

当前没有具体协议 Profile。MCP、A2A、HTTP、模型 SDK 和操作系统调用的物理 Profile，JSON-RPC、Protocol Buffers、SSE 与 Webhook 字段映射，适配器 API、进程隔离、凭据代理、事件存储、重试实现和部署拓扑仍保持开放。

Agent Card、工具列表和 schema 只是能力声明，不能缩写成“Agent 已有能力”。声明、协商、授权、会话上限、即时可调用性与调用结果的完整分层见非规范能力发现与协商研究提案；该提案不修改 ADP-CORE，也不创建能力制品或新规范。

并行调用和外部 Task 不能自行决定哪个候选可以提交。分支必须共享同一 Dromen 上限，非幂等副作用默认串行提交，竞争写入在执行前核对强前提，取消则只表示停止请求而非回滚。完整审查见非规范并行、推测执行与提交边界提案；该提案不修改 ADP-CORE，也不创建事务层或并行对象。

模型、控制面、凭据与实时句柄、适配进程、文件、网络、资源终止和观察也必须保持可审查的责任分离。完整审查见非规范模型、适配器与能力域隔离研究提案；模型不能接收原始凭据，token 不能透传，取消和超时不能冒充回滚，容器或 seccomp 名称也不能替代实际限制证据。该提案不修改 ADP-CORE，也不创建隔离规范或部署对象。

背景、权衡和权威资料见 ADR-0026 外部协议适配边界。
