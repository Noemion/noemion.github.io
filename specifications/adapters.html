外部协议适配

定义外部工具、Agent、模型服务与任务协议进入一次受限会话时必须保持的身份、权限、状态和安全边界。

ADP-CORE 0.1.0-draft12 ClausesWire Not ApplicableNo Adapter Code

它解决什么问题

外部协议可以传递请求、任务、消息与产物，但不能决定 Noemion 的对象身份、能力范围、目标满足或最终接受。

项目已经采用 ADP-CORE 的协议无关边界。未来适配必须遵守十二条规则，不得以外部协议的便利改写本地身份、权限或结果域。

权威边界：实现义务以 ADP-CORE 条款源为准。该规范没有物理编码，也没有创建适配器文件、媒体类型、网络协议或新组件。

十二条适配规则

规则直白要求必须拒绝
ADP-PIN-001固定协议、规范、绑定、schema、传输与扩展版本latest、默认版本、未知扩展和兼容猜测
ADP-PEE-001分开发现、声明、端点、认证主体、受众、租户与政策把 Agent Card 或工具清单当作身份与授权
ADP-CAP-001有效能力取五方交集，未知能力关闭失败协商或错误建议扩大当前 Dromen
ADP-INV-001每次调用绑定主体、输入、预算、截止点、幂等与观察责任用外部 task 或 trace ID 替代本地调用身份
ADP-MAP-001保留原协议事实、映射版本和有限损失清单静默丢字段、降精度或伪造通用等价
ADP-STA-001外部状态与本地五类结果域保持正交把 completed、HTTP 200 或 tool error 直接提升
ADP-ART-001消息与产物先成为有来源、有界的不可信候选远端名称、签名或完成状态直接形成可信对象
ADP-ERR-001按 DIA-CORE 分开协议、工具、任务、业务与本地错误解析错误消息或让错误建议提权
ADP-CAN-001分开取消请求、确认、停止观察与回滚证据取消、超时或断流冒充副作用已经消失
ADP-RTY-001重试需要幂等或去重证据并共享会话预算不确定结果下自动重复非幂等副作用
ADP-DEL-001流式和异步交付绑定授权、序列、去重、缺口与终结证据流关闭、通知缺失或轮询超时冒充完成
ADP-SEC-001凭据、网络目标、披露和资源保持最小化令牌透传、SSRF、跨租户枚举和无界保留

符合性材料

材料当前覆盖不能证明
ADP-THREAT12 类版本、身份、权限、副作用和安全威胁真实网络和实现防护已经有效
ADP-SCEN18 个支持、反例与边界场景场景本身是规范或协议测试
ADP-CORE 向量24 个提案包含 12 个允许包络与 12 个确定拒绝MCP、A2A、HTTP、SDK 或 Drasor 已经实现

当前仍未冻结

当前没有具体协议 Profile。MCP、A2A、HTTP、模型 SDK 和操作系统调用的物理 Profile，JSON-RPC、Protocol Buffers、SSE 与 Webhook 字段映射，适配器 API、进程隔离、凭据代理、事件存储、重试实现和部署拓扑仍保持开放。

Agent Card、工具列表和 schema 只是能力声明，不能缩写成“Agent 已有能力”。声明、协商、授权、会话上限、即时可调用性与调用结果的完整分层见非规范能力发现与协商研究提案；该提案不修改现行十二条适配规则，也不创建能力制品或新规范。

背景、权衡和权威资料见 ADR-0026 外部协议适配边界。
