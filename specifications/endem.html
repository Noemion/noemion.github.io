Endem 表示什么

Endem 是最小、独立有效且可验证的期望终态单元。它把一项自然语言目标变成可检查的对象—关系契约。

这份契约保存来源表达、已确认意义、中性可能事态、目标方向、比较规则和待确认信息。它不规定模型采用哪条提示、计划或推理过程。

权威边界：这是一份面向读者的解释。通用内容义务与条款 ID 以 END-CORE 内容标准为准；当前封闭字段集合以 END-P1 内容 Profile为准；实验性容器以 END-FMT 条款源为准。三层分别验证，END-FMT 0.1.0-draft 尚非稳定 ABI。

最小不变量：一个 Endem 必须且只能有一个根 skena 节点。根节点可以组合相互依赖的子命题；如果两个根可以独立实现、验收或失败，Ktisor 必须形成两个 Endem，再由 Synem 组合。

名称状态：六项职责已经进入现行草案，具体发行拼写和读音仍在研究。技术讨论应先说“来源表达、意义投影、事态、目标方向、满足判据、未决意义”等直白职责，再给出代码字段；读音建议不能替代独立人类听辨证据。

用一个健康目标读懂 Endem

以“让登记服务返回健康状态”为例，开发者先固定终态，而不是先写工具步骤。六个语义面分别保存形成和判断这项终态所需的不同事实。

目标文件内容由哪份标准定义

END-CORE 是 Endem 的通用内容标准，END-P1 是当前内容 Profile，END-FMT 是字节容器。它们共同定义一份实验性 .endem，但不能互相替代。

层次定义什么通过后仍不能说明
END-FMT 容器固定前导、记录目录、范围、确定性编码与有界读取内容字段闭合或语义成立
END-P1 内容 Profile六记录的字段、类型、枚举、顺序、引用、状态与资源上限所有 END-CORE 内容义务均已满足
END-CORE 内容标准六个语义面的职责、授权、关系不变量、状态、身份和符合性目标已经满足、签名有效、会话可运行或权威已经接受

ADR-0023借鉴 ELF 通用规范与平台补充的分层思想，但不复制机器指令、处理器 ABI 或传统链接语义。任何符合性报告都必须分别写出容器接受、Profile 接受和内容接受。

ADR-0030进一步明确：END-P1 内的权威名称只是选择器。授权决定、签名和证据必须在外部精确绑定，并且不改变 Endem 内容身份。

六个语义面的规范职责

语义面最低职责主要拒绝条件
rhem来源记号、主体、精确表达或内容摘要、语言、媒体类型、范围与版本。范围越界、内容摘要不匹配、编码无效、来源无法重放。
semion记号到符号、指称、关系、作用域、极性、量化、时间和来源位置的已确认投影。内容接受必须把它绑定到 AUT-CORE 约束的确定性规则或范围有限具名权威；规范称这项外部关系为语义授权。模型摘要被当作意义、指称无来源、作用域被默认、候选越权确认，或把意义确认误写成工具和运行权限。
skena一个根、中性可能事态及其对象—关系图，显式保存组合与极性。零个或多个根、只有实体没有关系、关系角色不完整或混入目标力量。
teliskine 使事态达到成立；mene 使事态在声明区间持续成立。用力量值重复表达否定、没有时间边界或把询问/许可压成目标。
krinphain 观察类型、结构比较、必需 Iknem、agno/fault 处理和决定权威。命题自证、只给分数没有结构比较、观察不足被当作满足或模型成为唯一权威。
apor候选解释、冲突、缺口、风险、解决策略和有权绑定的主体。只保留最高置信候选、无授权关闭歧义、待确认项在状态变化中消失。

rhem/semion/skena/telis/krin/apor 是固定语义顺序；没有待确认项时，apor 必须是显式空集合。END-P0 只验证编号 1–6 的结构记录；END-P1 已为未来首条实现路径的评审固定六记录内部字段、枚举、规范排序和引用闭包。任何后续 Profile 都必须恢复同一顺序、符号身份、关系位置与规范含义。

kine 当前存在一项待决定的措辞边界：krin 能判断终态，却不能单凭后态证明动作发生、状态转变或某主体造成结果。非规范状态变化与因果归因研究提案建议把它澄清为“要求事态达到成立”的目标方向；该建议仍处于研究阶段，不能冒充现行条款。

形式必须由结构显示

思想与方法基础用《逻辑哲学论》命题 3.141 解释“结构不是词语堆积”的问题来源。这里不从引文推导规范；ADR-0010、本规范和测试共同定义 Endem 的正式结构要求。

semion、skena 和适用 Iknem 中的 phain 使用同一组稳定符号身份。关系节点必须保存谓词身份、有序或具名角色、极性、作用域和来源映射。第一阶段复合节点只允许 all_of 与 any_of；否定保留在关系极性，条件和排他析取仍待确定。

规范载荷不得包含由作者填写的 logical_form、valid 或 true 自我声明。Ktisor 和独立 Theor 通过关系拓扑、角色数量、组合边与引用位置分别验证结构。只保存六段文本的实现不符合本规范。

五个结果域分别回答什么

同一次工作可以同时出现“会话完成、目标未满足、证据有效但覆盖不足”。结果必须带上所属问题，不能压成一个成功或失败状态。

问题结果不能替代
制品形成到哪一步nascent / coherent / attested（现行草案）不替代目标满足、外部陈述验证、环境授权或最终决定。
目标事态是否满足met / unmet / agno / fault不替代权威接受或会话终止。
具名权威怎样决定accepted / rejected / deferred不改写原满足结果。
一次会话怎样结束completed / failed / interruptedcompleted 不等于 met；failed 不等于 unmet。
证据是否有效且覆盖充分valid / invalid / revoked
sufficient / insufficient		记录有效性与证据集合覆盖度不能合并。

ADR-0015锁定这些禁止推导，但不新增 END-P1 字段。现行 attested 还是内容形成词表中的草案值；开发者必须另外保存有类型的外部陈述、验证政策、截止点、撤销状态和依赖方判断，不能把它理解为内容获得永久可信状态。完整限制见内容形成与外部陈述的两轴说明。

mene 怎样表达持续时间

mene 只有在时间范围和连续性政策已经确定时才可判断。它不读取运行机器当前时间，也不把采样频率当作目标语义。

部分允许设计必须拒绝或保留待确认
时间范围fixed 使用具名权威解析的 UTC 半开区间；elapsed 使用具名事件、固定时长和单调时钟。“现在起”“明天”、默认本地时区、日历月年、墙钟经过时长和未绑定事件。
连续性strict 要求全区间成立；budgeted 明确累计、单次和次数上限。把遥测空洞、缺少告警或几个健康采样点当作允许中断。
覆盖结果完整覆盖且预算内为 met；有效违约为 unmet；缺口为 agno；时钟契约故障为 fault。插值填洞、远端 Task 时间戳充当时钟权威，或验证器故障充当目标未满足。

完整决定见 ADR-0016。它不新增 END-P1 字段；未来 mene Profile 仍需物理字段、范围限制和正反字节向量。

否定与缺席怎样判断

否定目标保留同一关系、同一角色和顺序，只改变 skena 的极性。没有记录默认表示观察不足，不表示关系已经被观察为不成立。

依据判断边界
同构且显式 negative 的有效 phain可以支持 met必须绑定具名生产者、范围和方法。
同构且显式 positive 的反例unmet不能被更多空查询抵消。
查询未命中或日志为空默认 agno只有权威证明全集、路径、时间、枚举与损失边界均封闭时，完整缺席才可支持 met。
观察器或适配器未按契约完成fault不能改写成目标为假或没有事件。

完整决定见 ADR-0017。END-P1 只补齐原子负极性规范字节，不增加封闭声明字段；复合和量化否定仍待确定。

量化目标怎样固定成员范围

“所有”“任一”和数量条件必须先固定集合，再判断成员。查询结果、模型候选和工具搜索不能自行成为全集。

部分必须固定失败边界
集合一个关系模板、一个量化角色、成员权威、模式、截止点和身份规则缺失范围、动态选取有利子集或重复成员时拒绝
量词all / some / at_least / at_most / exactly上界和精确满足需要封闭全集；决定性见证或反例可以提前结束
计数只按不同规范成员身份计数，并保留成员级结果与 Iknem重复日志、重试、签名和证据不能增加成员数
空集合与变化空集合必须由具名权威明确允许不得默认采用空真；动态成员上的量化 mene 尚不支持

完整决定见 ADR-0018。它只确定抽象语义，不增加 END-P1 字段，也不表示量化求值组件已经存在。

测量目标怎样固定口径和阈值

分数不是事实本身。可测量目标必须先说明测量什么、测量谁、怎样取得和聚合观察，再说明阈值怎样比较。

部分必须固定失败边界
构念与总体关系、值角色、直接标准、固定或泛化总体、用途和权威代理基准不能自行成为能力本体；固定分数不能推广未见总体
数值与单位有限值、量纲、单位、换算版本和显示舍入量纲不兼容、隐式百分比缩放和比较前舍入必须拒绝
程序与聚合生产者、窗口、纳入排除、最小样本、缺失、均值/比例/分位数或模型估计参数事后换窗口、聚合器、样本或分桶不能改变判断
阈值与区间lt/le/gt/ge/eq、阈值单位和不确定区间区间跨边界为 agno；统计程序失败为 fault

完整决定见 ADR-0019。它不增加 END-P1 字段，也不表示遥测、基准或统计组件已经存在。

复合事态怎样组合判断

复合根仍然只能描述一个不可分终态。可以独立实现、版本化、验收或失败的部分必须拆成多个 Endem，再由 Synem 组合。

部分当前规则失败边界
结构有限无环的 all_of / any_of，每个节点至少两个不同子项重复叶、环、条件脚本和任意表达式必须拒绝
对齐每个 skena 关系叶对应一个 krin 叶判据判据不能增删或错配事态分支
结果all_of 的 unmet 和 any_of 的 met 是决定性结果没有决定性结果时，fault 与 agno 不能被改写为真假
短路保存决定依据、已求值叶、未求值叶、Iknem 和停止原因未求值不是 agno；已遇到的故障不能被删除

完整决定见 ADR-0020。它不增加 END-P1 字段，也不表示组合求值组件已经存在。

安全读取与资源限制

  • 所有 offset、length、count、index、alignment、解压大小和累计预算使用 checked arithmetic。
  • 读取器必须先验证目录和资源上限，再分配或解释大型载荷。
  • 未知且影响 semion、skena、telis、krin、apor、状态、权限、绑定或完整性的关键记录必须拒绝。
  • 未知可选调试记录可以跳过，但跳过不得把失败对象提升为 coherent 或 attested。
  • 解析失败不产生部分可信 Endem；诊断必须定位到字节范围、记录或字段组。

当前状态

当前策略：Endem 使用六个语义面和一个根 skena，并分开事态与方向、三类未知、否定与缺席、量化、测量和复合判断。END-P1 已登记成对的接受与拒绝字节向量；精确范围以 Profile、登记和版本化验证结果为准。

正在研究：具体发行拼写与读音，以及把 attested 从内容形成分类迁移为显式外部关系的方案。

待定内容：时间、封闭、量化、测量与组合的物理字段，以及条件适用性、统计模型登记、压缩、摘要、签名和稳定 ABI。两个独立实现和对抗测试也尚未建立。
