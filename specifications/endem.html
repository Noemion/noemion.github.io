Endem 表示什么

Endem 是最小、独立有效且可验证的期望终态单元。它把一项自然语言目标变成可检查的对象—关系契约。

这份契约保存来源表达、已确认意义、中性可能事态、目标方向、比较规则和未决信息。它不规定模型采用哪条提示、计划或推理过程。

权威边界：这是一份面向读者的解释。语义义务与条款 ID 以 END-CORE 条款源为准；实验性容器以 END-FMT 条款源为准。END-FMT 0.1.0-draft 尚非稳定 ABI。

最小不变量：一个 Endem 必须且只能有一个根 skena 节点。根节点可以组合相互依赖的子命题；如果两个根可以独立实现、验收或失败，Poiet 必须形成两个 Endem，再由 Synem 组合。

六个语义面

语义面最低职责主要拒绝条件
rhem来源记号、主体、精确表达或内容摘要、语言、媒体类型、范围与版本。范围越界、内容摘要不匹配、编码无效、来源无法重放。
semion记号到符号、指称、关系、作用域、极性、量化、时间和来源位置的已授权投影。模型摘要被当作意义、指称无来源、作用域被默认或候选越权确认。
skena一个根、中性可能事态及其对象—关系图，显式保存组合与极性。零个或多个根、只有实体没有关系、关系角色不完整或混入目标力量。
teliskine 使事态达到成立；mene 使事态在声明区间持续成立。用力量值重复表达否定、没有时间边界或把询问/许可压成目标。
krinphain 观察类型、结构比较、必需 Tekmor、agno/fault 处理和决定权威。命题自证、只给分数没有结构比较、观察不足被当作满足或模型成为唯一权威。
apor候选解释、冲突、缺口、风险、解决策略和有权绑定的主体。只保留最高置信候选、无授权关闭歧义、未决项在状态变化中消失。

rhem/semion/skena/telis/krin/apor 是固定语义顺序；没有未决项时，apor 必须是显式空集合。END-P0 只验证编号 1–6 的结构记录；END-P1 已为未来首条实现路径的评审固定六记录内部字段、枚举、规范排序和引用闭包。任何后续 Profile 都必须恢复同一顺序、符号身份、关系位置与规范含义。

形式必须由结构显示

思想与方法基础用《逻辑哲学论》命题 3.141 解释“结构不是词语堆积”的问题来源。这里不从引文推导规范；ADR-0010、本规范和测试共同定义 Endem 的正式结构要求。

semion、skena 和适用 Tekmor 中的 phain 使用同一组稳定符号身份。关系节点必须保存谓词身份、有序或具名角色、极性、作用域和来源映射；复合节点显式保存合取、析取、否定、条件或 Profile 允许的其他组合。

规范载荷不得包含由作者填写的 logical_form、valid 或 true 自我声明。Poiet 和独立 Theor 通过关系拓扑、角色数量、组合边与引用位置分别验证结构。只保存六段文本的实现不符合本规范。

从可能事态到实际观察

skena 不宣称事情已经发生。Praxor 把实际观察规范化为 phain，再由 Tekmor 绑定生产者、方法、环境、时间域、强度、限制和完整性。krin 只能比较声明范围内可对齐的关系位置。

telis · kine / mene
phain · 观察事态
Tekmor · 范围与依据
krin · 结构比较
met / unmet / agno / fault

aseme 表示没有任何允许的意义投影，apor 表示存在多个允许候选但无权选择，agno 表示意义和判据已经确定而观察不足。三者不得互相代替；fault 只表示求值过程没有按契约完成。

mene 怎样表达持续时间

mene 只有在时间范围和连续性政策已经确定时才可判断。它不读取运行机器当前时间，也不把采样频率当作目标语义。

部分允许设计必须拒绝或保留未决
时间范围fixed 使用具名权威解析的 UTC 半开区间；elapsed 使用具名事件、固定时长和单调时钟。“现在起”“明天”、默认本地时区、日历月年、墙钟经过时长和未绑定事件。
连续性strict 要求全区间成立；budgeted 明确累计、单次和次数上限。把遥测空洞、缺少告警或几个健康采样点当作允许中断。
覆盖结果完整覆盖且预算内为 met；有效违约为 unmet；缺口为 agno；时钟契约故障为 fault。插值填洞、远端 Task 时间戳充当时钟权威，或验证器故障充当目标未满足。

完整决定见 ADR-0016。它不新增 END-P1 字段；未来 mene Profile 仍需物理字段、范围限制和正反字节向量。

自然语言场景怎样检验六个语义面

同一套字段只有在能区分不同失败原因时才有意义。以下场景不是产品演示或可执行测试，而是用来暴露规范缺口的设计审查语料。

来源场景必须表达的区别不能偷换成什么
让登记服务返回健康状态正极性 skena、kine 与健康探测关系进程存在不等于服务健康
在发布窗口内持续保持主站可用mene、时间区间、允许中断与可用口径当前时间或监控默认值不能补齐未决边界
不得让匿名用户读取私有报告否定关系进入 skena，mene 只表示保持不新增第三种禁止方向，也不把无日志当作满足
把它发布给管理层“它”的来源范围、指称候选、解决权限与 apor模型最高概率候选不能成为确认指称
确认昨夜备份可以恢复可恢复事态、恢复观察和必需 Tekmor备份文件存在不等于可恢复；观察不足是 agno

完整预期分层、跨场景不变量和当前缺口见 END-SCEN 自然语言设计审查语料。十二个场景当前登记为非规范材料，不证明 Poiet、Theor、Praxor 或任何自然语言解析与时间求值能力已经存在。

状态机

状态成立条件允许的消费者不能推导
nascent结构合法，但仍可能有 apor、引用或确认事项。elenk、theor、人工或授权解析流程。不能执行发布；不能假设未决项会在运行中自动消失。
coherent必需引用、约束冲突、能力上限和验收关系都取得确定结果。tasse、theor、组合为 Synem。不证明来源可信、当前环境授权或结果已完成。
attested发布载荷冻结，签名请求与外部响应精确匹配。theor、隔离的 praxe。签名有效不证明语义正确，也不授予实时能力。

状态变化保持 Endem 的语义身份和来源链，但每次都会产生新的精确文件摘要。不得把旧状态字节当作新状态对象，也不得在 sphra 后原地改写被签载荷。

身份与完整性

属性用途不得替代
Exact Digest对完整制品字节计算，支持内容寻址和复现比较。语义等价、发布者身份或运行授权。
Semantic Key在等价规则冻结后标识跨布局保持的目标与契约。当前尚未冻结，不能提前宣称互操作。
Signature Envelope在被签载荷之外绑定候选包、算法、签名者和策略。结构、语义、权限和环境检查。
Tekmor Link把来源、绑定、构建、签名和运行依据关联到制品身份。Tekmor 内容真实或任务最终通过。

安全读取与资源限制

  • 所有 offset、length、count、index、alignment、解压大小和累计预算使用 checked arithmetic。
  • 读取器必须先验证目录和资源上限，再分配或解释大型载荷。
  • 未知且影响 semion、skena、telis、krin、apor、状态、权限、绑定或完整性的关键记录必须拒绝。
  • 未知可选调试记录可以跳过，但跳过不得把失败对象提升为 coherent 或 attested。
  • 解析失败不产生部分可信 Endem；诊断必须定位到字节范围、记录或字段组。

当前成熟度

六个语义面、一个根 skena、事态与方向分离、phain 结构比较、三类未知、三个制品状态和 .endem 扩展名已经采用。END-P1 又形成 2 个语义接受和 11 个预期拒绝字节向量；它们覆盖非空 apor、截断、目录乱序、非最短 CBOR 和四类资源边界。量化与时间、求值语言、压缩、摘要、签名和稳定 ABI 仍需新的 Profile；两个独立实现和对抗测试只能在代码阶段开启后建立。
