Endem

一件 Endem 说明目标来自哪里、表示什么、期望什么，以及怎样判断是否满足。

Endem 表示什么

一件 Endem 只描述一项能够独立理解和判断的目标。形成版保留实际输入的自然语言，供人检查意义投影与待确认项；最终发布版移除原文，只保留发布 Profile 允许的已确认目标结构。

Endem 不保存模型提示、推理过程或执行计划。它把来源表达、已确认意义、中性可能事态、目标方向、满足判据和未决意义分开，使形成者、检查者、运行系统和决定者能够各自核对自己的责任。

权威边界：通用内容义务与条款 ID 以 END-CORE 内容标准为准；来源保留字段由 END-P1 定义；实验性容器以 END-FMT 条款源为准。最终发布 Profile 尚未定义，END-FMT 0.1.0-draft 也尚非稳定 ABI。

名称状态：六项职责已进入现行草案，具体发行拼写和读音仍在研究。技术讨论应先说直白职责，再给出代码字段；读音建议不能替代独立人类听辨证据。

先分清内容、Profile 与容器

同一份实验性 .endem 必须分别通过容器、Profile 和内容检查。较低一层通过，不会自动提升后续结论。

形成版

发布版

移除原文和可逆重建材料，重新闭合引用并取得新的精确身份；当前 Profile 尚未定义。

受控伴随资料

在获准的审计中关联形成版与发布版，不进入公开包，也不提供运行权限。

ADR-0023定义三层接受；ADR-0036定义形成版、裁剪发布版与伴随资料的身份关系。形成版签名、证据或接受状态都不能由发布版继承。

六项职责共同维持什么不变量

  • 一个 Endem 必须且只能有一个根 skena。能够独立实现、验收或失败的目标分别形成 Endem，再由 Synem 组合。
  • rhem / semion / skena / telis / krin / apor 使用固定逻辑顺序；没有待确认项时，apor 是显式空集合。
  • semion、skena 与适用 Iknem 中的 phain 共享可比较的符号身份、关系和角色位置。
  • 关系拓扑、角色、极性、作用域和组合边显示逻辑形式。作者填写的 logical_form、valid 或 true 不能让内容自行成立。
  • 第一阶段 telis 只允许 kine 与 mene；禁止事项由 skena 的显式否定表达，不再增加重复极性的目标方向。
  • Ktisor 未来负责确定性写入，独立 Theor 负责只读检查；两条路径都必须从结构恢复约束，而不是把六项职责当作六段自由文本。

现行形成分类怎样阅读

精确内容不是在原身份中不断升级的可变状态机。六项职责发生变化时会产生新内容身份；签名、证明、验证结论和准入决定作为外部关系绑定该身份。

开发者当前应按“内容形成 + 外部关系”理解这三个值。外部陈述的增加、过期或撤销不改变内容字节；修改六项职责则产生新身份与显式派生关系。现行值尚未迁移，替代词也尚未通过人类读音验证和独立迁移决定。

安全读取与当前状态

  1. 固定 END-CORE、END-FMT 与 Profile 的精确版本，不使用“最新”或环境默认值解释正式输入。
  2. 先验证目录、范围和资源上限，再分配或解释载荷；offset、length、count、index 与累计预算使用 checked arithmetic。
  3. 未知且影响意义、事态、方向、判据、待确认项、权限或完整性的关键记录必须拒绝。
  4. 解析失败不产生部分可信 Endem；诊断定位字节范围、记录或字段组，并分别报告容器、Profile 与内容结果。
  5. 声明不超过已执行的检查。当前向量只能证明登记案例与草案关系一致，不能证明解析器、运行时或互操作已经实现。

当前策略：Endem 使用六项职责和一个根事态，分开事态与方向、形成未知、观察不足、求值故障和五个结果域。

正在研究：发行拼写与读音、attested 的迁移方案，以及来源裁剪发布 Profile。

待定内容：时间、封闭、量化、测量与组合的物理字段，条件适用性、统计模型登记、压缩、摘要、签名和稳定 ABI。Ktisor、Theor、Drasor 与两个独立读取实现均不存在。
