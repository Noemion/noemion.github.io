项目/规范参考指南/Dromen

一次会话的只读契约
Dromen

理解一个已经发布的目标制品怎样在不携带永久权限的前提下进入一次受控会话。

DRO-CORE 0.1.0-draft仅限一次会话不是持久文件尚无运行时

先给直白定义

一次会话的只读执行契约（设计阶段名称 Dromen）由 Drasor 从精确制品封存。它说明“这个精确目标，在这一次会话里，可以依据哪些政策、环境、能力和预算运行”。

Endem 或 Synem 保存持久目标；Dromen 只保存本次运行的边界。它不是文件、进程、模型上下文、凭据包、可恢复会话或最终结果。

用同一次发布理解会话契约

继续使用依赖制品与服务健康组成的 Synem。发布制品不会自行携带生产权限；Drasor 必须在会话开始前重新核对精确闭包、外部陈述、验证政策、撤销状态、目标环境、能力上限和预算，然后才建立本次 Dromen。

进入会话前本次发布案例不能混入 Dromen
固定主体绑定精确 Synem 内容身份，并逐项复核外部陈述目录名、远端“最新”对象、可修改制品副本
固定环境绑定部署区域、适配器、协议、模型后端和时钟来源运行途中发现的新工具或隐式模型切换

封闭预算声明调用、重试、时间、字节和成本上限无单位的“适量重试”或模型自行追加预算
分配观察责任预先说明谁观察制品摘要、部署响应和健康窗口工具成功、会话完成或远端 Task 状态直接充当最终决定

如果目标区域、适配器版本、政策、能力范围或具名权威发生实质变化，旧 Dromen 立即失效；系统必须结束或中断本次会话，再建立新契约，不能原地修补。

Drasor 是 Dromen 的唯一建立者，也是失效和销毁责任人。运行后端只消费获准的最小能力；Iknem 生产者记录结构化观察；具名验收权威最终决定是否接受。任何一个下游都不能反向改写 Dromen 或已签制品。

环境变化怎样处理

模型版本、适配器、协议、政策、能力范围、决定权威或隔离配置发生实质变化时，Drasor 使旧 Dromen 失效，并按预先声明规则中断或失败。旧观察和 Iknem 保留，不能被新会话覆盖。

短期凭据可以在独立能力域内轮换，但前提是资源、受众、范围和授权没有变化。一旦授权边界改变，就必须建立新的 Drase 会话。

它与 Agent 协议的关系

MCP 的能力协商、授权范围和长任务，A2A 的 Agent 与 Task 状态，都可以成为 Drasor 的外部输入。它们不会成为 Dromen 的权威，也不会直接改变 Endem、Synem、满足结果或最终决定。

因此，工具列表变化、模型切换、远端 Task 完成和 step-up 请求都必须先经过本地 Dromen 边界。系统不能因为协议返回成功就说目标已经满足，也不能因为远端请求更多权限就修改当前会话。

权威规范和验证资料

DRO-CORE 会话条款规定主体、政策、环境、能力、秘密、预算、激活、观察、只读和生命周期。 Dromen 威胁模型检查主体替换、权限放大、凭据泄漏、预算逃逸、契约漂移和会话复活。 Dromen 自然语言场景用支持案例、反例和边界场景检查会话职责；它们不是实现或符合性声明。 Dromen 提案向量记录允许契约与确定拒绝的资料一致性矩阵。 记忆、检查点与恢复研究提案分开会话历史、压缩、检查点、外部任务、恢复与重放；旧 Dromen 和权限不能复活。 模型、适配器与能力域隔离研究提案把模型、控制面、凭据、适配进程、文件、网络、资源终止和观察拆成可审查责任；不创建新隔离对象。

当前状态

当前策略：DRO-CORE 用核心规则定义一次会话的范围。未来实现不得把 Dromen 当作持久文件、凭据包或可恢复运行状态。

名称仍在研究：Dromen 是设计阶段名称，发行拼写、读音流畅度和与 Endem、Synem、Iknem 的口头区分度尚未通过真实使用者验证。面向开发者的说明必须先写“一次会话的只读契约”。

限制条件：当前 attested 主体条件仍是草案中的单值表达，不能完整表示同一内容的多项签名、不同验证政策、截止点和撤销结论。开发者应按内容形成与外部陈述的两轴模型阅读这一条件；正式字段迁移仍在研究，现行规范值尚未改变。

Dromen API、平台隔离方式、事件格式和运行后端仍待确定，需要分别设计与验证。非规范模型、适配器与能力域隔离研究提案已经分开部署责任与未来证据阶梯，但尚未成为规范要求。当前没有 Drasor、装载器、沙箱、凭据代理、预算器或运行时；精确场景与向量范围以对应源文件和版本化验证结果为准，提案向量只描述规范预期。
