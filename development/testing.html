测试与验证

Endem、Synem 与 Drasor 只有通过明确失败路径，才能获得对应范围的信任。

Malformed CorpusCross-readerReproducible

解析与资源安全

基本规则：“合法样例通过”只证明一条成功路径。畸形、冲突、越权、超限和不完整输入必须稳定、快速且不产生部分可信结果地失败。

  • 覆盖非法偏移、长度、计数、索引、对齐、重叠、环、自引用、超深图、压缩炸弹和恶意大计数。
  • 所有加法、乘法、对齐与区间端点使用 checked arithmetic；超限与溢出是规范错误，不是崩溃。
  • 未知关键结构必须拒绝；未知可选结构按规范跳过并保留诊断。读取受字节、节点、深度、内存和时间预算约束。
  • 错误必须定位到条款 ID 与字节、记录或语义路径；任何未执行层都不能被描述为通过。

形成、变换与复现必须分开

直接结论：相同输出只在已经声明的比较域中有意义。字节相同、回转成功、结构符合 schema 和两次结果一致，都不能自动证明来源相同、语义等价或目标正确。

验证主张必须固定什么最多能证明什么不能越过的边界
来源与文本变换原始字节身份、文本槽、严格解码 Profile、每次显式变换和损失指定输出确实由指定输入按记录步骤产生解码文本或显示视图不是原始文件；变换成功不证明可逆或意义相同
确定性形成解码输入、授权决定、配置、依赖、规范、Profile、格式和 Ktisor 版本的完整闭包同一封闭输入产生同一规范结果；在精确格式域中逐字节相同不得用隐式规范化构造“相同输入”，也不证明输入主张为真
显示与导出输入对象、渲染器或导出器版本、选项、删减和显示环境生成一份可追溯的派生视图派生视图不能反向重建原始来源、签名、权限或全部制品字节
回转与迁移源域、目标域、双向映射、保留属性、损失、版本和失败责任已声明属性在已声明路径中保持当前没有通用 round-trip 或跨 Profile 语义等价要求
独立复现精确输入闭包、两个独立产出路径、工具与环境约束两份输出逐字节相同或不同字节相同不证明设计正确；字节不同也不能自行判断哪份正确

分别报告 Endem 字节复现、工具二进制复现和发布签名真实性。至少使用两个工作目录和两个隔离环境；区域设置、时区、临时路径、归档顺序和时间戳不得污染结果。

双读取器契约

  • Ktisor 的生产解析器直接读取实际字节，并只在全部检查通过后产生内部验证句柄。
  • endem theor 使用不共享解析代码的独立 Theor，只输出有界视图、差异、引用、大小与跟踪信息。
  • 两者对共享向量逐字段比较解释、上限和失败分类；查看器输出、符号列表或格式化文本永不回流生产信任链。
  • 差异必须被视为实现或规范缺陷，不能用“多数实现一致”掩盖。

当前验证材料能证明什么

当前证据：登记、语义向量、专题提案向量、Iknem 证据向量、实验字节和来源样例可以比较预期接受或拒绝结果。这些材料只证明规范之间是否自洽，不代表 Ktisor、Theor、Drasor、求值器或决定引擎已经存在。

  • 现有验证覆盖已登记字段、引用、来源范围、排序、资源边界和结果域禁止转换案例。
  • 10 个 Endem 语义向量包含三个接受、七个确定性拒绝和一组外部语境身份等价检查；测试语境不是 Endem 字节或授权物理格式。
  • 规范向量描述未来实现必须共同处理的正反案例，但不证明任何组件已经满足这些要求。
  • 当前没有组件依赖树、可执行文件复现、实现级模糊测试、故障注入、签名或 SBOM 证据。
  • Ktisor 与 Theor 的实现必须分别接受同一组测试。

语义与歧义保留

  • 逐来源单元验证记号、符号、指称、关系角色、作用域、力量、满足条件和待确认分支得到保留。
  • 语义变异测试分别交换关系角色、翻转极性、改变 kine/mene 或弱化一条 krin，确认身份或验证结论发生预期变化。
  • 模型置信度不能关闭 apor；只有规则可重推导或具名语义决定才能确认语义。
  • 一个 Endem 只能有一个根 skena；多目标组合必须通过显式引用进入 Synem。
  • 违反 Profile 或无允许投影的输入必须稳定返回 aseme；观察不足必须返回 agno，不能混入 apor。
  • 十二个结果域提案向量分别执行 met/unmet/agno/fault、accepted/rejected/deferred、Drase 会话和证据状态的允许组合与拒绝路径；它们不模拟运行组件。
  • 十二个 mene 提案向量检查固定区间、经过时长、连续性预算、观测缺口和时钟故障的分类边界；它们不是计时器、监控器或求值组件。
  • 十二个否定与缺席提案向量检查显式负观察、空结果、封闭范围、正反例和观察故障；它们不是日志收集器、Drasor 或求值组件。
  • 十二个量化与成员资格提案向量检查集合范围、空集合、不同成员计数和决定性聚合；它们不是成员目录、Drasor 或求值组件。
  • 十二个测量与阈值提案向量检查构念与总体、单位换算、窗口、聚合器、不确定区间和阈值分类；它们不是采集器、统计引擎、Drasor 或求值组件。
  • 十二个复合事态与判据提案向量检查单根边界、有限无环拓扑、叶对齐、四结果传播和决定性短路；它们不是解析器、Drasor、运行时或求值组件。
  • 十二个 Synem 提案向量检查完整闭包、精确绑定、有限无环、权限交集、成员结果和激活四状态；它们不是 Pleko、解析器、Drasor 或运行时。
  • 十五个 Iknem 提案向量检查精确主体、有限范围、无环溯源、观察类别、完整性、外部有效性、相对 krin 的覆盖、决定权限和最小披露；它们不是采集器、验证器、归并器、撤销服务或决定引擎。

绑定、发布与签名

  • 引用可见性、版本、必需/可选依赖、ID 重映射、约束可满足性、权限交集和冲突矩阵全部有正反向量。
  • 链接顺序不改变规范结果；任何能力合并只能保持或收窄上限。
  • 当前没有可用的裁剪等价关系。未来只有版本化保留关系明确允许时，对应制品生产者才可变换；无法证明保留时保留内容或失败。主对象、调试伴随记录和其他派生物各有精确身份、访问权限与生命周期。
  • 私钥始终在外部签名系统；签名响应必须精确匹配请求、主体摘要、策略和发行身份。
  • END-P1 中的权威字段只定位候选权威；缺少或错绑外部授权决定时，独立文件不能越过 Profile 接受，也不能生成新的内容身份。

Drasor、控制平面与 Iknem

  • Drasor 未来必须重新验证收到的实际 Endem 字节和 Synem 精确成员闭包，不信任路径名、查看输出或缓存结论；Synem 物理字节尚未定义。
  • 控制平面持有实时能力句柄，并检查最小权限、读写分离、幂等、副作用、提示注入、污染返回和代理混淆风险。
  • 模型只能产生候选和能力请求；AUT-CORE 约束语义确认和能力授权，真实观察返回会话，具名权威或确定性规则再按独立结果域决定验收。
  • Iknem 绑定精确主体、有限范围、无环溯源、环境、方法、观察和限制；有效性由外部政策评估，覆盖只相对当前 krin 计算。覆盖不足产生 agno，决定保持 deferred 或由具名关闭失败政策拒绝，不能直接 accepted。
  • 恢复测试覆盖工具失败、环境漂移、重复失败、预算耗尽、取消竞态、状态冲突和人工判断分支。

模型与协议适配

  • MCP/A2A 适配器覆盖版本不兼容、未知必需字段、恶意工具或 Agent 描述、参数结构欺骗、状态伪造与静默降级；外部 completed 不能映射为本地 met 或 accepted。
  • 结构化输出测试必须包含结构合法但事实错误、引用悬空、跨字段冲突或权限越界的候选。
  • 模型任务执行多次独立运行，报告成功率、置信区间、模型、工具、策略身份和失败切片；固定随机种子或零温度不是跨版本确定性证明。
  • 模型评价器只产生建议性评价；机器可判定的 accepted 来自确定性规则，需要价值判断时由具名外部权威签署。

模型参与评测必须限定用途

直接结论：模型评分是有版本、有偏差且可能漂移的候选测量。它不是人工权威、统计区间、充分证据或最终决定。

检查必须保存失败时怎样处理
评测目的构念、用途、总体、评分说明和决定者缺少可观察标准时保留待确认，不用模型猜测
实际协议题目、候选顺序、提示、模型版本、设置、工具、重复和解析身份或设置无法固定时不形成可比较结果
偏差探针位置、长度、格式、来源、提示注入、重复和时间漂移反例探针失败使方法在对应范围失效
统计汇总estimand、评审者依赖、缺失、停止规则、区间和同分政策区间跨越阈值时保持 agno
证据与决定原始输出、model-candidate、人工校准、覆盖和具名权威多个模型投票不能自动产生 sufficient 或 accepted

完整的支持案例、反例、威胁与停止条件见模型参与评测与裁判边界研究提案。它当前非规范，不创建评测 CORE、裁判对象、命令或组件。

模型更新必须重新建立证据链

直接结论：训练完成、损失下降、固定种子或部署回滚都不是模型更好、可复现、可发布或目标满足的证明。

检查必须保存失败时怎样处理
数据与反馈资格精确快照、来源、用途、权利、同意、切片、合成来源和排除身份或资格不明时拒绝进入对应训练目的
训练活动基础模型、适配器、处理、代码、参数、环境、随机状态、停止条件和输出身份配置或实际执行无法闭合时不形成复现主张
独立复现软件闭包、数据顺序、硬件、数值路径、独立产出和逐字节比较只固定环境或种子时明确限定证据等级
行为评测新旧精确模型、用途、总体、污染检查、偏差反例、统计区间和退化切片固定基准提高不能掩盖子群或真实用途退化
发布与回滚具名权威、政策、受众、部署配置、截止点、撤回和运行监测训练或流水线成功不能自动产生 accepted

完整边界见模型训练与更新边界研究提案。它当前非规范，只使用直白职责短语，不创建训练 CORE、模型制品、训练平台、命令、组件或新专名。

外部机制只支持有限主张

权威机制可采用的验证纪律不能直接推出
GNU Guix challenge从独立来源取得构建结果并逐字节比较，同时保留差异供调查。相同字节不能证明源设计正确；不同字节不能自行判定哪一份可信。
GNU BFD canonical form通用内部表示必须披露源格式、目标格式和无法表示的信息。进入同一 canonical form 不能证明格式特有内容、来源和语义都被保留。
RFC 8785 JSON Canonicalization Scheme规范形式必须限定输入域、数值和字符串处理，并保留数组顺序。可哈希 JSON 不证明业务语义、权威或跨 Profile 等价。
OpenAI Structured Outputs把模型输出约束到精确 schema，减少结构不匹配并允许确定性结构检查。schema 合规只证明形状；字段中的事实、引用、授权和满足判断仍是不可信候选。
NIST AI 800-2 初稿与 NIST AI 800-3先固定评测目的、协议、模型版本、统计假设和限定用途，并分开固定基准与推广结论。基准名、模型裁判、一致率或点估计不能自动成为普遍能力和发布决定。
NIST AI 600-1、NIST SP 800-218A与 GNU Guix追踪训练与微调数据、模型派生物、训练流程和反馈；固定完整软件环境并单独比较输出。SBOM、provenance、相同环境、固定种子或训练成功都不能自动证明权重复现、行为安全或发布获批。
GNU Diffutils 与 Coreutils 随机来源分开“相同、不同、比较故障”，并记录候选顺序使用的随机来源。差异不选择正确一方；固定随机源也不证明模型评审没有位置偏差。

这些机制共同支持“先声明比较域，再解释结果”。它们不支持一个跨文本、对象、模型和版本的通用等价结论。
