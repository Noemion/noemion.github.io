测试与验证

说明对象安全、确定性编译、语义保真、链接、披露、信任和模型候选需要通过哪些测试。

安全与拒绝路径

读懂测试结论：“合法样例通过”只证明一条成功路径；安全核心还必须证明畸形、冲突、越权、超限和不完整输入能够稳定、快速且无部分产物地失败。

  • 对象解析测试覆盖非法偏移、长度、计数、索引、对齐、重叠、整数溢出、循环引用、超深图和恶意大计数。
  • 签名与完整性测试覆盖错误范围、错误 Merkle Proof、篡改、回滚和撤销。
  • 每个验证层独立报告；未执行或未通过的层不能被描述为通过。
  • 错误输入必须关闭失败，且诊断能定位到规范条款以及字节、记录或节点位置。

编译与复现

  • 基准样例、规范化文本往返转换、类型错误、约束冲突和覆盖缺口。
  • 生产 Reader/Writer 与不共享解析代码的独立 Reader 使用同一字节向量，逐字段比较解释、限制和失败分类。
  • 相同输入、工具版本和规范化构建环境的对象字节级复现；工具二进制复现与发布者签名真实性分别报告。
  • 至少在两个工作目录和两个隔离环境重复构建；归档顺序、locale、时区、固定构建纪元和临时路径不能污染结果。
  • 对象差异工具必须区分布局、调试、信任元数据和运行语义变化。

设计方案：语义与覆盖测试

  • Source-to-NIR 蕴含检查、反编译摘要和 Semantic Mutation Testing。
  • 逐来源单元验证目标、硬约束、条件、例外、路由、依赖和歧义得到保留。
  • 无法理解、冲突和未分类输入阻断发布；模型置信度不能替代覆盖证据。
  • 披露测试同时验证必需内容完整和无关节点最小化。

设计方案：链接、HOBJ 与 Fulfillment Runtime

  • local/export/import、必需/可选依赖、ID 重映射、约束可满足性、权限交集和语义冲突矩阵。
  • 主 HOBJ 与必需依赖闭包完整；可选依赖缺失行为明确。
  • strip 前后语义图、硬约束、依赖、验收契约和披露行为满足规范化属性等价；只有等价规则冻结后才比较运行语义哈希。
  • 预算不足、披露越权、会话重放和状态漂移必须安全拒绝或重新同步。

设计方案：Agent Harness 验证

  • 上下文测试验证版本、来源、适用范围、失效链接和逐步披露，不允许临时提示静默覆盖规范。
  • 能力测试覆盖参数结构、最小权限、读写分离、幂等性、副作用、提示注入和被污染的工具返回。
  • 反馈测试让智能体读取真实 UI 状态、日志、指标、追踪和测试结果，完成故障复现、修复验证和结果对照。
  • 恢复测试覆盖工具失败、环境漂移、重复失败、预算耗尽、状态冲突和需要人工判断的分支。
  • 端到端验收同时检查任务结果、过程合规和证据完整；智能体自述成功不能作为验收证据。

外部协议与模型适配测试

  • MCP/A2A 适配覆盖协议版本不兼容、未知必需字段、恶意 Tool/Agent Card、schema 欺骗、任务状态伪造与静默降级。
  • 授权测试覆盖 token audience、精确资源绑定、token passthrough、confused deputy、SSRF、重放、取消竞态和重复执行幂等性；仅在 OAuth Authorization Code Grant 用例中按客户端类型验证 PKCE。
  • 结构化输出测试必须包含完全符合 schema 但事实错误、引用悬空、跨字段冲突或权限越界的候选。
  • 模型和 Agent 任务执行多次独立运行，报告成功率、置信区间、模型/工具/策略身份和失败切片；seed 或 temperature=0 不作为跨版本确定性证明。
  • LLM 评价器只能产生 advisory assessment；机器 accepted 由确定性验证器形成，需要价值判断时由具名外部权威签署。

模糊测试从第一天开始

模糊测试至少覆盖 NOBJ/HOBJ 读取器、文本与 NSL Parser、重定位、证据账本、披露图、依赖闭包、Strip 和 Merkle Validator，并限制资源占用、递归深度和压缩扩张。完整性负向样例必须包含“攻击者修改载荷并同步重算对象内 Integrity Root”：在缺少受信外部 Exact File Digest、Package Subject Digest 或签名绑定时，该对象最多保持内部自洽，绝不能提升为可信状态。

后续计划：模型与跨平台矩阵

Horizon Engine 阶段增加来源隔离、困难负例、歧义、OOD、跨块关系、错误确定率、校准和量化退化；Agent Harness 阶段增加任务成功、越权拒绝、工具污染、长时恢复和人工升级测试；稳定阶段增加平台矩阵、性能预算和双构建复现。具体阈值须由基线与 ADR 决定。

证据等级与研究复现

等级适用结论最低材料
示例说明接口或候选语义明确输入、预期输出和非规范声明
回归证明已知行为未被破坏固定语料、版本、断言和失败诊断
性质与安全支持不变量和拒绝路径属性测试、模糊测试、资源限制、畸形与对抗案例
互操作支持配置或标准化主张至少两个独立实现或独立验证器、共享向量和差异报告
研究实验支持论文或模型效果结论假设、基线、数据许可、切分、消融、统计不确定性和失败分析

任何公开结果都应记录规范与工具版本、构建环境、随机性控制、硬件、原始数据摘要和复现实验步骤。内部通过但材料不可复现的结果不能升级为标准化或科学结论。
