CURRENT规范与安全二进制核心设计

Noemion 当前仍处于设计阶段。工作重点是收敛 GSIR、GOBJ、SSO 的最小规范边界，并为安全 Reader/Writer、验证器、合法样本和畸形语料定义可以实现和测试的契约；尚未发布可执行工具、稳定 ABI 或正式规范制品。

项目进度时间线

时间线不是完成百分比：阶段只在进入条件和退出证据可以核对时改变。当前没有足够依据承诺具体日期，因此不展示虚构进度或发布日期。

  1. 已确认

    问题、范围与信任原则

    项目定位、Strict 无模型边界、模型输出不可信、对象输入不可信、歧义优先保留等原则已经进入公开项目定义。

  2. 当前阶段

    规范与安全二进制核心设计

    正在收敛对象语义、格式边界、错误语义、资源限制、checked arithmetic、不变量以及合法与畸形样本的最低集合。

  3. 下一阶段

    安全二进制核心实现与验证

    在规范条款和 ADR 足够稳定后，实现 Checked Reader/Writer、结构验证器、Round-trip、Malformed 与 Fuzz 拒绝路径。

  4. 未来阶段

    文本 IR、Strict 编译与链接

    逐步进入规范化文本往返、确定性 Compiler Core、符号解析、重定位、归档、依赖闭包和链接一致性。

  5. 未来阶段

    可信装载、发布与模型辅助

    只有前序安全核心和一致性门通过后，才评估签名装载、最小 Runtime、SSO 发布以及提交不可信候选的 Assisted 前端。

当前阶段的进入条件

  • 项目问题、非目标、信任边界和第一阶段范围已经公开说明。
  • GSIR、GOBJ、SSO、Compiler Core、Linker/Loader 与工具职责已有可继续评审的设计入口。
  • 规范先行、安全核心先行、模型最后接入和失败关闭已成为项目级约束。

尚未满足的退出证据

  • 最小对象布局、类型边界、错误语义、限制值和演进规则尚未形成稳定规范与 ADR 集合。
  • 合法、边界、冲突和恶意对象语料尚未达到可以约束独立实现的覆盖程度。
  • 安全 Reader/Writer、验证器、字节级复现、Round-trip 与持续 Fuzz 尚无已发布实现和结果。
  • 因此当前不能宣称 Phase 0 或安全二进制核心已经完成，也不能给出稳定 ABI、版本或发布日期。

进入下一阶段的判断

当最小规范条款能够被正反样例和一致性测试约束，关键安全决定已经进入 ADR，且 Reader/Writer 的实现边界不再依赖未决语义时，项目才进入“安全二进制核心实现与验证”。详细阶段定义见实施路线图，证据要求见测试、Fuzz 与验证策略。
