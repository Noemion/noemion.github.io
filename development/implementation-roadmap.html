开发路线图

从最小规范和恶意语料开始，逐步推进确定性编译、链接、可信装载和模型辅助。

PhasesCompletion CriteriaReproducible

开发原则

规范先行

格式、类型系统、执行语义和信任边界先进入规范与 ADR，再由工具共享同一结构定义 registry。

安全核心先行

先让恶意对象无法越界、溢出、递归失控或产生部分可信结果，再建立更高层功能。

纵向切片

每阶段交付可验证输入输出、稳定错误、基准样例与拒绝路径，不以内库完成替代验收。

模型最后接入

Deterministic 链保持无模型；Horizon Engine 只提交不可信候选，不改变对象格式和可信核心。

设计方案：开发阶段与完成标准

阶段主要产物完成标准
Phase 0
规范与样本		格式、结构定义 registry、错误码、限制值、ADR、合法和畸形语料每个字段、索引、长度与演进规则都有条款和正反样本。
Phase 1
安全二进制核心		Checked 读取器/对象写入器、Header、Section、字符串与符号表畸形对象不能越界、溢出、无限递归或形成部分可信结果。
Phase 2
文本 IR 往返		规范化文本 NIR、汇编、反汇编和语义差异规范化往返转换一致，对象字节可复现。
Phase 3
Deterministic 编译		NSL Parser、Sema、实现化、证据账本、NOBJ 生成无模型环境可完成编译；相同规范化输入产生相同字节。
Phase 4
归档与链接		归档、符号解析、ID 重映射、重定位和冲突矩阵强依赖完整、无静默覆盖，链接顺序不改变规范结果。
Phase 5
HOBJ 与发布		披露图、strip、Coverage Proof、验证结论、候选包、签名请求与签名回填strip 前后运行语义相同；最终包绑定发布对象、依赖、策略、模型、证据和外部签名身份。
Phase 6
可信装载与运行		Merkle、权限保护、Loaded State、Agent Harness、最小 Fulfillment Runtime、Trace Stream篡改、回滚、越权披露、工具污染、预算不足和状态漂移产生明确拒绝或升级结果。
Phase 7
Model-assisted 前端		数据集、Horizon Engine、校准、OOD、量化、模型包和模型评估模型只能提交候选；关闭模型后 Deterministic 与对象格式不变。
Phase 8
稳定与一致性验证		端到端任务评估、跨工具一致性、平台矩阵、模糊测试、性能和复现报告规范套件、恶意语料、Agent 场景、平台矩阵和双构建复现全部通过。

设计方案：工具职责分组

  • 规范与对象：noemcertify、theoria、noemvalidate 与 noemtransform。
  • 文本 IR：morphe、noemdecode、noemformat 与 noemcompare。
  • 编译链接：noesis、noemanalyze、noemarchive、noemsymbols 与 synthesis。
  • 发布运行：noemreduce、noemcoverage、noembundle、noemexecute 与 noemobserve。
  • 模型工程：noemdataset、noemtrain、noemevaluate 与 noemquantize。

主产物顺序为“来源 → NOBJ → 链接对象/HOBJ → Release Object → Release Coverage Proof → Unsigned Package Candidate → 外部签名回填 → Signed Noemion Package → Loaded State → Run Evidence → Evidence Closure Report → Acceptance Decision → 离线评估”。工具名表达规划职责；具体 CLI、参数、归档扩展名和包格式尚未冻结，必须由各工具文档与 ADR 决定。

待定事项：仓库、接口与实现选择

仓库目录、构建系统、实现语言、稳定错误码、CLI 语法、文件扩展名、ABI 和规范演进策略仍需逐阶段确定；未经 ADR、规范条款和一致性证据批准的示例不得升级为稳定接口。

后续计划：Horizon Engine 与一致性验证

模型数据、训练、蒸馏和量化属于 Phase 7；跨平台一致性验证与性能预算属于 Phase 8。两者都不进入第一阶段。

交付物追溯

Phase 0–8 表示技术能力的开发顺序。每个实现任务都要关联对应规范、设计决定和测试结果；只有完成当前阶段标准的制品才进入后续开发或发布准备。

追溯对象必须链接到评审问题
研究问题或需求用例、现有技术、风险、规范范围问题是否真实且可证伪或验收
规范条款设计决定、正反样例、一致性测试是否无歧义、可实现、可测试
实现任务规范版本、接口、威胁与测试计划是否越过已批准边界或制造隐式语义
验证结果输入、环境、版本、基线、原始结果是否可复现并支持对应主张
论文/专利/软著/标准材料贡献记录、证据、权属、公开版本表述是否超出真实成果与审查状态
