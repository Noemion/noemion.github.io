开发路线图

路线图回答三个问题：当前可以推进什么，进入下一层需要什么证据，以及什么情况下应停止建设。

方案设计阶段证据驱动尚未实现

当前可以推进什么

直接结论：当前可以审查术语、规范、案例、威胁和验证设计。组件、运行时与外部协议适配仍需等待前一层留下可复核证据，并出现承担失败责任的真实消费者。

工作层现在怎样推进进入下一层需要什么证据何时停止或返回
术语与规范固定对象、意义、事态、判据、未知和结果域，补齐案例、反例与威胁不同读者能按同一条款定位责任；主要使用语言中的朗读、听写和相邻词区分通过人类验证普通工程词已经足够，或专名持续难读、易听错时，停止增加专名并重新命名
确定性形成与独立检查设计封闭输入、规范结果、分层失败和只读解释的验证方案两条独立读取路径、资源上限、畸形输入、故障注入、差分调查和双环境逐字节复现方案真实案例暴露对象边界错误时返回规范；内部一致性需求不自动增加公开动作
多目标组合与发布说明依赖闭包、权限交集、裁剪关系、派生身份和签名主体真实多目标消费者、冲突反例、顺序无关结果、来源保留关系和可复现发行方案单个 Endem 已经足够，或没有复用消费者时，省略通用组合与变换
受控运行与证据限定一次会话可使用的对象、政策、环境、能力、预算和观察责任实际字节重验、最小能力、终止与孤儿副作用测试，以及证据记录与决定权威分离控制无法强制、外部效果无法辨明或观察责任无法闭合时，关闭对应能力
模型与外部协议研究候选、计划、工具调用、远端状态和遥测怎样进入本地边界固定协议版本、降级与取消反例、提示注入测试、数据路径、无模型基线和可测净增益成熟外部工具已满足需求，或模型没有不可替代净增益时，不自建相应平台
设备内语义抽取模型以纯文本 Qwen3.5-0.8B-Base 为当前候选，保留原始词表和长上下文结构；本机只设计 LoRA/QLoRA、合并、量化与 CPU 评测闭环，最终训练计划迁移到可导出完整权重的外部算力环境精确数据与权重身份、隔离测试集、无模型和未微调基线、字段级抽取指标、长文档合并反例、量化回归、CPU 资源测量和外部训练权重导出验证微调没有可测净增益、量化后不满足资源上限、长文档覆盖不可接受，或外部服务不能导出可设备内运行的完整权重时，返回数据、模型或服务选择

确定性形成（form）只有受控来源、已确认意义、精确规范、Profile、配置和依赖能够影响结果同一封闭输入产生同一规范结果；任何失败都不留下部分制品
生产侧分层检查（lint）结构、Profile、引用、语义、资源和完整性分别报告，未执行层保持未知每个拒绝都有稳定失败位置与责任；检查成功不提升为目标满足或授权
独立只读解释（inspect）不复用形成侧解析器，不拥有写入、封装、密钥或运行权限任意不可信字节都能有界处理；两条路径的分歧能够定位和复现

form、lint 与 inspect 是现行设计标识，不是已发布接口。它们尚未完成正式发行前的读音、软件生态、包名与目标法域复核；若直白职责名称更容易理解和口头区分，发行名称仍可改变。Rust 与 C 的既有研究也只提供未来比较材料，不锁定实现语言或工具链版本。

GNU 技术提供哪些工程纪律

来源Noemion 借鉴的纪律采用限制
GNU readelf用独立读取路径直接检查实际字节，避免形成侧解析器的同一缺陷污染两条证据链不为每种视图创建一个公开程序，也不把查看结果写回制品
GNU BFD 信息损失说明跨格式表示必须显式披露来源域、目标域和无法保留的内容不把内部规范形式称为语义等价，也不在没有消费者时建立多格式通用表示
GNU Guix challenge从独立来源取得产物并逐字节比较，同时保留差异供调查相同字节不证明设计正确；不同字节也不自动选择可信一方
GNU 手册规范先定义术语，再按开发者要完成的工作和问题组织教程与参考不按内部组件清单组织公开学习路径，也不复制 GNU 的命令或格式

Section、Segment、符号、重定位、裁剪和调试分离的逐项适用性见GNU 与 ELF 适用性研究。该研究只提供比较材料，不定义 Noemion 格式或组件。

外部 Agent 与模型何时接入

外部机制只有在本地责任已经明确、版本能够精确固定且失败可以关闭时才进入适配研究。它们提供互操作事实，不替 Noemion 决定目标含义、证据范围、运行授权或最终接受。详细研究路由见Agent 系统边界指南。

开发问题与当前基线进入适配前要取得的证据外部机制不能证明什么
协议版本与状态：MCP 2025-11-25仍是官方 Current 版本；2026-07-28 候选版包含破坏性变化固定版本、能力与扩展，验证协商失败、降级、缓存、取消和未知状态；候选版在正式发布前只作为迁移风险工具返回、远端任务终态或协议成功不能产生本地 met、accepted 或权限扩大
身份与授权：NIST AI Agent Standards Initiative分别推进协议、身份基础设施和安全评价绑定实际行动者、被代表主体、对象、动作、目的、受众、截止点和具名政策，并验证逐级收窄的委托Agent Card、登录、scope、签名或人工确认界面不能自行成为语义授权
任务与副作用：A2A 1.0分别定义 Task、Message、Artifact、取消、绑定和版本保存请求、实际调用、已发生或未知的外部效果；只有具备幂等或去重证据时才允许自动重试远端 Task 状态不能替代 Endem 身份、evidence entry 证据、session contract 会话上限或本地结果域
模型与数据：规划、记忆、检索、训练和托管执行属于不同数据路径记录实际模型输入、预处理、模型与版本、保留和删除条件、无模型基线、评测总体与不确定性模型流畅度、基准得分、开放权重或可调用 API 不能证明目标忠实、服务控制或安全净增益
遥测：OpenTelemetry 语义约定 1.43.0已把 GenAI 约定移至独立仓库固定仓库提交与 Schema URL，限定最小字段、披露预算和接收方，再评估默认脱敏的单向导出Schema URL 与遥测记录不进入 Endem 编码、evidence entry 身份、授权决定或最终接受

当前模型策略：设备内阶段只验证纯文本小模型从数据到 CPU 量化推理的完整路径；外部算力阶段才比较全参数微调、长上下文强化和教师蒸馏。详见模型训练与更新边界研究。只提供封闭 API 且不能导出完整派生权重的服务，不能产生计划中的设备内发行候选。

当前限制条件

当前没有 Rust 组件、CLI、协议适配器、沙箱、签名 Profile、安装包或软件发行。开发者现在可以审查规范与验证设计，并通过开发与贡献入口提交事实、术语、案例和边界修订；路线图不承诺日期，也不把研究主题写成待实现功能清单。
