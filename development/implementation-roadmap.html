开发原则

一个原语

先证明一个 Endem 能稳定表示一个根终态，不为架构对称提前增加格式。

一个入口

公开应用只有 endem；子命令按消费者组织，信任域用独立实现和进程隔离。

失败优先

先定义畸形、冲突、越权、超限和未决输入怎样关闭失败，再实现成功路径。

模型最后

模型只提交候选；无模型路径必须能够形成、检查、组合、装载和验收规则型案例。

阶段、进入门与停止条件

阶段只交付什么进入下一阶段的验证条件停止或合并条件
Phase 0
词汇与规范骨架		Endem、Synem、Dromen、Tekmor；六个语义面、一个根 skena、形式显示、aseme/apor/agno；条款 ID 与正反向量关系角色、投影、满足状态与错误有可复查定义，每个规范“必须”都有测试或明确人工权威不能说明消费者或失败责任的名词删除
Phase 1
最小安全核心		poie/elenk/theor/peira，规范 Writer、生产 Parser、独立 TheorPoiet/Theor 差分一致、恶意语料关闭失败、资源有界、双环境逐字节复现若最小字段仍不能表达真实任务，回到规范，不叠加补丁格式
Phase 2
组合		pleko，引用解析、冲突矩阵、固定依赖与 Synem至少两个真实多 Endem 案例；链接顺序不改变规范结果；权限只能收窄没有真实复用消费者时不建设归档、动态链接或共享披露文件
Phase 3
发布		tasse/sphra、Debug Companion、外部签名请求与回填裁剪等价、闭包完整、签名主体精确、可复现发行与 SBOM通用 transform/strip/archive 仍无消费者时保持为内部类型化步骤
Phase 4
受控运行		praxe、隔离 Praxor、Dromen、Tekmor、规则型后端实际字节重验、能力最小化、事件完整、验收权威分离、恢复与升级路径若单一 Tekmor 无法覆盖不同权威，再按权限或生命周期拆分，不按流程步骤拆分
Phase 5
模型与协议适配		模型 Candidate Envelope、MCP/A2A gateway、版本化遥测 exporter相对无模型基线在错误确定率、上下文成本或任务成功率上证明净增益没有不可替代增益时停止模型训练平台建设，继续使用外部成熟栈

从 GNU 工具链借什么

来源采用不照搬
ELF / linker结构视图与装载视图分离、显式引用、类型化绑定、闭包、link map、失败即拒绝地址重定位、机器指令、弱符号默认选择、环境搜索和 lazy binding
GNU readelftheor 必须独立于生产解析库，使一个实现的错误不会污染两条证据链为每种查看方式创建一个稳定二进制
GNU BFD认识统一抽象的价值和信息损失风险Phase 1 不建立多格式 canonical layer；先把唯一格式做对
GNU 发行实践固定 locale、时区、路径与构建纪元；两环境复现；调试信息分离并绑定摘要把构建工具本身当作复现证明

GNU 官方资料明确说明 https://www.sourceware.org/binutils/docs/binutils/readelf.html 独立于 BFD；https://sourceware.org/binutils/docs/bfd.html 同时说明跨格式 canonical 转换可能丢失信息。这里借鉴的是验证与职责思想，不是命令清单。

面向 Agent 时代的适配边界

  • MCP 工具说明、注解和远端返回都只是 Praxor 外缘的不可信输入；能力授权来自 Dromen 与会话策略。参见 https://modelcontextprotocol.io/specification/2025-11-25/server/tools。
  • A2A Task、Message 与 Artifact 适合跨 Agent 传输和生命周期，但不能替代 Endem 身份、Synem 闭包或 Tekmor 验收。参见 https://a2a-protocol.org/v1.0.1/specification/。
  • 可靠 Agent 依赖清晰环境、可读工具、真实观察和反馈闭环；这些由 Praxor/Harness 负责，不通过增加薄命令包装解决。参见 https://openai.com/index/harness-engineering/。
  • 外部协议版本只约束适配器，不进入 Endem 编码、内部事件身份或 Acceptance Decision。

实现仓库基线

首个仓库保持单一工作区，按 spec/、vectors/、core/、theor/ 与 cli/ 分离。实现语言、构建系统、错误码和编码扩展由各自 ADR 决定；没有独立版本、权限隔离或真实消费者时不拆仓库。
