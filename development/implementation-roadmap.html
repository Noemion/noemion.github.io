开发原则

一个原语

先证明一个 Endem 能稳定表示一个根终态，不为架构对称提前增加格式。

一个入口

公开应用只有 endem；子命令按消费者组织，信任域用独立实现和进程隔离。

失败优先

先定义畸形、冲突、越权、超限和未决输入怎样关闭失败，再实现成功路径。

模型最后

模型只提交候选；无模型路径必须能够形成、检查、组合、装载和验收规则型案例。

下一条可执行路径

直接结论：第一实现只交付 poie、elenk、theor 与 peira。不同时创建工具家族，不接模型，不做通用二进制抽象层。

阶段、验证条件与停止条件

阶段只交付什么进入下一阶段的验证条件停止或合并条件
第 0 阶段
词汇与规范骨架		Endem、Synem、Dromen、Tekmor；六个语义面、一个根 skena、形式显示、三类未知、条款 ID 与正反向量关系角色、投影、满足状态与错误有可复查定义；每项规范要求都有测试或明确人工权威删除不能说明消费者或失败责任的名词
第 1 阶段
最小安全核心		poie/elenk/theor/peira、规范写入器、生产解析器和独立 TheorPoiet/Theor 差分一致、恶意语料关闭失败、资源有界、双环境逐字节复现若最小字段仍不能表达真实任务，回到规范，不叠加补丁格式
第 2 阶段
组合		pleko、引用解析、冲突矩阵、固定依赖与 Synem至少两个真实多 Endem 案例；链接顺序不改变规范结果；权限只能收窄没有真实复用消费者时不建设归档、动态链接或共享披露文件
第 3 阶段
发布		tasse/sphra、调试伴随记录、外部签名请求与响应裁剪等价、闭包完整、签名主体精确、可复现发行与 SBOM通用变换、裁剪或归档没有消费者时，只保留为内部步骤
第 4 阶段
受控运行		praxe、隔离 Praxor、Dromen、Tekmor 和规则型后端实际字节重验、能力最小化、事件完整、验收权威分离、恢复与升级路径若单一 Tekmor 无法覆盖不同权威，再按权限或生命周期拆分
第 5 阶段
模型与协议适配		模型候选封装、MCP/A2A 适配器和版本化遥测导出器相对无模型基线，在错误确定率、上下文成本或任务成功率上证明净增益没有不可替代增益时停止模型训练平台建设，继续使用外部成熟工具

第 0 阶段当前产物

工作包已经形成仍需完成
P0-W1 条款源END-CORE 0.1.0-draft、22 个唯一条款 ID、术语与成熟度登记。继续通过真实语料校正条款；物理格式另立规范。
P0-W2 威胁与限制10 类不可信输入威胁、强制缓解条款和 Profile 必需上限清单。用对象规模、复杂度和目标平台实验确定具体数值。
P0-W3 规范向量1 个接受向量、4 个直接拒绝向量、统一外壳和自动追溯检查。增加来源、未知状态、时间、资源边界和规范字节向量。
P0-W4 语言与构建只定义了比较维度，没有实验结果。用同一组字节向量比较候选语言、依赖、模糊测试和复现成本。

这些产物把设计变成了可引用的实施输入，但仍不证明存在可执行程序。只有 P0-W3 产生规范字节样本后，P0-W4 才能进行公平的跨语言读取实验。

从 GNU 工具链借什么

来源采用不照搬
ELF / 链接器结构视图与装载视图分离、显式引用、类型化绑定、闭包、绑定映射和失败即拒绝地址重定位、机器指令、弱符号默认选择、环境搜索和延迟绑定
GNU readelftheor 必须独立于生产解析库，使一个实现的错误不会污染两条证据链为每种查看方式创建一个稳定二进制
GNU BFD认识统一抽象的价值和信息损失风险第一阶段不建立多格式通用表示层；先把唯一格式做对
GNU 发行实践固定区域设置、时区、路径与构建纪元；两环境复现；调试信息分离并绑定摘要把构建工具本身当作复现证明

GNU 官方资料明确说明 https://www.sourceware.org/binutils/docs/binutils/readelf.html 独立于 BFD；https://sourceware.org/binutils/docs/bfd.html 同时说明跨格式 canonical 转换可能丢失信息。这里借鉴的是验证与职责思想，不是命令清单。

面向 Agent 时代的适配边界

  • MCP 工具说明、注解和远端返回都只是 Praxor 外缘的不可信输入；能力授权来自 Dromen 与会话策略。参见 https://modelcontextprotocol.io/specification/2025-11-25/server/tools。
  • A2A 1.0 的 Task、Message 与 Artifact 适合跨 Agent 传输和记录生命周期，但不能替代 Endem 身份、Synem 闭包或 Tekmor 验收。页面固定使用 v1.0.1 文档快照：https://a2a-protocol.org/v1.0.1/specification/。补丁号不进入协议协商。
  • 可靠 Agent 依赖清晰环境、可读工具、真实观察和反馈循环；这些由 Praxor 及其外侧控制平面负责，不通过增加薄命令包装解决。参见 https://openai.com/index/harness-engineering/。
  • OpenTelemetry 的生成式 AI 字段仍在演进，并可能携带输入、输出和工具数据。它只进入带版本、默认脱敏的导出器，不进入 Endem 编码、Tekmor 身份或最终决定。迁移说明：https://opentelemetry.io/docs/specs/semconv/registry/attributes/gen-ai/。
  • 所有外部协议版本只约束适配器，升级必须重新验证兼容性、安全边界和降级行为。

实现仓库基线

首个仓库保持单一工作区，按 spec/、vectors/、core/、theor/ 与 cli/ 分离。实现语言、构建系统、错误码和编码扩展由各自 ADR 决定；没有独立版本、权限隔离或真实消费者时不拆仓库。
