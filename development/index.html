项目/开发与贡献

开发与贡献

判断一项资料、术语、规范或验证修改是否在当前范围内，并在提交前留下能够支持主张的证据。

规范工作证据先行读音审查暂无组件代码

先判断贡献是否在当前范围内

当前可以改进研究、术语、架构、规范、威胁模型、验证方案和面向开发者的技术解释。当前公开范围不包括组件代码，不接受把未来接口、运行时或适配器写成实现任务。

准备修改什么当前需要留下什么何时停止
事实或技术解释具体主张、可复现问题、权威来源和受影响的开发者任务。修改会复制第二套规范、加入与项目结果无关的幕后描述，或只改变措辞却没有改善理解时。
术语、对象或发行名称普通工程词为何不足、独立职责、真实消费者、相邻词冲突；只有自造名称继续检查中文和英语中的首次朗读与听辨风险。普通词已经足够，专名没有独立生命周期，或自造名称持续难读、近音和易听错时。
规范义务或信任边界可被反例推翻的主张、适用对象、权威来源、支持案例、拒绝案例、威胁和失败责任。无法指出谁承担失败，或修改会把协议状态、签名、模型陈述、授权、满足和最终决定合并时。
场景、向量或验证设计对应条款、预期分类、稳定失败位置和测试通过后的声明上限。材料只增加数量，不能区分正确、拒绝、故障、未知或来源时。
组件、API、CLI 或运行原型当前不在公开工作范围；先用规范和验证方案说明不可替代需求、真实消费者与进入条件。组件实现尚未成为公开工作，或前一层证据仍不足时，不开始实现。

用一个边界修改检查贡献

假设准备把 A2A Task 的 completed 映射为 Noemion 的 met。这不是枚举改名，而是把外部协议状态提升为本地目标满足。

检查顺序本例应留下的结果
写出可证伪主张completed 只保存为外部任务终态，不直接产生 met。
确定权威顺序A2A 定义自己的 Task；ADP-CORE 保存外部状态；END-CORE 与精确 satisfaction_criteria 决定满足结果。
寻找反例远端任务已经完成，但 Artifact 指向错误版本，或本地安全检查拒绝发布。
同步唯一权威源先修改适用条款、登记、场景和向量，再更新解释页；研究提案不能直接成为规范。
限制公开声明最多说明已登记材料保持结果域分离；不能声称适配器已经实现或所有 A2A 对端均可互操作。

同一顺序也适用于名称、格式、权限、模型评测和开发者说明：先限定主张，再选择证据。测试数量不能代替失败条件。

按问题进入工作资料

项目现在进展到哪里查看已有成果、当前研究、近期可核对进展和仍然缺少的证据。 什么证据允许进入下一层查看术语、形成、组合、运行与外部协议研究的推进条件和停止条件。 一项修改需要哪些验证按变更对象选择支持案例、失败案例或未来实现级证据。

提交具体问题时，使用下方对应入口并说明对象、权威依据、失败条件、已验证范围和仍未验证的部分。

提交前留下最小证据

变更类型最低证据不能据此声称
面向开发者的解释现行权威链接、能够回答开发者问题的案例，以及与规范边界一致的结论。解释清楚不证明组件、隔离或互操作已经实现。
术语与命名职责和冲突分析、普通词替代方案、词首检查与关键字语料；自造名称再增加目标语言样本和真实人类记录。词源、搜索结果少、模型偏好或个人觉得顺口不能形成自造名称的发行结论。
规范、登记与向量条款、成熟度、威胁、场景、支持与拒绝向量同步，并运行与责任相匹配的检查。草案关系一致不等于格式、ABI、安全性或全部输入已经稳定。
外部技术与人工智能研究固定正式版本、原始来源、采用边界、反例、数据路径和本地无该机制时的基线。外部标准、开放协议或模型能力不能替 Noemion 生成语义、权限和实现证据。

GNU Manuals要求按读者问题组织资料、首次出现时定义专门术语，并使用主动语态明确责任主体。NIST SSDF 1.1提供可嵌入不同生命周期的高层安全实践；它不是 Noemion 已有实现的证明。NIST AI Agent Standards Initiative分别推进标准、协议、身份基础设施和安全评价，因此当前研究也继续分开这些证据。

怎样报告问题与敏感信息

  • 开发者资料问题可用

    通过资料问题表单报告事实、链接、术语或说明不清的问题。

  • 研究与规范问题可用

    通过研究与规范表单提交术语、对象边界、规范义务、威胁和验证方法。

  • 私密安全报告尚未提供

    不要在公开 Issue 中提交未公开漏洞、凭据、密钥、个人数据或可被滥用的样本；需要私密材料的问题应等待专用渠道。

当前可以证明什么

已有成果

当前仓库包含规范草案、ADR、登记、威胁模型、场景、向量、研究提案和面向开发者的公开资料，可检查它们在已覆盖关系上是否一致。

限制条件

producer、inspector、runner 与 endem CLI 都没有实现源码、可执行程序、安装包或发行入口。

名称与读音

普通职责词已经按词首、职责和关键字语料接受。Noemion 与 Endem 两个自造名称仍是待发行验证的设计标识。

进入下一阶段

组件实现进入公开路线并确定范围后，相关代码才成为可接受工作；规范和验证材料不能替代实现证据。
