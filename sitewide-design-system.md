# Noemion 全站设计系统

状态：已实施，随站点演进持续维护  
范围：全部正式页面、共享导航、目录、样式与交互

## 1. 设计目标

全站使用一致的页面结构、空间节奏与交互语言，同时保持不同内容类型的职责边界。桌面端以约 1200px 居中连续画布为基础，结合 64px 顶部导航、编辑式标题、细分隔线、高密度内容行和快速微交互；移动端按语义阅读顺序折叠为单列。

Noemion 的差异来自机器表征对象本身：薄荷、青蓝、朱橙和琥珀组成“语义频谱”，黑色记录面板表达可检查对象，等宽序号表达 Section、Symbol、Relocation 与阶段，珊瑚色脉冲表达尚未通过的设计门。状态文案必须区分已确认原则、设计提案、开放问题和未来能力。

## 2. 页面角色映射

| 页面角色 | 页面范式 | 布局契约 |
| --- | --- | --- |
| `portal` | 叙事型门户 | 左右首屏、论点双栏、横向入口行、三列焦点、长叙事画布 |
| `section` | 分组目录 | 大标题入口、分组章节、两列或横向内容条目、标题和箭头悬停反馈 |
| `content` | 显式组合的编辑式文章 | 全宽主题首屏；长篇内容使用约 800px 阅读列与约 380px 有职责的信息栏，版式类不解除正文行宽限制 |
| `tool-project` | 工具项目 | 从 Hero 起连续使用约 800px 六章主内容与约 380px 粘性工具信息栏 |
| `/docs/`、`docs-index`、`docs-topic` | 手册文档 | 320px 固定文档栏、约 880px 阅读区、紧凑标题和手册分页 |

页面职责决定内容结构；统一视觉不意味着把所有正文改造成同一模板。工具页继续保留六个项目章节，规范页继续保留规范性强度与失败语义，文档页继续提供连续阅读路径。

## 3. 全局导航

- 顶部栏以约 90% 的主题纸面和 12px 背景模糊保持粘性；滚动时允许下方页面色彩极浅地透入，底部只使用低对比分隔与轻微高光，不能切换为突兀的实色或重阴影状态。
- 桌面顶部导航包含项目、规范、工具、指南和开发五组。
- 悬停或键盘聚焦一级入口时，在导航下方展开卡片面板；每张卡片使用独立封面、标题、摘要和箭头，不显示装饰性标题序号。
- 移动端隐藏横向分组，显示品牌、阶段和目录按钮；目录使用同一份 `assets/directory.js` 数据，不维护第二套路由。
- 当前页面、悬停状态和键盘焦点必须清晰可辨，但不能通过大面积高饱和底色干扰正文。

## 4. 视觉元素

### 4.1 画布与网格

- 桌面主画布上限 1200px，整体居中，外侧背景与纸面通过轻微色差区分。
- 1200px 是画布而不是普通段落宽度。技术正文保持约 700–760px 阅读列；宽表、流程和对象结构才临时使用更宽空间。
- 长篇技术页和工具页可把画布稳定分成约 800px 主阅读列与约 380px 信息栏；信息栏从 Hero 起对齐并承载目录、状态、关键对象、输入输出或下一阅读入口，没有真实职责时回到单列。
- 内容区构成连续画布，并以 1px 低对比分隔线建立节奏。
- 关键介绍区可使用接近 1:1 或 5:4 的双栏；内容行使用“图形 / 文本 / 行动箭头”结构。
- 三列网格在中等宽度变为两列，移动端全部单列。

### 4.2 字体与层级

- 使用系统无衬线字体；标题使用 600–700 字重，正文使用 400–500 字重。
- 桌面长正文使用 17–18px 与约 1.75–1.8 行高；移动端使用 16–17px 与至少 1.7 行高。
- 桌面主标题约 48–64px，移动端约 40–48px；章节标题桌面约 30–42px。
- 标题使用 `text-wrap: balance`，正文使用 `text-wrap: pretty`，根元素保持字体抗锯齿。
- Section、版本、状态和对象字段使用等宽字体，不把等宽字体扩散到长正文。

### 4.3 色彩与表面

- 基础色为墨绿、纸白和低饱和灰绿；薄荷、青蓝、琥珀和朱橙组成语义频谱。
- 高饱和色只用于对象层次、阶段、状态、路径和关键交互，不用于大段正文。
- 卡片保持近方形或小圆角；圆形箭头是少量明确的行动符号。
- 深度表面使用透明多层阴影和细描边，不堆叠无语义的科技纹理。

### 4.4 对象语言

- 黑色对象记录面板表示 NIR、NOBJ 或 HOBJ 等可检查对象。
- 分层线、节点、记录行和偏移编号表达对象结构、符号、重定位和依赖闭包。
- 状态脉冲和证据状态只表达真实成熟度，不暗示未实现能力已经发布。
- 图形必须能对应正文中的真实技术概念，不能成为与内容无关的装饰。

## 5. 交互与动效

| 场景 | 行为 | 时长与曲线 |
| --- | --- | --- |
| 首屏进入 | 标题、说明、行动区和图形以透明度、4px 模糊和 12px 下移归位 | 360–420ms，组间约 90ms |
| 顶部菜单 | 悬停或聚焦改变底色并展开内容面板 | 180–200ms，`cubic-bezier(.2,0,0,1)` |
| 内容卡片 | 表面轻微提亮，标题变为朱橙并右移 4px | 150–180ms |
| 行动箭头 | 由 0.25 倍、透明和 4px 模糊切换到完整状态 | 180ms |
| 按压反馈 | 可点击按钮和入口缩放到 0.96 | 150ms，可中断 transition |
| 滚动进入 | 大型叙事分区执行一次 12px 与透明度进入 | 360ms |

禁止使用 `transition: all`、不可中断的交互关键帧、横向页面抖动、持续跟随鼠标的 3D 倾斜和无信息价值的循环动画。`prefers-reduced-motion: reduce` 下取消进入、滚动和图形循环，仅保留即时状态变化。

## 6. 文档页面

- 桌面端左侧文档栏固定为 320px，阅读区约 880px，总宽度与全站 1200px 画布对齐。
- 文档栏显示手册名称、分组、入口数量和当前页；当前入口使用珊瑚色左侧标记。
- 阅读区使用紧凑 Hero、30px 左右二级标题、手册分页、表格和代码块。
- 1217px 及以下隐藏固定文档栏，文档正文归中为单列；839px 及以下再由顶部移动目录接管导航。
- 手册正文以 Markdown 为权威源，布局、面包屑、分页和侧栏由 Jekyll 共享结构生成。

## 7. 响应式与可访问性

- 840px 及以上显示桌面顶部导航；只有 1218px 及以上的文档路由同时显示固定左栏。
- 1000px 以下正文双栏按语义顺序变为单列；839px 及以下隐藏桌面分组并显示移动目录。
- 560px 以下缩短首屏留白、减小标题和卡片高度；触控入口至少 44px。
- 任一宽度都不得产生页面级横向滚动；表格可在受限容器内横向滚动。
- 焦点样式、键盘导航、标题层级、跳到正文链接和深浅色对比必须持续可用。

## 8. 验收标准

1. 所有页面共享同一顶部品牌、阶段入口和目录数据；桌面一级入口可通过悬停与键盘聚焦展开。
2. `section`、`content`、`tool-project` 和文档路由呈现清楚不同但同源的页面范式。
3. 所有内容卡片、状态卡片和资源条目具有一致悬停或聚焦反馈，不存在 `transition: all`。
4. 桌面约 1512px 与移动约 390px 下无页面级横向溢出；文档左栏只在桌面显示，工具状态面板只在桌面保持粘性。
5. 深浅色和减少动态效果设置可读；视觉元素不改变 Noemion 的成熟度、权威性与证据边界。
6. 源码质量测试、构建产物质量测试、脚本语法检查和浏览器实测通过。
