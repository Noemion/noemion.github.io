# Noemion 全站设计系统

状态：Noemion 当前工程词汇
范围：正式页面、共享导航、目录、样式与交互

## 设计目标

全站以约 1200px 连续画布、清晰阅读列和低噪声交互呈现 Noemion 的工程边界。视觉语言围绕 Endem 命题投影、引用绑定、独立检查、能力控制和 Tekmor 证据，不再把传统对象文件面板作为品牌中心。

## 页面角色

| 角色 | 用途 | 布局 |
| --- | --- | --- |
| `portal` | 项目叙事与首条路径 | 双栏门户引言、论点行、三列焦点、生命周期 |
| `section` | 内容家族目录 | 大标题、分组摘要、清晰下游入口 |
| `content` | 架构、规范、开发专题 | 约 700–800px 阅读列，宽表局部扩展 |
| `tool-project` | Endem 应用专题 | 主阅读列与职责信息栏；名称保留为布局角色，不表示工具家族 |
| `docs` | 连续指南与手册 | 桌面 320px 文档栏 + 阅读区，窄屏单列 |

## 全局导航

- 品牌返回首页；一级入口按项目、规范、应用、指南和开发组织，Endem 作为具体应用名只出现在应用分组内部。
- 桌面面板与移动目录使用 `_data/navigation.yml` 同一数据源；手册目录由 Jekyll 直接生成。
- `TIMELINE` 入口从 `_data/site_header.yml` 读取并指向项目时间线，不在页头复制当前阶段。
- 旧制品与旧应用没有隐藏入口、重定向或兼容卡片。

## 视觉语汇

- 墨绿与纸白为基础；薄荷、青蓝、琥珀和朱橙分别用于来源、结构、验收和未决状态。
- 等宽短词用于 `rhem/semion/skena/telis/krin/apor`、子命令、版本、偏移和状态；长正文保持系统无衬线。
- Endem 使用五记录面板；Synem 使用交织引用；Dromen 使用封闭加载边界；Tekmor 使用事件与范围链。
- 视觉不能把 Dromen 画成文件，不能把 Tekmor 画成无条件证明，也不能把模型路径连到规范 Writer。

## 排版

- 桌面正文 17–18px、1.75–1.8 行高，阅读列约 700–760px。
- 主标题 48–64px，章节标题 30–42px；移动端分别收敛到约 40–48px 与 28–32px。
- 表格、代码和流程只在自己的容器横向滚动，页面本身不得溢出。
- 卡片使用小圆角与低对比分隔；阴影表达层级，不制造漂浮装饰。

## 交互

| 场景 | 行为 | 建议时长 |
| --- | --- | --- |
| 首屏进入 | 透明度、轻微下移和模糊归位 | 360–420ms |
| 菜单 | 可中断展开/收回 | 180–200ms |
| 链接卡片 | 表面/标题/箭头的明确属性变化 | 150–180ms |
| 按压 | 缩放至 0.96 后恢复 | 150ms |

禁止 `transition: all`、持续鼠标 3D、无意义循环和不可中断交互关键帧。减少动态效果时直接显示最终状态。

## 响应式与可访问性

- 840px 以上显示桌面导航；以下显示品牌、`TIMELINE` 和目录按钮。
- 1000px 以下双栏按语义顺序单列；固定文档栏只在足够宽时显示。
- 触控入口至少 44px；焦点样式、标题层级、跳到正文、深浅色对比和无 JavaScript 阅读持续可用。
- 图形提供有意义替代文本或明确标为装饰。

## 维护与验收

- `sitemap.md` 登记正式路由，目录数据与手册数据由测试校验。
- 页面、路由、名称或状态变化时同步 AGENTS、README、ADR、规范、测试和内容审计。
- 桌面与移动浏览器、源码与 `_site` 成品、深浅色与减少动态效果都必须验证。
- 视觉组件若无法对应真实 Endem 语义或读者任务，应删除而不是保留装饰。
