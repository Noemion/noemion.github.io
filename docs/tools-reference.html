Noemion 工具参考

按对象生命周期整理 23 个工具的职责、阶段和现有文档状态。

23 ToolsProject PagesUnreleased

工具链总览

已确认原则：每个工具入口都是项目说明页，不表示已有可执行程序。命令行接口、参数和文件扩展名尚未冻结。

工具按“规范与对象 → 文本 IR → 编译链接 → 发布运行 → 模型工程”逐步推进。每个工具必须说明上游输入、下游消费者、关键不变量、失败边界、阶段门和验证证据，不能只用一行命令描述代替设计。完整分类入口见工具目录。

规范与对象工具

noemconform规范、Golden、Malformed 和跨工具一致性套件。noemobj只读查看 Header、Section、符号、重定位和语义图。noemverify分层验证结构、语义、覆盖、策略和信任链。noemcopy受约束地复制、抽取、删除和重建对象内容。noemsize分析 Section、Segment、闭包和披露预算占用。

编译与链接

noemas将规范文本 GSIR 确定性汇编为对象。noemdis将对象反汇编为规范文本表示。noemfmt规范化 GSL 与文本 GSIR 的表现形式。noemdiff区分布局、调试、信任元数据和运行语义差异。noemc驱动前端并由确定性 Compiler Core 生成 GSIR/GOBJ。noemlint检查语义、覆盖和策略问题，不生成对象。noemar建立确定性对象归档和符号索引。noemnm检查符号定义、引用、版本和可见性。noemld解析符号、应用重定位、合并依赖闭包并输出链接对象或 SSO。

发布与运行

noemstrip分离开发调试信息并证明发布语义不变。noemcov审计 Source Unit 覆盖和发布闭包完整性。noempack组装签名对象、依赖、策略与模型指纹。noemrun可信验证、装载、渐进式披露与 Runtime 执行。noemtrace关联运行事件、调试伴随文件和源级因果链。

模型工程

未来阶段：以下工具只在确定性核心稳定后进入实施，模型输出始终视为不可信。

noemdata构造有许可、有血缘、无泄漏的数据集快照。noemtrain编排 NSFE 适配、多任务训练和教师蒸馏。noemeval评估错误确定、校准、OOD、切片和部署退化。noemquant导出、量化、端侧基准并封装模型包。

文档状态

设计提案：只有存在实质内容时才建立工具自己的 `docs/` 子树。目前 noemld 文档提供 12 个专题样板；其他工具先保持项目页，后续逐一 review 后决定拆分深度。

建立专题文档的最低条件是工具项目页已经稳定说明问题、职责和边界，并且至少存在需要独立维护的契约、流程、安全、测试或参考内容。专题页仍处于设计阶段时必须标明证据缺口，不得用文档数量暗示实现成熟。

开放问题：各工具 CLI、扩展名、稳定输入输出 schema、研究验证方法和发布顺序尚未冻结。
