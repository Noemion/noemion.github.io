## 如何使用

先建立共同背景，再进入权威规范或应用专题。推荐顺序是：确认问题 → 阅读 Noemion 入门 → 定位权威页 → 核对成熟度 → 追踪测试与失败证据。

指南解释概念和路径，不复制第二套技术定义。标为“待验证设计”“尚待确定”或“后续计划”的内容都不是稳定接口。

## 七个入口

- **入门指南：**Noemion 的项目边界、Endem 及其六个语义面和相关工程名词。
- **获取与使用：**当前可用性、计划流程和正式发布条件。
- **术语与读音验证：**怎样执行首次朗读、听写回填、职责匹配和发行名称门禁。
- **架构设计：**Ktisor、Theor、Drasor 与生命周期。
- **开发指南：**规范先行、仓库边界和验证纪律。
- **Endem 应用参考：**五个动作及其真实消费者。
- **规范参考：**权威顺序、成熟度、ADR 与开放问题。

研究、论文、专利、软著和标准提案引用 Noemion 时，必须记录路由、提交标识、成熟度和权威来源。愿景不能作为实验结论，候选编码不能作为冻结 ABI，规划流程不能作为已经发布的软件能力。
