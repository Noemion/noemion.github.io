## 如何使用这些指南与参考资料

先根据任务建立共同背景，再进入权威架构、规范或工具专题。任务型文档解释概念和阅读路径，不复制第二套技术定义。标为“待验证设计”“尚待确定”或“后续计划”的内容都不是稳定接口。

推荐顺序：确认问题 → 阅读背景 → 定位权威页 → 核对成熟度 → 追踪证据。

## 指南、参考资料与规范的边界

六类任务文档负责解释概念、提供阅读路径并说明当前可用性；架构关系由架构专题定义，规范性术语和约束仍以对应规范及经批准 ADR 为准。

## 成熟度与引用纪律

研究、论文、专利、软著和标准提案引用 Noemion 时，必须记录路由、版本或提交标识、成熟度和对应权威来源。项目愿景不能作为实验结论，候选类型名不能作为冻结 ABI，规划中的流程也不能作为已经发布的软件能力。

关键主张只有同时关联规范条款、ADR、测试、基准、数据来源和失败案例，才具备可审计的证据链；缺少证据的内容必须保持为待验证设计或开放问题。
