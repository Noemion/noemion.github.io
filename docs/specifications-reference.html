如何判断权威性

已确认原则：正式规范定义“必须是什么”；架构和项目页面解释关系；文档页提供阅读路径；FAQ 只回答常见问题。发生冲突时，以已批准规范和 ADR 为准。

快速判断：“已确认原则”表示项目已经采用的边界，“设计提案”表示仍可改变，“开放问题”表示没有唯一结论。只有带版本的规范条款和经批准 ADR 才能建立实现义务。

  • 规范性要求使用必须、不得、只有等明确约束，并能对应测试或明确说明不可自动判定。
  • 设计提案不能因为出现在示例、工具页、论文草稿或 FAQ 中而升级为规范。
  • 相同定义只保留一个权威页面，其他页面通过链接引用。
  • 实现行为、实验结果和专利文本都不能反向替代规范定义。

GSIR

GSIR 是生成式语义中间表示，表达目标、约束、偏好、歧义、推断权限、证据和验收条件。它不是二进制化 Prompt，也不是 Agent 关键词表。

开放问题：最小核心类型集、连续表示互操作、评价器接口和模型原生载荷边界尚未冻结。

阅读 GSIR 权威页面

GOBJ

GOBJ 是当前用于讨论可重定位生成式对象的工程名。对象包含安全可解析的 Header、Section、符号、类型化引用、完整性和装载视图。

设计提案：Section/Segment 双视图借鉴 ELF 工程策略，但不照搬传统机器指令语义。

开放问题：最终对象名称、ABI 字段、Section 编号和重定位标识尚未冻结。

阅读 GOBJ 权威页面

SSO

SSO 以内容寻址依赖闭包、Artifact Contract 和 Disclosure Graph 承载共享语义能力。运行时按请求、依赖和预算进行受控披露。

已确认原则：调试剥离不得改变运行语义哈希；权限扩大必须显式授权；硬约束冲突必须失败。

阅读 SSO 权威页面

开放问题与 ADR

开放问题集中保存在架构开放问题。格式、类型系统、执行语义、CLI、ABI、信任边界和兼容性选择一旦决定，应记录 ADR，说明背景、候选、决定、后果和替代方案。

面向标准化时，规范条款还需关联互操作 Profile、正反样例、一致性测试、安全分析和版本演进规则；面向研究和知识产权时，应关联假设、现有技术、实验结果、贡献与公开披露记录。

未来阶段：随着 Markdown 正式规范建立，HTML 将继续作为项目门户、解释层和阅读入口，而不承担唯一规范源。
