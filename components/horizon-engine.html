项目/组件/Horizon Engine

Horizon Engine

从长篇来源和锁定知识语料中提取带证据的候选语义，同时保留当前视域之外的未知内容。

Future Model StageUntrusted CandidatesSource GraphOOD

架构与边界

Horizon 表示对象周围尚未完全展开、但可能影响理解的背景和依赖。Horizon Engine 是后续模型阶段的设计，尚未实现；它拟使用预训练双向语言模型、层级来源图、知识检索路由、稀疏语义图和封闭任务头，输出 Candidate Envelope。

  • 允许输出有限概念 ID、文本跨度、关系、模态、作用域、置信信息、歧义，以及 NIL（知识库中无匹配）与 OOD（超出已知数据分布）候选。
  • 禁止自由文本生成器、聊天、网络、Shell、工具调用和文件写入；检索路由只访问运行前锁定、版本化、许可明确的离线索引或确定性前端提供的结果。
  • 每个 Candidate Envelope 携带来源身份与跨度、稳定来源单元 ID、候选类型、替代解释、证据、置信信息、未决状态和模型/窗口指纹。
  • Horizon Engine 不决定唯一语义，不直接生成 NIR、NOBJ 或 HOBJ，也不操纵二进制布局。

输入、输出与失败

类别对象边界
输入Source Package、Source Graph、Retrieval Corpus来源、许可、版本、切分与检索索引必须锁定；不隐式访问网络。
输入Model Package、Task Head Configuration模型、词表、本体库、任务头、量化与校准身份必须匹配。
输出Candidate Envelope保留来源、替代项、不确定性和失败状态；只供 Noesis Core 检查。
失败NIL、OOD、低置信、冲突、截断、资源超限、无效模型包必须形成稳定状态或诊断，不用默认候选填补未知。

存在必要性与停止条件

Horizon Engine 不是形成对象的必需组件：受控 NSL、Text NIR 或合格外部 Candidate Envelope 可以直接进入 Noesis Core。只有在长来源、跨章节关系、歧义发现和检索路由上，以来源隔离实验证明它相对规则与外部模型基线带来稳定收益，才值得承担训练、校准和模型供应链成本；不能证明时应省略或停止建设。

设计方案：层级语义编码

确定性结构前端
→ 来源图与稳定来源单元 ID
→ 局部双向语言编码器
→ 层级单元聚合器
→ 锁定知识检索路由
→ 稀疏语义图编码器
→ 封闭多任务头
→ Candidate Envelope
→ 确定性 Noesis Core

结构前端确定性识别文件、标题、段落、列表、代码、表格与引用；局部编码器处理受限窗口；层级聚合与稀疏图层传播跨块关系；Noesis Core 重新检查来源绑定、类型、歧义和编译来源覆盖。

设计方案：数据与评测

数据拟分为人工裁决的 Gold、经审计弱标注的 Silver、许可清晰的 Unlabeled（无标签语料）与 Adversarial（对抗反例）；切分按仓库、作者或文档族隔离。评测覆盖文本跨度、概念、关系、模态和否定范围、硬约束召回、OOD/NIL、错误确定率、校准、跨块召回、延迟与峰值内存。

任何候选数值门槛都必须在 Gold Pilot 建立统计依据后通过 ADR 固化；未经来源隔离评测和置信区间支持的百分比只能作为实验目标，不是规范要求。

待定事项：底座、训练与量化

MacBERT、DistilmBERT、ModernBERT 等只属于同条件选型对比候选；模型规模、教师模型、领域自适应预训练（DAPT）、任务自适应预训练（TAPT）、蒸馏范围、OOD 方法、量化格式和阈值必须在相同结构层、数据切片与目标硬件上比较。生成式大模型只可作为标注、External-generation 或能力上限对照。

后续计划：训练阶梯与模型包

未来按规则基线、局部语言基线、Noemion 结构层、领域继续预训练、蒸馏压缩、校准/OOD 与导出量化依次推进。模型包需要覆盖权重、词表、本体库、架构与任务头结构定义、量化配置、评测、校准和血缘；进入发布包时由 noembundle 纳入外部签名范围，并始终作为不可信输入验证。

待定事项：何时从头预训练

只有现有底座在合理微调和领域适配后仍系统性失败、消融定位到底座瓶颈、拥有许可清晰的大规模语料与长期盲测集、预算允许多次完整实验，且收益显著高于维护成本时，才进入立项评审。

研究假设、基线与验证要求

  • 层级聚合假设：相较仅做局部窗口编码，来源图与层级聚合能提高跨章节关系和硬约束召回；需用相同底座、数据和任务头做消融。
  • 封闭任务头假设：受限输出空间能降低错误确定与不可审计生成；需与生成式抽取和规则基线比较准确率、校准、OOD 与失败可解释性。
  • 检索路由假设：知识路由能在固定预算下降低无关上下文并保持必需证据；需测试遗漏、安全条款召回和延迟成本。
  • 模型底座假设：现有预训练双向底座足以支撑任务；只有系统性残差由底座导致时才考虑从头预训练。

验证报告必须说明数据许可与血缘、切分隔离、基线、消融、置信区间、失败切片、校准、OOD、量化退化、目标设备资源和模型指纹。单一综合分数不足以支持研究或产品结论。

数据、训练、评估与量化分别见noemdataset、noemtrain、noemevaluate和noemquantize；Horizon Object 是独立的共享对象形态，不是本组件的直接输出。
