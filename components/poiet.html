项目/系统组件/Poiet

Poiet

确定性地形成、验证、组合和冻结 Endem/Synem，是唯一能够写规范制品的组件。

Deterministic MakerSole WriterFail Closed

为什么 Poiet 不能省略

如果自然语言前端、模型、组合器和发布工具各自决定一套对象语义，Endem 将无法复现，也无法判断六个语义面是否在转换中保持。Poiet 集中执行投影、规范化、状态机和写入不变量，所有合格 Endem/Synem 都必须经过它。

Poiet 可以是一组库而不是独立服务，但模型、Praxor、Theor 和外部签名系统不能绕过它写入规范字节或提升 Endem 状态。

权威输入与生产者

输入生产者Poiet 检查
Rhem Source / 受控来源用户、规范作者或受控生成器编码、来源身份、投影决定、一个根 skena、telis、krin、apor 和版本。
Candidate + Semion Decision不可信模型/外部前端，以及有权确认的主体候选范围、替代项、权限、撤销与来源摘要。
Pleko Policy + Dependency Lock构建配置和依赖管理者内容身份、冲突、循环、能力收敛和可重放性。
Tasse Profile发布配置保留内容、调试外置、覆盖、预算和确定性元数据。
Signature Response外部签名系统原请求、候选载荷、算法、签名者、策略版本和签名有效性。

输出与消费者

输出实际消费者输出成立的前提
nascent Endem + Poie Tekmorelenk、theor、pleko、人工确认结构合法、来源可定位、semion/skena 关系位置一致、一个根 skena、telis 与 krin 完整；apor 项仍显式存在。
生产 Elenk Resultpleko、tasse、Praxor每个请求层明确 pass/fail/not-evaluated，并绑定实际字节与配置。
coherent Endem / Synem + Pleko Tekmortasse、theor必需引用与冲突已解析，权限未扩大。
Unsigned Candidate + Signing Request外部签名系统、sphra发布载荷冻结，闭包完整，裁剪保持规范属性。
attested Endem / Synem + Sphra Tekmortheor、PraxorSignature Response 精确绑定原候选和请求；被签字节未改变。

关键不变量

  • 相同规范化输入、配置、依赖和工具版本产生相同字节与稳定诊断。
  • 所有偏移、长度、计数、索引、对齐、解压和累计预算使用 checked arithmetic。
  • 未知关键记录、状态与内容不一致、悬空引用和权限扩大关闭失败。
  • 模型只能提交候选，不能决定字段、布局、绑定、签名范围或验收结果。
  • apor 项只能被授权绑定、保留为明确运行时未知或导致失败，不能静默删除。
  • sphra 只在被签载荷之外附加封装；私钥不进入 Poiet。

失败与拒绝责任

失败类别Poiet 必须返回不得做
Source invalid来源位置、编码或摘要诊断猜测缺失文本或调用模型修复。
Contract incomplete缺少的 rhem/semion/skena/telis/krin/apor 语义面和稳定错误码用默认提示补充来源、投影、可能事态、目标方向、满足判据或未决项。
Aseme违反 Profile、关系角色无效或没有允许投影的来源范围把畸形组合包装成合法 nascent Endem。
Pleko conflict参与成员、冲突字段、策略和 Pleko Tekmor按输入顺序、弱绑定或最后写入者静默选择。
Tasse changes semantics变化属性、被移除记录和拒绝结论只因文件更小就继续发布。
Sphra mismatch请求、载荷、算法、签名者或策略差异修改候选以适配签名响应。

当前成熟度与未冻结接口

Poiet 的唯一写入责任、确定性、状态机和失败边界已经采用，尚未发布实现。Rhem Source 语法、投影规则、规范记录、物理编码、生产 Verified Handle、Pleko Policy、Tasse Profile、Signing Request/Response 和错误码仍需规范与测试冻结。
