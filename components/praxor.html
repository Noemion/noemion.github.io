Praxor

在隔离权限域重新验证 attested Endem/Synem，建立 Dromen，受控调用后端和能力并形成 Tekmor。

Isolated DomainDromenLeast CapabilityTekmor

为什么 Praxor 不能并入模型

模型或其他求解后端能够提出候选和能力请求，但不能证明签名包安全、请求已授权、工具结果真实或任务已经完成。Praxor 把装载、会话、能力、观察、预算和决定策略放在后端外侧，并让每一步产生可定位 Tekmor。

Praxor 是 endem praxe 背后的隔离域。它可以使用 Poiet 的生产 elenk 接口，但不得链接 Poiet Writer 或修改 attested 制品；后端与能力适配器应进一步隔离。

权威输入与输出

方向内容边界
输入attested Endem/Synem 原始字节重新验证签名、状态、闭包和实际摘要，不复用过期结论。
输入Praxe Policy + krin固定能力上限、预算、停止、观察、证据、人工升级和决定权威；只能收紧制品要求。
输入Backend Binding + Capability Catalog模型/求解器与可用能力的身份、版本和参数结构；描述仍是不可信声明。
会话状态Dromen只读、一次性、不可序列化；不保存跨会话能力句柄。
输出Praxe Record候选、能力请求/观察、预算、错误、Run/Decision Tekmor 和最终状态。
消费者用户、CI、审计和指定外部权威模型或 Praxor 自述不能成为唯一决定依据。

Dromen 不是文件

Dromen 是 Praxor 从当前 attested 制品、策略、环境与授权能力计算出的会话视图。它只披露当前根 skena 所需的 semion 关系、skena 子图、telis、krin、apor、Synem 成员和能力参数结构。

  • Dromen 不进入 tasse、sphra、对象仓库或模型缓存。
  • Dromen 不包含凭据、Bearer token、文件描述符、网络连接或其他实时句柄。
  • 后端只能读取 Dromen 提供的只读值并返回类型化请求。
  • 环境、策略、对象或能力身份变化时，旧 Dromen 立即失效。
  • 会话结束后 Dromen 销毁；允许保留的信息进入有脱敏规则的 Tekmor。

运行循环

elenk attested bytes
build Dromen
backend candidate / request
policy elenk
execute or deny
phain → Tekmor
krin / continue / escalate
  1. Loader 验证实际输入并在失败时原子拒绝，不建立部分 Dromen。
  2. 控制平面按最小披露和能力上限创建 Dromen。
  3. 后端产生候选或 Capability Request；它不能直接访问环境。
  4. Praxor 校验参数、权限、预算、副作用、幂等和当前状态，再执行或拒绝。
  5. 真实结果先规范化为可与 skena 比较的 phain，再作为 Capability Observation 和 Praxe Tekmor 返回下一轮。
  6. 预注册策略读取 krin 和必需 Tekmor，先形成 met、unmet、agno 或 fault，再由具名权威形成 accepted、unsatisfied、pending-review、failed 或 interrupted。

外部协议适配

MCP、A2A 和供应商工具调用只连接 Praxor 外缘。远端工具说明、Agent Card、Task 状态、参数 schema 和返回内容都必须重新映射到本地 Capability Catalog、请求、观察和 Tekmor：

  • MCP 2025-11-25：https://modelcontextprotocol.io/specification/2025-11-25
  • A2A 1.0.1：https://a2a-protocol.org/v1.0.1/specification/
  • OpenAI Harness Engineering：https://openai.com/index/harness-engineering/

这些资料支持工具可读性、反馈循环和协议互操作，但不定义 Endem、Dromen、Tekmor、权限或最终验收。第一阶段不接入任何外部智能体协议或模型 SDK。

失败与拒绝责任

失败Praxor 必须记录后续状态
Load rejected对象身份、失败层、规则和位置failed；不创建 Dromen。
Capability denied请求摘要、策略版本、拒绝规则和当前状态继续、unsatisfied 或 pending-review，按预注册策略。
Backend failed后端身份、稳定错误类别、预算和安全诊断重试、failed 或 interrupted；不得提权。
Budget exhausted预算单位、限值、已用量和停止位置unsatisfied 或 interrupted。
State drift期望与实际对象、环境、依赖或能力身份旧 Dromen 失效；拒绝或重新建立会话。
Tekmor insufficient缺失的 krin 条件、证据强度和决定权威agno、pending-review 或 unsatisfied，不得 accepted。

当前成熟度与未冻结接口

隔离权限域、Dromen 非文件、后端不可信、能力句柄隔离和 Tekmor 不等于验收已经采用。Praxe Policy、Dromen API、Capability Request/Observation、Praxe Record、沙箱、预算单位、事件结构和 Decision Tekmor 尚未冻结。
