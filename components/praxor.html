Praxor

在隔离权限域重新检查待运行制品，只授予本次实现需要的能力，并记录有范围的证据。

Isolated DomainDromenLeast CapabilityTekmor

为什么 Praxor 不能并入模型

模型可以提出候选和能力请求，却不能证明请求已获授权、工具结果真实或任务已经完成。Praxor 因此把装载、能力、观察、预算和决定策略留在求解后端之外。

Praxor 是 endem praxe 背后的隔离域。它可以调用 Poiet 的生产验证接口，但不得链接 Poiet 写入器，也不得修改 attested 制品。求解后端与能力适配器还应进一步隔离。

权威输入与输出

方向内容边界
输入attested Endem/Synem 原始字节重新验证签名、状态、闭包和实际摘要，不复用过期结论。
输入运行策略与 krin固定能力上限、预算、停止、观察、证据、人工升级和决定权威；只能收紧制品要求。
输入后端绑定与能力目录模型或求解器与可用能力的身份、版本和参数结构；描述仍是不可信声明。
会话状态Dromen只读、一次性、不可序列化；不保存跨会话能力句柄。
输出运行记录候选、能力请求与观察、预算、错误、运行与决定类 Tekmor，以及彼此分离的满足、决定和会话结果。
消费者用户、CI、审计和指定外部权威模型或 Praxor 自述不能成为唯一决定依据。

Dromen 不是文件

Dromen 是 Praxor 从当前 attested 制品、策略、环境与授权能力计算出的会话视图。它只披露当前根 skena 所需的 semion 关系、skena 子图、telis、krin、apor、Synem 成员和能力参数结构。

  • Dromen 不进入 tasse、sphra、对象仓库或模型缓存。
  • Dromen 不包含凭据、Bearer token、文件描述符、网络连接或其他实时句柄。
  • 后端只能读取 Dromen 提供的只读值并返回类型化请求。
  • 环境、策略、对象或能力身份变化时，旧 Dromen 立即失效。
  • 会话结束后 Dromen 销毁；允许保留的信息进入有脱敏规则的 Tekmor。

  1. Loader 验证实际输入并在失败时原子拒绝，不建立部分 Dromen。
  2. 控制平面按最小披露和能力上限创建 Dromen。
  3. 后端产生候选或类型化能力请求；它不能直接访问环境。
  4. Praxor 校验参数、权限、预算、副作用、幂等和当前状态，再执行或拒绝。
  5. 真实结果先规范化为可与 skena 比较的 phain，再作为能力观察和 Praxe Tekmor 返回下一轮。
  6. 预注册策略读取 krin 和必需 Tekmor，先形成 met / unmet / agno / fault。具名权威再形成 accepted / rejected / deferred；会话终止另记为 completed / failed / interrupted。

mene 的时钟与覆盖责任

  • Praxor 对 fixed 读取已经授权的 UTC 区间，不用当前机器时区重新解释来源表达。
  • Praxor 对 elapsed 绑定具名起始事件，并使用声明的单调时钟测量经过时长；墙钟回拨不能缩短目标区间。
  • 观察适配器把覆盖区间、已知空洞、时钟分辨率、不确定度、同步和重启事件写入 Tekmor。
  • 离散采样只能支持采样时刻；覆盖空洞产生 agno，时钟契约故障产生 fault，不能默认为 met。
  • A2A Task、MCP 工具和远端遥测的时间戳只作为带来源事件，不能取得本地时钟权威。

外部协议适配

MCP、A2A、遥测系统和供应商工具只连接 Praxor 外缘。所有远端描述和返回内容都要先经过本地版本、权限、结构和来源检查。

外部接口进入 Praxor 的方式本地边界
MCP 2025-11-25 稳定规范工具说明、参数和结果映射为本地能力请求与观察。声明不授予权限；令牌必须绑定目标资源，不能转交给下游服务。后续版本只有完成兼容、安全与降级复核后才进入新适配基线。
A2A 1.0 版本化规范Task、Message 与 Artifact 映射为外部任务记录和候选结果。外部状态不改变 Endem；Artifact 不是 Tekmor，也不能形成最终决定。
OpenTelemetry GenAI 语义约定由可替换、带版本的导出器发送允许披露的运行观测。当前状态为 Development；输入、输出、工具参数和结果可能含敏感信息，默认不导出正文，也不把外部字段作为本地身份。

这些资料支持互操作和可观测性，却不定义 Endem、Dromen、Tekmor、权限或验收。第一阶段不接入外部智能体协议、遥测约定或模型 SDK。

负观察与封闭范围责任

  • Praxor 必须把显式负观察、查询未命中和观察器故障记录为三种不同事实。
  • 空日志、工具“未发现”或模型结论默认产生 agno，不能自动生成负极性 phain。
  • 从缺席推断前，控制平面必须核对具名权威、全集、全部角色路径、时间截止、完整枚举、迟到窗口和损失边界。
  • 同构正极性反例产生 unmet；观察器未按契约完成产生 fault。

这些是 ADR-0017 的未来运行责任。当前没有 Praxor、日志收集器、完整性证明器或求值器。

量化聚合责任

  • 装载时重新核对集合身份、成员权威、截止点、身份规则和唯一量化角色，不能由运行查询替换规范范围。
  • 把每个不同成员的观察分别形成结果与 Tekmor 链接，再按 ADR-0018 的决定性规则聚合。
  • 重复事件、重试和多份证据不得增加成员数；范围不完整为 agno，成员求值契约故障为 fault。
  • 当前没有成员目录、聚合器或量化求值器；这些只是未来 Praxor 的责任边界。

失败与拒绝责任

失败Praxor 必须记录后续状态
Load rejected对象身份、失败层、规则和位置会话 failed；不创建 Dromen，也不伪造满足结果。
Capability denied请求摘要、策略版本、拒绝规则和当前状态按预注册策略继续或 interrupted；若必需观察无法形成则为 agno，决定保持 deferred 或由具名政策拒绝。
Backend failed后端身份、稳定错误类别、预算和安全诊断会话 failed 或 interrupted；求值无法完成时为 fault，不得推成 unmet。
Budget exhausted预算单位、限值、已用量和停止位置会话 interrupted；满足结果按已有观察为 agno 或 fault。
State drift期望与实际对象、环境、依赖或能力身份旧 Dromen 失效，会话中断或失败；不从漂移推断目标为假。
Tekmor insufficient缺失的 krin 条件、证据强度和决定权威满足结果为 agno，决定为 deferred 或由具名关闭失败政策 rejected；不得 accepted。

当前成熟度与未冻结接口
