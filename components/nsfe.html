项目/组件/NSFE

NSFE

Neural Semantic Frontend 的封闭候选接口、Source Graph 架构与模型工程状态。

Untrusted CandidatesSource GraphOOD

已确认原则：架构与边界

NSFE 采用预训练双向语言底座，加自研层级 Source Graph 聚合、知识检索路由、稀疏语义图和封闭任务头；只有独立实验确认现有底座无法满足需求时，才考虑从头预训练语言底座。

  • 允许输出有限概念 ID、Span、Relation、Modality、Scope、置信度、Ambiguity 与 NIL/OOD 候选。
  • 禁止自由文本 Decoder、聊天、网络、Shell、工具调用和文件写入。
  • 每个候选携带来源跨度、稳定 Source Unit ID、候选类型、证据和模型/窗口指纹。
  • NSFE 不决定唯一语义，不直接生成 GSIR/GOBJ，也不操纵二进制布局。

设计提案：层级语义编码

Deterministic Structure Frontend
→ Source Graph + Stable Source Unit IDs
→ Local Bidirectional Language Encoder
→ Hierarchical Unit Aggregator
→ Knowledge Retrieval Router
→ Sparse Semantic Graph Encoder
→ Closed Multi-task Heads
→ Typed Candidates + Evidence + Uncertainty
→ Deterministic Compiler Core

结构前端确定性识别文件、标题、段落、列表、代码、表格与引用；局部 Encoder 处理受限窗口；层级聚合与稀疏图层传播跨块关系；Compiler Core 维护全局符号、覆盖和依赖状态。

设计提案：数据与评测

数据拟分为人工裁决 Gold、经审计 Silver、许可清晰 Unlabeled 与 Adversarial；切分按仓库、作者或文档族隔离。评测覆盖 Span/Concept/Relation、模态和否定范围、硬约束召回、OOD/NIL、错误确定率、校准、跨块召回、延迟与峰值内存。

任何候选数值门槛都必须在 Gold Pilot 建立统计依据后通过 ADR 固化；未经来源隔离评测和置信区间支持的百分比只能作为实验目标，不是规范要求。

开放问题：底座、训练与量化

MacBERT、DistilmBERT、ModernBERT 等只属于 bake-off 候选；模型规模、教师模型、DAPT/TAPT、蒸馏范围、OOD 方法、量化格式和阈值必须在相同结构层、数据切片与目标硬件上比较。生成式大模型只可作为标注、Generate-Strict 或能力上限对照。

未来阶段：训练阶梯与模型包

未来按规则基线、局部语言基线、Noemion 结构层、领域继续预训练、蒸馏压缩、校准/OOD 与导出量化依次推进。模型包需要覆盖权重、词表、ontology、架构与任务头 schema、量化配置、评测、校准、血缘和签名，并作为不可信输入验证。

开放问题：何时从头预训练

只有现有底座在合理微调和领域适配后仍系统性失败、消融定位到底座瓶颈、拥有许可清晰的大规模语料与长期盲测集、预算允许多次完整实验，且收益显著高于维护成本时，才进入立项评审。

研究假设、基线与验证要求

  • 层级聚合假设：相较仅做局部窗口编码，Source Graph 与层级聚合能提高跨章节关系和硬约束召回；需用相同底座、数据和任务头做消融。
  • 封闭任务头假设：受限输出空间能降低错误确定与不可审计生成；需与生成式抽取和规则基线比较准确率、校准、OOD 与失败可解释性。
  • 检索路由假设：知识路由能在固定预算下降低无关上下文并保持必需证据；需测试遗漏、安全条款召回和延迟成本。
  • 模型底座假设：现有预训练双向底座足以支撑任务；只有系统性残差由底座导致时才考虑从头预训练。

验证报告必须说明数据许可与血缘、切分隔离、基线、消融、置信区间、失败切片、校准、OOD、量化退化、目标设备资源和模型指纹。单一综合分数不足以支持研究或产品结论。
