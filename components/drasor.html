项目/系统组件/Drasor

Drasor

在隔离权限域重新检查待运行制品，只授予本次实现需要的能力，并记录有范围的证据。

Isolated DomainDromenLeast CapabilityIknem

为什么 Drasor 不能并入模型

模型可以提出候选和能力请求，却不能证明请求已获授权、工具结果真实或任务已经完成。Drasor 因此把装载、能力、观察、预算和决定策略留在求解后端之外。

Drasor 是 endem drase 背后的隔离域。它可以调用 Ktisor 的制品形成侧检查接口，但不得链接 Ktisor 写入器，也不得修改精确制品或外部签名陈述。求解后端与能力适配器还应进一步隔离。

权威输入与输出

方向内容边界
输入Endem 精确字节或 Synem 精确成员闭包，以及适用外部陈述重新验证内容身份、陈述主体、签名者、验证政策、截止点、撤销、闭包绑定和实际摘要，不复用过期结论；Synem 物理格式尚未定义。
输入运行策略与 krin固定能力上限、预算、停止、观察、证据、人工升级和决定权威；只能收紧制品要求。
输入后端绑定与能力目录模型或求解器与可用能力的身份、版本和参数结构；描述仍是不可信声明。
会话契约Dromen按 DRO-CORE 只读、一次性、不可序列化；不保存跨会话能力句柄。
输出运行记录符合 IKN-CORE 抽象边界的候选、能力请求、观察、预算、错误、运行与决定记录，以及彼此分离的满足、决定和会话结果；物理 Iknem 格式尚未定义。
消费者用户、CI、审计和指定外部权威模型或 Drasor 自述不能成为唯一决定依据。

Iknem 记录与评估边界

Drasor 可以生产有范围的记录，但不能给自己的记录填写“真实”“有效”“覆盖充分”或“最终接受”。

  • 记录种类分为 observation、derivation、attestation、appraisal 与 decision-record；来源类别另分为直接观察、确定性派生、外部断言、人工判断与模型候选。
  • 原始返回、规范化变换、phain 与主张分别保存；模型解释始终保持 model-candidate，不能升级为直接观察。
  • 签名和摘要只支持完整性与来源认证。验证者依据精确政策、参考值、环境和截止点形成 valid、invalid 或 revoked。
  • 证据集合只相对当前 krin 与观察责任形成 sufficient 或 insufficient；重复日志、更多签名或模型评分不能补齐缺失范围。
  • decision-record 只记录具名权威已经作出的决定，不会让 Drasor 获得决定权限。完整边界见 ADR-0022。

Dromen 是一次会话的只读契约

Dromen 是 Drasor 按 ADR-0024 从精确制品、适用外部陈述、当前验证结果、政策、环境、能力、预算和证据责任求交后封存的会话契约。现行 DRO-CORE 用 attested 概括其中一部分准入条件，但实现设计不能把多项外部关系压成内容自身的布尔状态。Dromen 只披露本次实现需要的非秘密语义与能力描述，不保存运行观察和最终结果。

  • Dromen 不进入派生制品、外部签名包络、对象仓库或模型缓存。
  • Dromen 不包含凭据、Bearer token、文件描述符、网络连接或其他实时句柄。
  • 后端只能读取 Dromen 提供的只读值并返回类型化请求；step-up 授权必须建立新会话。
  • 环境、政策、对象、权威、能力、预算或证据责任发生实质变化时，旧 Dromen 失效而不是原地更新。
  • 会话结束后 Dromen 与绑定实时能力都不可再用；允许保留的信息进入有范围和脱敏规则的 Iknem 或会话事件，但不能恢复旧权限。

复核外部陈述与当前政策
建立 Dromen

策略检查

  1. Loader 验证实际输入并在失败时原子拒绝，不建立部分 Dromen。
  2. 控制平面按最小披露和能力上限创建 Dromen。
  3. 后端产生候选或类型化能力请求；它不能直接访问环境。
  4. Drasor 校验参数、权限、预算、副作用、幂等和当前状态，再执行或拒绝。
  5. 真实结果先规范化为可与 skena 比较的 phain，再作为能力观察和 Drase Iknem 返回下一轮。
  6. 预注册策略读取 krin 和必需 Iknem，先形成 met / unmet / agno / fault。具名权威再形成 accepted / rejected / deferred；会话终止另记为 completed / failed / interrupted。

mene 的时钟与覆盖责任

  • Drasor 对 fixed 读取已经授权的 UTC 区间，不用当前机器时区重新解释来源表达。
  • Drasor 对 elapsed 绑定具名起始事件，并使用声明的单调时钟测量经过时长；墙钟回拨不能缩短目标区间。
  • 观察适配器把覆盖区间、已知空洞、时钟分辨率、不确定度、同步和重启事件写入 Iknem。
  • 离散采样只能支持采样时刻；覆盖空洞产生 agno，时钟契约故障产生 fault，不能默认为 met。
  • A2A Task、MCP 工具和远端遥测的时间戳只作为带来源事件，不能取得本地时钟权威。

外部协议适配

MCP、A2A、遥测系统和供应商工具只连接 Drasor 外缘。所有远端描述和返回内容都要先经过本地版本、权限、结构和来源检查，并遵守 ADP-CORE 外部协议适配边界。

外部接口进入 Drasor 的方式本地边界
MCP 2025-11-25 稳定规范工具说明、参数和结果映射为本地能力请求与观察。声明不授予权限；令牌必须绑定目标资源，不能转交给下游服务。后续版本只有完成兼容、安全与降级复核后才进入新适配基线。
A2A 1.0 版本化规范Task、Message 与 Artifact 映射为外部任务记录和候选结果。外部状态不改变 Endem；Artifact 不是 Iknem，也不能形成最终决定。
OpenTelemetry GenAI 语义约定独立仓库由可替换、带版本的导出器发送允许披露的运行观测。独立仓库仍在演进且 Schema URL 待定；输入、输出、工具参数和结果可能含敏感信息，默认不导出正文，也不把外部字段作为本地身份。

这些资料支持互操作和可观测性，却不定义 Endem、Dromen、Iknem、权限或验收。第一阶段不接入外部智能体协议、遥测约定或模型 SDK。

负观察与封闭范围责任

  • Drasor 必须把显式负观察、查询未命中和观察器故障记录为三种不同事实。
  • 空日志、工具“未发现”或模型结论默认产生 agno，不能自动生成负极性 phain。
  • 从缺席推断前，控制平面必须核对具名权威、全集、全部角色路径、时间截止、完整枚举、迟到窗口和损失边界。
  • 同构正极性反例产生 unmet；观察器未按契约完成产生 fault。

这些是 ADR-0017 的未来运行责任。当前没有 Drasor、日志收集器、完整性证明器或求值器。

量化聚合责任

  • 装载时重新核对集合身份、成员权威、截止点、身份规则和唯一量化角色，不能由运行查询替换规范范围。
  • 把每个不同成员的观察分别形成结果与 Iknem 链接，再按 ADR-0018 的决定性规则聚合。
  • 重复事件、重试和多份证据不得增加成员数；范围不完整为 agno，成员求值契约故障为 fault。
  • 当前没有成员目录、聚合器或量化求值器；这些只是未来 Drasor 的责任边界。

测量判断责任

  • 装载时重新核对构念、总体、方法与版本、窗口、单位、换算、纳入规则、最小样本、聚合器、阈值和不确定度政策。
  • 保留原始观察到规范单位、聚合结果和阈值分类的完整 Iknem 链；不得用显示舍入值、排名或模型置信度替换判断依据。
  • 固定基准只能支持固定项目；推广结论必须使用预注册统计模型。区间跨越阈值为 agno，测量契约故障为 fault。
  • 当前没有采集器、统计引擎或测量求值器；这些只是 ADR-0019 规定的未来 Drasor 责任边界。

复合判断责任

  • 装载时核对复合结构有限无环、只使用 all_of / any_of，并确认每个事态叶与一个判据叶对齐。
  • 先形成叶级 met / unmet / agno / fault，再按 ADR-0020 的固定矩阵组合；不得直接组合原始值或运行自由表达式。
  • 只有决定性反例或见证才允许短路，并保存决定依据、已求值叶、未求值叶、全部 Iknem 与停止原因。
  • 当前没有组合求值器；这些只是 ADR-0020 规定的未来 Drasor 责任边界。

Synem 激活责任

  • 只对固定、已重新验证的 Synem 闭包求值激活守卫，不在会话中发现或下载新成员。
  • 守卫绑定精确成员结果事件、结果域、期望值、截止点和授权策略；裸状态字符串不能决定激活。
  • 只记录 active / inactive / unresolved / error；未激活成员没有满足结果，激活也不创建能力句柄。
  • 依据变化、撤销或过期时重新检查或使会话失效，并保留追加事件。完整边界见 ADR-0021。

失败与拒绝责任

失败Drasor 必须记录后续状态
Load rejected对象身份、失败层、规则和位置会话 failed；不创建 Dromen，也不伪造满足结果。
Capability denied请求摘要、策略版本、拒绝规则和当前状态按预注册策略继续或 interrupted；若必需观察无法形成则为 agno，决定保持 deferred 或由具名政策拒绝。
Backend failed后端身份、稳定错误类别、预算和安全诊断会话 failed 或 interrupted；求值无法完成时为 fault，不得推成 unmet。
Budget exhausted预算单位、限值、已用量和停止位置会话 interrupted；满足结果按已有观察为 agno 或 fault。
State drift期望与实际对象、环境、依赖或能力身份旧 Dromen 失效，会话中断或失败；不从漂移推断目标为假。
Iknem insufficient缺失的 krin 条件、证据强度和决定权威满足结果为 agno，决定为 deferred 或由具名关闭失败政策 rejected；不得 accepted。

当前状态与待确定接口

未来 Drasor 必须按 DIA-CORE 保留协议错误与工具执行错误的来源差异。恢复分类只能帮助选择下一步，不能自动提权、扩大 Dromen、无限重试或把后端失败改写成目标不满足。

当前策略要求隔离权限域、把后端视为不可信、隔离能力句柄，并分开会话契约、证据、结果、时间、否定、量化、测量与复合判断。模型、控制面、凭据、适配进程、文件、网络、资源终止和观察怎样部署与验证，仍在非规范隔离研究提案中研究；容器、seccomp 或超时名称不构成实现证据。Drasor API、能力请求与观察事件、运行记录、沙箱、时间与预算计数器、封闭、测量与组合声明字段、事件编码和 Iknem 物理格式都是待定内容。
