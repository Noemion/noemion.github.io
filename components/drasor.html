项目/系统组件/Drasor

受限运行边界

Drasor 未来负责把一个已检查的目标放进一次性会话，限制模型能请求的能力，并把实际观察与最终决定分开记录。

受限运行边界一次性会话最小能力尚未实现

用一次依赖更新理解 Drasor

假设目标是“更新服务依赖，并确认当前版本可以发布”。模型可以建议改动、请求读取清单或运行测试，但它既没有默认文件权限，也不能把“命令退出成功”直接写成“目标已经满足”。

精确 Endem / Synem 与外部陈述
政策、环境、能力与预算求交
封存一次 Dromen
模型提出补丁或类型化能力请求
控制面授权并调用适配器
实际观察形成 Iknem
krin 判断满足情况
具名权威另行决定

测试服务报告完成，只能证明一次外部任务到达其终态；本地仍要核对锁文件、目标平台、安全政策和判据要求。Drasor 记录这条责任链，却不能给自己的记录填写“真实”“覆盖充分”或“最终接受”。

名称仍在研究：Drasor 是设计阶段名称，读音流畅度、口头区分度与发布拼写尚未经过使用者验证。讨论接口时应先说“受限运行边界”，再用 Drasor 指向这里的定义。

为什么运行边界不能并入模型

模型输出是候选，不是权限、观察或决定。若模型进程同时持有凭据、执行工具、解释结果并宣布完成，一次提示注入、工具描述污染或后端故障就可能跨过所有防线。Drasor 因此把装载、授权、能力句柄、预算、观察和停止策略留在求解后端之外。

参与者可以做什么不能自行决定什么
模型或求解器提出候选改动、解释和类型化能力请求实际权限、工具结果真实性、目标满足或最终接受
Drasor 控制面检查请求、政策、预算与当前状态，执行或拒绝已登记能力改写 Endem、扩大授权来源或代替具名决定权威
能力适配器在单一能力边界内访问文件、网络、进程或外部协议把远端描述当成本地授权，把返回文本当成已验证事实
判断与决定责任依据 krin 和有范围的 Iknem 形成满足结果；由具名权威形成决定由模型评分、会话成功或证据数量替代政策

运行前必须封闭什么

Drasor 不是把制品和凭据一起塞进提示词。它先重新验证实际输入，再把本次会话允许看见和使用的部分求交为只读 Dromen。

封闭内容来源变化后的处理
精确目标与依赖闭包Endem 精确字节；如适用，则为已固定且重新检查的 Synem 成员闭包身份或闭包变化使旧会话失效，不能在会话中偷偷下载新成员
适用外部陈述绑定精确主体的签名、来源、验证政策、截止点、撤销和验证结果到期、撤销或政策变化后重新验证，不复用旧结论
能力与环境具名政策、能力目录、目标资源、后端和适配器身份新增权限或 step-up 授权建立新会话，不原地扩大 Dromen
预算与停止条件时间、调用数、输出量、重试、副作用和终止策略预算耗尽按预注册策略中断，不能让模型无限重试
观察与决定责任krin、所需 Iknem 范围和具名决定权威判据、观察范围或决定者变化时重新建立会话

Dromen 只保存本次运行需要的非秘密语义与能力描述，不包含 Bearer token、文件描述符、网络连接或其他实时句柄，也不进入派生制品、签名包络、对象仓库或模型缓存。现行 DRO-CORE 仍使用 attested 概括一部分准入条件，但实现不能把多项外部关系压成内容自身的布尔状态。完整抽象契约见 DRO-CORE 与 ADR-0024。

一次能力请求怎样穿过边界

  1. 装载器检查实际对象、外部陈述、当前政策和环境；任一关键条件失败时原子拒绝，不留下部分 Dromen。
  2. 控制面只向后端披露完成当前步骤所需的语义和能力说明。后端返回候选或结构化请求，不能直接取得实时句柄。
  3. 控制面检查参数、调用者授权、目标资源、预算、副作用、幂等与会话状态，再把请求交给单一职责适配器或拒绝。
  4. 适配器保留原始返回和来源；确定性变换把它转换为可与 skena 比较的 phain，模型解释仍标为 model-candidate。
  5. 有范围的 observation、derivation、attestation、appraisal 与 decision-record 进入 Iknem。签名只证明来源和完整性，不能自动证明覆盖充分。
  6. 预注册策略读取 krin 和必需 Iknem，形成 met / unmet / agno / fault。具名权威再形成 accepted / rejected / deferred；会话终止另记为 completed / failed / interrupted。

时间覆盖、负观察、量化集合、测量不确定度和复合判断的细则属于 Endem 规范及其 ADR；Drasor 只承担未来执行边界，不在组件说明中复制另一套判断规则。Iknem 的记录、来源与评估分层见 IKN-CORE。

外部 Agent 协议只提供带来源的事实

当前 Agent 生态正把互操作、身份、授权和可观测性作为独立标准化问题处理。这支持 Drasor 把协议放在受控适配边缘，而不是让协议对象取得本地语义或决定权威。

外部接口可以带入什么仍由本地负责什么
MCP 2025-11-25 当前修订工具说明、资源、参数与返回内容能力登记、调用者授权和目标资源绑定；令牌透传被禁止。2026-07-28 候选版本在正式发布并完成兼容与安全复核前不改变当前适配基线。
A2A 1.0Task、Message、Artifact 与远端终态每次请求的授权、本地判据和最终决定；Task completed 不等于本地 met，取消请求也不保证任务已取消。
OpenTelemetry 语义约定 1.43.0GenAI 与 MCP 的可观测字段和稳定性标记敏感正文披露、字段到 Iknem 的映射和导出政策；多项 GenAI 属性仍处于 Development，因此当前不固定导出 Profile。

外部描述首先是不可信声明；外部返回首先是带来源记录。适配边界还须遵守 ADP-CORE 与 AUT-CORE，不得把远端身份、任务状态或 token 直接升级为本地权限。

隔离必须证明什么

“使用了容器、超时或 seccomp”只是机制名称。未来实现必须用威胁模型和测试证明能力不能越界、资源能够回收、失败不会伪装成目标结果。

边界最低工程责任需要的验证证据
文件与网络默认不可见，按目标资源和操作分别授权越权路径、符号链接、重定向、DNS 与目标替换的拒绝测试
凭据与实时句柄不进入提示、Dromen、日志或可序列化事件；适配器按调用使用泄漏扫描、目标绑定、撤销和会话结束后不可复用
进程与资源限制子进程、CPU、内存、输出、调用次数和嵌套并发耗尽、派生进程、阻塞 I/O 和终止后的回收测试
副作用与重试每项能力声明副作用、幂等键和重试政策重复提交、部分成功、超时后状态未知与补偿路径测试
观察与披露原始返回、确定性变换和模型解释分层；默认不导出敏感正文来源保留、脱敏、缺失覆盖和观察器故障的反例

GNU timeout 说明信号终止和前台子进程仍有边界条件；GNU Make 的 job slots 展示嵌套工作共享有限并发额度的做法；GNU Guix 的 guix shell --container 展示先隔离、再显式开放环境资源的纪律。这些先例能指导预算和能力设计，但都不能单独证明 Drasor 已实现安全隔离。

代表性失败怎样分类

失败必须保留的事实不能得出的结论
装载拒绝实际对象身份、失败层、规则与位置不能创建部分 Dromen，也不能伪造 unmet
能力被拒绝请求摘要、调用者、目标资源、政策版本和拒绝规则不能自动扩大权限；缺少必需观察通常留下 agno
后端或适配器故障组件身份、稳定错误类别、预算与已知副作用不能把工具故障改写为目标不满足
预算耗尽预算单位、限值、已用量和停止位置不能把会话 interrupted 写成任务完成
环境或政策漂移期望与实际身份、版本、权限和生效时间不能继续复用旧 Dromen 或旧验证结论
Iknem 范围不足缺失判据、覆盖范围、证据强度和决定权威不能仅因日志多、签名多或模型评分高而 accepted

当前状态与限制条件

当前已有成果是 endem drase 背后的 Drasor 职责边界、Dromen 会话契约、Iknem 记录分层、满足结果与决定结果的分离，以及模型后端和实时能力不得混为一体的架构约束。它可以调用 Ktisor 的制品形成侧检查接口，但不得链接 Ktisor 写入器，也不得修改精确制品或外部签名陈述。

当前没有 Drasor 实现、运行 API、沙箱、能力适配器、预算计数器或 Iknem 物理格式。模型、控制面、凭据、适配进程、文件、网络、终止与观察的部署方式仍在非规范隔离研究提案中研究；未来实现必须通过相应威胁测试后，才能陈述具体隔离能力。
