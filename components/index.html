项目/组件

组件

按系统职责进入 Compiler Core、链接与装载、对象格式、Runtime 和 NSFE。

CoreObjectsRuntime

为什么按组件分层

Noemion 必须把确定性对象生成、不可信对象处理、生成式求解和模型辅助分隔开，否则模型输出可能绕过格式、权限和完整性规则，系统也无法定位错误属于规范、编译、链接还是运行阶段。

核心组件

Compiler Core确定性解析、规范化、验证与 GSIR/GOBJ 生成。 Linker、Loader 与 Runtime符号、重定位、对象合并、可信装载和生成式求解边界。 对象格式GOBJ/SSO 的组织、链接、装载与完整性规则。 NSFE封闭任务头、Source Graph 聚合、检索路由与模型工程状态。

职责边界

对象格式负责组织、链接、装载与完整性；Runtime 负责生成式求解与 Lowering。模型不能决定二进制布局或直接生成 GOBJ。

概要设计与依赖方向

规范与 Profile
Compiler Core
对象与链接器
验证与 Loader
Runtime / NSFE

依赖方向必须从候选语义走向确定性对象，再走向已验证的运行绑定。Runtime 可以消费对象声明，但不得回写对象布局；NSFE 可以产生候选、聚合与路由信号，但其输出必须经过确定性边界才能进入对象或策略。

每个组件的权威说明必须定义输入输出、内部状态、关键算法或数据结构、失败条件、资源上限、可观测事件和验证策略。缺少其中任何一项时，只能视为职责级设计，不能推定接口或实现已经稳定。

工具入口

组件说明回答系统“由什么组成”；具体命令、输入输出和使用方式由工具项目页及其文档负责。
