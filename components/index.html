项目/组件

系统组件

Noemion 把确定性编译、对象处理、模型辅助、智能体控制、生成式求解和结果验收分成相互约束的组件。

NoesisNoemaHorizonHarnessFulfillment

为什么按组件分层

Noemion 必须把确定性对象生成、不可信对象处理、会话与能力控制、生成式求解和模型辅助分隔开，否则模型输出可能绕过格式、权限和完整性规则，系统也无法定位错误属于规范、编译、链接、装载、策略还是运行阶段。

核心组件

Horizon Engine从长篇来源中提取带证据的不可信候选，但不决定对象内容。 Noesis Core把受控来源或候选编译为可检验的 Noema IR，并生成确定性对象。 Noema Object System解析符号和依赖，链接、验证并装载 Noema Object。 Agent Harness装配上下文和受限能力，驱动可观察、可拒绝的智能体会话。 Fulfillment Runtime在已验证对象和验收契约下求解，并产生候选评价或能力请求。

组件为什么存在，什么时候可以省略

这些名称首先表示必须可审计的逻辑职责，不要求每项都部署成独立进程或服务。一个实现可以把多个职责放入同一二进制，但不能因此混淆输入、权限、失败和证据；不需要某项能力的场景也不应被迫加载对应组件。

组件不可替代的价值省略或合并条件不能被合并掉的边界
Horizon Engine在长来源中产生带来源、不确定性和替代解释的有限 Candidate Envelope。完全受控的 NSL、Text NIR 或外部候选路径可以不使用；若实验不能优于规则/外部基线，应停止建设。它始终是不可信前端，不能与 Noesis Core 合并成模型直接写对象。
Noesis Core以确定性规则完成来源绑定、类型、约束、歧义和规范对象写入，是形成合格 NIR/NOBJ 的信任转换点。可以作为编译器、汇编器共享的库，不必独立运行；凡生成合格对象都不能绕过。模型、Runtime 和 Harness 不能取得布局或确认语义的决定权。
Noema Object System把不可信对象安全地链接、验证和装载为不可变 Loaded State，并隔离失败原子性。只做编译或静态检查、不链接和不运行时可以不启用装载部分；实现可作为库嵌入工具。签名、结构、依赖、权限和环境装载检查不能交给 Runtime 猜测。
Agent Harness在多步、可调用外部能力的任务中统一会话、最小权限、真实观察、预算、恢复和 Acceptance Policy。无外部能力、无多步反馈且验收完全确定的单次求解可以退化为最小适配层，不必独立服务。只要存在外部副作用或动态权限，Runtime 就不能直接持有句柄或自行验收。
Fulfillment Runtime把已验证语义转化为具体候选，并把求解后端差异限制在受约束接口内。对象编译、查看、链接、验证和发布流程不需要启动 Runtime；不同后端可以实现同一逻辑接口。求解不能回写签名对象、扩大权限或把 Candidate Assessment 冒充 Acceptance Decision。

职责边界

Noema Object 是数据格式；Noema Object System 负责链接、验证和装载。Agent Harness 管理上下文、会话级能力、真实环境观察和最终验收流程。Fulfillment Runtime 在受限语义视图下求解并评价候选，但只能提出 Capability Request，不能直接持有外部能力或宣告任务最终通过。模型只能提交候选，不能决定对象布局、扩大权限或直接生成 NOBJ。

Evidence / Decision

组件关系不是“信任不断上升”的单向直线。Noesis Core 只能证明受控推导、结构和约束检查；Object System 只能证明链接、完整性和当前装载策略；Runtime 候选、工具结果和环境观察仍是不可信数据。Harness 与 Runtime 通过 Runtime Request、Capability Request 和 Capability Observation 循环协作，直到预先声明的策略形成 Acceptance Decision 或要求人工复核。

Horizon Engine 产生 Candidate Envelope，不直接产生 Horizon Object。Noesis Core 把共享内容编译为 NOBJ；Object System 可以把 NOBJ 与已有 HOBJ 链接为新的 HOBJ。HOBJ 是对象形态，Horizon Engine 是模型组件，两者只有语义主题关联，没有直接的生产者—产物关系。

每个组件的权威说明必须定义输入输出、内部状态、关键算法或数据结构、失败条件、资源上限、可观测事件和验证策略。缺少其中任何一项时，只能视为职责级设计，不能推定接口或实现已经稳定。对象格式本身属于NOBJ 规范，不作为运行组件重复登记。
