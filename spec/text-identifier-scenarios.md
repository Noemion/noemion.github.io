# Text and Identifier Natural-Language Design Scenarios

- 文档 ID：`TEXT-SCEN`
- 版本：`0.1.0-draft`
- 日期：2026-07-13
- 状态：非规范设计审查语料
- 适用范围：文本槽、UTF-8、来源变换、标识符、规范化、比较、范围、双向显示、不可见字符与模型输入

这些场景不是 Unicode 处理实现、编辑器测试或模型安全基准。它们用支持案例、反例和边界场景检查 TEXT-IDENTIFIER-CORE 能否解释现实文本问题；只有条款、登记验证和正反向量共同覆盖的内容才可能成为符合性要求。

### TEXT-SCN-001 — 同一字符串位于不同文本槽

- **场景：**诊断消息和机器码都显示 `missing-source`。
- **审查结论：**机器码按结构标识符比较，本地化消息只供阅读；字符串相同不能让两者互换职责。

### TEXT-SCN-002 — 无效 UTF-8 在解析前失败

- **场景：**来源文件包含孤立续字节，读取器打算用替换字符继续。
- **审查结论：**解码必须原子失败，不能先替换再形成 `rhem` 或诊断成功对象。

### TEXT-SCN-003 — 过长编码不能成为别名

- **场景：**攻击者用非最短 UTF-8 表示 ASCII 分隔符。
- **审查结论：**RFC 3629 解码边界先拒绝，不能让两个字节序列进入同一语法 token。

### TEXT-SCN-004 — 来源清单转义展开改变表示

- **场景：**`.ends` 字段包含两个可见字节 `\` 与 `n`，映射后 `rhem.content` 包含换行标量。
- **审查结论：**映射关系必须说明转义展开；END-P1 保存解码内容，不等于保存来源清单的原始字节。

### TEXT-SCN-005 — CRLF 与 LF 语义相同但字节不同

- **场景：**END-SRCM 允许两种行边界产生同一映射。
- **审查结论：**它们可以产生相同解码结果，但原始文件身份不同；没有原始字节引用时不得宣称逐字节来源保真。

### TEXT-SCN-006 — ASCII 标识符按字节区分大小写

- **场景：**`PolicyA` 与 `policya` 同时出现。
- **审查结论：**现行 Profile 把它们视为不同 ID；界面可以警告，但不能静默合并或按 locale 改写。

### TEXT-SCN-007 — 全角与同形字符不能进入现行 ID

- **场景：**候选关系 ID 使用全角拉丁字母或西里尔字母 `а` 冒充 ASCII `a`。
- **审查结论：**当前 ASCII 语法直接拒绝；不得用 NFKC 或同形骨架把它转换成已有 ID。

### TEXT-SCN-008 — NFC 与 NFD 来源看似相同

- **场景：**两个来源分别使用预组字符和字母加组合音标。
- **审查结论：**原始字节与标量序列不同。只有具名规范化 Profile 可以产生可比较派生文本，不能回写原来源。

### TEXT-SCN-009 — NFKC 会消除有意义区别

- **场景：**数学、兼容字符或全半角形式在来源中承担引用或排版意义。
- **审查结论：**不得盲目做 NFKC。若特定槽需要兼容映射，必须建立新输出身份并披露损失。

### TEXT-SCN-010 — 搜索命中不是精确引用

- **场景：**用户用不区分大小写和音标的搜索找到一个符号。
- **审查结论：**搜索只定位候选；最终绑定仍比较完整结构 ID 与内容身份。

### TEXT-SCN-011 — 标量范围不是用户可见字符数量

- **场景：**一个可见字素由基础字母和多个组合标记组成。
- **审查结论：**END-P1 范围按标量计数；界面选择可以使用字素，但必须映射回同一主体的标量边界。

### TEXT-SCN-012 — 规范化使旧范围失效

- **场景：**来源先记录标量范围，随后被 NFC 合成。
- **审查结论：**派生文本需要范围映射或新范围；旧 start/length 不能直接复用。

### TEXT-SCN-013 — 合法右到左正文保持逻辑顺序

- **场景：**阿拉伯语来源包含自然双向文本。
- **审查结论：**来源按逻辑顺序保存，显示遵守 UAX #9 并按槽隔离；不得粗暴强制全部从左到右。

### TEXT-SCN-014 — 方向覆盖符移动视觉边界

- **场景：**来源含方向覆盖控制，使引号或分隔符看起来位于另一位置。
- **审查结论：**解析仍按逻辑顺序；安全审查视图显示控制符名称和位置，结构字段因 ASCII 闭包直接拒绝。

### TEXT-SCN-015 — 零宽字符使两份文本看似相同

- **场景：**候选中插入零宽连接符、默认可忽略字符或非 ASCII 空白。
- **审查结论：**来源保留精确序列并生成风险清单；系统不静默删除，也不只凭存在该字符断言恶意。

### TEXT-SCN-016 — 变体选择符改变模型 token

- **场景：**审核者看到普通提示，模型实际输入附带不可见变体选择符。
- **审查结论：**模型输入身份、预处理链、隐藏字符清单和显示视图必须共同记录；输出仍是模型候选。

### TEXT-SCN-017 — 语言标签不能触发隐式语义规则

- **场景：**来源自报 `tr`，实现据此改变大小写比较和规范排序。
- **审查结论：**语言标签只是声明；规范 ID 与排序不受 locale 影响，语言相关处理必须使用独立 Profile。

### TEXT-SCN-018 — 安全显示与精确复制分开

- **场景：**界面为控制字符加可见标签，并把长内容截断。
- **审查结论：**显示视图绑定来源并披露转义和截断；授权或绑定操作必须使用独立的精确机器值复制路径。
