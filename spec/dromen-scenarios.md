# Dromen 自然语言设计场景

- 文档 ID：`DRO-SCEN`
- 版本：`0.1.0-draft`
- 日期：2026-07-13
- 状态：非规范设计审查语料
- 适用范围：主体、政策、环境、能力、秘密、预算、激活、观察、只读与生命周期边界

这些场景不是装载器、沙箱、能力代理、运行时或序列化会话格式。它们用支持案例、反例和边界场景检查 DRO-CORE 能否解释真实的会话边界问题。只有唯一条款、登记验证和正反提案向量共同覆盖的内容，才可能成为符合性义务。

### DRO-SCN-001 — 显示名不能选择会话主体

- **场景：**两个 attested Endem 使用相同的可读标题，但内容身份不同。
- **审查结论：**Drasor 必须绑定调用方与政策选定的精确身份。名称或搜索顺序不能替代选择权威。

### DRO-SCN-002 — 缓存验证在运行前已经过期

- **场景：**一个 Endem 昨天通过验证，但 attestation 在今天会话开始前已被撤销。
- **审查结论：**建立时必须按具名截止点重新检查并拒绝输入。系统不创建 Dromen，也不产生 `unmet`。

### DRO-SCN-003 — 最新政策不是稳定输入

- **场景：**某服务要求 Drasor 在每次工具调用时使用当时最新的授权政策。
- **审查结论：**会话必须绑定精确政策身份和截止点。后续变化使旧 Dromen 失效，或要求建立新会话。

### DRO-SCN-004 — 涉及权威的未决歧义阻止建立

- **场景：**`apor` 保留两个可能的资源所有者，但只有其中一方可以授权写入。
- **审查结论：**模型不能选择概率更高的一方。具名权威解决歧义，或政策证明该资源不在会话范围前，建立请求必须拒绝。

### DRO-SCN-005 — 适配器自述不能验证环境

- **场景：**远端工具自称只读，但其声明操作包含修改行为。
- **审查结论：**描述是不可信输入。环境绑定必须依据本地政策和实际观察的协议与操作事实。

### DRO-SCN-006 — 更换模型会改变实质后端绑定

- **场景：**会话从一个已登记模型开始，随后静默回退到工具和数据政策不同的另一个模型。
- **审查结论：**后端身份或必需政策属性变化时，旧 Dromen 失效。回退必须形成新的会话提案。

### DRO-SCN-007 — 能力取交集而不是并集

- **场景：**一个 Synem 成员允许读取仓库，另一个成员允许写入发行存储。
- **审查结论：**组合不会让每个成员获得两项权限。每项有效授予都是制品、政策、操作主体、环境与适配器上限的严格交集。

### DRO-SCN-008 — Step-up 授权不能修改运行契约

- **场景：**工具在 Drase 会话中途请求更大的文件系统访问范围。
- **审查结论：**会话必须停止，或在没有该能力时继续。批准会产生新会话提案，不能修补现有 Dromen。

### DRO-SCN-009 — 令牌轮换发生在 Dromen 之外

- **场景：**短期 MCP 访问令牌到期，但资源、受众与范围的授权没有变化。
- **审查结论：**独立能力域可以按政策轮换秘密，不把它存入 Dromen。受众或范围变化必须建立新会话。

### DRO-SCN-010 — 委托任务共享外层预算

- **场景：**模型创建十个子 Agent 任务，每个都要求新的调用和成本预算。
- **审查结论：**子预算必须是有限会话上限的严格子集。委托不能成倍增加权限或资源上限。

### DRO-SCN-011 — 激活不能发现新成员

- **场景：**一个 Synem 守卫成立，同时远端登记库提供了闭包中不存在的新版 Endem。
- **审查结论：**只有精确固定成员可以激活。新对象需要新的 Synem 和新的会话。

### DRO-SCN-012 — 工具成功只是带来源事件

- **场景：**MCP 工具返回成功，A2A 任务进入 completed 状态。
- **审查结论：**这些事件可以支持有范围观察，但不能直接产生 `met`、`sufficient` 或 `accepted`。

### DRO-SCN-013 — 观察计划必须分配每项责任

- **场景：**目标同时要求部署与健康观察，但会话只绑定了部署适配器。
- **审查结论：**适用 `krin` 责任没有获授权生产者，因此建立失败。系统不能先运行，再为了设计方便把缺失证据称为 `agno`。

### DRO-SCN-014 — 环境漂移使契约失效而不是修改契约

- **场景：**第一次操作后，绑定的沙箱配置或适配器版本发生变化。
- **审查结论：**Drasor 追加漂移事件，保留已有 Iknem，并按政策中断或失败。它不能原地更新旧 Dromen。

### DRO-SCN-015 — 诊断快照不能恢复权限

- **场景：**操作人员导出脱敏会话报告，并在另一台机器上导入。
- **审查结论：**报告可以作为 Iknem 支持审计，但不能重新创建 Dromen、恢复凭据或继续旧 Drase 会话。

## 当前未决

运行 API、事件编码、沙箱选择、凭据存储、协议适配进程、预算计数器、恢复和销毁机制仍未冻结。代码阶段开启前，项目只维护场景、条款、威胁、提案向量和未来验证条件。
