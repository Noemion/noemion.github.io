# Synem Core Threat Model

- 适用规范：`SYN-CORE 0.1.0-draft`
- 版本：`0.1.0-draft`
- 日期：2026-07-13
- 状态：组合闭包与条件激活威胁模型；没有组件实现证据

## 1. 保护目标

本模型保护闭包完整、精确绑定、有限无环、权限不扩大、成员结果不混同，以及条件激活不改写闭包或结果域。所有成员、关系、清单候选、缓存、路径、远端仓库响应、工具列表、模型输出与激活事件都视为不可信输入。

## 2. 威胁清单

### THR-SYN-001 — 环境依赖替换

攻击者借当前目录、搜索路径、网络最新版本、同名对象或输入顺序替换必需成员。`SYN-RES-001` 要求精确且唯一的内容绑定，并使同一正式输入得到同一闭包。

### THR-SYN-002 — 闭包截断与成员遗漏

攻击者只打包直接成员，省略传递依赖，或在运行时补入未审查对象。`SYN-CLS-001` 要求完整传递闭包并原子拒绝外部必需引用。

### THR-SYN-003 — 循环与资源放大

攻击者构造循环、高扇出、深链或可选依赖递归，耗尽解析和披露预算。`SYN-GRF-001` 要求第一阶段有限无环、显式预算和可选项不承担必需语义。

### THR-SYN-004 — 权限放大与冲突洗白

攻击者把成员需求解释为授权，采用权限并集，或借激活和外部 Agent 声明创建新句柄。`SYN-AUT-001` 只允许交集与收紧，并要求冲突关闭失败。

### THR-SYN-005 — 成员结果洗白

攻击者把一个成员的完成、满足、接受、签名或证据数量推广为整个 Synem 的结论。`SYN-STA-001` 保留成员结果域，并要求任何闭包级决定具名策略、范围和权威。

### THR-SYN-006 — 激活竞态与状态混同

攻击者用过期或撤销事件激活成员，把 `inactive` 写成 `met`，把 `unresolved` 写成 `agno`，或在会话中动态改变闭包与权限。`SYN-ACT-001` 固定激活引用、独立四状态、追加记录和重新检查义务。

## 3. 当前不解决

- Synem 物理容器和大规模性能上限；
- 远程对象仓库身份、镜像透明日志和供应链入侵；
- Drasor 沙箱逃逸、凭据泄漏和跨租户侧信道；
- 受限循环、分布式事务和动态调度。

这些边界保持开放，不能用本威胁清单声称实现已经安全。
