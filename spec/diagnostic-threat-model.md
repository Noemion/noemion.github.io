# Diagnostic Threat Model

- 文档 ID：`DIA-THREAT`
- 版本：`0.1.0-draft`
- 日期：2026-07-13
- 状态：跨 Endem、Synem、Dromen、Tekmor 与外部协议诊断威胁草案

诊断同时面向人、自动化和模型，因此也是输入面、权限面与信息披露面。本模型只定义未来实现必须关闭的威胁，不表示诊断生产器、渲染器、协议适配器或 CLI 已经存在。

### THR-DIA-001 — 解析人类消息取得机器语义

消费者通过匹配本地化文本、颜色或措辞决定重试、权限和结果，消息变化即可改变行为。`DIA-IDN-001` 要求唯一机器码与呈现分离。

### THR-DIA-002 — 诊断离开生产语境后被错误复用

缓存只保存路径和消息，随后把旧诊断应用到不同字节、Profile 或规范版本。`DIA-PIN-001` 要求生产者、操作、精确主体和版本绑定。

### THR-DIA-003 — 外部错误洗白为本地结果

HTTP、MCP、A2A、SDK 或退出状态被直接映射为 `unmet`、`accepted` 或其他本地结果。`DIA-LAY-001` 与 `DIA-EXT-001` 要求保留来源层次和结构化映射。

### THR-DIA-004 — 恶意位置污染终端或触发越界

文件名、URL、路径、来源范围或外部文本携带控制字符、超长深度或越界端点。`DIA-LOC-001` 要求类型化、有界、转义的位置。

### THR-DIA-005 — 并发与返回顺序改变主错误

多成员检查、Agent 调用或线程竞态让最后到达或最响亮的错误成为主诊断。`DIA-PRI-001` 要求规范优先级和稳定位置顺序。

### THR-DIA-006 — 恢复建议放大权限和副作用

模型把“重试”“修复”或远端建议解释为自动提权、删除、发布、关闭 `apor` 或无限循环。`DIA-REC-001` 要求有限恢复类别、前置条件和 Dromen 预算约束。

### THR-DIA-007 — 诊断泄露秘密与跨租户数据

工具返回、堆栈、环境、路径或请求正文把令牌、cookie、提示、用户数据和能力句柄写入公开日志。`DIA-SEC-001` 要求默认最小披露和独立调试访问域。

### THR-DIA-008 — 诊断风暴耗尽资源

循环图、重复成员、深层原因或格式化器自报错误造成无界数量、深度与输出。`DIA-BND-001` 要求先检查预算并保留唯一主诊断。

### THR-DIA-009 — 阻断错误与部分成功并存

组件报告错误却返回可继续使用的部分对象、能力或验证句柄，调用者选择成功分支。`DIA-ATM-001` 要求失败原子且未执行层显式为 `not-run`。

### THR-DIA-010 — 适配器改写诊断身份

文本、SARIF、HTTP、MCP 或遥测适配器为了目标格式方便而合并代码、改变层次或丢失原始来源。`DIA-IDN-001`、`DIA-LAY-001` 与 `DIA-EXT-001` 要求适配前后保持机器语义。

## 当前边界

当前威胁只映射规范条款与提案向量。错误注入、终端转义、并发稳定性、内存预算、协议降级、跨进程传输和秘密扫描必须等待用户开启代码阶段后另行授权并形成实现证据。
