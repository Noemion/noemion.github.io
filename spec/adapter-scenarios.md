# External Protocol Adapter Natural-Language Design Scenarios

- 文档 ID：`ADP-SCEN`
- 版本：`0.1.0-draft`
- 日期：2026-07-13
- 状态：非规范设计审查语料
- 适用范围：协议版本、对端、能力、调用、映射、状态、产物、错误、取消、重试、交付与安全边界

这些场景不是协议 Profile、适配器、网络服务、运行时或组件测试。它们用支持案例、反例和边界场景检查 ADP-CORE 能否解释现实适配问题；只有条款、登记验证和正反向量共同覆盖的内容才可能成为符合性要求。

### ADP-SCN-001 — 稳定版与候选版不能共用绑定

- **场景：**一个对端使用 MCP 2025-11-25 稳定版，另一个测试 2026-07-28 候选版。
- **审查结论：**两者必须采用不同版本绑定和扩展集合。候选版的无状态核心或 Tasks 扩展不能静默改变稳定绑定。

### ADP-SCN-002 — 未知扩展与默认版本关闭失败

- **场景：**服务端没有返回精确协议版本，并声明本地没有登记的新扩展。
- **审查结论：**适配不得开始调用；SDK 能够反序列化不表示扩展已经获准。

### ADP-SCN-003 — Agent Card 只是自我描述

- **场景：**A2A Agent Card 声称可以付款和审批，但认证主体只获准查询订单。
- **审查结论：**有效能力仍是查询交集；自报技能不是身份、语义权威或授权。

### ADP-SCN-004 — 端点受众或租户改变必须重验

- **场景：**发现结果把请求重定向到新域名，或回调带来另一个租户的 task ID。
- **审查结论：**必须重新核对认证主体、受众、租户和网络政策；原令牌与绑定不能沿用。

### ADP-SCN-005 — 协议能力不能扩大当前会话

- **场景：**远端在会话中新增 streaming、tasks 或文件上传能力。
- **审查结论：**当前 Dromen 只保留建立时的能力交集。需要扩权时建立新授权决定和新会话。

### ADP-SCN-006 — 外部任务始终追溯到一次本地调用

- **场景：**一次工具调用经过异步 task、流式事件、Webhook 和一次受控重试。
- **审查结论：**所有外部 ID 都附属于同一本地调用身份，并保留精确输入、预算和观察责任。

### ADP-SCN-007 — MCP 与 A2A 状态保留各自来源

- **场景：**MCP 工具返回错误结果，A2A task 返回 `input-required`。
- **审查结论：**本地视图可以统一查询，但必须保留协议、版本、原状态和映射版本，不能伪造两个状态等价。

### ADP-SCN-008 — 映射降精度必须写入损失清单

- **场景：**外部时间精度、媒体类型参数或未知字段无法进入本地有限视图。
- **审查结论：**必须声明省略或降精度及其影响；不能删除后仍声称映射完整。

### ADP-SCN-009 — 外部 completed 不等于 accepted

- **场景：**A2A task 完成并返回 Artifact。
- **审查结论：**它只说明外部任务状态。目标满足、证据有效、会话终止与最终接受仍由本地规则分别形成。

### ADP-SCN-010 — 取消或断流不证明副作用回滚

- **场景：**客户端发送取消后连接关闭，但远端已经开始发送邮件。
- **审查结论：**记录取消请求、确认和后续观察；没有停止或回滚证据时不能宣称副作用消失。

### ADP-SCN-011 — 外部 Artifact 只能先成为候选

- **场景：**远端返回名为 `goal.endem` 的文件并标记任务完成。
- **审查结论：**文件仍是不可信候选；在独立读取、内容、身份、证据和接受流程完成前不能成为 Endem 或 Tekmor。

### ADP-SCN-012 — 名称和远端 ID 不是内容身份

- **场景：**两次调用都返回 `report.json` 或同一个 artifact ID，但字节不同。
- **审查结论：**内容身份、生产调用和对端绑定必须分别保存，不能以名称覆盖已接受版本。

### ADP-SCN-013 — 协议工具任务与业务错误分开

- **场景：**一次 JSON-RPC 方法不存在，一次工具执行失败，一次业务规则拒绝付款。
- **审查结论：**三者分别映射到结构化来源层次；不得解析消息猜测，也不得直接产生 `unmet`。

### ADP-SCN-014 — 终态后的修订建立新调用

- **场景：**A2A task 已到终态，用户希望补充输入继续处理。
- **审查结论：**新工作必须有新的本地调用和外部任务关联；旧终态、Dromen 与权限不能复活。

### ADP-SCN-015 — 非幂等调用断连后不自动重试

- **场景：**付款请求发送后连接中断，无法确认服务端是否已经应用。
- **审查结论：**没有去重或事后核对证据时停止自动重试并升级人工复核。

### ADP-SCN-016 — 去重重试共享预算与调用身份

- **场景：**查询操作具有协议级去重键，第一次响应丢失。
- **审查结论：**重试保留本地调用身份、去重键、截止点和剩余 Dromen 预算，并核对原请求是否已经应用。

### ADP-SCN-017 — 推送乱序缺口不会伪造完整结果

- **场景：**Webhook 重复第 4 个事件、缺少第 3 个事件，并从错误租户发送终态通知。
- **审查结论：**跨租户通知拒绝；重复项去重；缺口保持不完整；终态不能覆盖交付证据缺失。

### ADP-SCN-018 — 令牌透传与 SSRF 默认拒绝

- **场景：**Agent Card 要求把用户令牌传给下游，并把回调地址指向云元数据服务。
- **审查结论：**凭据必须在独立能力域换取受众受限授权；网络政策拒绝私有地址、危险重定向和 DNS 重绑定。

## 当前未决

MCP、A2A、HTTP、模型 SDK 与操作系统调用的具体 Profile，JSON-RPC、Protocol Buffers、SSE 和 Webhook 字段映射，适配器 API、进程隔离、凭据代理、重试实现、事件存储和部署拓扑仍未冻结。
