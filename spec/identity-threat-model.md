# Exact Content Identity and Attestation Threat Model

- 文档 ID：`ID-THREAT`
- 版本：`0.1.0-draft`
- 日期：2026-07-13
- 状态：跨制品精确身份与签名边界威胁草案

精确身份同时穿过序列化、引用、签名、撤销、发布、证据与调试关系。攻击者不必攻破摘要算法，也可能通过换域、缩短显示、替换签名语境或复用旧验证状态造成对象混淆。本模型只定义未来实现必须关闭的威胁，不表示摘要器、签名器、验证器或发布系统已经实现。

### THR-ID-001 — 跨域摘要混淆对象

消费者只比较摘要字符串，把不同对象种类、规范或 Profile 中的值合并。`ID-DOM-001` 要求完整身份语境共同参与比较。

### THR-ID-002 — 摘要输入边界不一致或自包含循环

发布者与验证者对规范化、尾随数据、签名包络或排除范围理解不同，或身份字段把自己纳入摘要。`ID-BYT-001` 要求精确、版本化且非循环的字节输入。

### THR-ID-003 — 可变名称替代不可变引用

路径、URL、版本范围、`latest`、build ID 或外部 ID 在解析后改变目标。`ID-REF-001` 要求所有安全引用闭合到一个完整精确身份。

### THR-ID-004 — 算法降级与参数歧义

攻击者省略算法、改变输出长度、利用未知算法或让验证器尝试多种算法直到成功。`ID-ALG-001` 要求版本化算法政策、参数固定和关闭失败。

### THR-ID-005 — 短摘要碰撞选择错误对象

显示前缀、二维码缩写或 build ID 被用于绑定，碰撞时选择缓存中的第一个对象。`ID-DSP-001` 要求完整摘要参与安全比较。

### THR-ID-006 — 精确字节被提升为语义等价

相同摘要被解释为主张真实，不同摘要被解释为语义不同，或裁剪与迁移结果自动继承 Semantic Key。`ID-EQV-001` 保持精确身份与未定义等价关系分离。

### THR-ID-007 — 裸摘要签名被跨类型重放

攻击者把发布签名改作授权签名、替换载荷类型、受众或算法，或在不同包络结构间搬运签名。`ID-STM-001` 要求有类型、带用途并保护关键上下文的完整陈述。

### THR-ID-008 — 验证包络材料不足或反向改变主体

包络只保存密钥提示、缺少时间与状态材料，或追加证书后改变被签对象身份。`ID-ENV-001` 要求外置包络、精确陈述绑定和有限材料作用。

### THR-ID-009 — 有效签名洗白主张与权限

证书、签名或透明日志包含被解释为事实正确、目标满足、能力授权或最终接受。`ID-AUT-001` 要求密码验证、主体认证、授权、适用性与决定分开。

### THR-ID-010 — 旧验证状态绕过撤销与截止点

缓存成功被无限复用，密钥撤销、证书过期或政策变化没有进入具名截止点评估。`ID-VAL-001` 要求带状态新鲜度的时间语境。

### THR-ID-011 — build ID 或单次重跑冒充可复现性

相同源版本、相同日志、相同 build ID 或同一缓存的两个副本被宣称为独立逐字节复现。`ID-REP-001` 要求完整输入、独立产出和精确输出比较。

### THR-ID-012 — 派生制品继承来源信任

裁剪、压缩、调试或迁移制品沿用来源身份、签名、证据、接受状态或能力。`ID-REL-001` 要求每个制品独立身份和有损关系清单。

## 当前边界

当前威胁只映射规范条款与提案向量。摘要实现、密码库、证书链、密钥隔离、透明日志、撤销分发、可复现构建、调试伴随解析和发行签名测试，须在项目进入代码开发阶段并明确实现范围后验证。
