# Text and Identifier Threat Model

- 文档 ID：`TXT-THREAT`
- 版本：`0.1.0-draft`
- 日期：2026-07-13
- 状态：跨制品文本与标识符边界威胁草案

攻击者不必改变人眼看到的字形，也可能改变机器读取的码点、标识符绑定、来源范围、模型 token 或审核结论。本模型只定义未来实现必须关闭的威胁，不表示 Unicode 处理器、编辑器、同形检测器或模型输入网关已经实现。

### THR-TXT-001 — 文本槽职责混淆

来源正文、结构 ID、本地化消息和显示别名共享字符串类型后，消费者把显示内容用于机器绑定。`TXT-SLT-001` 要求每个槽固定用途与失败影响。

### THR-TXT-002 — 非唯一或宽容 UTF-8 解码

实现替换无效字节、接受过长编码或平台本地编码，使不同输入穿过同一验证路径。`TXT-ENC-001` 要求 RFC 3629 唯一解码并在变换前原子失败。

### THR-TXT-003 — 转换后伪称原始来源

换行统一、转义展开、去 BOM、规范化或脱敏后仍沿用输入身份和范围。`TXT-SRC-001` 要求原始字节、解码文本和变换结果分开。

### THR-TXT-004 — Unicode 或区域设置突破标识符闭包

全角字符、大小写折叠、土耳其语大小写或同形字符被映射到现有 ASCII ID。`TXT-IDN-001` 要求当前结构标识符保持封闭 ASCII 与原始字节比较。

### THR-TXT-005 — 隐式规范化改变内容

实现自行选择 NFC、NFKC 或拼接策略，丢失来源区别或产生跨实现身份漂移。`TXT-NRM-001` 要求槽绑定、版本化的显式变换。

### THR-TXT-006 — 宽松比较替代精确身份

搜索、排序、大小写、同形骨架或模型相似度命中被提升为对象相同。`TXT-CMP-001` 要求比较结果只能支持声明的比较域。

### THR-TXT-007 — 范围单位和主体漂移

字节、标量、字素、屏幕列和模型 token 混用，或变换后复用旧范围。`TXT-RNG-001` 要求主体、单位、半开区间和变换映射完整。

### THR-TXT-008 — 双向重排欺骗审查与解析

方向控制让分隔符、条件或字段边界的视觉位置与逻辑顺序不同。`TXT-BID-001` 要求逻辑存储、槽隔离与可见审查模式。

### THR-TXT-009 — 不可见字符和同形替换

零宽字符、变体选择符、非 ASCII 空白或混合脚本让两份不同文本看似相同。`TXT-HID-001` 要求风险清单和可见化，同时禁止把清单当作身份。

### THR-TXT-010 — 自报语言或默认 locale 改变规范结果

声明语言、媒体类型或进程区域设置触发大小写、排序、分词或转码。`TXT-MET-001` 要求元数据只是声明，规范处理使用显式 Profile。

### THR-TXT-011 — 人与模型实际看到不同文本

审核界面显示清洗文本，而 tokenizer 接收含隐藏序列或不同预处理结果的输入。`TXT-AIM-001` 要求绑定模型实际输入、变换、风险清单和显示视图。

### THR-TXT-012 — 显示与导出洗掉决定性差异

截断、脱敏、转义或方向隔离没有披露，复制出的值又被用于安全引用。`TXT-OUT-001` 要求视图 Profile、精确复制路径与损失影响。

## 当前边界

当前威胁只映射条款与提案向量。Unicode 数据更新、字体差异、编辑器交互、跨平台渲染、真实 tokenizer、模型攻击、模糊测试和性能上限必须等待用户明确开启代码阶段后另行授权。
