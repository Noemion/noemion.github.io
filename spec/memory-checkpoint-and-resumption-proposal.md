# 记忆、检查点与恢复边界研究提案

状态：非规范研究提案
日期：2026-07-13
决定状态：等待决定
研究范围：Agent 会话记忆、上下文压缩、检查点、外部任务句柄、恢复、重放与回滚

## 治理边界

本提案不构成 ADR、CORE 规范、内容 Profile 或实现要求，不进入 `registry.json`。
它不创建新制品、文件格式、扩展名、命令、组件、结果域、稳定接口或哲学专名。
当前项目尚未进入代码开发阶段；本文只检查现有责任能否承载未来 Agent 的持久状态与恢复流程。

如果用户接受本提案，后续也应优先把唯一义务归还 TEXT-IDENTIFIER、ID、DRO、ADP、AUT、IKN、DIA、END 与 SYN。
只有真实消费者和验证证明这些责任无法分担时，才重新讨论独立规范。

## 直接结论

会话记忆只是后续调用的候选输入，不是事实、证据或权威。
上下文压缩只是有损或无损的显式变换，不是原始历史的等价替身。
检查点只是对某一截止点状态的有范围描述，不是 Dromen、凭据包或可恢复权限。
外部任务句柄只定位协议域中的任务，不证明任务历史完整、结果可信或本地目标满足。
恢复是在当前政策、环境、主体、能力和预算下重新核对后继续工作，不是复活旧会话。
重放是一次新的动作尝试，必须处理未知副作用、幂等、去重和共享预算。
回滚只能选择或恢复声明范围内的状态，不能撤销已经发生的外部副作用，也不能恢复旧授权。

因此，Noemion 不应建立一个通用的“保存并恢复 Agent”对象，把历史、计划、运行状态、权限、证据和外部任务压在一起。
当前更可辩护的方向是保留不同事实域，以精确身份、来源、截止点、损失和重新验证关系连接它们。

## 发现的真实缺口

DRO-CORE 已规定 Dromen 不能序列化、转移、恢复或复活。
ADP-CORE 已把外部任务状态、重试、取消和本地结果分开。
TEXT-IDENTIFIER-CORE 已要求模型实际输入及其变换可追溯。
ID-CORE、IKN-CORE 与 AUT-CORE 已分别约束身份、证据和授权。

现有资料仍没有在一处回答以下横切问题：

1. 对话历史、压缩摘要、计划状态、运行状态、工作区快照和外部 Task 分别保存什么？
2. 检查点怎样固定精确输入、截止点、已完成与未完成动作，以及未知副作用？
3. 从检查点继续时，哪些信息可以复用，哪些政策、能力、对象和环境必须重验？
4. 流式连接中断、历史缺失、任务 TTL 到期或远端重复通知时，怎样避免误判和重复副作用？
5. “恢复到旧版本”怎样与“撤销外部世界已发生的效果”分开？
6. 缓存、摘要或持久工作区怎样避免取得身份、证据或授权权威？

如果这些问题不显式分开，Agent 框架的 `session`、`state`、`task`、`resume` 与 `rollback` 很容易被误读为同一个可靠恢复机制。

## 八类状态必须分开

| 类别 | 它回答什么 | 最低绑定 | 不能推出 |
| --- | --- | --- | --- |
| 会话或对话历史 | 此前向模型提供或由模型产生过什么 | 会话、消息身份、角色、顺序、时间、来源 | 内容真实、指令获权、历史完整 |
| 压缩摘要或缓存 | 哪些输入经过什么变换被保留 | 原输入身份、变换、模型与 tokenizer、损失、输出身份 | 与原历史语义等价、可替代证据 |
| 计划与控制检查点 | 截止点前控制平面认为做到哪里 | 精确目标、计划修订、截止点、已知事件、未决项、生成者 | 动作确已完成、目标满足、权限可恢复 |
| 外部任务句柄 | 怎样再次定位 MCP、A2A 或其他协议任务 | 协议与版本、对端、租户、任务 ID、创建语境、TTL | 历史完整、结果可信、本地满足 |
| 持久工作区或快照 | 哪些文件、数据或环境状态被保留 | 内容身份、范围、快照方法、排除项、时间、来源 | 外部世界回滚、凭据仍有效、结果可复现 |
| 事件、日志或轨迹 | 生产者报告哪些事件发生 | 生产者、操作、顺序或时间、完整性边界、丢失信息 | 完整因果链、证据充分、最终决定 |
| Iknem | 哪个主体以何方法对何范围提出证据主张 | IKN-CORE 的主体、范围、方法、观察、限制与溯源 | 跨范围真值、自动满足、自动接受 |
| 实时能力与秘密 | 当前允许行动者实际使用什么 | 当前主体、对象、动作、范围、截止点、最小能力域 | 可序列化、可跨会话继承、可由检查点重建 |

“持久”只描述保留期限，不描述可信度。
“可恢复”只描述某种继续能力，不描述恢复对象与授权范围。
任何接口都必须先说清恢复的是输入历史、控制状态、文件快照、外部任务定位，还是一次新的执行尝试。

这些资料共同显示：现实系统已经把历史存储、运行暂停、外部任务、流式续接、文件保留和版本回滚实现为不同机制。
Noemion 可以借鉴这种分离，但不能复制任何供应商或工具的对象模型作为本项目规范。

## MCP 版本与无状态核心的影响

截至 2026 年 7 月 13 日，MCP 官方仍把 2025-11-25 标为当前正式版本。
2026-07-28 发布候选版已经公开，但计划在 7 月 28 日才成为最终版；本提案只把它用于版本漂移研究。

候选版移除协议级 `initialize/initialized` 会话，并把 Tasks 作为扩展处理。
这不会消除状态，反而要求应用明确回答：任务句柄由谁保存、绑定哪个协议版本和对端、何时到期、怎样重新授权、断线期间丢了什么。

因此，Noemion 不能把“协议无状态”写成“执行无状态”。
也不能把 2025-11-25 实验性 Tasks 与候选扩展假定为线格式兼容。
未来 ADP Profile 必须逐版本冻结，并把跨版本迁移和损失作为显式失败边界。

## 来自 GNU 的可用边界

GNU Make 提供一个直接反例：失败配方可能已经改变目标文件。
如果只看文件存在或时间戳更新，就可能把部分写入或损坏目标误认为最新结果。
`.DELETE_ON_ERROR` 的价值不在于规定 Noemion 删除文件，而在于揭示“保留状态”和“状态有效”是两个判断。

`.PRECIOUS` 进一步说明保留中间目标可以是有意策略。
但保留只改变中断后的处置，不证明目标完整、可复现或适合继续。
未来检查点必须声明完成条件、原子提交边界和未知副作用，不能用“文件还在”作为恢复证据。

GNU Guix 的 profile generation 则提供另一类边界。
回滚是选择此前登记的 generation，并保留可追踪的版本关系。
它不是时间倒流：外部系统、网络调用、通知、消费次数和撤销事件不会随 profile 自动逆转。

Noemion 只借鉴具名版本、事务边界和显式回滚关系。
它不把 Make 时间戳、Guix store、generation 编号或命令语义写入 Endem、Dromen 或 Iknem。

## 恢复必须是重新验证，不是复活

检查点可以帮助控制平面重建候选上下文，但不能重建旧 Dromen。
恢复时至少必须重新核对：

1. 精确 Endem 或 Synem 及其 attest 状态；
2. 请求主体、实际行动者、被代表主体、租户与权威；
3. 政策、截止点、撤销与同意仍然适用；
4. 协议、对端、适配器、工具 schema、模型与环境没有越界漂移；
5. 当前能力仍是各方约束的交集，秘密和实时句柄由最小能力域重新取得；
6. 预算扣减、已消费授权次数和已发生副作用没有被检查点回退；
7. 观察与证据责任仍可履行；
8. 未知结果的调用已有幂等、去重或人工处置策略。

需要继续执行时，Drasor 应形成新的会话语境和适用的 Dromen。
新的 Dromen 可以精确引用旧检查点、历史或外部任务作为输入来源，但不会继承旧 Dromen 的身份、能力或授权。

## 继续、重放、重试与回滚必须分开

| 操作 | 精确含义 | 首要风险 | 最低条件 |
| --- | --- | --- | --- |
| 继续 | 从仍有效的外部任务或控制位置取得后续状态 | 句柄过期、历史缺失、主体漂移 | 重新绑定对端、任务、租户、截止点和当前授权 |
| 恢复 | 从持久状态重建候选上下文并开始新的受控会话 | 旧权限复活、快照过期 | 全量重新验证并建立新 Dromen |
| 重试 | 对已知失败或未完成请求再次尝试 | 重复副作用、预算逃逸 | 幂等或去重证据、共享预算、明确重试分类 |
| 重放 | 按历史记录再次发出动作或重算结果 | 历史不完整、输入漂移、重复消费 | 精确输入、版本、环境、授权和副作用处置 |
| 回滚 | 选择或恢复声明范围内的先前状态 | 假装撤销外部效果、恢复已撤销凭据 | 回滚范围、补偿责任、不可逆效果清单 |
| 补偿 | 以新动作处理既有副作用 | 补偿失败或造成新效果 | 新授权、新会话、独立观察与失败语义 |

取消不是回滚，回滚不是补偿，重连不是无损续接。
这些操作可以组合，但每个事实必须保持自己的身份和结果。

## 支持案例与反例

### 案例一：压缩对话后继续研究

系统保存原消息身份，并记录压缩所用模型、tokenizer、提示、截止点、删除项和摘要身份。
新调用把摘要作为候选输入，同时允许审查者返回原始来源。

支持结论：压缩可以降低上下文成本。
它不能宣称与原历史等价，也不能让摘要中的模型判断取得事实或授权地位。

### 案例二：摘要遗漏否定约束

原始要求是“不得向外部发送”，摘要只保留“生成并分享报告”。

反例：这是有权威差异的装配损失。
恢复必须阻断或回到原始来源，不能因为摘要是系统生成就优先采用。

### 案例三：审批暂停后政策变化

SDK 可以持久化暂停状态并在稍后继续，但等待期间政策撤销了目标权限。

正确分层：暂停记录仍可读取；旧批准不再适用；旧 Dromen 不能复活。
如要继续，必须形成新请求、新授权判断和新会话。

### 案例四：MCP Task 在断线后 completed

客户端以任务 ID 重新查询，远端报告 completed，但断线期间的历史不可用。

正确分层：外部任务状态可以记录为 completed。
本地目标仍依据可得产物、完整性边界和 Iknem 判断；缺失历史不得自动补成完整证据。

### 案例五：A2A 推送重复到达

同一终态通知因至少一次交付而到达两次。

反例：第二次通知不能再次消费授权、发送付款或生成双份外部副作用。
适配器必须按精确任务和事件关系去重，并保留重复事实。

### 案例六：未知结果的付款调用

请求已发送，连接在响应前中断；系统不知道外部是否提交成功。

反例：不能盲目重放。
必须先查询有权威的外部状态或进入人工处置；没有幂等或去重证据时，重试保持阻断。

### 案例七：工作区快照恢复

持久沙箱保存了文件，但访问令牌已撤销，远端数据也已更新。

正确分层：文件快照可以作为新会话输入。
令牌必须重新取得，远端对象重新绑定，快照不能证明当前环境或权限未变。

### 案例八：Guix generation 回滚

控制平面选择上一份声明配置，但此前版本已经发送外部通知。

正确分层：配置关系可以记录为 rolled back。
通知不会消失；若需要撤回或补偿，必须形成新的授权动作和观察。

### 案例九：Make 中间文件被保留

中断后文件因 `.PRECIOUS` 保留，但只写入了一部分。

反例：保留策略不等于有效检查点。
没有原子完成标记或独立验证时，该文件不能支持继续构建或满足判断。

### 案例十：缓存命中产生相同输出

模型或工具缓存返回此前结果，当前输入表面相同，但政策、时间范围或外部数据已变。

反例：缓存键必须覆盖声明语境；命中只是来源事实。
它不能替代当前授权、环境核对或证据新鲜度判断。

### 案例十一：检查点分叉并发恢复

两个工作进程从同一检查点恢复，各自认为一次性授权尚未消费。

反例：检查点不能回退权威状态。
消费计数和去重必须由当前权威域原子判断；其中一个分支应被拒绝或转为新的决定。

### 案例十二：TTL 删除后否认历史

会话历史和外部 Task 因保留期限到期被删除。

反例：删除只说明当前不可取回。
它不能证明事件从未发生，也不能撤销已有证据、授权消费或外部副作用。

## 威胁到失败责任的映射

| 威胁 | 典型表现 | 唯一主责任 |
| --- | --- | --- |
| 旧权限复活 | 从检查点重建 Dromen、令牌或批准 | DRO 建立新会话；AUT 重新判断授权 |
| 跨租户串接 | 相同 session/task ID 被另一主体继续 | ADP/AUT 绑定协议、对端、租户、主体与对象 |
| 摘要洗白 | 压缩删除否定、限制或来源 | TEXT-IDENTIFIER 固定变换、损失与实际模型输入 |
| 历史冒充证据 | 对话或日志被当作完整真实观察 | IKN 固定范围、方法、完整性与限制 |
| 盲目重放 | 未知结果调用被自动再次执行 | ADP 要求幂等或去重证据；DRO 约束预算 |
| 句柄漂移 | 旧 ID 指向过期、替换或错误任务 | ADP 固定版本、对端、任务创建语境与 TTL |
| 缓存提权 | 缓存命中跳过当前政策或环境核对 | DRO/AUT 在使用时重验；ID 只记录派生关系 |
| 快照夹带秘密 | 持久工作区保存令牌或能力句柄 | DRO 保持秘密外置；最小能力域负责重新取得 |
| 分叉双消费 | 多个恢复分支重复消费一次性决定 | AUT 原子消费决定；ADP 按调用身份去重 |
| 回滚幻觉 | 恢复文件后声称外部副作用已撤销 | ADP 保留外部状态；IKN 记录实际观察 |
| 缺失洗白 | 断线、压缩或 TTL 删除被解释为没有事件 | DIA 报告缺失；IKN 把覆盖保持不足 |
| 状态越级 | Task completed 或 session resumed 变成 met | END/IKN 保留满足与证据责任 |

## 失败域与结果隔离

| 事件 | 历史或检查点 | 外部任务 | 会话终止 | 满足 | 最终决定 |
| --- | --- | --- | --- | --- | --- |
| 摘要遗漏权威约束 | 有损且不可安全使用 | 不改变 | 新会话不得开始 | 不改变 | 不改变 |
| 旧授权已经撤销 | 历史仍可保留 | 可继续查询只读状态 | 旧会话不能恢复 | 依据既有证据 | 可 deferred |
| 断线且结果未知 | 记录未知截止点 | working 或未知 | interrupted | 通常 `agno` | 可 deferred |
| 重试被去重 | 记录新尝试与去重关系 | 保持协议事实 | completed/failed 按本地会话 | 仍需观察 | 不自动 accepted |
| Task completed 但产物不可验证 | 历史不变 | completed | completed 或 failed 按本地契约 | `agno` 或 `fault` 按失败来源 | 由权威决定 |
| 回滚配置成功但外部效果保留 | 新旧快照关系成立 | 外部状态不回写 | 新会话独立终止 | 依据当前事态 | 由权威决定 |

DIA 可以报告 `checkpoint stale`、`history incomplete`、`task expired` 或 `outcome unknown` 一类诊断概念，但诊断不能自动重试、恢复权限或改写任何结果域。
本提案不登记具体机器码。

## 候选责任的唯一主归属

| 责任 | 主归属 | 说明 |
| --- | --- | --- |
| 历史、摘要、压缩和实际模型输入 | TEXT-IDENTIFIER-CORE | 固定槽、来源、变换、模型、tokenizer、损失和复核视图 |
| 快照、检查点、派生状态和原对象的精确关系 | ID-CORE | 名称、路径、session ID 和 task ID 只定位候选 |
| 新会话建立、当前能力交集、预算和秘密外置 | DRO-CORE | Dromen 永远不被检查点恢复或复活 |
| 外部任务句柄、轮询、重连、重试、去重和交付 | ADP-CORE | 按精确协议版本与对端保留外部事实和映射损失 |
| 重新授权、撤销、次数消费和分叉竞争 | AUT-CORE | 恢复不能回退决定状态或扩大范围 |
| 观察、历史完整性主张和证据覆盖 | IKN-CORE | 日志、快照与远端历史只有在声明范围内才支持判断 |
| 目标和闭包身份 | END-CORE / SYN-CORE | 恢复不改变目标内容，不把控制位置写入目标 |
| 主阻断诊断与安全披露 | DIA-CORE | 报告陈旧、缺失、冲突和未知结果，不执行恢复副作用 |

可变计划、检查点索引和会话历史存储属于 Drasor 外侧控制平面的未来实现职责。
这不使控制平面成为新的语义权威，也不表示当前存在存储器、恢复器或运行时。

## 语言哲学边界

维特根斯坦《逻辑哲学论》3.32 区分记号与其指号方式。
这支持一个有限工程类比：同一段保留字节、同一个 session ID 或 task ID，不会仅凭形式相同就保留原来的作用、对象和权威语境。

该类比不能定义持久化格式、分布式一致性、事务隔离、幂等算法或恢复协议。
正式边界仍必须由精确身份、截止点、权限、外部副作用、威胁和验证证据定义。

## 若接受后的最小规范动作

1. 在 TEXT-IDENTIFIER-CORE 中明确压缩摘要必须绑定原输入、变换、模型、tokenizer、损失和实际复核视图。
2. 在 ID-CORE 中明确检查点、快照和压缩结果是新对象或记录，各有身份并以显式关系连接来源。
3. 在 DRO-CORE 中明确恢复只能重新验证并建立新会话，不能重建旧 Dromen、秘密或能力。
4. 在 ADP-CORE 中明确外部任务句柄的协议、对端、租户、TTL、历史缺失、重连、重复通知和未知结果边界。
5. 在 AUT-CORE 中明确恢复、分叉和重放不能回退撤销、截止点、授权次数或消费状态。
6. 在 IKN-CORE 中明确历史、日志、快照与 Task 记录只有在声明完整性和范围内才可支持证据覆盖。
7. 形成摘要遗漏、权限复活、跨租户串接、未知结果重放、重复通知、检查点分叉和回滚幻觉的正反向量。

这些动作都要求先由用户接受责任分配，再通过 ADR 或现有规范修订进入权威材料。
不得直接把本提案文字复制为条款。

## 采用门槛与停止条件

在以下条件满足前，本提案保持非规范：

- 用户接受八类状态分离和现有 CORE 的责任分配；
- 至少一个真实消费者需要跨进程、跨设备或长时间暂停后恢复；
- 检查点能绑定精确目标、计划修订、截止点、未知副作用和已消费授权；
- 恢复流程能证明重新验证，而不是反序列化旧能力；
- 外部 Task Profile 能处理版本、对端、租户、TTL、历史缺失和重复通知；
- 未知结果调用有幂等、去重或人工处置策略；
- 压缩、缓存、快照和日志的损失与完整性可以被审查；
- 正反向量能够区分继续、恢复、重试、重放、回滚和补偿。

若未来消费者只需要短会话内的临时历史，不需要持久检查点或外部任务恢复，本提案可以完全省略。
若恢复必须携带实时令牌、私钥或能力句柄，设计必须拒绝，不能用加密存储把跨会话权限包装成安全恢复。
