# External Protocol Adapter Threat Model

- 文档 ID：`ADP-THREAT`
- 版本：`0.1.0-draft`
- 日期：2026-07-13
- 状态：Drasor 外部协议适配威胁草案

外部协议把不可信对端、网络、任务、消息与产物连接到一次受限会话，因此同时跨越身份、权限、副作用、证据和披露边界。本模型只定义未来实现必须关闭的威胁，不表示 MCP、A2A、HTTP、模型 SDK 或任何适配器已经实现。

### THR-ADP-001 — 浮动版本改变同一绑定的含义

配置使用 `latest`、服务端默认或 SDK 包版本，远端升级后同一调用开始接受新字段、新状态或新扩展。`ADP-PIN-001` 要求固定协议、规范、绑定、schema、传输和扩展版本。

### THR-ADP-002 — 自描述对端取得身份与授权

恶意 Agent Card、工具清单、模型名称或注册信息把自报技能伪装成已认证主体和已授权能力。`ADP-PEE-001` 要求分开发现、声明、端点、认证主体、受众、租户和政策决定。

### THR-ADP-003 — 协议协商绕过 Dromen 能力上限

远端新增工具、任务、回调或采样能力后，当前会话在没有新授权的情况下使用它。`ADP-CAP-001` 要求协议、对端、适配器、Dromen 与政策能力取交集。

### THR-ADP-004 — 外部任务身份混淆本地调用

多个会话复用 request、task、message 或 trace ID，重试和回调无法追溯到精确主体与输入。`ADP-INV-001` 要求每次调用绑定本地会话、调用、主体、对端、输入、预算和观察责任。

### THR-ADP-005 — 统一对象模型静默丢失协议事实

适配器把不同协议的状态、时间、媒体类型和错误折叠成通用布尔值，并声称无损。`ADP-MAP-001` 要求保留原始身份、映射版本和显式损失清单。

### THR-ADP-006 — 外部完成状态洗白为本地结果

A2A `completed`、MCP 工具成功、HTTP 200 或 SDK stop reason 被直接解释成 `met`、`accepted`、Drase `completed` 或 Iknem `valid`。`ADP-STA-001` 要求外部状态域与本地结果域正交。

### THR-ADP-007 — 远端产物越权成为规范制品或证据

消息、Artifact、ToolResult、模型文本、文件名或远端签名直接替换 Endem、Synem、Iknem 或已接受输出。`ADP-ART-001` 要求外部产物保持有来源、有界的不可信候选。

### THR-ADP-008 — 错误文本驱动权限或业务判断

消费者解析本地化消息，把协议错误、工具错误、任务错误和业务错误合并，并据此提权、重试或改变结果。`ADP-ERR-001` 要求通过 DIA-CORE 保留结构化来源。

### THR-ADP-009 — 取消与断流伪造停止或回滚

发送取消、收到 `cancelled`、超时或连接关闭后，系统假定远端副作用已经停止、回滚或不可见。`ADP-CAN-001` 要求分开取消请求、确认和后续观察，并保持终态不可变。

### THR-ADP-010 — 不确定传输结果导致重复副作用

连接在响应前中断，适配器自动重复付款、发布、删除或消息发送，或通过新调用身份隐藏重复。`ADP-RTY-001` 要求幂等分类、去重证据、事后核对和共享预算。

### THR-ADP-011 — 推送乱序缺口与跨租户注入

重复、迟到、缺失、伪造或跨租户的 SSE、Webhook、推送和轮询结果改变本地状态，流关闭又被误认为完成。`ADP-DEL-001` 要求认证、序列、去重、缺口、终结与资源证据。

### THR-ADP-012 — 令牌透传网络绕过与无界披露

上游令牌被传给下游，回调访问本地元数据服务，DNS 或重定向改变目标，任务列表泄露其他租户，长寿命输出无限保留。`ADP-SEC-001` 要求凭据、受众、网络目标、披露和资源最小化。

## 当前边界

当前威胁只映射规范条款与提案向量。协议互操作、网络故障注入、令牌代理、SSRF 防护、并发去重、Webhook 服务、事件存储、沙箱和真实副作用测试，须在项目进入代码开发阶段并明确实现范围后验证。
