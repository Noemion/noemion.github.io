# Endem Natural-Language Scenario Corpus

- 语料 ID：`END-SCEN`
- 版本：`0.1.0-draft`
- 日期：2026-07-13
- 状态：非规范性设计审查语料
- 用途：检验六个语义面、未知分类、根唯一性和验收边界是否能解释真实自然语言场景
- 实现状态：没有解析器、组件或符合性声明；本文件不是可执行测试

## 1. 使用边界

本语料把自然语言案例拆成预期的工程问题，不规定 Ktisor 语法、`.endem` 字节或模型提示。案例中的结构只是审查目标：若现行规范无法不含糊地解释一个案例，应修订术语、ADR、条款或开放问题，不能让模型猜测后把结果写成规范事实。

本语料参考《逻辑哲学论》中事态由对象结合构成（2.01）、图像以要素关系描述可能情况（2.1–2.15）、图示形式由结构显示而非自我陈述（2.17–2.172）、名称与对象配置相对应（3.203–3.21）、理解命题需要知道其成立情况（4.024），以及判断真假之前必须先规定判定条件（4.063）。这些命题只提供问题结构，不直接推出软件字段或现实世界本体。

## 2. 场景矩阵

### SCN-001 — 使一个可观察事态成立

- **来源表达：**“让登记服务返回健康状态。”
- **预期分层：**`rhem` 保存提交后的解码表达与来源；`semion` 绑定“登记服务”“返回”“健康状态”；`skena` 表达 `returns(registry_service, healthy)`；`telis` 使用 `kine`。
- **判断边界：**`krin` 必须指定健康探测的关系位置、允许观察、必需 Iknem 和决定权威。服务进程存在不等于返回健康状态。
- **审查结论：**支持一个根、正极性事态与达到成立方向。

### SCN-002 — 在明确区间保持事态成立

- **来源表达：**“在发布窗口内持续保持主站可用。”
- **预期分层：**`skena` 表达 `available(primary_site)`；`telis` 使用 `mene`，并引用已确定的发布窗口。
- **未决边界：**时区、窗口起止、允许中断和“可用”的测量口径若未确定，必须进入 `apor`，不能由当前时间或监控默认值补齐。
- **审查结论：**现行核心能说明方向分离，但 END-P1 尚未冻结时间区间与连续测量语义。

### SCN-003 — 禁止事项由否定事态表达

- **来源表达：**“不得让匿名用户读取私有报告。”
- **预期分层：**`skena` 表达否定关系 `not can_read(anonymous_user, private_report)`；`telis` 使用 `mene`，表示在适用范围内保持该否定事态成立。
- **判断边界：**`krin` 需要授权策略、实际访问观察和覆盖范围；没有访问日志不等于禁止事项已经满足。
- **审查结论：**不得新增 `avoid`、`forbid` 等第三种目标方向，也不得把否定只留在来源文本中。

### SCN-004 — 指称歧义必须保留

- **来源表达：**“把它发布给管理层。”
- **预期分层：**`rhem` 保存“它”的精确范围；`semion` 只能登记可能指向的报告、构建物或其他候选；`apor` 保存候选、影响关系、解决方式和决定主体。
- **失败边界：**输入没有任何允许候选时是 `aseme`；存在多个允许候选但无人有权选择时是 `apor`。模型最高概率候选不能成为确认指称。
- **审查结论：**nascent Endem 可以显式携带未决项，但不能伪装成已经可发布的 coherent 闭包。

### SCN-005 — 观察不足不同于未满足

- **来源表达：**“确认昨夜备份可以恢复。”
- **预期分层：**`skena` 表达 `recoverable(last_night_backup)`；`telis` 根据请求语境进入待确认方向；`krin` 要求一次受控恢复观察及其 Iknem。
- **判断边界：**只看到备份文件存在时返回 `agno`，因为观察不足；恢复试验明确失败时才是 `unmet`。
- **审查结论：**`phain` 必须与“可恢复”关系对齐，文件存在、摘要有效和恢复成功是不同主张。

### SCN-006 — 求值故障不同于观察不足

- **来源表达：**沿用 SCN-005，但恢复验证器因存储服务故障无法运行。
- **预期分层：**事态、方向与判断契约不变；验证器、依赖或执行环境异常记录为 `fault`。
- **失败边界：**不得把工具故障写成备份不可恢复，也不得把故障隐藏为普通 `agno`。
- **审查结论：**`agno` 描述证据不足，`fault` 描述求值过程没有按契约完成。

### SCN-007 — 权威不足不能确认语义

- **来源表达：**“把响应时间控制在合理范围。”
- **预期分层：**“合理范围”需要具名权威、适用服务、统计窗口、分位数和阈值。模型或遥测平台可以提出候选，不能确认这些值。
- **失败边界：**没有任何允许的定义来源时返回 `aseme`；存在多个业务允许阈值但尚未决定时进入 `apor`。
- **审查结论：**类型正确、历史均值或行业惯例都不能替代语义授权。

### SCN-008 — 两个独立根必须拆分

- **来源表达：**“生成安全评审报告并部署修复后的服务。”
- **预期分层：**生成报告和部署服务可以独立版本化、失败和验收，因此必须形成两个 Endem；只有真实依赖需要解析时才由 Synem 表达关系。
- **失败边界：**不得用一个复合根掩盖两套权限、证据和失败责任，也不得仅因同一句出现“并”就机械拆分。
- **审查结论：**拆分依据是独立身份与生命周期，不是表面连接词。

### SCN-009 — 会话结束、满足判断与最终决定必须分开

- **来源表达：**“确认昨夜备份可以恢复。”恢复试验按计划完成，但实际数据损坏；安全负责人随后依据预注册政策作出否定决定。
- **预期分层：**Drase 会话终止为 `completed`，因为测试过程正常结束；`krin` 求值为 `unmet`，因为有效恢复观察反驳可恢复事态；权威决定为 `rejected`，并绑定安全负责人、政策版本和原满足结果。
- **对照失败：**验证器崩溃时会话可以是 `failed`、满足结果是 `fault`、决定是 `deferred`。缺少恢复观察时满足结果是 `agno`；这些情况都不能写成备份不可恢复。
- **审查结论：**`completed` 不等于 `met` 或 `accepted`，`failed` 不等于 `unmet`，有效 Iknem 也不自动表示证据覆盖充分。

### SCN-010 — 固定发布窗口必须先解析为确定区间

- **来源表达：**“北京时间 7 月 20 日凌晨 1 点到 2 点持续保持主站可用。”
- **预期分层：**具名发布负责人把年份、时区规则和两个时刻解析为确定 UTC 瞬间；`mene` 使用 `fixed` 与半开区间 `[start,end)`；连续性政策明确为 `strict` 或完整的 `budgeted`。
- **失败边界：**缺少年份、使用“本地时间”、时区数据库规则冲突或夏令时导致同一墙钟值有多个候选时进入 `apor`；Ktisor 不能读取当前机器时区替用户选择。
- **审查结论：**来源中的民用时间保留在 `rhem/semion`，规范时间范围只使用已授权的确定瞬间。

### SCN-011 — 离散健康采样不能证明连续可用

- **来源表达：**沿用 SCN-010；监控只在窗口开始、中点和结束前各记录一次健康状态，中间没有覆盖保证。
- **预期分层：**三个健康点可以形成有效 Iknem，但覆盖并集仍有空洞；严格持续目标的满足结果必须是 `agno`。
- **对照失败：**覆盖完整且观察到服务中断时为 `unmet`；监控时钟或适配器违反契约时为 `fault`。不得用插值、最后值延续或“没有告警”填补空洞。
- **审查结论：**观测采样是证据方法，不是连续性政策；记录有效与覆盖充分继续分开。

### SCN-012 — 经过时长必须使用具名事件与单调时钟

- **来源表达：**“部署完成后持续保持健康十分钟。”
- **预期分层：**`mene` 使用 `elapsed`，起点绑定具名部署完成事件，时长使用固定长度单位，Drasor 以声明的单调时钟测量并记录重启边界。
- **失败边界：**部署完成事件没有唯一身份时进入 `apor`；墙钟回拨不能缩短十分钟；单调时钟跨未声明重启或时钟读取失败时结果为 `fault`。
- **审查结论：**绝对时刻与经过时长属于不同时间域，外部 Agent 的完成时间戳不能自动成为本地起始事件或时钟权威。

### SCN-013 — 否定不创造第二套关系

- **来源表达：**“撤销匿名用户读取私有报告的能力。”
- **预期分层：**`semion` 只授权一个 `can_read(actor, resource)` 关系；`skena` 引用相同关系和角色，并把极性设为 `negative`；`telis` 使用 `kine` 使该负事态成立。
- **失败边界：**不得另造 `cannot_read`、`not_can_read` 或 `forbid` 谓词，不得删除关系节点表示否定，也不得交换 actor 与 resource 后声称语义等价。
- **审查结论：**否定改变同一可能事态的极性，不改变关系身份、角色位置或目标方向的职责。

### SCN-014 — 没有访问日志不等于没有访问

- **来源表达：**“发布窗口内没有匿名用户读取过私有报告。”审计系统只保存部分服务的成功访问日志，当前查询没有匹配记录。
- **预期分层：**查询结果只能形成“已搜索记录中未发现匹配项”的 Iknem；由于服务路径、丢失量和时间覆盖不完整，负目标的满足结果为 `agno`。
- **对照失败：**若任何有效记录显示匿名读取，结果为 `unmet`；采集器违反声明契约时为 `fault`。不得把空结果、没有告警或模型未发现异常升级为 `met`。
- **审查结论：**记录缺席只描述数据集，不能自动描述现实事态不存在。

### SCN-015 — 封闭范围才允许从缺席推断

- **来源表达：**沿用 SCN-014，但具名审计权威证明所有读取决策都经过同一不可绕过入口，完整枚举发布窗口内全部路径，并记录零丢失、截止点和已知排除项。
- **预期分层：**Iknem 固定被枚举主体、关系、角色路径、时间区间、方法、生产者和完整性计数；覆盖充分且没有正极性反例时，缺席可以在这个有限范围内支持 `met`。
- **失败边界：**任何旁路、未知丢失、迟到记录、过滤条件、未封闭主体集合或撤销的完整性声明都会使结果回到 `agno`；一个正极性反例足以产生 `unmet`。
- **审查结论：**封闭的是明确观察范围，不是整个世界；它不能把局部完整性推广为无限否定。

### SCN-016 — 全部生产节点健康需要固定全集

- **来源表达：**“确认本次发布涉及的所有生产节点都健康。”
- **预期分层：**具名发布权威在截止点前给出唯一生产节点清单；关系模板固定为 `healthy(node)`，只量化 `node` 角色，量词为 `all`。
- **判断边界：**一个节点的有效 `unmet` 足以使聚合结果为 `unmet`；只有完整清单中每个不同节点均为 `met` 才能得到 `met`。未封闭清单或任一节点观察不足为 `agno`，观察契约故障为 `fault`。
- **审查结论：**“所有”首先要求回答成员是谁，而不是先运行一次搜索再把返回项称为全集。

### SCN-017 — 任一区域就绪可以由一个见证确定

- **来源表达：**“只要任一获准区域已经就绪，就可以进入下一步评审。”
- **预期分层：**量词为 `some`，成员资格仍绑定具名区域权威和截止点；一个不同区域的有效 `met` 是决定性见证。
- **判断边界：**已有见证时，其他区域的故障不撤销该语义结果；没有见证时，只有封闭清单且全部有效 `unmet` 才能得到 `unmet`，开放或不完整范围只能得到 `agno`。
- **审查结论：**存在性见证可以提前决定成立，但“没有找到见证”仍不等于不存在。

### SCN-018 — 至少三个批准按不同主体计数

- **来源表达：**“至少三名不同的授权评审人批准该发布。”
- **预期分层：**量词为 `at_least 3`；成员身份由评审权威目录确定，关系模板为 `approved(reviewer, release)`，只量化 `reviewer` 角色。
- **判断边界：**同一评审人的重复点击、重试事件、多份签名包或多条日志只计一个成员；三个不同成员的有效 `met` 才是决定性满足证据。
- **审查结论：**基数属于成员身份，不属于事件、记录或证据数量。

### SCN-019 — 空集合和成员漂移不能静默决定结果

- **来源表达：**“所有当班值守人都已确认”，但截止点上的当班名单为空，且运行期间名单可能变化。
- **预期分层：**空集合必须由具名排班权威明确允许；缺少该政策时不能利用空真值产生 `met`。若目标是 `mene`，还必须定义成员何时加入、退出及各自适用区间。
- **失败边界：**查询暂时返回空、目录同步失败、名单在求值中变化或默认接受空集时，不能产生确定满足；量化 `mene` 在成员变化语义冻结前保持不支持。
- **审查结论：**形式逻辑揭示了空集问题，但软件规范必须另行决定授权、时间和安全后果。

### SCN-020 — p95 响应时间需要完整测量契约

- **来源表达：**“发布窗口内成功请求的 p95 响应时间不高于 300 毫秒。”
- **预期分层：**`krin` 固定生产请求总体、成功请求的纳入规则、发布窗口、延迟起止事件、毫秒单位、p95 的排序或插值方法、最小样本数、最大估计误差和 `le` 比较器。
- **失败边界：**不得把平均值替换 p95、静默排除慢失败、混合不同分桶边界，或先把秒值舍入再比较。有效区间整体不高于 300ms 才是 `met`，整体高于才是 `unmet`，跨越边界为 `agno`。
- **审查结论：**阈值只有和总体、窗口、方法、单位与不确定度一起出现时才是可判断条件。

### SCN-021 — 固定基准准确率不等于普遍能力

- **来源表达：**“该模型在基准 v3 上的准确率至少为 80%”，并希望据此声称它对同类未见任务也达到 80%。
- **预期分层：**固定题集上的 `benchmark accuracy` 与对相关项目总体的 `generalized accuracy` 是两个 estimand；后者还需总体定义、抽样假设、重复试验结构、预注册统计模型和不确定区间。
- **失败边界：**固定基准的点估计不能升级为普遍能力；模型裁判、自报置信度、事后挑选题目或忽略重复试验相关性都不能关闭缺口。区间跨越 80% 时为 `agno`。
- **审查结论：**AI 评估分数是有假设的测量结果，不是模型能力本体或自动验收结论。

### SCN-022 — 单位可换算不等于可以随意换算

- **来源表达：**“响应时间不高于 0.3 秒”，观察以毫秒产生。
- **预期分层：**秒与毫秒量纲兼容，固定版本的精确线性换算可以得到同一比较值；规范仍保存原始单位、规范单位和换算依据。
- **失败边界：**时间与字节不能比较；百分数与比例不能隐式缩放；摄氏与华氏等非线性或带偏移换算必须固定方向和定义域；比较前舍入无效。
- **审查结论：**单位工具可以核对换算，却不能选择业务构念、总体、统计方法或阈值。

### SCN-023 — 缺样本和统计程序故障必须分开

- **来源表达：**沿用 SCN-020，但发布窗口只收集到 80 个有效请求，契约最低需要 100 个；另一次评估中分位数程序自身失败。
- **预期分层：**前者保留已有观察但覆盖不足，结果为 `agno`；后者没有按声明程序完成，结果为 `fault`。两者都不能根据当前点估计猜测 `met` 或 `unmet`。
- **失败边界：**“无记录值”、遥测间隙、被删失请求、样本不足、程序异常和真实违约必须分别保存，不能统一填零或丢弃。
- **审查结论：**统计精度不足是证据问题，程序失败是求值问题，都不是目标事实本身。

### SCN-024 — 同一服务终态可以使用复合根

- **来源表达：**“发布接口可以访问，并且返回的响应符合版本 3 模式。”
- **预期分层：**可达性与响应模式共同定义同一个接口终态，使用同一 `kine` 和发布验收权威；`skena` 以 `all_of` 组合两个关系叶，`krin` 以相同拓扑组合两个叶判断。
- **失败边界：**任一叶有效 `unmet` 都决定总体 `unmet`；没有反例时，故障优先于观察不足，只有两叶都为 `met` 才能得到总体 `met`。
- **审查结论：**复合根不是多个目标的容器，而是一个不可分终态的显式内部结构。

### SCN-025 — 独立报告与部署必须拆成两个 Endem

- **来源表达：**“生成发布报告并部署服务。”
- **预期分层：**报告与部署具有不同产物、生产者、替换方式、失败责任和验收时点，因此分别形成 Endem，再由 Synem 表达依赖或共同交付关系。
- **失败边界：**不得因为一句话中出现“并”就把两个生命周期压进一个 `all_of`，也不得通过拆分一个不可分接口终态绕过整体判据。
- **审查结论：**拆分依据是独立身份和责任，不是语法连词数量。

### SCN-026 — 决定性短路仍要保存覆盖

- **来源表达：**“主通道或已批准备用通道至少有一个可用。”主通道已经得到有效 `met`，备用通道尚未求值；此前另一个已求值诊断叶记录过故障。
- **预期分层：**`any_of` 可以由主通道决定总体 `met`；结果保存决定叶、全部已求值叶及其 Iknem、故障记录、未求值备用叶和短路原因。
- **失败边界：**备用叶不能被伪写成 `agno`；总体 `met` 不能删除已经发生的故障。若没有任何 `met`，则不能在剩余叶未求值时短路。
- **审查结论：**短路只减少工作，不减少可追溯事实，也不把“未执行”变成一种判断结果。

### SCN-027 — 条件目标暂不进入组合语言

- **来源表达：**“如果审批通过就部署，否则只生成报告。”
- **预期分层：**当前 `all_of / any_of` 不能表达激活条件和不适用分支。设计者必须拆成具有明确依赖的 Endem，或把适用性决定保留在 `apor`。
- **失败边界：**不得把条件写成自由脚本、把未激活分支当作 `met`，也不得用 `agno` 冒充“不适用”。
- **审查结论：**条件组合需要独立的适用性与激活语义，不能借布尔语法提前固化。

### SCN-028 — 权威名称不能替代授权决定

- **来源表达：**“匿名用户不能读取私有报告。”END-P1 的投影写有 `authority:security-owner`，但没有能绑定当前来源、候选关系、显示视图、目的与截止点的授权决定。
- **预期分层：**容器和 Profile 可以接受这些规范字节；完整内容接受保持未满足或未求值，制品继续是 `nascent`。权威名称只帮助定位待核对主体，不能自证该主体已经看过并批准当前候选。
- **失败边界：**不得从组织名称、登录状态、本地角色、旧缓存或另一份 Endem 的批准推导当前 `semion` 已授权。
- **审查结论：**文件内选择器与文件外授权事实必须显式分开。

### SCN-029 — 精确伴随绑定允许内容接受继续

- **来源表达：**沿用 SCN-028；外部验证语境固定 `AUT-CORE 0.1.0-draft`，并取得一个 `grant`，它精确绑定当前来源身份、提交文本、候选内容、`semion` 关系位置、安全显示、授权主体、作用范围、目的、截止点与剩余空 `apor`。
- **预期分层：**符合性报告可以在容器与 Profile 接受之后继续检查 END-CORE 内容；授权绑定只证明该投影在声明范围和截止点内获准，不证明负事态为真、目标满足或制品最终接受。
- **失败边界：**候选、语义位置、目的、租户、截止点或显示视图任一不匹配都不能复用该决定；`deny` 和 `defer` 也不能被解释为有限 `grant`。
- **审查结论：**外部伴随关系补齐验证前置条件，但不进入目标文件自证链。

### SCN-030 — 两个授权决定不创造两个内容身份

- **来源表达：**同一份规范 `.endem` 字节先由安全负责人批准，之后又由合规负责人按另一政策批准；随后第一项授权被撤销。
- **预期分层：**三次事件产生不同的授权伴随关系和截止点评估，但 Endem 精确内容身份保持不变。若任何负责人要求修改六个语义面，修改后的字节形成新的内容身份。
- **失败边界：**不得把授权主体、批准时间、撤销状态或签名材料混入内容摘要输入，也不得在语义字段改变后继续引用旧身份。
- **审查结论：**内容回答“对象是什么”，伴随决定回答“谁在何时、何范围内允许怎样使用它”。

## 3. 跨场景不变量

1. `rhem` 保存提交后的来源表达，不把改写后的文本冒充该来源。
2. `semion` 的每项确认投影都必须来自确定性规则或范围有限的具名决定。
3. `skena` 只表达对象关系及其极性，不携带希望、禁止、优先级或置信度。
4. `telis` 只说明达到成立或保持成立，不复制事态极性。
5. `krin` 在判断之前固定可接受观察、比较关系、证据范围和决定权威。
6. `apor`、`aseme`、`agno` 与 `fault` 分别表示未决选择、无允许意义、观察不足和求值故障，不能合并。
7. 外部模型、MCP 工具、A2A Agent、签名身份和遥测平台只能提供带来源的候选或观察，不能自行确认语义或验收。
8. 两个能够独立实现、失败或验收的根必须拆成两个 Endem；组合不是把多个目标塞回一个对象。
9. 制品生命周期、满足判断、权威决定、会话终止、Iknem 有效性和证据覆盖度分别记录；外部协议状态不得跨域覆盖。
10. `mene` 只接受具名权威解析的 `fixed` 区间或绑定具名事件的 `elapsed` 时长；相对日期和默认时区不能进入规范语义。
11. 连续性使用 `strict` 或完整 `budgeted` 政策；遥测空洞不属于允许违约。
12. 观测覆盖空洞产生 `agno`，有效违约产生 `unmet`，时钟或求值契约故障产生 `fault`。
13. 否定作用于同一关系身份与角色位置，不能通过删除节点、改名谓词或新增目标方向表达。
14. 没有记录不是负极性 `phain`；开放或覆盖不足的观察只能产生 `agno`。
15. 从缺席推断负事实必须绑定具名权威声明的封闭观察范围、完整枚举方法、截止点、路径覆盖和丢失计数。
16. 量化必须固定一个关系模板、一个被量化角色、成员资格权威、截止点和成员身份规则，运行工具不能另选子集。
17. 基数只按不同成员身份计数；重复观察、事件、签名或证据不能增加成员数。
18. 决定性见证或反例可以提前结束聚合；否则范围不完整为 `agno`，成员求值契约故障为 `fault`。
19. 空集合必须明确授权；动态成员和量化 `mene` 继续等待成员有效区间与变化政策。
20. 测量构念、直接可观察标准、固定或泛化总体和预期用途必须先于基准、指标或分数。
21. 数值必须绑定量纲、单位、换算版本与显示舍入边界；比较前不得舍入或隐式缩放。
22. 窗口、纳入排除、最小样本、聚合器及其算法和误差必须固定并可重放。
23. 不确定区间完全位于边界一侧才可确定 `met/unmet`；跨界为 `agno`，程序失败为 `fault`。
24. 复合根只容纳一个不可分终态；可独立版本化、验收或失败的部分必须拆成 Endem，再由 Synem 组合。
25. 第一阶段组合只允许有限无环的 `all_of / any_of`，并要求 `skena` 叶与 `krin` 叶一一对齐。
26. 四结果组合必须保留未知与故障；只有决定性 `unmet` 或 `met` 才允许短路。
27. 短路结果必须保存决定依据、已求值叶、未求值叶与全部已遇到的故障。
28. END-P1 中的权威与证据名称只是待核对选择器，不能证明外部授权、证据或最终决定已经存在。
29. 完整内容接受必须显式核对精确版本的外部前置条件；缺失、错绑、撤销或未求值不能由环境默认补齐。
30. 授权、签名、证据与决定变化不改变同一规范字节的 Endem 内容身份；语义字节变化必须产生新身份。

## 4. 当前暴露的规范缺口

- `mene` 的时间范围、连续性与覆盖分类已经由 ADR-0016 设计冻结；物理字段、闰秒、循环日历、跨重启时钟关联和多生产者归并仍未冻结。
- 否定关系、缺席证据和封闭观察范围已经由 ADR-0017 设计冻结；复合否定和封闭声明的物理编码仍未冻结。
- 单变量量化、成员范围、不同成员计数与决定性聚合已经由 ADR-0018 抽象冻结；物理字段、嵌套或多变量量化、量化否定的组合规范以及滚动成员 `mene` 仍未冻结。
- 指称候选怎样参与尚未 coherent 的关系图仍需更精确的类型边界。
- 测量构念、固定或泛化总体、单位、聚合器、阈值和不确定度已经由 ADR-0019 抽象冻结；物理字段、统计模型登记、单位词典和多重比较政策仍未冻结。
- 单根复合事态、`all_of / any_of` 四结果传播和决定性短路已经由 ADR-0020 抽象冻结；跨 Endem 条件激活由 ADR-0021 与 SYN-CORE 定义。单根条件节点、排他析取、复合否定、物理字段与规范字节仍未冻结。
- 满足结果和最终决定的语义已经由 ADR-0015 分开；结果事件编码、时间域、重放键、聚合规则与决定类 Iknem 格式仍未冻结。
- 自然语言中的并列不等于多根；ADR-0020 已用终态身份、方向、生命周期、权威和失败责任定义拆分判据。

这些缺口保持开放。代码阶段开启前，项目只补充案例、反例、规范文字和验证方案，不创建解析器或组件测试。
