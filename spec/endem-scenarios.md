# Endem Natural-Language Scenario Corpus

- 语料 ID：`END-SCEN`
- 版本：`0.1.0-draft`
- 日期：2026-07-13
- 状态：非规范性设计审查语料
- 用途：检验六个语义面、未知分类、根唯一性和验收边界是否能解释真实自然语言场景
- 实现状态：没有解析器、组件或符合性声明；本文件不是可执行测试

## 1. 使用边界

本语料把自然语言案例拆成预期的工程问题，不规定 Poiet 语法、`.endem` 字节或模型提示。案例中的结构只是审查目标：若现行规范无法不含糊地解释一个案例，应修订术语、ADR、条款或开放问题，不能让模型猜测后把结果写成规范事实。

本语料参考《逻辑哲学论》中事态由对象结合构成（2.01）、图像以要素关系描述可能情况（2.1–2.15）、图示形式由结构显示而非自我陈述（2.17–2.172）、名称与对象配置相对应（3.203–3.21）、理解命题需要知道其成立情况（4.024），以及判断真假之前必须先规定判定条件（4.063）。这些命题只提供问题结构，不直接推出软件字段或现实世界本体。

## 2. 场景矩阵

### SCN-001 — 使一个可观察事态成立

- **来源表达：**“让登记服务返回健康状态。”
- **预期分层：**`rhem` 保留原句与来源；`semion` 绑定“登记服务”“返回”“健康状态”；`skena` 表达 `returns(registry_service, healthy)`；`telis` 使用 `kine`。
- **判断边界：**`krin` 必须指定健康探测的关系位置、允许观察、必需 Tekmor 和决定权威。服务进程存在不等于返回健康状态。
- **审查结论：**支持一个根、正极性事态与达到成立方向。

### SCN-002 — 在明确区间保持事态成立

- **来源表达：**“在发布窗口内持续保持主站可用。”
- **预期分层：**`skena` 表达 `available(primary_site)`；`telis` 使用 `mene`，并引用已确定的发布窗口。
- **未决边界：**时区、窗口起止、允许中断和“可用”的测量口径若未确定，必须进入 `apor`，不能由当前时间或监控默认值补齐。
- **审查结论：**现行核心能说明方向分离，但 END-P1 尚未冻结时间区间与连续测量语义。

### SCN-003 — 禁止事项由否定事态表达

- **来源表达：**“不得让匿名用户读取私有报告。”
- **预期分层：**`skena` 表达否定关系 `not can_read(anonymous_user, private_report)`；`telis` 使用 `mene`，表示在适用范围内保持该否定事态成立。
- **判断边界：**`krin` 需要授权策略、实际访问观察和覆盖范围；没有访问日志不等于禁止事项已经满足。
- **审查结论：**不得新增 `avoid`、`forbid` 等第三种目标方向，也不得把否定只留在来源文本中。

### SCN-004 — 指称歧义必须保留

- **来源表达：**“把它发布给管理层。”
- **预期分层：**`rhem` 保存“它”的精确范围；`semion` 只能登记可能指向的报告、构建物或其他候选；`apor` 保存候选、影响关系、解决方式和决定主体。
- **失败边界：**输入没有任何允许候选时是 `aseme`；存在多个允许候选但无人有权选择时是 `apor`。模型最高概率候选不能成为确认指称。
- **审查结论：**nascent Endem 可以显式携带未决项，但不能伪装成已经可发布的 coherent 闭包。

### SCN-005 — 观察不足不同于未满足

- **来源表达：**“确认昨夜备份可以恢复。”
- **预期分层：**`skena` 表达 `recoverable(last_night_backup)`；`telis` 根据请求语境进入待确认方向；`krin` 要求一次受控恢复观察及其 Tekmor。
- **判断边界：**只看到备份文件存在时返回 `agno`，因为观察不足；恢复试验明确失败时才是 `unmet`。
- **审查结论：**`phain` 必须与“可恢复”关系对齐，文件存在、摘要有效和恢复成功是不同主张。

### SCN-006 — 求值故障不同于观察不足

- **来源表达：**沿用 SCN-005，但恢复验证器因存储服务故障无法运行。
- **预期分层：**事态、方向与判断契约不变；验证器、依赖或执行环境异常记录为 `fault`。
- **失败边界：**不得把工具故障写成备份不可恢复，也不得把故障隐藏为普通 `agno`。
- **审查结论：**`agno` 描述证据不足，`fault` 描述求值过程没有按契约完成。

### SCN-007 — 权威不足不能确认语义

- **来源表达：**“把响应时间控制在合理范围。”
- **预期分层：**“合理范围”需要具名权威、适用服务、统计窗口、分位数和阈值。模型或遥测平台可以提出候选，不能确认这些值。
- **失败边界：**没有任何允许的定义来源时返回 `aseme`；存在多个业务允许阈值但尚未决定时进入 `apor`。
- **审查结论：**类型正确、历史均值或行业惯例都不能替代语义授权。

### SCN-008 — 两个独立根必须拆分

- **来源表达：**“生成安全评审报告并部署修复后的服务。”
- **预期分层：**生成报告和部署服务可以独立版本化、失败和验收，因此必须形成两个 Endem；只有真实依赖需要解析时才由 Synem 表达关系。
- **失败边界：**不得用一个复合根掩盖两套权限、证据和失败责任，也不得仅因同一句出现“并”就机械拆分。
- **审查结论：**拆分依据是独立身份与生命周期，不是表面连接词。

### SCN-009 — 会话结束、满足判断与最终决定必须分开

- **来源表达：**“确认昨夜备份可以恢复。”恢复试验按计划完成，但实际数据损坏；安全负责人随后依据预注册政策作出否定决定。
- **预期分层：**Praxe 会话终止为 `completed`，因为测试过程正常结束；`krin` 求值为 `unmet`，因为有效恢复观察反驳可恢复事态；权威决定为 `rejected`，并绑定安全负责人、政策版本和原满足结果。
- **对照失败：**验证器崩溃时会话可以是 `failed`、满足结果是 `fault`、决定是 `deferred`。缺少恢复观察时满足结果是 `agno`；这些情况都不能写成备份不可恢复。
- **审查结论：**`completed` 不等于 `met` 或 `accepted`，`failed` 不等于 `unmet`，有效 Tekmor 也不自动表示证据覆盖充分。

### SCN-010 — 固定发布窗口必须先解析为确定区间

- **来源表达：**“北京时间 7 月 20 日凌晨 1 点到 2 点持续保持主站可用。”
- **预期分层：**具名发布负责人把年份、时区规则和两个时刻解析为确定 UTC 瞬间；`mene` 使用 `fixed` 与半开区间 `[start,end)`；连续性政策明确为 `strict` 或完整的 `budgeted`。
- **失败边界：**缺少年份、使用“本地时间”、时区数据库规则冲突或夏令时导致同一墙钟值有多个候选时进入 `apor`；Poiet 不能读取当前机器时区替用户选择。
- **审查结论：**来源中的民用时间保留在 `rhem/semion`，规范时间范围只使用已授权的确定瞬间。

### SCN-011 — 离散健康采样不能证明连续可用

- **来源表达：**沿用 SCN-010；监控只在窗口开始、中点和结束前各记录一次健康状态，中间没有覆盖保证。
- **预期分层：**三个健康点可以形成有效 Tekmor，但覆盖并集仍有空洞；严格持续目标的满足结果必须是 `agno`。
- **对照失败：**覆盖完整且观察到服务中断时为 `unmet`；监控时钟或适配器违反契约时为 `fault`。不得用插值、最后值延续或“没有告警”填补空洞。
- **审查结论：**观测采样是证据方法，不是连续性政策；记录有效与覆盖充分继续分开。

### SCN-012 — 经过时长必须使用具名事件与单调时钟

- **来源表达：**“部署完成后持续保持健康十分钟。”
- **预期分层：**`mene` 使用 `elapsed`，起点绑定具名部署完成事件，时长使用固定长度单位，Praxor 以声明的单调时钟测量并记录重启边界。
- **失败边界：**部署完成事件没有唯一身份时进入 `apor`；墙钟回拨不能缩短十分钟；单调时钟跨未声明重启或时钟读取失败时结果为 `fault`。
- **审查结论：**绝对时刻与经过时长属于不同时间域，外部 Agent 的完成时间戳不能自动成为本地起始事件或时钟权威。

### SCN-013 — 否定不创造第二套关系

- **来源表达：**“撤销匿名用户读取私有报告的能力。”
- **预期分层：**`semion` 只授权一个 `can_read(actor, resource)` 关系；`skena` 引用相同关系和角色，并把极性设为 `negative`；`telis` 使用 `kine` 使该负事态成立。
- **失败边界：**不得另造 `cannot_read`、`not_can_read` 或 `forbid` 谓词，不得删除关系节点表示否定，也不得交换 actor 与 resource 后声称语义等价。
- **审查结论：**否定改变同一可能事态的极性，不改变关系身份、角色位置或目标方向的职责。

### SCN-014 — 没有访问日志不等于没有访问

- **来源表达：**“发布窗口内没有匿名用户读取过私有报告。”审计系统只保存部分服务的成功访问日志，当前查询没有匹配记录。
- **预期分层：**查询结果只能形成“已搜索记录中未发现匹配项”的 Tekmor；由于服务路径、丢失量和时间覆盖不完整，负目标的满足结果为 `agno`。
- **对照失败：**若任何有效记录显示匿名读取，结果为 `unmet`；采集器违反声明契约时为 `fault`。不得把空结果、没有告警或模型未发现异常升级为 `met`。
- **审查结论：**记录缺席只描述数据集，不能自动描述现实事态不存在。

### SCN-015 — 封闭范围才允许从缺席推断

- **来源表达：**沿用 SCN-014，但具名审计权威证明所有读取决策都经过同一不可绕过入口，完整枚举发布窗口内全部路径，并记录零丢失、截止点和已知排除项。
- **预期分层：**Tekmor 固定被枚举主体、关系、角色路径、时间区间、方法、生产者和完整性计数；覆盖充分且没有正极性反例时，缺席可以在这个有限范围内支持 `met`。
- **失败边界：**任何旁路、未知丢失、迟到记录、过滤条件、未封闭主体集合或撤销的完整性声明都会使结果回到 `agno`；一个正极性反例足以产生 `unmet`。
- **审查结论：**封闭的是明确观察范围，不是整个世界；它不能把局部完整性推广为无限否定。

## 3. 跨场景不变量

1. `rhem` 保存来源事实，不把改写后的文本冒充原句。
2. `semion` 的每项确认投影都必须来自确定性规则或范围有限的具名决定。
3. `skena` 只表达对象关系及其极性，不携带希望、禁止、优先级或置信度。
4. `telis` 只说明达到成立或保持成立，不复制事态极性。
5. `krin` 在判断之前固定可接受观察、比较关系、证据范围和决定权威。
6. `apor`、`aseme`、`agno` 与 `fault` 分别表示未决选择、无允许意义、观察不足和求值故障，不能合并。
7. 外部模型、MCP 工具、A2A Agent、签名身份和遥测平台只能提供带来源的候选或观察，不能自行确认语义或验收。
8. 两个能够独立实现、失败或验收的根必须拆成两个 Endem；组合不是把多个目标塞回一个对象。
9. 制品生命周期、满足判断、权威决定、会话终止、Tekmor 有效性和证据覆盖度分别记录；外部协议状态不得跨域覆盖。
10. `mene` 只接受具名权威解析的 `fixed` 区间或绑定具名事件的 `elapsed` 时长；相对日期和默认时区不能进入规范语义。
11. 连续性使用 `strict` 或完整 `budgeted` 政策；遥测空洞不属于允许违约。
12. 观测覆盖空洞产生 `agno`，有效违约产生 `unmet`，时钟或求值契约故障产生 `fault`。
13. 否定作用于同一关系身份与角色位置，不能通过删除节点、改名谓词或新增目标方向表达。
14. 没有记录不是负极性 `phain`；开放或覆盖不足的观察只能产生 `agno`。
15. 从缺席推断负事实必须绑定具名权威声明的封闭观察范围、完整枚举方法、截止点、路径覆盖和丢失计数。

## 4. 当前暴露的规范缺口

- `mene` 的时间范围、连续性与覆盖分类已经由 ADR-0016 设计冻结；物理字段、闰秒、循环日历、跨重启时钟关联和多生产者归并仍未冻结。
- 否定关系、缺席证据和封闭观察范围已经由 ADR-0017 设计冻结；复合否定、量化否定和封闭声明的物理编码仍未冻结。
- 指称候选怎样参与尚未 coherent 的关系图仍需更精确的类型边界。
- `krin` 尚未冻结统计窗口、阈值、单位、容差和复合观察表达式。
- 满足结果和最终决定的语义已经由 ADR-0015 分开；结果事件编码、时间域、重放键、聚合规则与决定类 Tekmor 格式仍未冻结。
- 自然语言中的并列不等于多根，仍需以独立身份、权限、失败和验收责任定义拆分判据。

这些缺口保持开放。代码阶段开启前，项目只补充案例、反例、规范文字和验证方案，不创建解析器或组件测试。
