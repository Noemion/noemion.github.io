# Exact Content Identity and Attestation Natural-Language Design Scenarios

- 文档 ID：`ID-SCEN`
- 版本：`0.1.0-draft`
- 日期：2026-07-13
- 状态：非规范设计语料
- 适用范围：精确身份、不可变引用、签名陈述、验证材料、撤销、可复现性与伴随制品关系

这些场景不是签名格式、证书 Profile、密码实现、透明日志或组件测试。它们用支持案例、反例和边界场景检查 ID-CORE 能否解释现实发布问题；只有条款、登记验证和正反向量共同覆盖的内容才可能成为符合性要求。

### ID-SCN-001 — 同一摘要文本位于不同对象域

- **场景：**一项 Endem 引用和一项 evidence 引用偶然显示相同摘要文本。
- **场景结论：**对象种类、规范、表示、Profile 与身份域不同，因此不能合并为同一对象。

### ID-SCN-002 — Profile 变化建立新身份语境

- **场景：**相同载荷被解释为两个不同 Profile 的对象。
- **场景结论：**即使字节相同，完整身份语境也不同；不能沿用旧 Profile 的接受结果。

### ID-SCN-003 — 追加签名不能改变被签字节身份

- **场景：**发布者对一个 `.endem` 添加证书、签名和透明日志材料。
- **场景结论：**包络外置并拥有自己的身份；原 `.endem` 精确身份保持不变，摘要输入范围没有循环。

### ID-SCN-004 — closure 只按完整身份绑定成员

- **场景：**两个仓库都有名为 `policy` 的 Endem，显示版本相同但摘要不同。
- **场景结论：**名称与仓库只帮助找候选；闭包必须绑定唯一允许的完整身份。

### ID-SCN-005 — URL 和 build ID 不能选择执行主体

- **场景：**发布 URL 的内容被替换，但页面仍显示原 build ID。
- **场景结论：**contract 不能据此建立；必须重新取得并核对完整身份。

### ID-SCN-006 — 算法迁移不覆盖旧身份

- **场景：**政策停止接受旧摘要算法并登记新算法。
- **场景结论：**新摘要形成新的完整身份表达；旧材料保留历史状态，不能静默改写或降级。

### ID-SCN-007 — 短摘要只帮助人阅读

- **场景：**两个对象共享界面显示的十二位前缀。
- **场景结论：**界面必须扩展显示或要求完整值，不能按列表顺序、路径或缓存选择对象。

### ID-SCN-008 — 相同精确身份不证明目标满足

- **场景：**验证器确认取到的字节与发布引用完全一致。
- **场景结论：**这只证明对象匹配；内容接受、`met` 与最终决定仍走各自规则。

### ID-SCN-009 — 裁剪结果不能自称语义等价

- **场景：**未来派生处理删除调试和来源辅助信息后得到更小制品。
- **场景结论：**两个制品各有精确身份；在单独等价规范确定之前，只能声明有损变换关系。

### ID-SCN-010 — 发布签名覆盖用途和受众

- **场景：**发布责任人签署一份供公共发布审核使用的 Endem 陈述。
- **场景结论：**签名覆盖对象身份、陈述类型、发布用途、受众、主体与政策，不能转作 runner 能力授权。

### ID-SCN-011 — 载荷类型变化使签名失效

- **场景：**攻击者保持载荷字节不变，却把陈述类型从发布证明改为授权证明。
- **场景结论：**类型与上下文属于受保护输入，修改后验证必须失败。

### ID-SCN-012 — 离线包络包含有限验证材料

- **场景：**审核者在证书过期后离线验证历史发布签名。
- **场景结论：**包络需要证书、可信时间或日志材料和适用状态证据；`kid` 与签名值本身不足。

### ID-SCN-013 — 供应商签名不能决定事实正确

- **场景：**供应商签署“设备健康”的外部声明。
- **场景结论：**签名可支持来源绑定；声明是否适用、真实并足以满足 `satisfaction_criteria` 仍由 evidence、政策和决定权威判断。

### ID-SCN-014 — 撤销改变评估不改变历史字节

- **场景：**签名密钥在发布后被撤销。
- **场景结论：**对象身份、陈述字节和签名值不变；以撤销生效后的截止点评估时，有效性结论改变。

### ID-SCN-015 — 透明日志包含不是最终接受

- **场景：**签名事件拥有有效包含证明和集成时间。
- **场景结论：**它支持有限的公开记录与时间关系，不能单独证明主张真实、密钥未撤销或制品可接受。

### ID-SCN-016 — 两条独立路径逐字节复现

- **场景：**两个独立环境使用同一精确输入闭包、工具版本与封装参数生成输出。
- **场景结论：**只有两个输出完整身份相同且逐字节比较成功，才支持声明范围内的可复现性；相同 build ID 不足。

### ID-SCN-017 — 调试伴随文件独立绑定主制品

- **场景：**调试信息被分离为伴随文件。
- **场景结论：**主制品和调试文件各有身份，伴随关系精确指向双方并记录分离规则，文件名相同不构成绑定。

### ID-SCN-018 — 压缩与迁移不能继承信任

- **场景：**一个已签制品被压缩、重编码或迁移到另一表示。
- **场景结论：**派生物需要新身份和关系证明；来源签名、evidence、接受状态与能力不会自动转移。

## 当前开放项

摘要与签名算法基线、算法登记表、摘要文本语法、物理签名包络、证书与透明日志 Profile、撤销分发、Semantic Key、跨版本等价、调试伴随格式和长期归档尚未确定。当前只有案例、条款、威胁、提案向量和未来验证条件，没有身份或签名组件实现。
