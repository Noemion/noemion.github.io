# Endem Core Threat Model

- 适用规范：`END-CORE 0.1.0-draft`
- 版本：`0.1.0-draft`
- 日期：2026-07-13
- 状态：第一阶段威胁模型；随格式 Profile 和实现证据继续收敛

## 1. 保护目标

第一阶段保护以下属性，而不是保护某个尚不存在的程序：

1. 来源范围不会被越界引用或错误重放；
2. 确认语义不会被模型候选、默认值或未授权输入替换；
3. 一个根 `skena`、事态中性和六语义面分层不会被畸形结构绕过；
4. 解析失败不会产生部分可信 Endem；
5. 资源耗尽、整数回绕和未知关键结构会稳定关闭失败；
6. 精确摘要、语义等价、签名和 Iknem 不会被互相冒充；
7. 生产路径与独立 Theor 不会共享同一解析故障根；
8. 外部任务完成、会话结束、证据有效与最终接受不会被互相冒充；
9. 默认时区、墙钟回拨、相对日期与遥测空洞不会把持续目标洗白；
10. 空日志、部分搜索、单路径拒绝或模型“未发现”不会被洗白为负事实；
11. 有利子集、重复见证、空集合和变化中的成员范围不会被洗白为量化满足；
12. 代理构念、样本选择、窗口、单位、聚合、舍入或不确定度不会被操纵为有利阈值结果。

保密性只覆盖来源、调试记录和未来运行证据的披露边界。第一阶段不把加密存储或机密计算纳入可信核心。

## 2. 攻击者与信任边界

系统把所有对象字节、来源文件、模型输出、远端协议描述、工具返回、缓存、路径名和构建元数据视为不可信。攻击者可以控制这些输入、输入顺序、大小、重复内容、编码、图结构和扩展记录，也可以重复提交旧输入。

攻击者不能直接修改已审查的规范、当前 Profile、Ktisor/Theor 可执行文件或外部签名私钥。供应链替换、构建机入侵和密钥泄漏属于后续发行威胁模型，但本规范仍要求通过摘要、复现和权限隔离为这些审查保留边界。

## 3. 威胁清单

### THR-END-001 — 结构算术回绕

攻击者构造偏移、长度、计数、索引、对齐或解压大小，使中间计算回绕并落入看似合法的范围。读取器必须通过 `END-SAF-001` 的受检算术和 `END-ERR-001` 的原子失败阻断。验证计划是边界向量、性质测试、未定义行为检查和覆盖引导模糊测试。

### THR-END-002 — 资源放大

攻击者用巨大计数、深层组合、高扇出图、重复字符串或压缩载荷迫使读取器在确认输入合法前分配资源。`END-SAF-002` 要求 Profile 提供有限上限并在分配、递归或解压前检查。END-P0 已为第一轮实验给出保守数值，但这些数值不是生产规模证明，也不得被环境静默提高。

### THR-END-003 — 未知关键结构降级

攻击者把影响语义、权限、状态或完整性的内容放进读取器不认识的记录，并诱使实现按可选扩展跳过。`END-SAF-003` 要求未知关键结构关闭失败；只有已登记为可跳过且不改变规范含义的记录才能忽略。

### THR-END-004 — 歧义清洗

模型或调用者只提交最高置信解释，隐藏其他允许投影，或把观察不足改写成语义已确认。`END-APR-001`、`END-UNK-001` 与 `END-AUT-001` 要求保留 `apor` 并区分 `aseme`、`agno` 和 `fault`。`SV-REJECT-APOR-001` 与 `SV-REJECT-AUTHORITY-001` 提供首批回归输入。

### THR-END-005 — 载荷自证

攻击者设置 `valid`、`true`、`logical_form`、模型评分或自由文本结论，诱使读取器跳过关系结构验证。`END-KRN-001`、`END-KRN-002` 与 `END-STR-001` 要求满足判断来自可对齐的关系位置和有范围 Iknem，并区分反驳、观察不足与求值故障，而不是相信载荷自述。

### THR-END-006 — 身份域混淆

攻击者使用名称、短校验值、构建 ID、签名存在或旧 Iknem 代替当前完整字节摘要，或用相同摘要声称两个不同布局语义等价。`END-ID-001` 分离精确摘要、语义等价键、签名包络与证据链接；Semantic Key 在等价规则冻结前必须禁用。

### THR-END-007 — 解析器单一故障根

Ktisor 写入器、生产读取器与查看器共享解析、生成代码或错误分类，使同一错误在所有路径中得到一致但错误的解释。`END-VER-001` 要求 Theor 独立读取；后续通过依赖检查、变异测试和故障注入验证，而不是仅比较两个命令名。

### THR-END-008 — 状态或签名洗白

攻击者把 `coherent`、`attested`、签名数量、Iknem 数量、外部 Task 的 `completed` 或本地会话结束解释为语义正确、目标满足或最终接受。`END-STA-001`、`END-STA-002` 与 `END-DEC-001` 要求每个结果域只支持有限推导，Drasor 仍需重新验证实际字节、策略、当前能力、满足结果和决定权威。

### THR-END-009 — 环境依赖劫持

攻击者通过当前目录、搜索路径、网络“最新版本”、区域设置或未记录默认值改变同一正式输入的解析结果。`END-CON-001` 与 `END-DET-001` 要求固定规范、Profile、依赖与环境输入，并在两套隔离环境比较结果。

### THR-END-010 — 诊断失去可追溯性

实现只返回自由文本错误、错误位置受攻击者控制，或在失败后暴露可继续使用的部分状态。`END-ERR-001` 要求稳定错误类别、主条款 ID、受限位置和失败原子性；诊断字符串本身不得进入信任判断。

### THR-END-011 — 时间与覆盖洗白

攻击者或失配适配器使用当前机器时区、相对日期、可回拨墙钟、稀疏采样、遥测空洞、重叠窗口或远端 Agent 时间戳，使未覆盖的持续目标看似成立。`END-TIM-001` 固定时间范围和时钟权威，`END-TIM-002` 分离违约预算与观测缺口，`END-TIM-003` 要求空洞为 `agno`、时钟契约故障为 `fault`。未来验证必须覆盖夏令时歧义、墙钟回拨、跨重启、窗口边界、缺口、重叠和预算超限。

### THR-END-012 — 缺席与否定洗白

攻击者、模型或不完整适配器删除关系节点、改写谓词、交换角色，或用空查询、部分日志、单路径拒绝、未知丢失和外部 Agent 的“未发现”声称负目标已经满足。`END-NEG-001` 固定同一关系与角色上的显式极性，`END-NEG-002` 分离负观察与记录缺失，`END-NEG-003` 只允许在具名封闭观察范围内从缺席推断。未来验证必须覆盖旁路、过滤、迟到、去重、丢失计数、范围撤销、角色错位和正极性反例。

### THR-END-013 — 量化范围与计数洗白

攻击者、模型或查询适配器选择有利子集、重复同一成员或见证、把日志行数当成员数、利用默认空集合，或在 `find` 式搜索的起点、裁剪、提前退出和并发变化未封闭时声称完成了全称或基数判断。`END-QNT-001` 固定集合、成员权威、截止点和身份规则，`END-QNT-002` 固定量词、闭包与空集合政策，`END-QNT-003` 只按不同成员身份聚合并保留成员级结果。未来验证必须覆盖重复身份、成员漂移、不完整枚举、决定性短路、空集合和故障优先级。

### THR-END-014 — 测量与阈值洗白

攻击者、模型、基准运行器或遥测管线把代理任务冒充目标构念，从固定题集推广普遍能力，事后选择有利样本、时间窗口、平均值或分位数，排除失败记录，混用单位与分桶边界，比较前舍入，隐藏误差区间，或把模型自报置信度当作统计不确定度。`END-MSR-001` 固定构念与总体，`END-MSR-002` 固定数值和单位，`END-MSR-003` 固定可重放程序，`END-MSR-004` 要求区间相对阈值决定结果。未来验证必须覆盖单位冲突、非线性换算、样本不足、重复试验、窗口漂移、量化误差、缺失值、固定与泛化 estimand 以及多阈值选择。

### THR-END-015 — 复合判断洗白

攻击者、模型或策略适配器把多个独立目标塞进一个根以共享权限或验收，把一个不可分终态拆开以绕过整体判据，嵌入任意布尔脚本，利用重复叶、环或求值顺序改变结果，把 `agno` 或 `fault` 当作假，或在短路后删除故障与未求值分支。`END-CMP-001` 固定 Endem 与 Synem 边界，`END-CMP-002` 固定有限无环结构和叶对齐，`END-CMP-003` 固定四结果矩阵，`END-CMP-004` 保存决定依据与求值覆盖。未来验证必须覆盖拓扑错配、重复引用、环、全排列结果一致性、决定性与非决定性短路以及故障保留。

### THR-END-016 — 权威名称与伴随决定洗白

攻击者在 END-P1 写入可信外观的 `projection.id`、`decision_authority` 或 `required_iknem`，再把名称命中、缓存、环境角色或另一对象的授权决定冒充当前投影已经获准；也可能通过改变授权状态重新计算 Endem 内容身份，破坏引用稳定性。`END-CON-006` 要求显式报告外部前置条件，`END-AUT-002` 禁止名称自证授权，`END-ID-002` 分开内容身份与决定，`END-FMT-015` 把 P1 字段限制为待核对选择器。未来验证必须覆盖缺失、错绑、过期、撤销、跨对象重放与同字节多授权关系。

## 4. Profile 必须给出的有限上限

任何读取 Profile 在允许实现解析不可信字节前，都必须给出以下有限、非零上限：

| 上限标识 | 约束对象 | 检查时机 | 当前数值状态 |
| --- | --- | --- | --- |
| `max_artifact_bytes` | 输入制品总字节 | 读取内容前 | END-P0 / END-P1：16 MiB |
| `max_record_count` | 物理记录数量 | 目录展开前 | END-P0：64；END-P1：6 |
| `max_graph_nodes` | 语义与事态节点；结构读取时保守计全部 CBOR 数据项 | 按声明数量循环或图分配前 | END-P0 / END-P1：整份制品累计 65,536 |
| `max_graph_edges` | 关系与组合边；结构读取时保守计数组元素数和映射 pair 数 | 按声明数量循环或图分配前 | END-P0 / END-P1：任一容器及整份制品累计 262,144 |
| `max_nesting_depth` | 组合、引用与递归结构 | 递归下降前 | END-P0：64；END-P1：16 |
| `max_string_bytes` | 单个字符串或标识 | 分配与解码前 | END-P0 / END-P1：1 MiB |
| `max_decompressed_bytes` | 解压后总大小 | 解压开始前 | END-P0 / END-P1：16 MiB；两者都禁止压缩 |
| `max_total_allocation_bytes` | 单次整份制品解析的累计分配 | 每次分配前；跨记录不清零 | END-P0 / END-P1：64 MiB |

没有完整上限集合的 Profile 无效。将上限设为实现整数最大值不等于给出了安全预算。END-P0 数值只用于结构实验；END-P1 在此基础上把记录数收紧为 6、嵌套深度收紧为 16。后续提高必须采用新的 Profile 身份，并用目标消费者、测试语料、最坏复杂度、32/64 位平台差异和停止建设条件支持。

## 5. 明确不在第一阶段解决

- 恶意但已授权的决定主体；
- 运行期能力沙箱逃逸；
- 远端 MCP/A2A 服务、模型供应商或网络身份欺骗；
- 签名私钥泄漏、透明日志和撤销分发；
- 侧信道、机密计算和多租户隔离；
- 超出 SYN-CORE 抽象有限性要求的大规模 Synem 数值预算与性能拒绝服务。

这些问题分别属于 Drasor、协议适配、发行供应链和后续 Synem Profile。当前 `spec/synem-threat-model.md` 已覆盖闭包截断、循环和资源放大类别，但尚未冻结大规模数值上限。不得为了完整感把它们塞进最小 Endem 读取核心。
