# Authority and Authorization Decision Threat Model

- 文档 ID：`AUT-THREAT`
- 版本：`0.1.0-draft`
- 日期：2026-07-13
- 状态：权威、授权决定、委托、同意和能力边界威胁草案

自然语言投影、Agent 委托和工具权限都可能在“已经登录”“用户点过允许”或“外部协议返回成功”的表象下越权。本模型只定义未来实现必须关闭的威胁，不表示权威目录、政策求值器、同意界面、能力代理或决定服务已经实现。

### THR-AUT-001 — 可变政策与语境漂移改写旧授权

攻击者让旧请求在新的 `latest` 政策、租户、目的或对象下继续使用。`AUT-CTX-001` 要求完整权威语境和实质变化失效。

### THR-AUT-002 — 认证成功冒充决定权限

任何已登录用户、签名者、服务账户或自报 Agent 被当作有权确认意义、授予能力或验收制品。`AUT-PRN-001` 分开主体、行动者、角色资格和当前决定权。

### THR-AUT-003 — 通配符、前缀与环境权限扩大范围

宽泛角色、资源前缀、JSON 包含或操作系统环境权限被解释为所有字段和动作均获授权。`AUT-SCP-001` 要求封闭范围和默认不授予。

### THR-AUT-004 — 模型候选与批量界面洗白语义

模型最高概率、结构合法、截断显示或一个“全部接受”按钮消除多个高风险 `apor`。`AUT-SEM-001` 要求来源、候选、语义位置、显示视图和具名权威精确绑定。

### THR-AUT-005 — 模糊或部分决定被解释为完整允许

系统只保存“已批准”、把部分范围当作原请求，或把缺少决定的 `defer` 当作允许。`AUT-DEC-001` 固定请求、决定结果、限制和缩小后的精确范围。

### THR-AUT-006 — 委托、冒充与循环隐藏实际行动者

子 Agent 获得父级没有的权限，中间服务冒充用户，或委托链通过循环和续期无限延伸。`AUT-DEL-001` 要求保留 subject/actor、有限无环链和逐级收窄。

### THR-AUT-007 — 重复身份与顺序操纵多人决定

同一控制主体使用多个账户满足法定人数，重复签名增加票数，或最后一票覆盖预注册否决。`AUT-MUL-001` 要求不同主体规则、门槛、否决和冲突政策预先固定。

### THR-AUT-008 — 显示不一致与暗黑模式伪造同意

界面隐藏写权限、接收方或期限，用默认选中、捆绑权限或不可达拒绝路径取得点击。`AUT-CNS-001` 绑定完整安全视图、机器对象、目的和撤销后果。

### THR-AUT-009 — 过期缓存与撤销失明延长授权

一次批准永久复用，角色或政策已经变化，离线缓存仍忽略撤销与状态新鲜度。`AUT-TIM-001` 要求截止点、适用期、状态来源和追加式历史。

### THR-AUT-010 — 决定跨对象、目的或会话重放

发布批准被用于能力授予，旧版本 Endem 的点击被用于新版本，或重试换 ID 重复外部副作用。`AUT-RPL-001` 要求请求身份、精确对象、用途、使用次数和去重规则共同绑定。

### THR-AUT-011 — 协议 scope、凭据与环境能力被并集放大

MCP 挑战、Agent 技能、token 存在、root 环境或子 Agent 预算被当作新的授权来源。`AUT-CAP-001` 只允许所有本地上限的交集，并要求 step-up 建立新会话。

### THR-AUT-012 — 授权结果洗白真值、证据与验收

访问 `grant`、人工确认、签名有效或外部 `AUTH_REQUIRED` 已解决被直接映射为 `met`、`sufficient`、`accepted` 或 `completed`。`AUT-SEP-001` 保持所有结果域和权威主体正交。

## 当前边界

当前威胁只映射规范条款与提案向量。身份提供方、政策引擎、法定人数服务、同意界面、撤销分发、令牌交换、能力代理、平台权限和真实副作用测试，须在项目进入代码开发阶段并明确实现范围后验证。
