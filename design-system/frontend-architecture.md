# 前端模块架构

本文件定义全站共享前端代码的职责、接口和加载边界。`sitemap.md` 仍是正式 HTML 路由的唯一注册表；本文件不建立第二套路由权威源。

## 分层与职责

| 层 | 权威输入 | 产出 | 失败责任 |
| --- | --- | --- | --- |
| Jekyll 内容层 | 页面 Front Matter、`sitemap.md`、`_data/navigation.yml`、`_data/manuals.yml` | 可直接阅读、可无脚本导航的 HTML | 构建或质量测试失败 |
| 路由模型 | 当前 URL 与站点根 URL | 规范化路由、所属模块、当前项判断 | 返回保守的项目模块，不改写页面 |
| 导航数据仓库 | 构建生成的 `/assets/navigation-data.json` | 一次获取并缓存的不可变导航模型 | 保留已输出的一级链接并记录错误 |
| 同步移动目录守卫 | 页面根、1000px 以下断点、原生 `details`、透明遮罩与目录滚动容器 | 首次打开保护、背景手势冻结、滚轮与触摸边界约束 | 统一覆盖手机横竖屏，不改变正文定位并阻止菜单手势穿透 |
| 移动页头布局控制器 | 页头可用宽度、品牌与行动入口的固有尺寸、字体和视口变化 | 在第一行与面包屑行之间选择导航按钮位置 | 无脚本时保留可用的第一行入口，不遮挡或缩小触控目标 |
| 摘要栏布局控制器 | 容器可用宽度、摘要栏需求、主内容最低可读宽度、字体和动态视口 | 在粘性双栏与自然单栏之间选择并持续重算 | 无脚本或增强失败时保留完整的单栏阅读顺序 |
| 共享布局观察器 | 需要测量的容器、字体就绪状态与可选媒体状态 | 合并尺寸变化并在同一动画帧触发布局重算 | 不支持 `ResizeObserver` 时退回窗口尺寸监听，仍保留静态布局 |
| 交互控制器 | 已存在的语义 HTML 与导航模型 | 即时启动的展开、关闭动画与焦点状态 | 降级为普通链接和原生 `details` |
| 正文增强 | 已完成的正文 DOM | 表格滚动容器或长文目录 | 不改变正文内容与可读性 |

## 接口规则

- URL 规范化、精确当前页判断和模块选择是无状态纯逻辑，不放进有状态 UI 类；`aria-current="page"` 只能使用精确当前页判断。
- `GlobalNavigation`、`DirectoryNavigation` 与 `MobileDirectoryController` 只拥有各自交互状态，不读取页面正文或修改路由数据。移动目录、移动页头和桌面导航复用入口提供的同一个紧凑布局媒体状态。`MobileHeaderLayout` 只测量共享页头；`SummaryRailLayout` 只测量声明了该布局的容器及 CSS 需求；两者通过 `LayoutObserver` 复用尺寸观察、字体就绪、媒体变化、帧合并和降级逻辑，不能各自复制观察循环，也不能把单个视口测量写回固定断点。
- `NavigationStore` 是导航 JSON 的唯一读取接口；同一页面中的多个消费者复用同一个 Promise，不重复请求。
- 手册桌面目录由 `_includes/docs-rail.html` 在构建期生成。移动目录从这份已存在的语义 DOM 建立只读模型，不再向每页嵌入第二份手册 JSON。
- 页面 HTML 只负责其主题、页面角色和正文结构；全局外壳、导航和增强行为不得复制进具体页面。

## 渐进式披露与加载

1. 首屏 HTML 直接包含品牌、五个一级任务入口、`TIMELINE` 入口、手册目录和无脚本站点地图入口。
2. 1000px 以下由共享入口在页面启动时并行加载路由模型与纯文字移动目录；普通页面同时读取导航数据，手册页直接复用已输出的文档栏。首次点击通常只切换已建立的控制器，同步守卫仍覆盖脚本尚未就绪的极短窗口。
3. 1000px 及以上不预建隐藏的移动目录，也不预取导航 JSON。顶部二级卡片在第一次精确指针悬停、带明确 Tab 意图的键盘聚焦，或无悬停粗指针平板第一次主动点击一级入口时，才并行加载导航数据、控制器和专属 SVG 封面；触摸产生的焦点本身不能提前触发该路径。布局跨过 1000px 进入移动端时再准备文字目录。移动端不导入桌面导航控制器，也不请求或合成封面资源；横屏平板只为实际打开、即将可见的富菜单加载封面。
4. 只有页面声明摘要栏布局时才加载摘要栏控制器；只有存在手册表格，或未使用摘要栏且至少有六个二级章节时，才加载正文增强模块。
5. 任一增强失败都不得移除正文、一级链接、手册链接或浏览器原生 `details` 行为。
6. 入口、动态模块、模块间共享依赖和导航 JSON 使用同一构建版本参数；不得让入口更新后继续命中旧的传递依赖缓存。

## 删除与合并条件

- 没有独立状态或第二个消费者的逻辑不建立类。
- 没有正式页面使用的目录分组从 `_data/navigation.yml` 删除，不保留占位配置。
- 新模块只有在能独立加载、独立测试或被多个消费者复用时才建立；否则合并到最近的职责所有者。
- 正式路由变化先修改 `sitemap.md` 和页面源，再同步导航数据；导航数据不能反向成为路由权威源。
