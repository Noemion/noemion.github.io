# 门户页设计

适用范围：`page_role: portal`，当前为根 `index.html`。

## 阅读任务

门户页负责让首次访问者依次理解现实缺口、项目结论、对象工具链、当前成熟度和下一步入口。它是叙事页面，不承担规范字段定义或完整手册职责。

## 页面节奏

1. 左右分栏首屏：左侧主张与状态，右侧语义频谱和对象记录。
2. 双栏核心论点：强结论与解释分离。
3. 居中章节过渡：用短句连接问题与解决路径。
4. 横向阅读入口：图形、标题、摘要和箭头形成稳定扫描节奏。
5. 三列设计焦点：只展示真实成熟度和当前工作。
6. 生命周期、证据门、名称边界与下一步入口依次收束。

## 视觉元素

- 首屏标题桌面约 48–68px，字重对比明确，重点短语可使用朱橙下划线。
- 首屏之后的九个同级章节标题统一去除逗号和句末句号，以标题左右各一段的双色细横线表达章节停顿；单行和双行标题使用同一结构，移动端只缩短横线，不退回标点装饰。
- 右侧对象图必须表达 Source、Compiler Core、GSIR/GOBJ、链接或装载关系。
- 首屏图形左侧使用“底层数据流场”而不是无语义的色带：`SOURCE`、`BIND`、`RELOC`、`VERIFY` 四条轨道以不同速度把数据包推向确定性核心，轨道末端的状态读数显示 `DATAFLOW / ACTIVE`。渐变只用于表现数据密度、类型和校验状态，不能暗示真实运行遥测或已经实现的编译器。
- “表达方式，不等于语义身份”卡片左侧缩略图中的可见标签统一使用简洁英文；此规则不扩展到首屏主图或页面中文叙事正文。
- 语义频谱使用低幅度位移、旋转与缩放形成可感知但克制的缓慢漂移；背景网格缓移，数据流场通过轨道高光和数据包位移表达输入收敛，Compiler Core 连接线流动、虚线轨道缓慢旋转，对象面板只做约 10px 轻浮动。所有循环在减少动态效果时停止并保留静态轨道结构。
- 560px 以下首屏图形区不得沿用桌面叠放坐标：Source 标签位于上部，Compiler Core 位于中部，GOBJ 从约 272px 处开始，三者保持清晰的纵向处理顺序；不得让 `AMBIGUITY` 遮挡 GOBJ 标题或对象记录。
- 横向入口和治理网格使用细分隔线，不使用无信息价值的大圆角卡片。
- “四条路径理解系统”的全部横向入口在悬停或键盘聚焦时使用 5px Noemion 四色频谱描边、缩略图轻微放大、标题右移和箭头交叉切换。频谱框以首尾等价的 0–360° 角度渐变持续循环，不能用背景位置回跳造成每轮卡顿。右侧圆形箭头在明暗主题中都必须保留清晰静态圆环；交互时从 12 点方向以约 550ms 顺时针补画较粗圆环。文字和缩略图不能被动效遮盖，卡片布局不发生尺寸变化。

## 内容边界

- 第一屏必须明确当前仍处于设计与安全核心阶段。
- 不显示虚构下载、客户、背书、性能或兼容性结果。
- 哲学名称只说明问题框架和工程类比，不能替代技术定义。
- 更详细的首页内容映射见 `homepage-design.md`。
