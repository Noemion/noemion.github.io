# 内部工具视觉签名

本文件为每个 `tools/<tool>/index.html` 定义独立视觉方向。共享布局、六章语义结构、可访问性和动效仍以 `tool-project.md` 为准；这里的差异只用于帮助读者快速识别工具职责，不能暗示实现成熟度。

工具路径由默认布局自动写入 `body[data-tool-id]`。共享 CSS 使用工具 ID 选择对应的色彩、签名文字、网格角度和焦点位置；不得在单个 HTML 页面内添加样式。

### noemconform

- 签名：`SPEC / CONFORM`
- 色彩：规范绿、系统蓝、合规黄绿。
- 母题：垂直校准线和一致性检查点。
- 内容强调：规范条款、Golden、Malformed 与跨实现一致性。

### noemobj

- 签名：`OBJECT / INSPECT`
- 色彩：检视青、对象蓝、偏移橙。
- 母题：对象记录、字段窗口和偏移切片。
- 内容强调：Header、Section、Symbol、Relocation 与只读安全。

### noemverify

- 内容强调：结构、语义、策略、覆盖和信任结论分离。

## 共同行为

- 工具签名只替换 Hero 对象面板文字，不替代页面 H1。
- 工具色彩用于 Hero、章节短线、状态面板和有限悬停反馈，不改变正文对比度。
- 手册页面只继承当前工具的强调色；侧栏宽度、Markdown 排版和动态目录保持统一。
- 所有工具继续使用 0.96 按压反馈、具体属性过渡和减少动态效果支持。
