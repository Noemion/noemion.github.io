# Endem 应用视觉签名

本文件为唯一公开应用 `endem` 及其八个动作定义视觉方向。共享布局、六章语义结构、可访问性和动效仍以 `tool-project.md` 为准；动作差异只帮助读者辨认当前任务、产物和权限边界，不能暗示已经实现，也不能把动作包装为独立产品。

正式应用入口是 `/endem/index.html`，手册位于 `/endem/docs/`。页面可以由默认布局写入 `body[data-endem-action]`；共享 CSS 根据动作选择有限的色彩、签名文字、网格角度和焦点位置。不得在单个 HTML 或 Markdown 页面内添加样式。

## 共同制品语言

- Endem 记录面板固定以 `rhem / semion / skena / telis / krin / apor` 六行作为识别骨架；`semion` 与 `skena` 必须通过共享符号和关系位置显示投影，不能只画六条无关联文字。运行证据用 `phain ↔ skena` 显示结构比较，`telis` 独立显示目标方向。
- Synem 使用明确的引用边和闭包外框，不能画成会在运行时无限扩张的网络。
- Dromen 使用有边界的装载区域、能力票据和预算刻度，不能暗示全局权限。
- Tekmor 使用“主体 + 范围 + 方法 + 结果 + 限制”五部分标记，不能只显示通过图标。
- Noemion 只出现在品牌导航和项目归属，不进入动作签名、格式标签或对象类型。

## 动作签名

### poie

- 签名：`SOURCE / POIE`
- 色彩：来源薄荷、成形蓝、诊断橙。
- 母题：来源片段沿稳定顺序进入投影语义骨架，再形成单个 Endem 边界。
- 内容强调：来源定位、确定性投影、一个根 `skena`、`apor` 保留与 Poiet 的字节生成权。
- 不得暗示：模型直接生成 `.endem`、自由文本天然有效或来源推断等于原文。

### elenk

- 签名：`STRUCTURE / ELENK`
- 色彩：检查绿、规则紫、拒绝红。
- 母题：从边界、字段、引用、语义到策略的分层检查线。
- 内容强调：稳定错误类别、精确定位、checked arithmetic、资源上限和失败即停止。
- 不得暗示：检查通过等于目标已经实现或最终验收通过。

### pleko

- 签名：`REFERENCE / PLEKO`
- 色彩：链接深蓝、冲突红、闭包薄荷。
- 母题：多个 Endem 的显式引用在固定顺序中解析为 Synem 外框。
- 内容强调：内容身份、必需/可选关系、冲突拒绝、闭包完整性与确定性 Link Map。
- 不得暗示：环境搜索、弱优先级、延迟碰运气或模型选择依赖。

### tasse

- 签名：`SYNEM / TASSE`
- 色彩：封装紫、清单靛、摘要金。
- 母题：已解析 Synem 进入规范容器，清单、主体摘要和可剥离视图保持分层。
- 内容强调：规范排序、精确摘要域、可复现字节、发布材料和调试伴随信息边界。
- 不得暗示：封装自动建立发布者信任或签名属于语义身份。

### sphra

- 签名：`IDENTITY / SPHRA`
- 色彩：信任金、包络紫、撤销红。
- 母题：确定性产物保持不动，外部信任包络在其摘要之外闭合。
- 内容强调：签名者身份、密钥权限、算法、时间、撤销与多签策略。
- 不得暗示：签名修复无效产物、时间进入 Endem 身份或 Poiet 持有发布密钥。

### theor

- 签名：`BYTES / THEOR`
- 色彩：检视青、偏移橙、只读蓝。
- 母题：独立观察窗直接切开不可信字节，显示字段、引用、闭包和 Tekmor 范围。
- 内容强调：Theor 与 Poiet 代码独立、只读权限、直接解析、边界检查和差分验证。
- 不得暗示：Theor 调用生产解析器、修复输入、写回文件、访问密钥或启动 Praxor。

### praxe

- 签名：`DROMEN / PRAXE`
- 色彩：运行蓝、能力绿、拒绝红。
- 母题：Synem 依次通过检查、策略和能力门进入有边界的 Dromen，事件向 Tekmor 流出。
- 内容强调：Praxor 独立权限域、最小能力、预算、可观察事件、人工升级和不可修改已封装制品。
- 不得暗示：通用 shell、无限会话、模型自扩权或 Praxor 自行宣告验收。

### peira

- 签名：`VECTOR / PEIRA`
- 色彩：一致性绿、Malformed 红、差分青。
- 母题：Golden 与 Malformed 向量同时进入 Poiet、Theor 和 Praxor 的独立测试路径。
- 内容强调：字节复现、跨实现一致性、模糊测试、权限负向测试、双环境构建和回归定位。
- 不得暗示：只测试成功路径、Theor 与 Poiet 使用同一实现，或模型评语替代可重复断言。

## 权限域视觉规则

1. Poiet、Theor、Praxor 和密钥域使用不同边界线，不得放在同一无边界容器中。
2. `poie/elenk/pleko/tasse` 可以共享 Poiet 色系，但必须显示不同输入和产物。
3. `sphra` 的密钥域只接受确定性产物摘要，不能画出反向修改 Endem/Synem 的箭头。
4. `theor` 只能从制品指向只读视图，不能存在回写箭头。
5. `praxe` 只能从已检查 Synem 建立 Dromen；能力从策略域单向授予并可撤销。
6. Tekmor 从具体检查或运行事件产生，范围标记始终可见；不得使用一个全局“Verified”覆盖所有结论。

## 共同行为

- 动作签名只替换应用引言中的制品面板文字，不替代页面 H1，也不改变唯一应用名 Endem。
- 动作色彩用于应用引言、章节短线、状态面板和有限悬停反馈，不改变正文对比度。
- 手册页面只继承当前任务的强调色；侧栏宽度、Markdown 排版和动态目录保持统一。
- 所有交互继续使用 0.96 按压反馈、具体属性过渡和减少动态效果支持。
- 页面或目录不得出现并列工具卡片；需要导航八个动作时使用同一应用内的任务列表或手册目录。
- 视觉状态必须区分“规范已冻结”“实现中”“可运行”“有 Tekmor”和“验收通过”，不能把其中任意两个状态合并。
