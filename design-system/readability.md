# 长内容可读性

适用范围：技术说明、规范解读、架构文章、Endem 应用介绍和手册正文。页面所属角色的设计文档继续决定内容结构；本文件只定义跨角色的阅读尺度、信息栏职责和版式边界。

## 首要原则

- 哲学准确性优先，可读性紧随其后。思想来源、工程类比、规范要求和验证事实必须分层表达；读者不需要先接受某种哲学解释，才能理解页面的工程结论。
- 每页的标题、导语和第一节必须共同回答三个问题：这里讨论什么、直接结论是什么、这与读者有什么关系。目录页可以简短，专题页不得用专名清单代替结论。
- 新造词第一次出现时采用“直白中文定义（专名）”或“专名，即直白中文定义”的形式。状态词、策略名和英文接口名同样先说明职责；不能连续堆出三个以上未解释的英文名。
- 阅读路径优先于版式变化。连续论证不得为了填满画布被拆成无上下文的卡片，也不得为了视觉对称被拉成超长文本行。
- 1200px 是页面画布，不是段落宽度。普通正文保持约 700–760px 的舒适阅读列；中文正文不得长期占满 1000px 以上宽度。
- 桌面正文基线为 17–18px、约 1.75–1.8 行高。长篇技术页的二级标题以 30–34px 为主，避免标题远大于正文却没有形成更清楚的层级。
- 当前共享实现以桌面 18px / 1.78 为长正文默认值；应用状态栏、导航、徽标、表格元数据等辅助界面可以保持 13–16px，但不能反向缩小主阅读列正文。
- 每段只承担一个主要论点。定义、原因、机制、限制和例子按自然论证顺序连续出现；只有精确映射、比较、状态和短步骤才改用表格、卡片或横向行。
- 一个句子只推进一个主要判断。包含三个以上并列动作、条件、例外或状态时优先拆句；必须保持一一对应关系时改用列表或表格。
- 首次出现的缩写或项目术语先给直白解释，再给精确定义；提高可读性不能删除规范性强度、不变量、失败语义、成熟度或权威来源。
- 规范解释页的徽标只承担版本、当前策略、待定内容和限制条件，不把条款数、向量数或“没有某组件”的英文库存标签当作首屏摘要。章节标题回答开发者的问题或说明规则职责，不用“十条规则”“十二条条款”等会随规范演进漂移的数量组织阅读；精确范围以条款源、登记和验证结果为准。
- 规范解释页只连续展开当前对象的定义、案例、判断顺序、失败结果与现行条款。模型评测、训练、因果、恢复、删除等相邻研究若没有改变当前条款，应按开发者问题路由到各自资料，不在同一页面复制完整论证；研究链接必须同时说明当前结论上限。
- 中文入口页和专题页的徽标必须直接说明读者任务、工程边界或项目状态，不使用 `Why`、`Scope`、`Evidence`、`Design Stage` 等通用英文模板词制造分类感。CORE、Profile、协议名、版本和不可替代的正式标识可以保留；保留它们时，首屏导语仍须先给直白职责。
- 页面根语言保持 `zh-CN`。一段不可替代的外语自然语言需要原文呈现时，在最小准确片段上声明对应 `lang`；可准确翻译的状态和说明优先直接使用中文。专有名词、规范标识、代码、缩写和已经进入中文开发语境的技术词不机械添加语言切换，也不为视觉统一翻译机器标识。
- `lang` 只帮助浏览器、屏幕阅读器和其他辅助技术选择语言与发音规则，不能证明 Noemion 专名在真实会议中的首次朗读、听辨和回写表现。网页语言可访问性检查与发行名称的人类读音门禁必须分别记录。
- 页面外框保持居中，正文不强制居中。共享内边距使用 `clamp(14px, 4vw, 56px)` 随手机、平板和桌面可用宽度连续变化，不在设备断点处突然放大；章节、FAQ 回答和手册正文可以限制最长行宽，但阅读列必须从内容区左侧起排，右侧留出自然余量。
- 连续画布的边框不得随整块内容的位移动画离开相邻表面。首页动态封面后的第一章节保持固定位置并明确绘制上边线；需要渐入时只处理不会破坏框线连续性的内容层。

## 哲学引文

- 引文只在能缩短解释、澄清一个区分或指出问题边界时使用；不把名言当作页面装饰。
- 每段引文保持简短，并标出作者、书名和命题编号或页码。引文后立即说明“它启发了什么工程问题”和“它不能直接决定什么”。
- 同一命题的完整解释集中维护在思想基础或 ADR 页面。门户、应用、组件和手册优先链接权威解释，不重复摘引。
- 《逻辑哲学论》的事实、事态、图像、显示与可说边界属于问题来源。Endem 的六个语义面、状态、错误和编码仍只由 ADR、规范与测试定义。

## 动态摘要栏阅读框架

- 使用共享摘要栏的长篇技术页以约 284px 摘要列和至少 700px 主阅读列作为当前最低双栏需求，中间以一条低对比分隔线建立连续画布；Endem 应用页继续遵循自身信息栏契约。
- 信息栏从首屏开始承担真实职责，不是为消除空白而添加的装饰。可包含主题封面、成熟度、可用性、关键对象、章节目录、输入输出摘要和下一阅读入口。
- 共享摘要栏只在实际容器能同时容纳摘要栏、间距与主内容最低可读宽度时进入双栏；字体载入、容器尺寸和动态视口变化后必须重新计算。双栏时使用 `position: sticky`，自身内容不能超过实时可视高度，超出时只允许摘要栏内部滚动；单栏和无脚本状态按 DOM 顺序自然展开。
- 信息栏内容不得重复主栏整段正文，也不得包含内部制作说明。没有足够有用信息时回到单列阅读，不保留空栏。
- 页面角色引言与正文使用同一列线：标题和摘要对齐主阅读列，主题图形或状态摘要对齐信息栏。不能在引言结束后突然改变左右栏比例。
- 宽表格、对象布局、流程图和大型对照可以临时跨越完整画布；其前后解释仍回到主阅读列，并提供局部横向滚动而不是扩大所有段落。

## 普通专题

- `content-stack`、`content-band`、`content-split`、`content-grid`、`content-rows` 和 `content-wide` 只决定章节组织，不得解除正文最大行宽。
- 未使用摘要栏的单列专题、目录正文和手册阅读区保持外层画布居中，但受限正文从内容区左侧起排，最大行宽留下的空间自然留在右侧。不得为了形式上的对称把段落、列表或答案强制放到画布中央。`content-wide` 的表格、流程和映射、目录卡片以及手册表格和代码仍可跨越完整可用宽度。
- `content-rows` 的整行分隔可以横跨画布，但每行使用独立的序号/标签列与受限文字列；正文文字不直接铺满 1080px。
- `content-split` 的标题侧栏用于章节定位，右侧论述保持约 680–760px；镜像版式不得让读者在左右两边来回寻找下一段。
- 未使用摘要栏的普通专题中，直接语义章节的 `h2` 达到 6 个时，共享脚本必须在专题引言后自动生成一份页内目录；少于 6 个时不生成，避免短页面增加无意义的导航层。摘要栏中的少量稳定重点入口由作者维护，不再叠加自动目录。
- 目录只读取正文中真实存在的章节标题。作者只需写有意义的 `h2`，可在需要长期外链时提供稳定 `id`；生成器必须保留已有 `id`，缺失时才使用确定性的顺序标识，不得要求作者在正文之外手抄第二份链接清单。
- 自动目录必须带有可理解的导航标签，链接文字与对应标题一致，跳转位置避开固定页头。窄屏时目录只在自身横向滚动，不得撑宽页面，也不得改变章节的 DOM 阅读顺序。
- 自动目录适用于没有手册布局的 `page_role: content` 页面。手册使用自身动态目录；Endem 应用页使用固定信息栏和六章阅读结构，不叠加这份自动目录。

## Endem 应用页

- 六个语义章节继续位于主阅读列，形成从定位、状态、能力、输入输出、边界到资源的连续说明。
- 右侧信息栏从应用引言起形成一块连续应用面板：上部显示 Endem 封面与签名，下部显示可用状态、开发阶段、关键输入输出和手册入口。
- “当前状态”仍是正文第二章的语义内容；桌面可在信息栏中呈现，移动端必须按第二章顺序回到主内容，不得只留一份脱离阅读顺序的副本。
- 应用正文使用 17–18px、约 30–32px 行高；单段建议控制在约 700–740px。输入输出表可以使用主栏完整宽度。
- 信息栏固定期间，后续主栏章节自然滚动；相关资源结束后不再用空白高度维持两栏等高。
- 应用页只有在可用画布同时容纳约 800px 主栏和约 380px 信息栏时才使用双栏。当前视口边界为 1218px；更窄时按正文顺序改为单栏，不能隐藏信息栏却继续保留它的空列。

## 手册

- 手册保留左侧动态目录与右侧阅读区，不再叠加第二个固定信息栏。
- 阅读区内普通正文继续限制在约 720–760px；表格、代码和对象结构可以使用阅读区完整宽度。
- Markdown 作者只负责语义内容；行宽、字号、目录固定、当前章节提示和响应式行为由 Jekyll 布局及共享样式统一生成。
- 每张由 Markdown 生成的表格必须拥有自己唯一的 `.table-wrap.manual-table-wrap` 滚动容器。增强脚本只包装尚未包装的表格，保留原生 `table` 语义，不产生嵌套滚动层，也不要求作者书写 HTML 容器。
- 手册表格以约 680px 的最小内容宽度保存列关系；容器不超过阅读区宽度，并使用局部 `overflow-x: auto` 与横向过滚动约束。桌面能完整容纳时不制造额外滚动，窄屏上的触摸、触控板或滚轮操作只移动该表格，不得带动整页横向位移。
- 1218px 及以上可显示 320px 动态目录和约 880px 阅读区。1000–1217px 必须隐藏目录侧栏，把手册变成最大 1000px、两侧至少 18px 留白的居中单列阅读页；该区间不是把双栏硬塞进窄画布，也不能为隐藏侧栏保留空白占位。1000px 以下继续沿用移动目录和单列正文。

## FAQ

- FAQ 的问题标题保持清楚的可操作层级，答案正文使用 17px、约 1.75 行高，并以最大 760px 的左侧锚定阅读列展开；不得为了与问题行等宽而把答案拉满画布，也不得用左右自动外边距制造卡片内部的大块空白。
- 答案中的段落、列表与链接沿用正文对比度和自然换行。折叠组件只负责渐进披露，不得用更小字号隐藏重要限制，也不得把一个需要连续论证的长篇章节拆成大量问答。

## 响应式与验收

- 普通专题摘要栏在实际可用宽度不足时取消粘性并按语义顺序合并为单列；Endem 应用页仍按自身布局契约处理。移动端正文使用 16–17px、至少 1.7 行高。
- 390px 下不通过缩小字体维持桌面行数；表格和代码只在自身容器滚动。
- 桌面验收必须测量普通段落实际宽度、字号和行高，不能只检查页面是否溢出。
- 每类页面至少检查首屏、长段落、列表、表格、最后一个章节和粘性摘要栏的起止状态；摘要栏还要在临界宽度两侧、短视口和字体就绪后验证分栏、回收和真实溢出。FAQ 检查展开答案宽度，未使用摘要栏的长专题检查 5 个与 6 个 `h2` 的目录阈值，手册覆盖 999px、1000px、1217px 与 1218px。深浅主题均须保持正文与次级文字对比度。
