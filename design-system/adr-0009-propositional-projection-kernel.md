# ADR-0009：Endem 命题投影语义核

- 状态：Superseded by ADR-0010；五组字段与目标力量模型已废止
- 日期：2026-07-12
- 影响范围：Endem 顶层语义、最小性、成形、绑定、验收、规范向量、Reader 视图和 Runner 决定
- 取代：ADR-0008 的“决定一：Endem 是唯一根制品”中的五字段语义与根 `aim` 规则
- 保留：ADR-0008 的 Endem/Weave/Frame/Witness 名称、状态机、单一 CLI、组件与权限拓扑

## 问题

ADR-0008 使用 `say / aim / must / done / open` 把来源、目标、约束、验收和未决项分开。这比自由文本目标更安全，但它仍是一套需求工程分类，不足以成为自然语言目标的底层语义：

1. `say` 之后直接进入 `aim`，没有表示记号怎样取得意义、怎样绑定对象、怎样形成命题关系。
2. `aim` 与 `must` 经常表达同一个可能事态的不同言语力量或规范强度，却被拆成两个互不共享结构的容器。
3. `done` 同时混入命题的满足条件、观察方法、证据要求和最终决定权威，无法区分“命题是什么意思”“当前是否有足够证据”和“谁有权接受”。
4. 旧结构没有要求命题与目标事态具有可比较的关系结构，因而可能退化为五段带标签的文本。
5. 如果新增一个名为 `logical_form` 的说明字段来补救，系统只会宣称自己具有逻辑形式，而不是由实际结构显示它。

《逻辑哲学论》把世界区分为事实而不是事物的堆积（1–2），把图像和命题理解为元素按确定关系形成的事实（2.14–2.18、3.14），并把理解命题关联到“若它为真，事情会怎样”（4.022–4.024）及真值条件（4.431）。它同时指出逻辑形式由结构显示，不能再由命题说出（2.172、4.12–4.1212）。这些命题不能直接推出软件格式，但足以否决旧五字段作为最终底层语义。

## 决定一：采用五组投影语义

每个 Endem 按固定顺序保存 `say / mean / case / when / open`：

| 组 | 规范职责 | 不得退化为 |
| --- | --- | --- |
| `say` | 保存可重定位的来源记号、语言、媒体、主体、范围、版本和精确摘要。 | 已确认的解释、目标或事实。 |
| `mean` | 保存来源记号到符号、指称、关系、作用域、极性、量化、时间和言语力量的已授权投影。 | 自然语言改写、模型摘要或无来源的标签。 |
| `case` | 保存一个根满足命题及其对象—关系图，描述应成为、保持或不得成为现实的可能事态。 | 实体清单、步骤、计划或默认世界知识。 |
| `when` | 把 `case` 节点绑定到可执行的满足条件、观察类型、必需 Witness、未知处理和决定权威。 | 命题自称为真、模型评分或最终结果本身。 |
| `open` | 保存投影、指称、关系、作用域、力量、测量或权威中仍未解决的有限缺口及其候选和解决权限。 | “不可说之物”的编码、概率最高猜测或运行时静默默认。 |

名称保持小写 ASCII。开发阶段不保留 `aim / must / done` 顶层别名，也不接受旧顺序；旧名称只允许出现在历史 ADR 的迁移说明中。

## 决定二：结构显示形式

`mean` 和 `case` 必须使用相同的稳定符号身份与可比较关系位置。一个关系节点至少具有谓词身份、有序或具名角色、极性、作用域和来源映射；复合节点显式记录合取、析取、否定、条件或规范允许的其他组合。

规范载荷不得包含一个仅由作者填写的 `logical_form`、`valid` 或 `true` 自我声明。对象—关系拓扑、角色数量、组合边和引用位置本身显示形式；Core 与独立 Reader 分别验证这套结构。任何实现如果只能输出五段文本而不能比较关系位置，就没有实现 Endem 语义核。

## 决定三：一个根满足命题

一个 Endem 必须且只能有一个根 `case` 节点。根节点可以组合多个相互依赖的子命题；如果两个根可以独立版本化、实现、验收或失败，就必须拆成两个 Endem，再由 Weave 组合。

目标强度不是新的顶层容器。Phase 1 的确定性 Profile 至少区分：

- `seek`：希望某个可能事态成为现实；
- `keep`：要求某个事态在声明区间持续成立；
- `avoid`：要求某个事态在声明区间不成立。

询问、断言、承诺、许可和授权等言语力量不能被强行压入这三个值。它们在获得独立 ADR、满足条件和权限模型前必须留在 `open` 或被当前 Profile 拒绝。

## 决定四：意义、满足与决定分离

`case` 描述可能事态，不宣称该事态当前已经发生。`when` 定义怎样把外部观察映射到满足判断；Witness 记录观察及其范围；决定者依据预先声明的规则作出运行结论。

确定性判断至少区分：

- `met`：适用观察满足全部必需条件；
- `unmet`：适用观察反驳至少一项必需条件；
- `unknown`：意义已经确定，但观察不足以支持 `met` 或 `unmet`；
- `error`：观察、求值器、环境或规则本身无效，不能作出满足判断。

这些是工程上的满足状态，不冒充《逻辑哲学论》的真值理论。`unknown` 不能被当作 `unmet`，`error` 也不能被当作普通未知。最终 `accepted` 还要求 Witness 完整、决定权威匹配并且没有阻断性 `open`。

## 决定五：无意义、歧义与未知分别处理

- 记号组合违反当前 Profile 的结构规则，或没有任何允许的投影时，`form` 以 `no-sense` 类错误拒绝；不得把畸形输入包装成合法 `open` Endem。
- 存在两个或更多可表达候选，但没有规则或授权决定能够选择时，候选进入 `open`。
- 投影和满足条件已经确定，只是尚无外部观察时，状态是运行期 `unknown`，不是语义 `open`。
- 无法形式化的价值判断只能记录“哪一项工程判断尚无表达或决定方法”，不能声称 Endem 已经编码不可说的内容。

## 与《逻辑哲学论》的采用边界

严格采用的是结构约束，不是哲学权威：

| 原书区分 | Endem 采用 | Endem 不采用 |
| --- | --- | --- |
| 事实由对象的结合构成（2.01、2.03） | `case` 使用对象—关系结构。 | 不断言世界存在不可再分的逻辑原子。 |
| 图像元素与被图示对象相关联（2.13–2.1515） | `mean` 保存可审计的符号、指称和来源投影。 | 不假设自然语言与现实天然一一同构。 |
| 命题是有结构的事实（3.14–3.1432） | 关系位置和组合边是规范内容。 | 不把词袋、向量或 JSON 字段数量当作命题结构。 |
| 理解关联到真时的情况（4.022–4.024） | `case` 与 `when` 明确可能事态和满足条件。 | 不把生成式目标缩成封闭二值真值表。 |
| 逻辑形式显示而非说出（2.172、4.12–4.1212） | 结构由 Core/Reader 验证，不设自证字段。 | 不把哲学语句写进载荷作为有效性证明。 |
| 为可说者划界（4.112–4.116、7） | `open`、`no-sense` 与 `unknown` 分开。 | 不声称格式能够保存神秘或不可说之物。 |

使用的中文版本为：路德维希·维特根斯坦著，贺绍甲译，《逻辑哲学论》，商务印书馆，2009 年第 1 版；本次按命题编号复核 1–3、4.002–4.1212、4.431–4.463 与 6.5–7。开放校本用于交叉核对编号与英译：

https://www.wittgensteinproject.org/w/index.php/Tractatus_Logico-Philosophicus_%28English%29

## 第一条实现切片

第一阶段改为：

```text
Source Card
  → deterministic projection rules / named decisions
  → say + mean + case + when + open
  → endem form
  → open .endem
  → endem check
  → endem see
  → endem test
```

最小向量必须覆盖：同义不同表达映射到同一 `case` 结构、同词不同指称、关系角色互换、否定与作用域、`seek/keep/avoid` 差异、无意义组合、可表达歧义、观察未知、求值错误以及 Core/Reader 对结构的差分判断。

## 后果与未冻结内容

- ADR-0008 继续决定名称、状态机、应用和权限拓扑；它的旧五字段与根 `aim` 规则被本 ADR 取代。
- 现有公开文档、示例、测试和视觉必须在同一变更中迁移，不提供旧字段兼容入口。
- `say / mean / case / when / open`、一个根 `case`、形式由结构显示、`open/unknown/no-sense` 分离已经采用。
- 物理记录编号、文本语法、符号 ID 编码、量化表达、时间模型、`when` 求值语言、Phase 1 以外的言语力量和稳定 ABI 仍未冻结。
