# ADR-0010：原生词汇与事态分层

- 状态：Accepted
- 日期：2026-07-12
- 影响范围：Endem 顶层语义、制品、组件、命令、状态、路由、视觉与测试
- 取代：ADR-0008 的现行制品/组件/动作名称与生命周期状态；ADR-0009 的现行五组语义、力量值和判断词
- 保留：Endem、`.endem`、单一 `endem` CLI、确定性写入、独立只读解释、隔离实现、模型不可信与外部签名边界

## 问题

ADR-0009 的 `say / mean / case / when / open` 比需求分类更接近命题结构，但仍有两个根本问题。

第一，名称借用了主流语言已经占用或高度通用的词。`case` 是 C、C++、Java、ECMAScript 与 Swift 的关键字；`open` 是 Swift 声明关键字；`when` 是 Kotlin 硬关键字。`Core / Reader / Runner / Frame / Witness` 和 `form / check / bind / pack / seal / see / run / test` 也主要来自既有编译器与通用软件词汇，不能为新领域形成清楚、可检索的本体。

第二，`case` 同时保存可能事态和 `seek / keep / avoid` 目标力量，混合了“事情怎样可能成立”与“主体要求它发生、持续或不发生”。`avoid` 又会与关系极性或命题否定形成双重编码。Witness 虽记录观察，却未要求观察形成可与目标事态比较的关系结构。

## 决定一：采用六个原生语义面

每个 Endem 按固定顺序保存 `rhem / semion / skena / telis / krin / apor`。这些词使用古希腊语词根重新造词，不是对哲学概念的直接软件化。

| 语义面 | 构词来源 | 规范职责 | 直白工程副标题 |
| --- | --- | --- | --- |
| `rhem` | *rhēma*，说出的内容 | 保存可重定位来源记号、主体、媒介、范围、版本与摘要 | 来源表达 |
| `semion` | *sēmeion*，记号 | 保存记号到稳定符号、指称、关系、角色、作用域、时间与来源位置的已授权投影 | 意义投影 |
| `skena` | *skēnē*，场景 | 保存一个根、中性且可比较的可能事态图；图由对象、关系、角色、组合与极性构成 | 可能事态 |
| `telis` | *telos*，趋向的终点 | 保存目标对 `skena` 的适合方向和时间形态，不改写事态内容 | 目标方向 |
| `krin` | *krinein*，区分与判断 | 规定观察类型、结构比较、证据强度、未知处理和决定权威 | 满足判据 |
| `apor` | *aporia*，无通路与疑难 | 保存仍可表达但尚未获授权决定的投影、指称、测量或权威缺口 | 未决边界 |

名称保持小写 ASCII，不翻译、不复数化。载荷没有旧键别名，读取器遇到旧键必须拒绝。

## 决定二：把可能事态与目标方向分开

`skena` 只表达一个可能成立或不成立的对象—关系结构，不携带请求、承诺、命令或价值。一个关系节点至少保存谓词身份、具名或有序角色、极性、作用域和来源投影；复合节点显式保存合取、析取、否定与条件。

`telis` 对整个 `skena` 施加目标方向。第一阶段只冻结两个值：

- `kine`：使 `skena` 在声明边界达到成立；词根来自 *kinesis*，变化；
- `mene`：使 `skena` 在声明区间持续成立；词根来自 *menein*，保持。

禁止事项通过 `skena` 中的显式否定或不存在关系表达，再由 `kine` 或 `mene` 指定达到或维持；不再设置与极性重复的 `avoid`。询问、承诺、许可和授权在获得独立权限与成功条件前进入 `apor` 或被 Profile 拒绝。

## 决定三：观察必须形成可比较的 Phain

运行观察先规范化为 `phain`，即“显现出来的结构化观察”。`phain` 使用与适用 `skena` 相同的稳定符号身份、关系角色和作用域坐标；无法形成这种对应时，`krin` 不能把自由文本、模型评分或日志数量当作满足证据。

`phain` 不独立成为文件制品。它连同生产者、方法、环境、时间域、强度、限制和完整性封装在 Tekmor 中。`krin` 比较 `skena` 与适用 `phain`，产生 `met / unmet / agno / fault`：

- `agno`：意义与判据已确定，但适用观察不足；
- `fault`：观察、求值器、环境或规则无效。

无法形成任何 Profile 允许投影的输入以 `aseme` 拒绝；存在多个允许投影但无权选择时进入 `apor`。`aseme / apor / agno` 分别表示无可允许意义、语义未决和观察不足，禁止合并。

## 决定四：重铸制品与运行名词

| 名称 | 构词 | 直白工程副标题 | 边界 |
| --- | --- | --- | --- |
| Endem | `end` + `-eme` | 最小目标制品 | 一个根 `skena` 与六个语义面 |
| Synem | `syn-` + `-eme` | 多目标解析闭包 | 两个或更多 Endem 的已解析组合；单 Endem 时省略 |
| Dromen | *drōmenon*，被实行之物 | 会话实现态 | 一次运行内建立，不是文件或永久权限 |
| Tekmor | *tekmōr*，迹象或凭据 | 有范围的证据记录 | 封装 `phain`、来源、强度、限制与决定依据，不自动证明结论 |

`.synem` 与 `.tekmor` 只是保留的候选扩展名，在物理格式 ADR 前不得宣称冻结。Dromen 永不使用文件扩展名。

## 决定五：重铸组件与动作

三个实现域使用“制作—观察—实行”的关系：

| 组件 | 构词 | 职责 |
| --- | --- | --- |
| Poiet | *poiein*，制作 | 唯一规范写入者与生产验证域 |
| Theor | *theōrein*，观察 | 独立、只读且资源有界的第二解释路径 |
| Praxor | *praxis* + 行动者后缀 | 隔离实现域，持有会话级最小能力 |

唯一 CLI 仍为 `endem`，动作改为：

| 动作 | 词根 | 直白职责 |
| --- | --- | --- |
| `poie` | *poiein* | 形成 Endem |
| `elenk` | *elenchos* | 生产验证与反驳检查 |
| `pleko` | *plekein* | 解析组合并形成 Synem |
| `tasse` | *tassein* | 排序并冻结发布闭包 |
| `sphra` | *sphragis* | 核对并附加外部签名包络 |
| `theor` | *theōrein* | 由 Theor 独立只读解释 |
| `praxe` | *praxis* | 由 Praxor 隔离实现目标 |
| `peira` | *peira* | 运行一致性、安全与复现试验 |

动作不是八个产品。Poiet 承担 `poie / elenk / pleko / tasse` 并验证 `sphra` 响应；Theor 独立承担 `theor`；Praxor 独立承担 `praxe`；`peira` 比较全部实现域。

## 决定六：采用不与语义面重名的生命周期

Endem/Synem 使用三个英文状态值：

- `nascent`：结构合法，但 `apor`、引用或确认仍可能未决；
- `coherent`：全部必需投影、引用、冲突、能力上限与判据已取得确定结论；
- `attested`：不可变发布载荷已与外部签名响应精确绑定。

状态值不复用语义面，也不暗示信任等级。`attested` 不证明语义正确、当前环境授权或目标已经满足。

## 名称审计与使用边界

截至 2026-07-12，主流语言关键字集合中没有本 ADR 的现行领域词。PyPI、npm 与 crates.io 的精确包名查询显示 `endem`、`rhem`、`semion`、`skena`、`telis`、`krin`、`apor`、`synem`、`dromen`、`tekmor`、`poiet`、`theor`、`praxor` 以及八个动作当时均未同时形成强占用；相邻词、大小写变体和其他行业名称仍然存在。

该结果只用于降低工程混淆，不构成商标、域名或包名权利。正式发行仍须执行法域商标、近音、分类、命令、MIME、扩展名和法律复核。

## 后果

- 所有现行规范、页面、手册、路由、目录、示例、视觉、测试和项目简介在同一变更中迁移。
- `/specifications/weave.html`、`/specifications/witness.html`、`/components/core.html`、`/components/reader.html` 与 `/components/runner.html` 删除，使用 Synem、Tekmor、Poiet、Theor、Praxor 新路由，不提供重定向。
- ADR-0008 与 ADR-0009 保留为被取代的历史记录；旧词只允许出现在明确标为 Superseded 的 ADR 和名称迁移表。
- ADR-0011 已采用实验性固定前导、目录和六个记录编号；记录内部字段、`phain` 编码、求值语言、量化与时间模型、扩展 registry 和稳定 ABI 仍未冻结。
