# 分析哲学视觉语言

适用范围：全站由线条、节点、关系图和短标签构成的哲学相关视觉。它把项目的思想来源转成阅读辅助，不把哲学名称当作技术证明。

## 采用边界

- 视觉只表达分析哲学中与项目直接相关的区分：表达与意义、命题与满足条件、部分与整体、规则与使用、主张与证据、理解活动与理解对象。
- 哲学图形提供问题框架，工程含义仍由 Endem、Synem、Tekmor、组件边界、规范条款和测试定义；不得用图形暗示某个哲学命题已经直接推出技术结论。
- 优先使用细线、节点、括号、箭头和几何关系；不使用哲学家肖像、古典柱式、雕塑、羊皮纸或与项目语义无关的名言装饰。
- 可见短标签保持简洁，可使用 `EXPRESSION / MEANING / OBJECT` 等英文关系词；中文正文继续负责准确解释。

## 基础关系图

| 关系图 | 含义 | 使用模块 |
| --- | --- | --- |
| 三节点收敛 | 不同表达、意义身份与对象指向需要区分 | 项目背景、首页 |
| 平行命题线 | 命题结构与满足条件必须分别可查 | 规范、指南 |

| 问题—命题—答案缺口 | 常见问题应回到明确命题和权威入口 | 常见问题 |

## 视觉实现

- 模块页面引言的彩色几何面内部最多显示三个节点和两条关系线；节点使用深色核心与浅色环，关系线保持低透明度，不与标题争夺层级。
- 节点位置和连线角度随模块母题变化，但同一模块全部页面保持一致。目录页完整显示，专题页在较小几何面内复用同一关系。
- 文档折页以平行细线表示连续论证，以折角表示可索引和回查；Endem 手册只按动作职责更换强调色。
- 首页使用 `rhem → semion → skena → telis → krin → apor` 语义轨道、Endem 记录和 `source → poie → Endem` 关系；关系位置必须可见，不把六个词画成彼此无关的文本标签。证据视觉还必须显示 `phain` 与 `skena` 的同构比较，而不是只放一个通过标记。

## 动效与可访问性

- 哲学关系线和节点默认静态，避免把概念关系做成无意义粒子动画。
- 只允许页面已有的数据流、卡片和状态组件执行克制动效；减少动态效果时关系图仍完整可见。
- 关系图是辅助图形，使用伪元素时不进入无障碍树；对应含义必须由页面引言摘要、章节标题或正文说明。
- 深色模式降低彩色面亮度并保留节点环对比；移动端减少线段长度，不删除关系中的关键节点。
