# 目录与聚合页设计

适用范围：`page_role: section`，包括项目背景、架构、规范、组件、资源、FAQ、开发和新闻入口。

## 阅读任务

目录页负责解释模块职责、当前状态和内容地图，让读者快速选择下一页。它可以简洁，但不能只堆链接，也不能伪造专题页深度。

## 页面结构

- 顶部使用全宽 Hero：标题、模块说明、状态徽标和 Noemion 对象图形。
- 首个章节说明如何使用本模块或推荐阅读顺序。
- 后续章节按任务或主题分组，使用两列内容入口；单个入口自动占满整行。
- 大型目录可在入口前使用流程或责任表解释分组逻辑；Endem 不使用并列应用目录。
- 入口数量达到两位数时提供渐进增强的搜索与分组筛选；关闭 JavaScript 时仍显示完整目录，筛选不能改变正式路由或卡片原始分组。
- 项目动态页在没有正式公告时仍使用横向记录行组织真实的阶段、规范和验证进展；每行必须链接到可核验的项目页面，不生成虚构日期、封面或发布记录。
- 每个目录页从 `<body data-site-module>` 取得所属模块的 Hero 轮廓、对象标签和章节短标；完整母题用于建立模块识别，入口卡的矩形命中区保持不变。模块映射与使用边界见 [`geometric-layouts.md`](geometric-layouts.md)。

## 卡片与标题

- `.page-link` 高度以 152–176px 为基线，标题约 19–21px。
- 悬停时表面轻微提亮、左侧出现朱橙标记、标题右移 4px。
- 圆形箭头由透明、模糊和 0.25 倍缩放切换到完整状态。
- 卡片摘要只回答点击后能解决的问题，不复制目标页正文。

## 响应式

- 桌面两列，中等和移动宽度单列。
- 分组标题和卡片之间保留清晰边界，不能因单列化丢失所属分组。
- 表格只在自身容器滚动，目录页整体不得横向滚动。
