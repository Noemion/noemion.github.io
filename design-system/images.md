# Noemion 图片与视觉素材

本文件定义站点图片的选择、落库、裁切、可访问性和维护规则。图片是内容层，不是为了填满空白；页面角色与文字结构仍由对应设计文档决定。

## 选择顺序

1. 先检索许可清晰、可本地保存和裁切的素材库；使用前记录作者、原始 URL、许可和允许的改编方式。
2. 已有项目素材能够准确表达内容时优先复用，但不同工具不得机械使用同一封面。
3. 现成素材过于泛化或无法准确表达 Noemion 的对象语义时，生成项目定制图片。生成图不得伪装为产品截图、实验结果或已经实现的硬件。
4. 不使用第三方热链，不把外部设计参考、模仿目标或生成过程写入公开页面。

## 表达与文字

- 图片必须承担概念解释、页面辨识或内容氛围中的至少一项职责；纯装饰图使用空 `alt`，有独立信息的图片使用简洁中文替代文本。
- 图片内部尽量不生成文字。必须出现视觉标签时优先使用英文，并把需要准确阅读、翻译或动态更新的文字放在 HTML 层。
- 学术图片、历史资料和真实产品截图不得通过生成式视觉替代；此类素材必须保留准确来源与说明。

## 文件与性能

- 成品保存在 `assets/images/`，使用描述性英文文件名。保留明确的 `width`、`height`、`decoding` 与合适的 `loading` 属性，避免布局跳动。
- 按实际最大显示尺寸压缩；普通内容图原则上不超过 1600px 长边。只有真实需要透明通道时使用 PNG，摄影或复杂渐变优先使用压缩 JPEG/WebP。
- 关键首屏图可使用 `fetchpriority="high"`，非首屏图片使用延迟加载。不得为了图片引入新的运行时依赖。

## 裁切与界面

- 每张图记录主体焦点，使用 `object-fit: cover` 和明确的 `object-position`，在约 1512px、900px、390px 三档检查主体没有被错误裁掉。
- 浅色模式图片使用 `rgba(0, 0, 0, 0.1)` 的 1px 内描边；深色模式使用 `rgba(255, 255, 255, 0.1)`，不得使用带色中性色代替。
- 悬停仅允许可中断的轻微缩放、亮度或位移，明确列出过渡属性；不得使用 `transition: all`。减少动态效果下取消图片缩放和进入动画。
- 图片与文字叠加时必须使用稳定遮罩保证对比度，重要文字不能依赖图片本身提供可读性。

## 验收

- 图片不存在断链，尺寸属性与实际宽高一致，页面无布局偏移或横向溢出。
- 深浅主题描边正确，叠字对比度清晰，键盘焦点不被图片层遮挡。
- `prefers-reduced-motion: reduce` 下没有持续或悬停缩放。
- 生成图不被描述为产品实拍、完成成果或验证证据。
