# Endem 应用页设计

适用范围：`page_role: tool-project`，当前仅 `endem/index.html`。`tool-project` 是现有布局角色标识，不表示继续建设工具家族。

本页定义唯一公开应用 `endem` 的共同结构；五个动作的色彩、签名和边界见 [`internal-tools.md`](internal-tools.md)。正文尺度与信息栏遵守 [`readability.md`](readability.md)。

## 阅读任务

应用页必须说明 Endem 为什么存在、当前能否使用、五个动作怎样分工、三个信任域为何不能合并，以及哪些历史职责被取消或并入子命令。它不是已经发布的 CLI 手册。

## 固定结构

1. 先确认当前可用性
2. 用一个开发任务建立应用边界
3. 按工作选择设计入口
4. 说明任务怎样跨越不同信任职责
5. 给出停止与责任交接条件
6. 限定外部工程先例的采用范围
7. 继续阅读

## 桌面布局

- 约 800px 主阅读列与约 380px 信息栏从应用引言对齐。
- 主栏正文 17–18px、约 30–32px 行高；表格可使用主栏完整宽度。
- `.tool-status-panel` 位于粘性信息栏，显示“未发布”、当前资料范围和正式发行前仍需确定的内容；布局按类名识别职责，不依赖章节序号。
- 信息栏不得复制主栏长段落；进入全宽“继续阅读”前自然离开，不覆盖资源入口。
- Endem 应用使用前向平行四边形表达受约束转换，动作差异只在色彩、网格和短签名体现。

## 状态表达

- 当前必须明确“尚未进入代码开发阶段”和“没有可执行组件”；规范草案、参数和 ABI 仍未稳定发布。
- `.endem` 与子命令当前只承担已接受的设计职责；具体发行拼写和读音仍受 ADR-0034 约束，也不得因此虚构安装命令或完成度。
- 输入、输出、消费者和失败责任优先使用短表；不能只列功能名。
- 一个 CLI 不得被画成一个信任域：Ktisor、独立 Theor 与隔离 Drasor 必须明确分开。

## 响应式与手册边界

- 当视口窄于 1218px 时取消双栏与粘性定位，状态面板按 DOM 顺序回到任务说明之前。这个边界保证扣除页面留白后的可用画布，仍能容纳约 800px 主栏和约 380px 信息栏；不能为了保留桌面形式而裁掉信息栏或留下空列。
- `endem/index.html` 是 HTML 应用页；`endem/docs/*.md` 是手册并遵循通用 Markdown 规则。
- 没有独立维护价值时不新增手册主题，也不为目录对称创建空页面。
