# Endem 应用页设计

适用范围：`page_role: tool-project`，当前仅 `endem/index.html`。`tool-project` 是现有布局角色标识，不表示继续建设工具家族。

本页定义唯一公开应用 `endem` 的共同结构；八个动作的色彩、签名和边界见 [`internal-tools.md`](internal-tools.md)。正文尺度与信息栏遵守 [`readability.md`](readability.md)。

## 阅读任务

应用页必须说明 Endem 为什么存在、当前能否使用、八个动作怎样分工、三个信任域为何不能合并，以及哪些历史职责被取消或并入子命令。它不是已经发布的 CLI 手册。

## 固定结构

1. 它解决什么问题
2. 当前状态
3. 它怎样工作
4. 它读取什么，产生什么
5. 它不会做什么
6. 继续阅读

## 桌面布局

- 约 800px 主阅读列与约 380px 信息栏从应用引言对齐。
- 主栏正文 17–18px、约 30–32px 行高；表格可使用主栏完整宽度。
- 当前状态位于粘性信息栏，显示“未发布”、Phase 1 范围、关键输入输出和手册入口。
- 信息栏不得复制主栏长段落；进入全宽“继续阅读”前自然离开，不覆盖资源入口。
- Endem 应用使用前向平行四边形表达受约束转换，动作差异只在色彩、网格和短签名体现。

## 状态表达

- 当前必须区分“本地候选已运行”和“尚无公开安装包”；规范草案、参数和 ABI 仍未稳定发布。
- `.endem` 与子命令名是已接受词汇，不得因此虚构安装命令或完成度。
- 输入、输出、消费者和失败责任优先使用短表；不能只列功能名。
- 一个 CLI 不得被画成一个信任域：Poiet、独立 Theor 与隔离 Praxor 必须明确分开。

## 响应式与手册边界

- 当视口窄于 1218px 时取消双栏与粘性定位，状态面板回到正文第二章。这个边界保证扣除页面留白后的可用画布，仍能容纳约 800px 主栏和约 380px 信息栏；不能为了保留桌面形式而裁掉信息栏或留下空列。
- `endem/index.html` 是 HTML 应用页；`endem/docs/*.md` 是手册并遵循通用 Markdown 规则。
- 没有独立维护价值时不新增手册主题，也不为目录对称创建空页面。
