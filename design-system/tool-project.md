# 工具项目页设计

适用范围：`page_role: tool-project`，即 `tools/<tool>/index.html`。

本文件定义所有工具项目页的共同结构；每个具体工具的色彩、签名文字、图形母题和内容强调点见 [`internal-tools.md`](internal-tools.md)。共同结构不得抹平工具差异，工具差异也不得改变六章语义契约。

正文尺度与固定信息栏同时遵守 [`readability.md`](readability.md)。

## 阅读任务

工具项目页用于说明工具为什么存在、当前能否使用、主要能力、输入输出、处理边界和相关资源。它不是 CLI 手册，也不能把规划中的工具写成已发布程序。

## 固定语义结构

页面必须按以下顺序保留六章：

1. 它解决什么问题
2. 当前状态
3. 它怎样工作
4. 它读取什么，产生什么
5. 它不会做什么
6. 继续阅读

## 桌面布局

- 页面使用约 800px 主阅读列与约 380px 工具信息栏，两栏从 Hero 开始保持同一列线。
- Hero 左侧标题与摘要对齐阅读列；右侧工具专属封面、签名与状态面板组成连续信息栏，不在 Hero 后突然切换宽度。
- 主栏正文使用 17–18px、约 30–32px 行高，普通段落控制在约 700–740px；表格可以使用主栏完整宽度。
- “当前状态”位于右侧粘性信息栏，顶部与 64px 导航对齐，使用语义频谱细线标识阶段，并补充可用性、关键输入输出和实际存在的手册入口。
- “它解决什么问题—它不会做什么”共同位于 `.tool-project-body` 两栏容器；外层不得使用会建立滚动上下文的 `overflow: hidden/auto` 截断粘性定位。状态卡在该容器仍处于阅读范围时保持在导航下方约 88px，并在进入全宽“继续阅读”前随容器自然离开，不覆盖资源入口。
- 信息栏不得复制主栏长段落；内容超出可视高度时只在栏内滚动。主栏结束后不通过额外空白强行保持两栏等高。
- 最后的“继续阅读”可横跨完整画布，形成后续阅读入口。

## 状态表达

- 当前未发布时必须明确没有可执行程序，并说明 CLI、参数和扩展名尚未冻结。
- 状态面板使用产品语言说明当前是否可用、开发依赖、下一步工作和验证目标，不展示内部管理术语或虚构安装按钮。
- 输入、产物和失败边界优先使用表格、清单和短流程。
- “继续阅读”以入口卡片结束，不在卡片之后追加候选接口、冻结状态或开发约束；这些信息按职责进入“当前状态”或“它不会做什么”。

## 响应式

- 1000px 以下取消双栏与粘性定位，状态面板回到正文第二章。
- 移动端保持六章顺序，状态信息不得因布局变化被移到页面末尾。

## 项目页与帮助手册边界

- `tools/<tool>/index.html` 始终是 HTML 项目页，不转换为 Markdown。
- 只有 `tools/<tool>/docs/*.md` 属于帮助手册，并遵循纯通用 Markdown 正文规则。
- 工具手册继承当前工具的颜色与签名，但继续使用统一的手册布局、动态目录和分页。
- 没有实质手册内容时不创建 `docs/`，也不为目录对称创建空 Markdown。
