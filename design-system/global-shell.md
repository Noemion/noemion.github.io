# 全局外壳与导航

适用范围：默认布局、站点头部、页脚、全局导航、移动目录、文档侧栏和目录数据。

## 布局基线

- 桌面画布宽度上限为 1200px，居中放置；顶部导航高度为 64px。
- 头部、正文和页脚的左右边界必须对齐，文档页的 320px 侧栏与 880px 阅读区共同占满画布。
- 页面外侧使用低对比灰绿背景，内容画布使用纸白或深色纸面。
- 1000px 以下隐藏固定文档栏，但顶部五组任务导航继续保留到 840px；只有 840px 以下才隐藏桌面分组导航，并显示带“导航”文字的移动入口，不能只显示缺乏含义的汉堡图标。

## 顶部导航

- 一级入口按读者任务组织，不机械复制文件夹名称。
- 右侧项目状态入口必须明确表达“当前阶段”与阶段值两层含义；当前显示“当前阶段 / 设计”，完整无障碍名称和提示为“当前项目阶段：规范与安全核心设计”，链接到开发与路线图入口，不能让读者误认为是当前页面状态或稳定版本标记。
- 桌面端悬停和键盘聚焦都必须展开纵向内容面板；触发器使用 `aria-haspopup` 并同步 `aria-expanded`，当前页使用 `aria-current`。
- 每个一级分组必须先说明其在 Noemion 信息架构中的用途，再展示对应入口；不能按视频、文章等外部内容形态机械组织。
- 展开面板包含入口专属 SVG 封面、标题、摘要和行动箭头，不显示分组或卡片序号；21 个入口分别表达目标身份、信任流水线、目标图、Section 表、依赖闭包、工具职责、阅读路径或证据状态，不能复用同一封面换色冒充定制。
- 菜单内容按 45ms 依次进入；卡片悬停使用彩色轮廓、轻微横移、封面缩放和箭头反馈。封面只使用项目自有图形语汇，不能引用外部站点图片、文案或链接。
- 移动目录复用 `assets/directory.js` 的同一数据，不维护第二套链接。

## 目录数据

- 普通页面目录由 `DIRECTORY_MODULES` 提供。
- 手册目录由 Jekyll 扫描带 `manual_id` 的 Markdown 页面并输出 JSON，JavaScript 只负责渲染。
- 当前页高亮必须使用规范化 URL，正确处理 `/index.html` 与目录形式。
- 动态数据不可用时允许回退到最小静态入口，但不得产生错误链接。

## 可访问性与状态

- 跳到正文链接、键盘焦点和 `aria-expanded` 状态必须保留。
- 移动目录按钮操作面积至少 44×44px，展开时汉堡图标切换为关闭状态。
- 交互过渡使用 150–200ms，声明具体属性；减少动态效果时立即切换。
- 页头保持粘性，但不得形成横向滚动或遮挡锚点定位。
- 共享 CSS 与导航脚本 URL 必须携带当前 Pages 构建版本；每次部署后浏览器应立即取得匹配的 HTML、样式和目录逻辑，不能因旧缓存隐藏导航或保留过期交互。
- 品牌星形标记使用主题响应配色：浅色模式深底浅字，深色模式浅底深字；内部 `N` 与底板必须保持清晰对比。站点 favicon 是独立 SVG，不用于页头标记。
