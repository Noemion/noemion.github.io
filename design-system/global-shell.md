# 全局外壳与导航

适用范围：默认布局、站点头部、页脚、全局导航、移动目录、文档侧栏和目录数据。

## 布局基线

- 桌面画布宽度上限为 1200px，居中放置；顶部导航高度为 64px。页头、各页面角色正文、页脚、文档侧栏与移动目录必须共同读取 `--site-frame-*`、`--site-header-height`、`--docs-rail-width` 和 `--docs-content-width`，不得各自复制 1200/36/16/18/8px 画框公式；1000px 以下只切换共享画框边距变量。
- 头部、正文和页脚的左右边界必须对齐，文档页的 320px 侧栏与 880px 阅读区共同占满画布。
- 共享画布的水平起点统一按可用内容宽度计算，不得让顶部导航使用百分比居中、正文或侧栏另用 `vw` 定位；经典滚动条占据宽度时也必须保持同一左右边界。
- 浅色模式的页面外侧使用低对比灰绿背景，内容画布使用纸白。深色模式以首页封面边缘的 `#060f0d` 深墨绿作为主纸面，外层背景、次级纸面、抬升纸面、导航、代码面和分隔线只在同一墨绿色相内调整明度与饱和度，不混入独立的中性灰黑表面。
- 1000px 以下隐藏固定文档栏与依赖悬停的五组桌面任务导航，统一显示带“导航”文字的移动入口，不能只显示缺乏含义的汉堡图标。这个断点同时覆盖 iPhone 14 Pro 的 852px 横屏宽度，不能让手机横屏退回没有背景滚动隔离的桌面下拉菜单。

## 顶部导航

- 顶部栏保持粘性，并使用约 82% 的主题抬升纸面、18px 背景模糊和轻微饱和度补偿；顶部栏、品牌单元和右侧行动入口统一使用直角边框，不建立圆角缺口或额外角部遮挡。边缘只保留低对比分隔线、极浅高光和一层克制投影。页面滚动时，下方内容的色彩应通过半透明表面自然显现，使顶部栏与当前内容保持连续关系；不得在滚动阈值处突然切换为实色、重阴影或高对比分隔。
- 1000px 以下的页头从页面加载起使用稳定的高不透明纸面，不使用会为固定子元素建立新定位容器的 `backdrop-filter`；不得在菜单开关时临时切换页头滤镜或定位。
- 一级入口按读者任务组织为项目、规范、应用、指南和开发，不机械复制文件夹名称；架构设计是“项目”分组中的明确入口，Endem 作为具体应用名只出现在“应用”分组内部。
- 右侧只显示一个 `TIMELINE` 行动入口，链接到 `development/current-stage.html`。可见内容不附加图形，也不包含 Endem、当前阶段名、版本或完成比例；文字本身承担完整导航含义。入口必须铺满所属页头单元，不建立带外边距、描边或阴影的内层卡片；默认背景透明并继承页头表面，只用左右分隔线说明单元边界，避免常态色块被误解为当前选中项。悬停时才显示导航纸面与强调软色的低对比混合，不浮动整块入口。文字、链接、标题和无障碍名称统一读取 `_data/site_header.yml`，不在共享布局中复制。
- 桌面端悬停和键盘聚焦都必须展开纵向内容面板；鼠标进入后约 20ms 自动展开，离开分组后保留约 120ms 的移动容错，让光标能稳定进入卡片且不会在相邻分组间留下旧面板。触发器使用 `aria-haspopup` 并同步 `aria-expanded`，当前页使用 `aria-current`。
- 达到桌面宽度但仍为 `(hover: none) and (pointer: coarse)` 的横屏平板，不把触摸产生的焦点解释为键盘操作，也不在初始状态加载二级卡片或 SVG 封面。第一次主动触摸一级入口时阻止立即跳转，按需建立即将显示的富菜单；菜单打开后再次触摸标题可进入目录，也可直接选择卡片。外接键盘以 Tab 产生明确键盘意图后，仍可聚焦并展开完整菜单。
- 每个一级分组必须先说明其在 Noemion 信息架构中的用途，再展示对应入口；不能按视频、文章等外部内容形态机械组织。
- 桌面展开面板包含入口专属 SVG 封面、标题、摘要和行动箭头，不显示分组或卡片序号；入口分别表达 Endem 六个语义面、Synem 闭包、Ktisor/Theor/Drasor 边界、阅读路径、当前阶段或证据状态，不能复用同一封面换色冒充定制。移动目录只呈现文字层级，不创建封面 DOM，也不请求或合成 `nav-covers.svg`。
- 动态创建 SVG 时必须使用 SVG 命名空间；验收不能只确认封面容器和文字覆盖层存在，还要确认内部图形实际渲染。
- 面板使用约 180ms 的淡入、上移与轻缩放组合，不以模糊滤镜延迟文字清晰度；菜单内容按 15ms 依次进入，五张卡片在约 230ms 内全部稳定。卡片悬停、键盘聚焦或表示当前页时外层边界必须保持原位，四色频谱以约 4.8s 一周的线性节奏沿边界持续循环，只允许内部文案与箭头轻移、封面缩放。箭头默认指向右侧；进入状态后，箭头外圈从顶部起笔并以约 280ms 顺时针补画，仍略慢于彩色轮廓显现。封面只使用项目自有图形语汇，不能引用外部站点图片、文案或链接。
- 移动目录与桌面目录复用 `_data/navigation.yml` 生成的数据；手册移动目录读取已由 Jekyll 输出的文档栏，不维护第二套链接。

## 目录数据

- 普通页面目录由 `_data/navigation.yml` 提供；共享入口在 1000px 以下启动时获取数据并建立纯文字目录，移动端首次点击通常不等待模块或目录数据加载。精确指针桌面提前准备同一份数据，但只在实际打开时生成当前分组；不预建隐藏的移动目录或其他未打开分组。
- 手册目录由 Jekyll 扫描带 `manual_id` 的 Markdown 页面并直接输出语义 HTML；JavaScript 只在需要时复用这份 DOM 添加移动交互。
- 当前页高亮必须使用规范化 URL，正确处理 `/index.html` 与目录形式。
- 动态数据不可用时保留服务端输出的一级入口、手册目录与 `sitemap.md` 完整路由入口，不得产生错误链接。

## 全站页脚与主题

- 所有页面使用同一共享页脚：上层按项目、阅读、开发和发现分为四列，下层提供全部页面、文档、Endem、开发、FAQ、许可证与 `GitHub·Pages` 入口；其中 `GitHub·Pages` 固定指向 `https://github.com/Noemion`。只能链接真实存在的 Noemion 内容，不得用不存在的订阅、账户或词典功能填充版面。
- 页脚在快捷链接与主题选择所在底栏自然结束，不再追加页面标题、开发阶段或安装状态等重复说明带；具体状态只在承担该信息职责的正文区域呈现。
- 页脚右下角使用至少 40px 高的胶囊主题按钮，1000px 以下触屏布局提高到至少 44px，显示当前选择值与 `Theme`；点击后在按钮上方展开 128px 宽的 `Light / Dark / System` 菜单，当前项显示勾选。菜单以 180ms 淡入、去模糊、上移和 0.95→1 缩放进入，关闭动画为 150ms，按压缩放固定为 0.96。
- 主题选择使用站点级持久化键保存，并在共享 CSS 加载前恢复。`Light` 与 `Dark` 固定主题；`System` 根据 `prefers-color-scheme` 解析，并在操作系统设置变化时实时更新。任何页面跳转后都必须保留上次选择。
- 主题菜单支持点击外部和 `Escape` 关闭，支持方向键、Home、End 和 `menuitemradio` 状态；按钮最小触控高度为 44px（移动端）。太阳与月亮图标同时保留在 DOM 中，以 opacity、4px blur 和 0.25→1 scale 交叉切换。
- 800px 以下四列折为两列，560px 以下折为单列；1000px 以下页脚上层入口与底栏入口都提供至少 44×44px 触控区域，主题菜单项至少 44px 高，主题按钮在手机端占满页脚底行，菜单向上展开，不产生横向溢出。

## 可访问性与状态

- 页面不渲染额外的“跳到正文”入口；主内容继续使用唯一 `main` 地标，键盘焦点和 `aria-expanded` 状态必须保留。
- 1000px 以下的非首页使用共享动态页头：控制器按品牌、`TIMELINE`、84px 导航按钮的固有宽度与页头实时可用宽度判断位置。三者能完整放下时，导航按钮留在第一行，面包屑单独占第二行；放不下时，导航按钮才进入面包屑所在第二行并为路径保留空间。窗口、字体和视口变化后必须重新计算，不能用另一个经验断点代替实际测量。首页没有面包屑，继续保留单行移动头部。无论按钮位于哪一行，展开面板都从第二行下方进入。
- 520px 以下的首页单行头部必须为品牌、`TIMELINE` 与 84px 导航入口保留互不重叠的网格列。品牌图标不得收缩，品牌文字必须裁切在自身列内；393px 宽度应完整显示 `Noemion`，更窄设备才允许文字省略。时间线卡片可以压缩字号和间距，但不能覆盖品牌列。
- 移动目录按钮操作面积至少 44×44px，展开时汉堡图标以 180ms 过渡为关闭状态。首次点击必须立即开始菜单面板的 180ms 淡入和下移归位动画，不得等待模块、数据或动画延时；再次点击、点击透明遮罩或按 `Escape` 时立即开始同等时长的反向动画。图标与面板动画保持同步，并允许用户在关闭动画期间反向打开。大面积菜单不使用模糊滤镜，避免移动浏览器在首帧创建昂贵的离屏表面。
- 移动目录打开期间只冻结背景输入，不改写根元素或 `body` 的定位、尺寸、overflow、滚动坐标或页头定位。透明固定遮罩负责隔离主体点击、滚轮、触摸和滚动按键，目录面板作为独立固定合成层显示在遮罩上方；正文保持原位置和原有绘制状态。目录高度由实际条目决定，只在内容超过可用高度时以动态视口余量为上限并启用独立纵向滚动，同时使用 `overscroll-behavior: none`。单指位移由守卫限定在目录自身范围，不依赖 WebKit 的根滚动链或旧式 `-webkit-overflow-scrolling`；双指缩放仍由浏览器处理。
- 移动目录展开后与页头、面包屑和正文统一保留左右各 8px，不使用 `100vw` 另算宽度，避免窄屏桌面浏览器的经典滚动条把面板推离共享画布。
- 交互过渡使用 150–200ms，声明具体属性；减少动态效果时立即切换。
- 提高对比度时页头退回不透明纸面并增强分隔；减少透明度时关闭模糊并使用主题实色，导航职责与布局保持不变。
- 页头保持粘性，但不得形成横向滚动或遮挡锚点定位。
- 共享 CSS 与导航脚本 URL 必须携带当前 Pages 构建版本；每次部署后浏览器应立即取得匹配的 HTML、样式和目录逻辑，不能因旧缓存隐藏导航或保留过期交互。
- 品牌星形标记使用主题响应配色：浅色模式深底浅字，深色模式浅底深字；内部 `N` 与底板必须保持清晰对比。站点 favicon 是独立 SVG，不用于页头标记。
