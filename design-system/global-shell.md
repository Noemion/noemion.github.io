# 全局外壳与导航

适用范围：默认布局、站点头部、页脚、全局导航、移动目录、文档侧栏和目录数据。

## 布局基线

- 桌面画布宽度上限为 1200px，居中放置；顶部导航高度为 64px。
- 头部、正文和页脚的左右边界必须对齐，文档页的 320px 侧栏与 880px 阅读区共同占满画布。
- 页面外侧使用低对比灰绿背景，内容画布使用纸白或深色纸面。
- 1000px 以下隐藏固定文档栏，但顶部五组任务导航继续保留到 840px；只有 840px 以下才隐藏桌面分组导航，并显示带“导航”文字的移动入口，不能只显示缺乏含义的汉堡图标。

## 顶部导航

- 顶部栏保持粘性，并使用约 90% 的主题纸面、12px 背景模糊和轻微饱和度补偿；边缘只保留低对比分隔线与一层极浅的底部高光。页面滚动时，下方内容的色彩应通过半透明表面自然、克制地显现，使顶部栏与当前内容保持连续关系；不得在滚动阈值处突然切换为实色、重阴影或高对比分隔。
- 一级入口按读者任务组织为项目、规范、工具、指南和开发，不机械复制文件夹名称。
- 右侧项目状态入口只显示一个“4 字阶段名 + 阶段”的白色状态卡片，不再额外显示“当前阶段”标签或分隔线。阶段文字右侧放置不超过 40px 的短活动轨道，彩色活动段从轨道外进入并从另一侧离开，以仅改变 `transform` 和 `opacity` 的循环表达“正在进行”；轨道与卡片本身不循环位移，不使用百分比或填充长度暗示虚构完成度。白色表面保留纯黑透明描边和轻微阴影，深色模式仍保持白底。阶段值、链接和无障碍名称统一读取 `_data/project_timeline.yml`，无障碍名称仍由配置标签与 `current_stage_id` 对应节点标题组合；减少动态效果时停止循环并保留静态活动段。当前配置必须链接到 `development/current-stage.html`，不能链接到泛化开发首页，也不能让读者误认为是当前页面状态或稳定版本标记。
- 桌面端悬停和键盘聚焦都必须展开纵向内容面板；鼠标进入后约 40ms 自动展开，离开分组后保留约 120ms 的移动容错，让光标能稳定进入卡片且不会在相邻分组间留下旧面板。触发器使用 `aria-haspopup` 并同步 `aria-expanded`，当前页使用 `aria-current`。
- 每个一级分组必须先说明其在 Noemion 信息架构中的用途，再展示对应入口；不能按视频、文章等外部内容形态机械组织。
- 展开面板包含入口专属 SVG 封面、标题、摘要和行动箭头，不显示分组或卡片序号；22 个入口分别表达目标身份、信任流水线、目标图、Section 表、依赖闭包、工具职责、阅读路径、当前阶段或证据状态，不能复用同一封面换色冒充定制。
- 面板使用约 200ms 的淡入、去模糊、上移与轻缩放组合；菜单内容按 45ms 依次进入。卡片悬停使用彩色轮廓、轻微横移、封面缩放和箭头反馈。封面只使用项目自有图形语汇，不能引用外部站点图片、文案或链接。
- 移动目录复用 `assets/directory.js` 的同一数据，不维护第二套链接。

## 目录数据

- 普通页面目录由 `DIRECTORY_MODULES` 提供。
- 手册目录由 Jekyll 扫描带 `manual_id` 的 Markdown 页面并输出 JSON，JavaScript 只负责渲染。
- 当前页高亮必须使用规范化 URL，正确处理 `/index.html` 与目录形式。
- 动态数据不可用时允许回退到最小静态入口，但不得产生错误链接。

## 全站页脚与主题

- 所有页面使用同一共享页脚：上层按项目、阅读、开发和发现分为四列，下层提供 `Browse All`、文档、工具、开发、FAQ 与许可证入口。只能链接真实存在的 Noemion 内容，不得用不存在的订阅、账户或词典功能填充版面。
- 页脚在快捷链接与主题选择所在底栏自然结束，不再追加页面标题、开发阶段或安装状态等重复说明带；具体状态只在承担该信息职责的正文区域呈现。
- 页脚右下角使用 36px 高的胶囊主题按钮，显示当前选择值与 `Theme`；点击后在按钮上方展开 128px 宽的 `Light / Dark / System` 菜单，当前项显示勾选。菜单以 180ms 淡入、去模糊、上移和 0.95→1 缩放进入，关闭动画为 150ms，按压缩放固定为 0.96。
- 主题选择使用站点级持久化键保存，并在共享 CSS 加载前恢复。`Light` 与 `Dark` 固定主题；`System` 根据 `prefers-color-scheme` 解析，并在操作系统设置变化时实时更新。任何页面跳转后都必须保留上次选择。
- 主题菜单支持点击外部和 `Escape` 关闭，支持方向键、Home、End 和 `menuitemradio` 状态；按钮最小触控高度为 44px（移动端）。太阳与月亮图标同时保留在 DOM 中，以 opacity、4px blur 和 0.25→1 scale 交叉切换。
- 800px 以下四列折为两列，560px 以下折为单列；主题按钮在手机端占满页脚底行，菜单向上展开，不产生横向溢出。

## 可访问性与状态

- 跳到正文链接、键盘焦点和 `aria-expanded` 状态必须保留。
- 839px 以下的非首页使用两层固定头部：第一行是品牌与阶段，第二行把面包屑路径和 84×48px 导航按钮放在同一表面；首页没有面包屑，继续保留单行移动头部。展开面板从第二行下方进入，不能让导航按钮脱离路径栏孤立悬挂。
- 移动目录按钮操作面积至少 44×44px，展开时汉堡图标切换为关闭状态。再次点击按钮、点击面板外、按 `Escape` 或向下滚动正文都会触发收回；关闭使用 180ms 的透明度、4px 模糊和 12px 上移过渡，减少动态效果时立即关闭。
- 移动目录打开期间必须锁定根页面滚动，目录容器自身保持纵向滚动、触摸惯性与 `overscroll-behavior: contain`；在目录卡片上拖动只能滚动目录内容，不能把手势链传给后方正文。滚动锁在关闭动画完成后才解除。
- 交互过渡使用 150–200ms，声明具体属性；减少动态效果时立即切换。
- 页头保持粘性，但不得形成横向滚动或遮挡锚点定位。
- 共享 CSS 与导航脚本 URL 必须携带当前 Pages 构建版本；每次部署后浏览器应立即取得匹配的 HTML、样式和目录逻辑，不能因旧缓存隐藏导航或保留过期交互。
- 品牌星形标记使用主题响应配色：浅色模式深底浅字，深色模式浅底深字；内部 `N` 与底板必须保持清晰对比。站点 favicon 是独立 SVG，不用于页头标记。
